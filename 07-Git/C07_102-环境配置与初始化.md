## 1. Git 简介

### 1.1 什么是 Git

Git 是一个分布式版本控制系统，用于追踪文件变化、协调多人协作开发。

### 1.2 Git vs 其他 VCS

| 特性 | Git | SVN | Mercurial |
|------|-----|-----|----------|
| 类型 | 分布式 | 集中式 | 分布式 |
| 分支 | 轻量级快速 | 较慢 | 较快 |
| 网络依赖 | 低 | 高 | 低 |

## 2. 安装 Git

### 2.1 Windows 安装

从官网下载安装包：`https://git-scm.com/download/win`

### 2.2 Linux 安装

```bash
# Debian/Ubuntu
sudo apt update
sudo apt install git

# CentOS/RHEL
sudo yum install git
```

### 2.3 macOS 安装

```bash
# 使用 Homebrew
brew install git
```

### 2.4 验证安装

```bash
git --version
```

## 3. 仓库初始化

### 3.1 初始化本地仓库

在当前目录初始化一个新的 Git 仓库：

```bash
# 进入项目目录
cd <项目目录>

# 初始化 Git 仓库
git init
```

这会创建一个 `.git` 目录，包含所有 Git 仓库需要的数据。

### 3.2 配置初始信息

```bash
# 全局配置用户名
git config --global user.name "Your Name"

# 全局配置邮箱
git config --global user.email "your.email@example.com"

# 查看配置
git config --list
```

### 3.3 初始化现有项目

对于已有代码的项目：

```bash
cd existing-project
git init
git add .
git commit -m "Initial commit"
```

## 4. 基本配置

### 4.1 配置文件位置

- **系统级**：`/etc/gitconfig`
- **用户级**：`~/.gitconfig` 或 `~/.config/git/config`
- **仓库级**：`.git/config`

### 4.2 常用配置项

```bash
# 设置默认编辑器
git config --global core.editor vim

# 设置默认分支名
git config --global init.defaultBranch main

# 启用颜色输出
git config --global color.ui auto

# 设置代理（可选）
git config --global http.proxy http://proxy.example.com:8080
```

### 4.3 SSH 密钥配置

```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your.email@example.com"

# 查看公钥
cat ~/.ssh/id_ed25519.pub
```

### 4.4 关联远程仓库

```bash
# 添加远程仓库
git remote add origin https://github.com/username/repo.git

# 查看远程仓库
git remote -v
```

## 5. 常见配置问题与解决方案

### 5.1 行尾符问题

Windows 和 Unix 系统使用不同的行尾符，可能导致问题：

```bash
# 提交时转换为 LF，检出时转换为 CRLF
git config --global core.autocrlf true

# Linux/Mac
git config --global core.autocrlf input

# Windows
git config --global core.autocrlf true
```

### 5.2 中文文件名问题

```bash
# 让 Git 正确显示中文
git config --global core.quotepath false
```

### 5.3 忽略文件不生效

```bash
# 清除 Git 缓存
git rm -r --cached .
git add .
```

## 6. 最佳实践

- 首次使用时配置好用户名和邮箱
- 使用 SSH 密钥连接远程仓库，避免每次输入密码
- 创建 `.gitignore` 文件排除不需要版本控制的文件
- 经常进行提交，保持提交信息的清晰和一致性
- 使用分支开发新功能，完成后合并到主分支
