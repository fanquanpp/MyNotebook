## 1. 配置系统概述

### 1.1 配置项变量

配置项变量（`config.*`）控制 Ren'Py 的执行器行为，通常在 `init python` 或 `define` 语句中设置：

```renpy
define config.rollback_enabled = False
define config.name = "My Visual Novel"
define config.version = "1.0.0"
```

### 1.2 配置时机

```renpy
# 正确: 在 init 块中配置
init python:
    config.window_title = "我的视觉小说"

# 错误: 在游戏开始后修改
label start():
    $ config.name = "Changed"  # 可能导致未定义行为
```

### 4.1 基本配置


### 4.2 AFM 回调

```renpy
define config.afm_callback = my_afm_check        # AFM 安全检查回调

init python:
    def my_afm_check():
        if voice.get_playing():
            return False
        return True
```

## 9. 构建发布

### 9.1 构建命令

```bash
# 在 Ren'Py Launcher 中
# 1. 选择你的项目
# 2. 点击 "Build Distributions"
# 3. 选择发布格式
# 4. 点击 "Build"

# 命令行构建
renpy分布.py "<path_to_project>" --destination "<output_dir>"
```

### 9.2 发布格式

```renpy
# 指定发布格式
define config.distribution_formats = ["zip"]
define config.destination_directory = "build"
```

### 9.3 构建配置

```renpy
# game/options.rpy 中的配置
define build.name = "MyVisualNovel"
define build.version = "1.0.0"

define build.classifier = ["Visual Novel", "Game"]

# 包含额外文件
define build.exclude_patterns = ["*.bak", "*.tmp", ".git/*"]

# 包含额外目录
init python:
    build.include_directory("game/extra_content")
```

## 13. 最佳实践

1. **配置时机**: 所有 `config.*` 变量应在 `init` 块中设置
2. **版本管理**: 在存档中存储版本号，确保兼容性
3. **平台适配**: 针对不同平台调整配置
4. **构建优化**: 使用 `build.exclude_patterns` 排除不必要文件
5. **测试发布**: 在目标平台上测试构建结果
