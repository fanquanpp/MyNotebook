## 1. 屏幕系统概述 (Screen System Overview)

### 1.1 屏幕系统简介

屏幕系统是 Ren'Py 中用于创建自定义用户界面的强大工具。它允许开发者创建各种界面元素，如菜单、对话框、HUD 等。

### 1.2 屏幕的作用

- **创建自定义界面**: 完全控制游戏的视觉外观
- **处理用户输入**: 响应鼠标点击、键盘输入等
- **显示动态内容**: 根据游戏状态显示不同内容
- **组织 UI 元素**: 提供多种布局方式

### 1.3 屏幕与标签 (tag) 的关系

```renpy
# ==========================================
# 示例 1-1: 屏幕与标签的关系
# ==========================================

# 当使用 show screen 或 hide screen 时
# tag 用于标识和引用屏幕

screen main_menu():
    tag menu       # 定义此屏幕的标签为 "menu"
    # ...

screen settings():
    tag settings   # 定义此屏幕的标签为 "settings"
    # ...

# 显示屏幕 - 使用 tag 引用
show screen menu
hide screen settings

# 等价于
show menu
hide settings
```

## 2. 屏幕定义基础 (Screen Definition Basics)

### 2.1 基本语法

```renpy
# ==========================================
# 示例 2-1: 最简单的屏幕定义
# ==========================================

screen hello():
    # screen 语句开始一个屏幕定义
    # hello 是屏幕名称（唯一标识符）

    # 屏幕内容 - 使用 Ren'Py 的屏幕语言
    text "Hello, World!"  # 显示一个文本
    # 这是一个文本控件

# 执行流程:
# 1. 当调用 "show screen hello" 或 "show hello" 时
# 2. Ren'Py 创建屏幕实例
# 3. 执行屏幕内容，显示所有控件
# 4. 屏幕保持显示，直到调用 "hide screen hello"

# ==========================================
# 示例 2-2: 带按钮的屏幕
# ==========================================

screen greet():
    text "欢迎来到游戏！"

    # button 控件 - 可点击的按钮
    button:
        # button 需要包含内容，通常是 text
        text "开始游戏"

        # action 指定点击按钮时执行的动作
        action Start()

# 解释:
# - button 是一个容器控件
# - text 是 button 的子控件
# - action=Start() 是 Ren'Py 内置动作，开始新游戏

# ==========================================
# 示例 2-3: 使用 textbutton（文本按钮）
# ==========================================

screen simple_menu():
    # textbutton 结合了 button 和 text
    # 更简洁的语法

    textbutton "开始游戏":
        action Start()

    textbutton "加载游戏":
        action ShowMenu("load")

    textbutton "退出":
        action Quit()

# 推荐: 优先使用 textbutton，语法更简洁
```

### 2.2 屏幕名称与标签详解

```renpy
# ==========================================
# 示例 2-4: 屏幕的 tag 和 name
# ==========================================

# 屏幕名称 (screen_name):
# - 屏幕定义的名称
# - 用于引用屏幕定义

# 屏幕标签 (tag):
# - 屏幕实例的标识符
# - 可以与屏幕名称不同
# - 用于 show/hide 操作

screen main():           # name = "main"
    tag menu            # tag = "menu"
    text "主菜单"

# 引用方式:
show screen main    # 显示 name="main" 的屏幕
show menu          # 显示 tag="menu" 的屏幕
hide menu          # 隐藏 tag="menu" 的屏幕

# 注意: tag 必须是唯一的
screen first():
    tag unique_tag
    text "第一个"

screen second():
    tag unique_tag  # 错误！tag 已存在
    text "第二个"
```

## 3. 布局系统 (Layout System)

### 3.1 布局概述

布局用于组织屏幕上的控件位置。Ren'Py 提供多种布局方式。

### 3.2 垂直布局 (vbox)

```renpy
# ==========================================
# 示例 3-1: vbox 垂直布局
# ==========================================

screen vbox_example():
    # vbox 将子控件垂直排列
    vbox:
        # spacing 设置控件之间的间距
        spacing 20

        # 第一个子控件
        text "第一项"

        # 第二个子控件
        text "第二项"

        # 第三个子控件
        text "第三项"

# 布局效果:
# [  第一项  ]
# [ 20px 间距 ]
# [  第二项  ]
# [ 20px 间距 ]
# [  第三项  ]

# ==========================================
# 示例 3-2: vbox 的对齐方式
# ==========================================

screen vbox_alignment():
    vbox:
        # xalign: 0.0=左对齐, 0.5=居中, 1.0=右对齐
        xalign 0.5    # 整个 vbox 在屏幕上水平居中

        spacing 10

        # 每个子控件可以独立设置 xalign
        text "居中文本" xalign 0.5
        text "左对齐文本" xalign 0.0
        text "右对齐文本" xalign 1.0
```

### 3.3 水平布局 (hbox)

```renpy
# ==========================================
# 示例 3-3: hbox 水平布局
# ==========================================

screen hbox_example():
    hbox:
        spacing 20    # 子控件之间的间距

        text "左侧"
        text "中间"
        text "右侧"

# 布局效果:
# [ 左侧 ] [ 20px 间距 ] [ 中间 ] [ 20px 间距 ] [ 右侧 ]

# ==========================================
# 示例 3-4: hbox 的对齐方式
# ==========================================

screen hbox_alignment():
    hbox:
        # yalign: 0.0=顶部对齐, 0.5=垂直居中, 1.0=底部对齐
        yalign 0.5    # 整个 hbox 垂直居中

        spacing 15

        text "上" yalign 0.0
        text "中" yalign 0.5
        text "下" yalign 1.0
```

### 3.4 网格布局 (grid)

```renpy
# ==========================================
# 示例 3-5: grid 网格布局
# ==========================================

screen grid_example():
    # grid cols rows: 创建 cols 列 rows 行的网格
    grid 3 2:          # 3列, 2行
        spacing 10     # 单元格之间的间距

        text "1-1"
        text "1-2"
        text "1-3"
        text "2-1"
        text "2-2"
        text "2-3"

# 布局效果:
# [ 1-1 ] [ 1-2 ] [ 1-3 ]
# [ 2-1 ] [ 2-2 ] [ 2-3 ]
```

### 3.5 固定布局 (fixed)

```renpy
# ==========================================
# 示例 3-6: fixed 固定位置布局
# ==========================================

screen fixed_example():
    # fixed 允许使用 xpos, ypos 精确定位
    fixed:
        text "左上角" xpos 0.1 ypos 0.1
        text "右上角" xpos 0.8 ypos 0.1
        text "中心" xpos 0.5 ypos 0.5
        text "左下角" xpos 0.1 ypos 0.8

# ==========================================
# 示例 3-7: 固定大小区域
# ==========================================

screen fixed_size():
    fixed:
        # xsize, ysize 设置固定大小
        xsize 300
        ysize 200

        # area 设置可用区域
        area (0, 0, 300, 200)

        text "这段文字会在 300x200 的区域内显示"
```

### 3.6 框架布局 (frame)

```renpy
# ==========================================
# 示例 3-8: frame 带边框的容器
# ==========================================

screen frame_example():
    frame:
        # frame 自动添加背景和边框
        padding (20, 20)    # 内边距

        # 内容
        vbox:
            text "框架内的内容"
            text "第二行"
```

### 3.7 视图端口 (viewport) 和滚动

```renpy
# ==========================================
# 示例 3-9: viewport 可滚动区域
# ==========================================

screen scrollable_example():
    viewport:
        # scrollbars: 添加滚动条
        scrollbars "vertical"

        # xsize, ysize 设置视口大小
        xsize 300
        ysize 200

        # vbox 包含大量内容
        vbox:
            for i in range(20):
                text "项目 #{i+1}"
```

### 3.8 布局组合示例

```renpy
# ==========================================
# 示例 3-10: 复杂布局组合
# ==========================================

screen inventory_screen():
    # 主容器 - 固定布局
    fixed:
        # 顶部标题
        text "背包" size 30 xalign 0.5 ypos 0.05

        # 物品网格 - 3列网格
        grid 3 4:
            xpos 0.1
            ypos 0.15
            spacing 10

            # 假设有12个物品槽
            for i in range(12):
                frame:
                    xsize 80
                    ysize 80

                    # 物品图标
                    if i < len(inventory):
                        add inventory[i].image

                    # 物品数量
                    if inventory[i].count > 1:
                        text "[inventory[i].count]" align (0.95, 0.95)

        # 底部按钮栏 - 水平布局
        hbox:
            yalign 0.95
            xalign 0.5
            spacing 20

            textbutton "使用":
                action ShowScreen("use_item")
            textbutton "丢弃":
                action ShowScreen("discard_item")
            textbutton "返回":
                action Return()
```

```renpy
screen imagebutton_demo():

        # idle: 鼠标悬停外时显示的图像

        hover "buttons/start_hover.png"

screen auto_imagebutton():
```

### 4.4 输入控件 (input)

```renpy
# ==========================================
# 示例 4-6: input 文本输入
# ==========================================

screen name_input():

        input:
            # default: 默认显示的文本
            default "匿名"

            # length: 最大输入长度
            length 20

            length wrap

            # 允许的字符
            allow "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ"

            # 排除的字符
            exclude " \t\n"
```

## 5. 屏幕动作 (Screen Actions)

### 5.1 动作分类

| 类别 | 动作 | 描述 |
| :--- | :--- | :--- |
| **游戏控制** | Start() | 开始新游戏 |
| **游戏控制** | Quit() | 退出游戏 |
| **游戏控制** | MainMenu() | 返回主菜单 |
| **游戏控制** | GameMenu() | 显示游戏菜单 |
| **存档** | Load() | 加载存档 |
| **存档** | Save() | 保存游戏 |
| **存档** | ShowMenu("save") | 显示保存界面 |
| **存档** | ShowMenu("load") | 显示加载界面 |
| **流程控制** | Jump(label) | 跳转到标签 |
| **流程控制** | Call(label) | 调用标签 |
| **流程控制** | Return() | 返回 |
| **变量控制** | SetVariable(var, value) | 设置变量 |
| **变量控制** | ToggleVariable(var) | 切换变量 |
| **屏幕控制** | ShowScreen(screen) | 显示屏幕 |
| **屏幕控制** | HideScreen(screen) | 隐藏屏幕 |
| **通用** | Function(func, *args) | 调用函数 |
| **通用** | NullAction() | 空动作 |

### 5.2 动作详解

```renpy
# ==========================================
# 示例 5-1: Start 和 Quit 动作
# ==========================================

screen start_screen():
    textbutton "开始游戏":
        action Start()    # 开始新游戏（从 start 标签）

    textbutton "退出":
        action Quit()      # 退出游戏

# ==========================================
# 示例 5-2: Jump 和 Call 动作
# ==========================================

screen chapter_select():
    textbutton "第一章":
        action Jump("chapter1")

    textbutton "第二章":
        action Jump("chapter2")

    textbutton "查看结局（需要解锁）":
        # 带条件
        action If(ending_unlocked, Jump("ending"), NullAction())

# ==========================================
# 示例 5-3: SetVariable 和 ToggleVariable
# ==========================================

screen settings():
    textbutton "音乐 [settings.music_on ? 'ON' : 'OFF']":
        # ToggleVariable 会切换布尔值
        action ToggleVariable("settings.music_on")

    textbutton "音效音量: [settings.sfx_volume]":
        # SetVariable 设置具体值
        action ShowScreen("sfx_volume_slider")
```

### 5.3 动作组合

```renpy
# ==========================================
# 示例 5-4: 动作列表
# ==========================================

screen confirm_dialog():
    textbutton "确定":
        # 列表中的动作按顺序执行
        action [
            SetVariable("confirmed", True),
            Return()  # 返回上一层
        ]

# ==========================================
# 示例 5-5: 条件动作
# ==========================================

screen save_button():
    textbutton "保存游戏":
        # If(condition, true_action, false_action)
        action If(save_exists,
                  Save(),              # 如果有存档则保存
                  ShowScreen("no_save")) # 否则显示提示
```

### 5.4 自定义动作函数

```renpy
# ==========================================
# 示例 5-6: 使用 Function 调用自定义函数
# ==========================================

init python:

screen custom_action_demo():
```

## 6. 屏幕语句 (Screen Statements)

### 6.1 语句详解

| 语句 | 描述 | 示例 |
| :--- | :--- | :--- |
| **`screen`** | 定义屏幕 | `screen name():` |
| **`has`** | 指定容器 | `has vbox:` |
| **`use`** | 包含屏幕 | `use navigation` |
| **`default`** | 设置默认值 | `default value = 5` |
| **`for`** | 循环 | `for i in range(10):` |
| **`if/elif/else`** | 条件 | `if flag:` |
| **`on`** | 事件处理 | `on show:` |
| **`showif`** | 条件显示 | `showif visible:` |
| **`python`** | Python 代码 | `python:` |

### 6.2 has 语句

```renpy
# ==========================================
# 示例 6-1: has 语句简化嵌套
# ==========================================

# 不使用 has 的写法
screen menu_v1():

        vbox:
            textbutton "选项1":
                action Return()
            textbutton "选项2":
                action Return()

# 使用 has 直接指定容器
screen menu_v2():

            textbutton "选项1":
                action Return()

                action Return()
```

### 6.3 use 语句（屏幕复用）

```renpy
# ==========================================
# 示例 6-2: use 语句复用屏幕
# ==========================================

# 定义可复用的导航屏幕
screen navigation():
    vbox:
        textbutton "开始":
            action Start()
        textbutton "加载":
            action ShowMenu("load")
        textbutton "设置":
            action ShowMenu("preferences")

# 在其他屏幕中复用
screen main_menu():
    # 添加背景
    add "bg_menu.png"

    # 标题
    text "我的游戏" size 48

    # 使用 navigation 屏幕
    use navigation

# ==========================================
# 示例 6-3: 传递参数给 use
# ==========================================

screen list_item():
    # 参数通过 use 传递
    default item

    text "[item.name]"
    text "[item.description]" size 16

screen item_list():
    vbox:
        for item in items:
            # 传递 item 给 list_item 屏幕
            use list_item(item)
```

## 7. 屏幕变种 (Screen Variants)

### 7.1 变种概述

屏幕变种允许为不同设备定义不同的界面。

### 7.2 变种使用

```renpy
# ==========================================
# 示例 7-1: 基本变种定义
# ==========================================

# PC 版本的界面
screen main_menu():
    variant "pc"     # 或省略，因为 "pc" 是默认变种

    frame:
        has hbox

        textbutton "开始":
            action Start()

# 触摸设备的界面
screen main_menu():
    variant "touch"

    # 触摸设备使用更大的按钮
    textbutton "开始":
        action Start()
        text_size 36
        xsize 400
        ysize 100
```

### 9.3 定时器

```renpy
# ==========================================
# 示例 9-3: 倒计时屏幕
# ==========================================

screen countdown():
    default time_left = 10

    text "剩余时间: [time_left]" size 48

    # repeat=True: 重复执行
    # action 必须返回一个值来更新 time_left
    timer 1.0 repeat True action SetScreenVariable("time_left", time_left - 1)

    # 当 time_left <= 0 时触发
    if time_left <= 0:
        text "时间到！" size 60
        timer 2.0 action Return()
```

## 11. 性能优化

### 11.1 屏幕性能建议

```renpy
# ==========================================
# 性能优化指南
# ==========================================

# 1. 避免不必要的屏幕
# 不推荐: 在屏幕中嵌套大量条件
screen bad_example():
    if show_element_1:
        text "..."
    if show_element_2:
        text "..."
    # ... 更多条件

# 推荐: 使用 showif 或分层屏幕
screen good_example():
    text "..."
    showif show_element_1:
        text "..."

# 2. 减少复杂计算
# 不推荐: 在屏幕中使用复杂表达式
text "[sum([i*2 for i in range(1000)])]"

# 推荐: 在 Python 代码中预计算
init python:
    expensive_value = sum([i*2 for i in range(1000)])
```

## 13. 总结

### 13.1 关键要点

- **屏幕是 Ren'Py UI 的核心**: 所有界面元素都在屏幕中定义
- **布局系统**: vbox、hbox、grid、fixed 等用于组织控件
- **丰富控件**: text、button、imagebutton、input 等
- **动作系统**: Start、Jump、SetVariable 等处理交互
- **屏幕复用**: use 语句可以复用屏幕组件
- **样式系统**: 统一管理界面外观
- **设备适配**: 屏幕变种支持不同平台

### 13.2 学习建议

- **从简单屏幕开始**: 先创建基本菜单
- **理解布局**: 掌握 vbox、hbox 的使用场景
- **实践动作**: 练习各种动作的组合使用
- **创建可复用组件**: 将常用 UI 提取为独立屏幕
- **性能意识**: 注意屏幕的性能影响
