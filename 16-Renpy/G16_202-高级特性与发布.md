## 1. 屏幕系统 (Screen System)

屏幕系统是 Ren'Py 最强大的部分之一，用于创建自定义用户界面。

### 1.1 基础屏幕定义

```renpy
screen main_menu():
    vbox:
        xalign 0.5
        yalign 0.5
        spacing 20

        text "游戏标题" size 48 bold True

        imagebutton:
            idle "button_idle.png"
            hover "button_hover.png"
            action Start()

        imagebutton:
            idle "button_idle.png"
            hover "button_hover.png"
            action ShowMenu("load")

        imagebutton:
            idle "button_idle.png"
            hover "button_hover.png"
            action Quit()
```

### 1.7 界面变种 (Screen Variants)

Ren'Py 支持为不同设备定义不同界面：

```renpy
screen hello_world():
    variant "small"
    text "Hello, World." size 30
```

常见变种：`pc`, `touch`, `mobile`, `tablet`, `phone`, `medium`, `small`

## 3. Python 集成 (Python Integration)

### 3.1 基础用法

```renpy
$ player_name = "Alice"

python:
    def calculate_damage(attacker, defender):
        base_damage = attacker.strength * 2
        defense = defender.defense * 0.5
        return max(1, base_damage - defense)
```

### 3.2 init python 块

```renpy
init python:
    import random
    import math

    class Inventory:
        def __init__(self):
            self.items = []

        def add_item(self, item):
            self.items.append(item)

        def remove_item(self, item):
            if item in self.items:
                self.items.remove(item)
```

### 3.3 性能优化

- **使用 `init python:` 预计算数据**
- **避免在热路径中使用复杂计算**
- **使用局部变量减少属性查找开销**
- **合理使用 `default` 和 `define` 语句**
- **使用 `store` 模块组织代码**

## 5. 构建与发布 (Build & Distribution)

### 5.1 构建流程

1. **准备资源**: 确保所有资源文件都已就绪
2. **设置项目信息**: 在 `options.rpy` 中配置游戏信息
3. **构建发布版本**: 使用 Ren'Py Launcher 的 "Build Distributions"
4. **测试发布版本**: 在目标平台上测试游戏

### 5.2 多平台构建

| 平台 | 输出格式 |
| :--- | :--- |
| **Windows** | .exe 文件 |
| **macOS** | .app 包 |
| **Linux** | 可执行文件和所需库 |
| **Web** | Web 版本 (HTML5) |
| **Android** | .apk 文件 |
| **iOS** | iOS 应用 |

### 5.3 资源优化

- **图像压缩**: 使用适当的图像格式和压缩率
- **音频优化**: 调整音频质量和格式
- **脚本编译**: 编译 .rpy 文件为 .rpyc
- **资源混淆**: 启用资源加密保护
- **文件大小优化**: 移除未使用的资源

## 7. 性能优化与调试

### 7.1 性能优化

- **图像缓存**: 合理使用 `cache` 属性
- **纹理打包**: 使用纹理图集减少 Draw Call
- **代码优化**: 减少 Python 代码在热路径中的使用
- **内存管理**: 及时释放不需要的资源
- **渲染优化**: 减少屏幕元素数量
- **脚本优化**: 避免不必要的计算

### 7.2 调试技巧

| 技巧 | 说明 |
| :--- | :--- |
| `renpy.trace()` | 启用跟踪调试 |
| `renpy.log()` | 记录日志信息 |
| `shift+D` | 打开开发者控制台 |
| 性能分析 | 使用 `renpy.profile()` 分析性能 |
| 变量检查 | 使用 `renpy.get_variables()` 查看变量状态 |
| 错误处理 | 使用 `try-except` 捕获异常 |

### 7.3 常见问题解决

- **内存泄漏**: 检查循环引用和未释放的资源
- **性能瓶颈**: 使用性能分析工具定位问题
- **兼容性问题**: 测试不同平台和设备
- **打包错误**: 检查资源文件路径和依赖
- **回滚问题**: 避免使用不可回滚的操作
