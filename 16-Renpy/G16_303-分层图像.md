## 1. 分层图像概述 (Layered Images Overview)

### 1.1 什么是分层图像

分层图像是 Ren'Py 中用于创建复杂角色精灵的强大工具。它允许开发者将角色分解为多个图层（如身体、头部、表情、服装等），并根据游戏状态动态组合这些图层。

### 1.2 分层图像的优势

```renpy
# ==========================================
# 示例 1-1: 分层图像 vs 传统方式
# ==========================================

# 传统方式 - 为每个组合创建单独图像
# 需要: happy_standing.png, happy_sitting.png, sad_standing.png, sad_sitting.png...
# 问题: 大量图像文件，难以维护

# 分层图像方式 - 组合图层
# 只需: base.png, happy.png, sad.png, standing.png, sitting.png
# 优势:
# - 减少图像文件数量
# - 方便替换和修改
# - 动态组合不同状态
```

### 1.3 执行流程

```renpy
# ==========================================
# 示例 1-2: 分层图像的加载流程
# ==========================================

# 当使用 show eileen happy standing 时:
# 1. Ren'Py 查找 layeredimage "eileen" 的定义
# 2. 找到 "always" 组，添加 base.png
# 3. 找到 "expression" 组的 "happy" 属性，添加 happy.png
# 4. 找到 "pose" 组的 "standing" 属性，添加 standing.png
# 5. 组合所有图层并显示

# 解析流程图:
# show eileen happy standing
#         ↓
# layeredimage "eileen" 定义
#         ↓
# always: base.png (始终显示)
#         ↓
# group expression:
#   attribute happy → happy.png (显示)
#   attribute sad → (不显示)
#         ↓
# group pose:
#   attribute standing → standing.png (显示)
#         ↓
# 最终组合: base.png + happy.png + standing.png
```

## 2. 基本语法 (Basic Syntax)

### 2.1 基本定义结构

```renpy
# ==========================================
# 示例 2-1: 最基本的分层图像定义
# ==========================================

layeredimage character_name:
    # always 组: 始终显示的图层
    always:
        "base_layer.png"  # 基础身体

    # group: 图层组
    # expression: 组名
    # auto: 自动处理图层命名
    group expression auto:
        # 自动查找: expression_neutral.png, expression_happy.png...
        # 会创建 neutral, happy 等属性

    group pose auto:
        # 自动查找: pose_standing.png, pose_sitting.png...
```

### 2.2 完整定义示例

```renpy
# ==========================================
# 示例 2-2: 完整的分层图像定义
# ==========================================

layeredimage eileen:
    # always 组 - 始终显示的基础图层
    always:
        "eileen_base.png"      # 基础身体轮廓
        "eileen_hair.png"      # 头发（始终显示）

    # group 组 - 可切换的图层组
    # auto 属性 - 自动根据文件名创建属性
    group expression auto:
        # 期望文件: eileen_expression_neutral.png
        #          eileen_expression_happy.png
        #          eileen_expression_sad.png
        # 创建属性: neutral, happy, sad

    group pose auto:
        # 期望文件: eileen_pose_standing.png
        #          eileen_pose_sitting.png
        # 创建属性: standing, sitting

    group clothes auto:
        # 期望文件: eileen_clothes_casual.png
        #          eileen_clothes_formal.png
        # 创建属性: casual, formal
```

## 3. 图层组详解 (Layer Groups)

### 3.1 always 组

```renpy
# ==========================================
# 示例 3-1: always 组的使用
# ==========================================

layeredimage character:
    # always 组的图层始终显示
    always:
        "body_base.png"      # 身体基础（必须显示）
        "hair_back.png"      # 后层头发
        "body_outline.png"   # 轮廓线

    # 其他图层组...
```

### 3.2 group 组

```renpy
# ==========================================
# 示例 3-2: 带属性的 group 组
# ==========================================

layeredimage eileen:
    always:
        "eileen_base.png"

    # group 组需要显式定义属性
    group expression:
        # 定义属性
        attribute neutral:
            "eileen_neutral.png"    # 中性表情

        attribute happy:
            "eileen_happy.png"      # 开心表情

        attribute sad:
            "eileen_sad.png"       # 悲伤表情

        attribute angry:
            "eileen_angry.png"     # 愤怒表情

    # 属性可以按顺序匹配文件
    group clothes:
        attribute casual:
            "eileen_casual.png"

        attribute formal:
            "eileen_formal.png"
```

### 3.3 auto 属性

```renpy
# ==========================================
# 示例 3-3: auto 属性的作用
# ==========================================

# 不使用 auto - 需要手动定义每个属性
layeredimage eileen_v1:
    always:
        "eileen_base.png"

    group expression:
        attribute neutral:
            "eileen_neutral.png"
        attribute happy:
            "eileen_happy.png"

# 使用 auto - Ren'Py 自动从文件名创建属性
# 文件命名规范: group_attribute.png
layeredimage eileen_v2:
    always:
        "eileen_base.png"

    # auto 会自动查找 eileen_expression_neutral.png 等
    group expression auto:
        # Ren'Py 会自动创建 neutral, happy 等属性
        # 期望文件: eileen_expression_*.png
```

### 3.4 default 属性

```renpy
# ==========================================
# 示例 3-4: 设置默认属性
# ==========================================

layeredimage eileen:
    always:
        "eileen_base.png"

    group expression:
        # default 属性 - 当不指定时使用的默认属性
        attribute neutral default:
            "eileen_neutral.png"

        attribute happy:
            "eileen_happy.png"

        attribute sad:
            "eileen_sad.png"

# 使用:
# show eileen - 显示 neutral (默认)
# show eileen happy - 显示 happy
```

## 4. 图层属性 (Layer Attributes)

### 4.1 attribute 语句

```renpy
# ==========================================
# 示例 4-1: attribute 语句详解
# ==========================================

layeredimage character:
    always:
        "base.png"

    group expression:
        # 基本属性
        attribute happy:
            "happy.png"

        # 多个图层 (attribute 块可以包含多个文件)
        attribute angry:
            "angry_base.png"
            "angry_effect.png"    # 愤怒特效

        # 条件属性
        attribute injured if show_injury:
            "injury_effect.png"
```

### 4.2 if 条件

```renpy
# ==========================================
# 示例 4-2: if 条件显示图层
# ==========================================

layeredimage eileen:
    always:
        "eileen_base.png"

    # 根据变量条件显示
    group accessories:
        attribute glasses if wearing_glasses:
            "eileen_glasses.png"

        attribute hat if wearing_hat:
            "eileen_hat.png"

        attribute earring if has_earring:
            "eileen_earring.png"

# 使用:
$ wearing_glasses = True
show eileen - 会显示 glasses

$ wearing_glasses = False
show eileen - 不会显示 glasses
```

### 4.3 图层变换

```renpy
# ==========================================
# 示例 4-3: 为图层添加变换
# ==========================================

layeredimage character:
    always:
        "body_base.png"

    group expression:
        attribute happy:
            # 变换语法
            "happy.png"
            xoffset 5          # 水平偏移
            yoffset -3         # 垂直偏移
            zoom 1.05         # 缩放

        attribute sad:
            "sad.png"
            xoffset 0
            yoffset 2          # 悲伤时略微下垂
```

## 5. 使用分层图像

### 5.1 基本使用

```renpy
# ==========================================
# 示例 5-1: 显示分层图像
# ==========================================

# 定义分层图像
layeredimage eileen:
    always:
        "eileen_base.png"
    group expression auto
    group pose auto

# 使用 show 语句显示
show eileen
# 等价于: show eileen neutral standing (默认属性)

show eileen happy
# 显示: base + happy + standing (默认pose)

show eileen sad sitting
# 显示: base + sad + sitting

show eileen formal
# 显示: base + neutral (默认) + formal
```

### 5.2 动态属性切换

```renpy
# ==========================================
# 示例 5-2: 动态切换属性
# ==========================================

# 表情切换
show eileen happy
e "今天真开心！"

show eileen sad
e "等等，发生什么了..."

show eileen angry
e "太过分了！"

# 姿势切换
show eileen standing
e "我站着。"

show eileen sitting
e "我坐下了。"

# 服装切换
show eileen casual
e "穿着休闲装。"

show eileen formal
e "穿着正式服装。"
```

### 5.3 组合使用

```renpy
# ==========================================
# 示例 5-3: 复杂组合
# ==========================================

layeredimage character:
    always:
        "base.png"
    group expression:
        attribute neutral default
        attribute happy
        attribute sad
        attribute angry
    group pose:
        attribute standing default
        attribute sitting
        attribute walking
    group clothes:
        attribute casual default
        attribute formal
        attribute swimwear

# 所有组合都可以使用:
show character                    # neutral + standing + casual
show character happy              # happy + standing + casual
show character sad walking         # sad + walking + casual
show character angry sitting formal  # angry + sitting + formal
```

## 7. 实际应用示例

### 7.1 角色系统

```renpy
# ==========================================
# 示例 7-1: 完整的角色系统
# ==========================================

# 定义主要角色
layeredimage protagonist:
    always:
        "protagonist_body.png"
        "protagonist_hair_back.png"

    group expression:
        attribute neutral default:
            "protagonist_neutral.png"
        attribute happy:
            "protagonist_happy.png"
        attribute sad:
            "protagonist_sad.png"
        attribute angry:
            "protagonist_angry.png"
        attribute surprised:
            "protagonist_surprised.png"
        attribute embarrassed:
            "protagonist_embarrassed.png"

    group pose:
        attribute standing default:
            "protagonist_standing.png"
        attribute sitting:
            "protagonist_sitting.png"
        attribute walking:
            "protagonist_walking.png"
            pause 0.15
            "protagonist_walking2.png"
            pause 0.15
            repeat

    group clothes:
        attribute casual default:
            "protagonist_casual.png"
        attribute school:
            "protagonist_school.png"
        attribute formal:
            "protagonist_formal.png"

# 使用角色系统
label start():
    scene bg classroom

    show protagonist school
    # 显示: body + hair + neutral + standing + school

    p "大家好，我是主角！"

    show protagonist happy school
    p "今天真开心！"

    show protagonist sitting school
    p "先坐下来休息一下。"

    return
```

### 7.2 动态换装

```renpy
# ==========================================
# 示例 7-2: 动态换装系统
# ==========================================

layeredimage doll:
    always:
        "doll_base.png"

    group outfit:
        attribute dress default:
            "doll_dress.png"
        attribute uniform:
            "doll_uniform.png"
        attribute swimwear:
            "doll_swimwear.png"

    group accessory:
        attribute none:
            null    # null 表示不显示任何图像
        attribute hat:
            "doll_hat.png"
        attribute bow:
            "doll_bow.png"

# 换装脚本
label dress_up():
    $ current_outfit = "dress"
    $ current_accessory = "none"

    show doll

    "今天穿连衣裙出门。"

    $ current_outfit = "uniform"
    show doll uniform
    "还是穿校服吧。"

    $ current_accessory = "hat"
    show doll uniform hat
    "戴上帽子防晒。"

    return
```

## 8. 性能优化

### 8.1 优化技巧

```renpy
# ==========================================
# 性能优化指南
# ==========================================

# 1. 减少图层数量
# 不推荐: 10+ 个图层组
# 推荐: 3-5 个必要的图层组

# 2. 使用共享图层
# 将多个属性共享的图像放在 always 组

# 3. 图像尺寸优化
# 分层图像的每个图层应该使用合适的大小
# 避免使用过大的图像

# 4. 合理使用 auto
# auto 会预加载所有可能的图层
# 如果图层过多，考虑手动定义属性
```

### 8.2 缓存策略

```renpy
# ==========================================
# 分层图像缓存说明
# ==========================================

# Ren'Py 会自动缓存分层图像
# 缓存策略:
# - 访问过的属性组合会被缓存
# - 下次使用时直接调用缓存
# - 减少重新组合图层的开销

# 建议:
# - 避免在短时间内频繁切换大量属性
# - 批量切换属性时使用同一个 show 语句
```

## 9. 常见问题

### 9.1 图像找不到

```renpy
# ==========================================
# 常见问题: 属性文件命名
# ==========================================

# 问题: 属性图像不显示
# 可能原因:

# 1. 文件命名错误
# 错误: expression_happy.png
# 正确: character_name_expression_happy.png
#      或在 group expression 下定义:
#      attribute happy:
#          "happy.png"

# 2. 文件路径错误
# 确保文件在 game/images/ 目录下

# 3. auto 命名规范
# group expression auto 期望:
# {character}_{expression}_{attribute}.png
```

### 9.2 图层顺序

```renpy
# ==========================================
# 问题: 图层显示顺序错误
# ==========================================

# 解决: 调整图层组的定义顺序
# 后定义的组显示在上层

layeredimage character:
    always:
        "body.png"          # 最底层

    group clothes:
        "clothes.png"      # 在 body 上面

    group hair:
        "hair_front.png"   # 最上层

# 如果 hair_back.png 应该在下层:
layeredimage character:
    always:
        "hair_back.png"    # 先定义，在下层
        "body.png"

    group hair:
        "hair_front.png"   # 后定义，在上层
```

## 11. 总结

### 11.1 关键要点

- **分层图像是创建复杂角色的利器**: 通过组合图层实现不同状态
- **always 组**: 始终显示的基础图层
- **group 组**: 可切换的图层组
- **attribute**: 定义可切换的属性
- **auto 属性**: 自动从文件名创建属性
- **条件显示**: 使用 `if` 根据变量控制显示
- **动态切换**: 通过 show 语句切换属性

### 11.2 学习建议

- **从简单开始**: 先创建 2-3 个图层组的角色
- **理解图层顺序**: 后定义的组显示在上层
- **命名规范**: 遵循 `{name}_{group}_{attribute}.png` 规范
- **实践组合**: 练习不同属性的组合使用
- **性能意识**: 避免过多的图层组
- **参考示例**: 学习官方示例的分层图像实现

通过掌握分层图像，开发者可以创建出更加生动、灵活的角色精灵，为视觉小说增添丰富的视觉表现。
