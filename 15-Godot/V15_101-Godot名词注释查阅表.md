## 1. 基础概念类

### 1.1 Godot

**名称**：Godot 游戏引擎（Godot Engine）

**首次出现位置**：C15_101-概述与环境.md 第1章

**定义**：
Godot 是一款功能开源完整的 2D 和 3D 游戏引擎，提供场景系统、节点树、物理引擎等完整游戏开发功能。

### 1.2 GDScript

**名称**：GDScript 脚本语言（GDScript）

**首次出现位置**：C15_101-概述与环境.md 第1.1章

**定义**：
GDScript 是 Godot 引擎专用的高级动态类型脚本语言，语法深受 Python 启发，与引擎深度集成。

### 1.3 节点

**名称**：节点（Node）

**首次出现位置**：C15_101-概述与环境.md 第2.1章

**定义**：
节点是 Godot 中的基本构建单元，每个节点都有特定功能，可以添加到场景树中形成游戏。

### 1.4 场景

**名称**：场景（Scene）

**首次出现位置**：C15_101-概述与环境.md 第2.2章

**定义**：
场景是节点的树形组织结构，可以包含子节点，本质上是一个可重用的游戏对象模板。

### 1.5 节点树

**名称**：节点树（Node Tree）

**首次出现位置**：C15_101-概述与环境.md 第2.1章

**定义**：
节点树是 Godot 中组织的层次结构，场景中的所有节点按父子关系形成树形结构。

### 1.6 引擎

**名称**：引擎（Engine）

**首次出现位置**：C15_101-概述与环境.md 第1章

**定义**：
引擎是指提供游戏运行时核心功能的软件框架，Godot 引擎包含渲染、物理、音频等子系统。

## 2. 节点系统类

### 2.1 节点类型

**名称**：节点类型（Node Types）

**首次出现位置**：C15_102-基础语法.md 第2章

**定义**：
Godot 内置多种节点类型，如 Node2D、Spatial、Area2D、CharacterBody 等，每种类型提供不同功能。

### 2.2 Node2D

**名称**：2D节点（Node2D）

**首次出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
Node2D 是 2D 游戏开发的基础节点，支持 2D 变换，如位置、旋转、缩放。

### 2.3 Spatial

**名称**：3D节点（Spatial）

**首次出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
Spatial 是 3D 游戏开发的基础节点，支持 3D 变换，是所有 3D 节点的基类。

### 2.4 CharacterBody

**名称**：角色体（CharacterBody）

**首次出现位置**：C15_102-基础语法.md 第3.2章

**定义**：
CharacterBody 是用于实现玩家或 NPC 等受控角色的节点，提供移动和碰撞检测功能。

### 2.5 RigidBody

**名称**：刚体（RigidBody）

**首次出现位置**：C15_102-基础语法.md 第3.3章

**定义**：
RigidBody 是受物理引擎控制的物体节点，用于实现物理模拟的物体。

### 2.6 Area2D / Area3D

**名称**：区域节点（Area）

**首次出现位置**：C15_102-基础语法.md 第3.1章

**定义**：
Area 节点用于检测物体的进入和离开，常用于触发器、碰撞检测等。

### 2.7 StaticBody

**名称**：静态物体（StaticBody）

**首次出现位置**：C15_102-基础语法.md 第3.3章

**定义**：
StaticBody 是物理引擎中的静止物体节点，用于实现墙壁、地面等不移动的物体。

### 2.8 KinematicBody

**名称**：运动体（KinematicBody）

**首次出现位置**：C15_102-基础语法.md 第3.2章

**定义**：
KinematicBody 是通过代码控制的运动物体节点，不受物理力影响，需要手动处理碰撞。

### 2.9 Sprite

**名称**：精灵节点（Sprite）

**首次出现位置**：C15_102-基础语法.md 第2.2章

**定义**：
Sprite 是用于显示 2D 图像的节点，支持纹理切换、动画等功能。

### 2.10 Label

**名称**：文本标签（Label）

**首次出现位置**：C15_102-基础语法.md 第2.2章

**定义**：
Label 是用于显示纯文本的 2D 节点，不支持富文本格式。

### 2.11 Control

**名称**：控件节点（Control）

**首次出现位置**：C15_102-基础语法.md 第4章

**定义**：
Control 是 UI 控件的基类，提供了用户界面开发的基础，如 Button、Label、Panel 等。

### 2.12 Camera2D / Camera3D

**名称**：摄像机节点（Camera）

**首次出现位置**：C15_102-基础语法.md 第2.3章

**定义**：
Camera 节点用于定义视口视角，2D 游戏中决定显示哪部分画面，3D 游戏中决定观察方向。

## 3. 脚本系统类

### 3.1 脚本

**名称**：脚本（Script）

**首次出现位置**：C15_101-概述与环境.md 第1.2章

**定义**：
脚本是附加到节点上的代码文件，用于定义节点的行为和逻辑，GDScript 是 Godot 的主要脚本语言。

### 3.2 extends

**名称**：继承关键字（Extends）

**首次出现位置**：C15_102-基础语法.md 第1.1章

**定义**：
extends 关键字用于声明脚本继承的节点类型，确定脚本可以访问的成员和方法。

### 3.3 func

**名称**：函数定义（Function）

**首次出现位置**：C15_102-基础语法.md 第1.3章

**定义**：
func 关键字用于定义函数，是 GDScript 中声明函数的核心语法。

### 3.4 var

**名称**：变量声明（Variable）

**首次出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
var 关键字用于声明变量，GDScript 是动态类型语言，变量类型由值自动决定。

### 3.5 export

**名称**：导出变量（Export）

**首次出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
export 关键字使变量在编辑器中可见可编辑，方便在不修改代码的情况下调整参数。

### 3.6 static

**名称**：静态变量（Static）

**首次出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
static 关键字用于声明静态变量或函数，属于类而非实例，所有实例共享。

### 3.7 signal

**名称**：信号（Signal）

**首次出现位置**：C15_102-基础语法.md 第1.4章

**定义**：
signal 关键字用于声明信号，是 Godot 观察者模式实现，用于节点间通信。

### 3.8 onready

**名称**：就绪初始化（On Ready）

**首次出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
onready 关键字用于在节点进入场景树后初始化变量，确保获取到子节点引用。

### 3.9 _ready

**名称**：就绪回调（Ready Callback）

**首次出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
_ready 是节点进入场景树时调用的回调函数，常用于初始化。

### 3.10 _process

**名称**：进程回调（Process Callback）

**首次出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
_process 是每帧调用的回调函数，delta 参数表示两帧时间间隔，用于游戏逻辑更新。

### 3.11 _physics_process

**名称**：物理进程回调（Physics Process Callback）

**首次出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
_physics_process 是物理引擎每帧调用的回调，用于处理物理相关的逻辑更新。

### 3.12 get_node

**名称**：获取节点（Get Node）

**首次出现位置**：C15_102-基础语法.md 第2.2章

**定义**：
get_node 是通过节点路径获取子节点引用的方法，返回目标节点的引用。

## 4. 信号系统类

### 4.1 信号

**名称**：信号机制（Signal）

**首次出现位置**：C15_101-概述与环境.md 第2.3章

**定义**：
信号是 Godot 的观察者模式实现，允许节点在特定事件发生时通知其他节点。

### 4.2 emit_signal

**名称**：发射信号（Emit Signal）

**首次出现位置**：C15_101-概述与环境.md 第2.3章

**定义**：
emit_signal 用于发射信号，触发所有连接该信号的回调函数。

### 4.3 connect

**名称**：连接信号（Connect）

**首次出现位置**：C15_101-概述与环境.md 第2.3章

**定义**：
connect 用于将信号的回调函数连接到另一个节点，实现信号通知。

### 4.4 信号参数

**名称**：信号参数（Signal Parameters）

**首次出现位置**：C15_101-概述与环境.md 第2.3章

**定义**：
信号可以携带参数，在发射时传递给回调函数，实现信息传递。

### 4.5 自定义信号

**名称**：自定义信号（Custom Signal）

**首次出现位置**：C15_101-概述与环境.md 第2.3章

**定义**：
使用 signal 关键字声明的信号为自定义信号，区别于内置节点的内置信号。

## 5. 物理系统类

### 5.1 物理引擎

**名称**：物理引擎（Physics Engine）

**首次出现位置**：C15_302-物理系统.md 第1章

**定义**：
物理引擎是 Godot 中处理碰撞检测和物理模拟的系统，支持 2D 和 3D 物理。

### 5.2 碰撞

**名称**：碰撞（Collision）

**首次出现位置**：C15_302-物理系统.md 第2章

**定义**：
碰撞是指两个具有碰撞体的物体在空间上重叠时发生的事件。

### 5.3 CollisionShape

**名称**：碰撞形状（Collision Shape）

**首次出现位置**：C15_302-物理系统.md 第2章

**定义**：
CollisionShape 是定义碰撞检测范围的节点，支持多种几何形状如矩形、圆形、胶囊等。

### 5.4 CollisionShape2D

**名称**：2D碰撞形状（Collision Shape 2D）

**首次出现位置**：C15_302-物理系统.md 第2章

**定义**：
CollisionShape2D 是 2D 物理中用于定义碰撞区域的几何形状。

### 5.5 Area

**名称**：区域节点（Area Node）

**首次出现位置**：C15_302-物理系统.md 第2章

**定义**：
Area 节点用于检测物体的进入和离开，常用于触发器、碰撞检测等。

### 5.6 PhysicsMaterial

**名称**：物理材质（Physics Material）

**首次出现位置**：C15_302-物理系统.md 第3章

**定义**：
PhysicsMaterial 用于定义物理材质的属性，如摩擦系数、弹性等。

### 5.7 move_and_slide

**名称**：移动滑动（Move and Slide）

**首次出现位置**：C15_102-基础语法.md 第3.2章

**定义**：
move_and_slide 是 CharacterBody 的内置方法，用于处理角色移动和碰撞滑动。

### 5.8 move_and_collide

**名称**：移动碰撞（Move and Collide）

**首次出现位置**：C15_102-基础语法.md 第3.2章

**定义**：
move_and_collide 是 KinematicBody 的内置方法，返回碰撞信息。

### 5.9 射线投射

**名称**：射线投射（Raycasting）

**首次出现位置**：C15_302-物理系统.md

**定义**：
射线投射是从一点向特定方向发射射线，检测碰撞的技术，常用于视线检测、子弹轨迹等。

### 5.10 RayCast2D / RayCast3D

**名称**：射线投射节点（Ray Cast Node）

**首次出现位置**：C15_302-物理系统.md

**定义**：
RayCast 节点是 Godot 内置的射线投射实现，可配置起点、方向、长度等属性。
