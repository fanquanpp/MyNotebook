# Godot 专有名词解释

## 1. 基础概念类

### 1.1 Godot

**英文名称**：Godot Engine

**中文译名**：Godot 游戏引擎

**出现位置**：C15_101-概述与环境.md 第1章

**定义**：
Godot 是一款功能开源完整的 2D 和 3D 游戏引擎，提供场景系统、节点树、物理引擎等完整游戏开发功能。

### 1.2 GDScript

**英文名称**：GDScript

**中文译名**：GDScript 脚本语言

**出现位置**：C15_101-概述与环境.md 第1.1章

**定义**：
GDScript 是 Godot 引擎专用的高级动态类型脚本语言，语法深受 Python 启发，与引擎深度集成。

### 1.3 节点

**英文名称**：Node

**中文译名**：节点

**出现位置**：C15_101-概述与环境.md 第2.1章

**定义**：
节点是 Godot 中的基本构建单元，每个节点都有特定功能，可以添加到场景树中形成游戏。

### 1.4 场景

**英文名称**：Scene

**中文译名**：场景

**出现位置**：C15_101-概述与环境.md 第2.2章

**定义**：
场景是节点的树形组织结构，可以包含子节点，本质上是一个可重用的游戏对象模板。

### 1.5 节点树

**英文名称**：Node Tree

**中文译名**：节点树

**出现位置**：C15_101-概述与环境.md 第2.1章

**定义**：
节点树是 Godot 中组织的层次结构，场景中的所有节点按父子关系形成树形结构。

### 1.6 引擎

**英文名称**：Engine

**中文译名**：引擎

**出现位置**：C15_101-概述与环境.md 第1章

**定义**：
引擎是指提供游戏运行时核心功能的软件框架，Godot 引擎包含渲染、物理、音频等子系统。

## 2. 节点系统类

### 2.1 节点类型

**英文名称**：Node Types

**中文译名**：节点类型

**出现位置**：C15_102-基础语法.md 第2章

**定义**：
Godot 内置多种节点类型，如 Node2D、Spatial、Area2D、CharacterBody 等，每种类型提供不同功能。

### 2.2 Node2D

**英文名称**：Node2D

**中文译名**：2D节点

**出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
Node2D 是 2D 游戏开发的基础节点，支持 2D 变换，如位置、旋转、缩放。

### 2.3 Spatial

**英文名称**：Spatial

**中文译名**：3D节点

**出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
Spatial 是 3D 游戏开发的基础节点，支持 3D 变换，是所有 3D 节点的基类。

### 2.4 CharacterBody

**英文名称**：CharacterBody

**中文译名**：角色体

**出现位置**：C15_102-基础语法.md 第3.2章

**定义**：
CharacterBody 是用于实现玩家或 NPC 等受控角色的节点，提供移动和碰撞检测功能。

### 2.5 RigidBody

**英文名称**：RigidBody

**中文译名**：刚体

**出现位置**：C15_102-基础语法.md 第3.3章

**定义**：
RigidBody 是受物理引擎控制的物体节点，用于实现物理模拟的物体。

### 2.6 Area2D / Area3D

**英文名称**：Area

**中文译名**：区域节点

**出现位置**：C15_102-基础语法.md 第3.1章

**定义**：
Area 节点用于检测物体的进入和离开，常用于触发器、碰撞检测等。

### 2.7 StaticBody

**英文名称**：StaticBody

**中文译名**：静态物体

**出现位置**：C15_102-基础语法.md 第3.3章

**定义**：
StaticBody 是物理引擎中的静止物体节点，用于实现墙壁、地面等不移动的物体。

### 2.8 KinematicBody

**英文名称**：KinematicBody

**中文译名**：运动体

**出现位置**：C15_102-基础语法.md 第3.2章

**定义**：
KinematicBody 是通过代码控制的运动物体节点，不受物理力影响，需要手动处理碰撞。

### 2.9 Sprite

**英文名称**：Sprite

**中文译名**：精灵节点

**出现位置**：C15_102-基础语法.md 第2.2章

**定义**：
Sprite 是用于显示 2D 图像的节点，支持纹理切换、动画等功能。

### 2.10 Label

**英文名称**：Label

**中文译名**：文本标签

**出现位置**：C15_102-基础语法.md 第2.2章

**定义**：
Label 是用于显示纯文本的 2D 节点，不支持富文本格式。

### 2.11 Control

**英文名称**：Control

**中文译名**：控件节点

**出现位置**：C15_102-基础语法.md 第4章

**定义**：
Control 是 UI 控件的基类，提供了用户界面开发的基础，如 Button、Label、Panel 等。

### 2.12 Camera2D / Camera3D

**英文名称**：Camera

**中文译名**：摄像机节点

**出现位置**：C15_102-基础语法.md 第2.3章

**定义**：
Camera 节点用于定义视口视角，2D 游戏中决定显示哪部分画面，3D 游戏中决定观察方向。

## 3. 脚本系统类

### 3.1 脚本

**英文名称**：Script

**中文译名**：脚本

**出现位置**：C15_101-概述与环境.md 第1.2章

**定义**：
脚本是附加到节点上的代码文件，用于定义节点的行为和逻辑，GDScript 是 Godot 的主要脚本语言。

### 3.2 extends

**英文名称**：Extends

**中文译名**：继承关键字

**出现位置**：C15_102-基础语法.md 第1.1章

**定义**：
extends 关键字用于声明脚本继承的节点类型，确定脚本可以访问的成员和方法。

### 3.3 func

**英文名称**：Function

**中文译名**：函数定义

**出现位置**：C15_102-基础语法.md 第1.3章

**定义**：
func 关键字用于定义函数，是 GDScript 中声明函数的核心语法。

### 3.4 var

**英文名称**：Variable

**中文译名**：变量声明

**出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
var 关键字用于声明变量，GDScript 是动态类型语言，变量类型由值自动决定。

### 3.5 export

**英文名称**：Export

**中文译名**：导出变量

**出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
export 关键字使变量在编辑器中可见可编辑，方便在不修改代码的情况下调整参数。

### 3.6 static

**英文名称**：Static

**中文译名**：静态变量

**出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
static 关键字用于声明静态变量或函数，属于类而非实例，所有实例共享。

### 3.7 signal

**英文名称**：Signal

**中文译名**：信号

**出现位置**：C15_102-基础语法.md 第1.4章

**定义**：
signal 关键字用于声明信号，是 Godot 观察者模式实现，用于节点间通信。

### 3.8 onready

**英文名称**：On Ready

**中文译名**：就绪初始化

**出现位置**：C15_102-基础语法.md 第1.2章

**定义**：
onready 关键字用于在节点进入场景树后初始化变量，确保获取到子节点引用。

### 3.9 _ready

**英文名称**：Ready Callback

**中文译名**：就绪回调

**出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
_ready 是节点进入场景树时调用的回调函数，常用于初始化。

### 3.10 _process

**英文名称**：Process Callback

**中文译名**：进程回调

**出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
_process 是每帧调用的回调函数，delta 参数表示两帧时间间隔，用于游戏逻辑更新。

### 3.11 _physics_process

**英文名称**：Physics Process Callback

**中文译名**：物理进程回调

**出现位置**：C15_102-基础语法.md 第2.1章

**定义**：
_physics_process 是物理引擎每帧调用的回调，用于处理物理相关的逻辑更新。

### 3.12 get_node

**英文名称**：Get Node

**中文译名**：获取节点

**出现位置**：C15_102-基础语法.md 第2.2章

**定义**：
get_node 是通过节点路径获取子节点引用的方法，返回目标节点的引用。
