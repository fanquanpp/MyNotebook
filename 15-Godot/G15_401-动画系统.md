## 1. 动画系统概述

Godot 引擎提供了强大的动画系统，用于创建和控制游戏中的动画效果。动画系统支持骨骼动画、属性动画、精灵动画等多种类型的动画，为游戏开发者提供了灵活、高效的动画创作工具。

### 1.1 核心概念

- **动画 (Animation)**: 定义了一组属性随时间变化的序列
- **动画播放器 (AnimationPlayer)**: 负责播放和控制动画
- **动画树 (AnimationTree)**: 用于管理复杂的动画状态和过渡
- **动画状态机 (AnimationStateMachine)**: 用于控制动画的状态转换
- **动画混合 (Animation Blending)**: 平滑过渡不同的动画状态

### 1.2 动画类型

Godot 支持多种类型的动画：

- **属性动画**: 动画化节点的属性，如位置、旋转、缩放等
- **骨骼动画**: 动画化骨骼的姿态，用于角色动画
- **精灵动画**: 动画化精灵的帧，用于 2D 角色和特效
- **材质动画**: 动画化材质的属性，如颜色、纹理偏移等
- **音频动画**: 同步音频和动画

## 2. 动画播放器 (AnimationPlayer)

**继承:** AnimationMixer < Node < Object

AnimationPlayer 节点用于播放动画，包含 AnimationLibrary 资源的字典和动画过渡之间的自定义混合时间。
某些方法和属性使用单个键直接引用动画。这些键的格式为动画库的键，后跟正斜杠，然后是动画库内动画的键，例如 `"movement/run"`。如果动画库的键为空字符串（称为默认库），则省略正斜杠，与动画库使用相同的键。
**AnimationPlayer** 比 **Tween** 更适合用于复杂动画，例如不规则计时的动画。如果用动画轨道编辑器比用代码实现更方便，也可以优先选择使用 **Tween**。
更新动画的目标属性是在处理帧中进行的。

### 2.1 创建动画

1. **通过编辑器创建动画**:
   - 在场景树中添加 `AnimationPlayer` 节点
   - 点击 `AnimationPlayer` 节点，在底部的动画编辑器中点击 `New Animation`
   - 为动画命名，并设置动画长度
   - 在时间轴上添加关键帧，调整节点的属性
2. **通过代码创建动画**:

```gdscript
# 创建动画播放器
var animation_player = AnimationPlayer.new()
add_child(animation_player)

# 创建动画
var animation = Animation.new()
animation.length = 1.0  # 动画长度（秒）
animation.loop = true  # 循环播放

# 添加位置关键帧
animation.track_insert_key(0, "Node2D:position", Vector2(0, 0))
animation.track_insert_key(1.0, "Node2D:position", Vector2(100, 100))

# 添加旋转关键帧
animation.track_insert_key(0, "Node2D:rotation", 0)
animation.track_insert_key(1.0, "Node2D:rotation", PI)

# 添加动画到播放器
animation_player.add_animation("move", animation)
```

### 2.2 播放动画

```gdscript
# 播放动画（不带参数播放当前动画）
animation_player.play()

# 播放动画并设置参数
# play(name, custom_blend, custom_speed, from_end)
# name: 动画名称
# custom_blend: 混合时间（秒），-1 表示使用默认混合时间
# custom_speed: 播放速度，1.0 为正常速度
# from_end: 是否从末尾开始播放
animation_player.play("move", 0.2, 1.0, false)

# 暂停动画
animation_player.pause()

# 停止动画
# stop(keep_state) - keep_state 表示是否保留当前动画状态
animation_player.stop(false)

# 继续动画
animation_player.play()

# 倒退播放
animation_player.play_backwards("move")

# 从特定段落播放
# play_section(name, start_time, end_time, custom_blend, custom_speed, from_end)
animation_player.play_section("move", 0.5, 1.5)

# 使用标记播放段落
animation_player.play_section_with_markers("move", "start_marker", "end_marker")

# 添加到队列
animation_player.queue("walk")
animation_player.queue("idle")

# 清除队列
animation_player.clear_queue()

# 获取队列
var queue = animation_player.get_queue()
```

### 2.3 动画参数

```gdscript
# 设置动画速度
animation_player.playback_speed = 1.5  # 1.5 倍速
# 获取当前动画
var current_animation = animation_player.current_animation

# 获取动画时长
var animation_length = animation_player.get_animation_length("move")

# 设置动画位置
animation_player.seek(0.5)  # 跳转到动画的 0.5 秒处
```

### 2.4 动画信号

```gdscript
# 连接动画信号
animation_player.animation_finished.connect(_on_animation_finished)
animation_player.animation_started.connect(_on_animation_started)
animation_player.animation_changed.connect(_on_animation_changed)

# 处理动画完成信号
func _on_animation_finished(anim_name):
    print("Animation finished: " + anim_name)

# 处理动画开始信号
func _on_animation_started(anim_name):
    print("Animation started: " + anim_name)

# 处理动画改变信号
func _on_animation_changed(anim_name):
    print("Animation changed: " + anim_name)
```

## 3. 动画树 (AnimationTree)

### 3.1 动画树概述

`AnimationTree` 节点用于管理复杂的动画状态和过渡，特别适合角色动画系统。它基于动画状态机，允许创建复杂的动画逻辑。

### 3.2 创建动画树

1. **添加 AnimationTree 节点**:
   - 在场景树中添加 `AnimationTree` 节点
   - 设置 `AnimationPlayer` 属性为场景中的 `AnimationPlayer` 节点
   - 在 `Tree Root` 中选择动画树的类型，如 `AnimationNodeStateMachine`
2. **配置动画状态机**:
   - 在动画树编辑器中，添加状态节点，每个状态对应一个动画
   - 添加过渡节点，定义状态之间的转换条件
   - 设置状态的参数和过渡的条件

### 3.3 动画树代码控制

```gdscript
# 获取动画树
var animation_tree = $AnimationTree

# 设置动画参数
animation_tree.set("parameters/playback/seek", 0.0)
animation_tree.set("parameters/playback/speed", 1.0)

# 触发状态转换
animation_tree.set("parameters/state_request", "run")

# 获取当前状态
var current_state = animation_tree.get("parameters/current_state")

# 设置混合参数
animation_tree.set("parameters/blend_amount", 0.5)
```

### 3.4 动画混合

动画混合允许平滑过渡不同的动画状态，创造更自然的动画效果。

```gdscript
# 创建混合节点
var blend_node = AnimationNodeBlendSpace2D.new()

# 添加动画到混合空间
blend_node.add_blend_point(preload("res://animations/idle.tres"), Vector2(0, 0), 0)
blend_node.add_blend_point(preload("res://animations/walk.tres"), Vector2(1, 0), 1)
blend_node.add_blend_point(preload("res://animations/run.tres"), Vector2(2, 0), 2)

# 设置混合参数
animation_tree.set("parameters/blend_position", Vector2(1.5, 0))  # 混合 walk 和 run 动画
```

## 4. 精灵动画

### 4.1 AnimatedSprite2D

`AnimatedSprite2D` 节点用于播放 2D 精灵动画，适合角色动画和简单特效。

```gdscript
# 获取 AnimatedSprite2D 节点
var animated_sprite = $AnimatedSprite2D

# 设置动画
animated_sprite.play("walk")

# 停止动画
animated_sprite.stop()

# 切换动画
animated_sprite.animation = "run"

# 连接动画完成信号
animated_sprite.animation_finished.connect(_on_animation_finished)

func _on_animation_finished():
    print("Sprite animation finished")
```

### 4.2 SpriteFrames

`SpriteFrames` 资源用于存储精灵动画的帧数据。

```gdscript
# 创建 SpriteFrames 资源
var sprite_frames = SpriteFrames.new()

# 添加动画
var walk_frames = [preload("res://textures/walk1.png"), preload("res://textures/walk2.png")]
sprite_frames.add_animation("walk")
for i in range(walk_frames.size()):
    sprite_frames.add_frame("walk", walk_frames[i], 0.1)  # 帧，持续时间

# 设置到 AnimatedSprite2D
$AnimatedSprite2D.sprite_frames = sprite_frames
```

## 5. 骨骼动画

### 5.1 Skeleton2D/3D

`Skeleton2D` 和 `Skeleton3D` 节点用于管理骨骼动画，支持蒙皮网格和反向运动学 (IK)。

```gdscript
# 获取骨骼节点
var skeleton = $Skeleton3D

# 设置骨骼姿势
skeleton.set_bone_pose(0, Transform3D.IDENTITY)

# 获取骨骼姿势
var bone_pose = skeleton.get_bone_pose(0)

# 重置骨骼姿势
skeleton.reset_bone_poses()
```

### 5.2 AnimationPlayer 与骨骼动画

使用 `AnimationPlayer` 来控制骨骼动画：

```gdscript
# 创建骨骼动画
var animation = Animation.new()
animation.length = 1.0

# 添加骨骼旋转关键帧
animation.track_insert_key(0, "Skeleton3D:bones/Arm.R/rotation", Vector3(0, 0, 0))
animation.track_insert_key(1.0, "Skeleton3D:bones/Arm.R/rotation", Vector3(0, 0, PI/2))

# 添加动画到播放器
$AnimationPlayer.add_animation("wave", animation)

# 播放骨骼动画
$AnimationPlayer.play("wave")
```

## 6. 动画状态机

### 6.1 创建状态机

动画状态机用于管理复杂的动画状态转换，适合角色动画系统。

1. **在 AnimationTree 中创建状态机**:
   - 设置 `Tree Root` 为 `AnimationNodeStateMachine`
   - 添加状态节点，每个状态对应一个动画
   - 添加过渡节点，定义状态之间的转换条件
2. **状态机代码控制**:

```gdscript
# 获取状态机
var state_machine = $AnimationTree.get("parameters/playback")

# 请求状态转换
$AnimationTree.set("parameters/state_request", "idle")

# 获取当前状态
var current_state = $AnimationTree.get("parameters/current_state")

# 设置状态机参数
$AnimationTree.set("parameters/transition_duration", 0.2)  # 过渡时间
```

### 6.2 状态机参数

状态机可以使用参数来控制动画的行为。

```gdscript
# 设置状态机参数
$AnimationTree.set("parameters/move_speed", 1.0)  # 移动速度
$AnimationTree.set("parameters/turn_amount", 0.0)  # 转向角度
# 在状态机中使用参数
# 例如，根据 move_speed 参数混合 idle、walk 和 run 动画
```

## 7. 动画混合空间

### 7.1 2D 混合空间

`AnimationNodeBlendSpace2D` 用于基于二维参数混合多个动画。

```gdscript
# 创建 2D 混合空间
var blend_space = AnimationNodeBlendSpace2D.new()

# 设置混合空间参数
blend_space.set_min_space(Vector2(0, -1))
blend_space.set_max_space(Vector2(2, 1))

# 添加动画点
blend_space.add_blend_point(preload("res://animations/idle.tres"), Vector2(0, 0), 0)
blend_space.add_blend_point(preload("res://animations/walk.tres"), Vector2(1, 0), 1)
blend_space.add_blend_point(preload("res://animations/run.tres"), Vector2(2, 0), 2)
blend_space.add_blend_point(preload("res://animations/walk_back.tres"), Vector2(1, -1), 3)
blend_space.add_blend_point(preload("res://animations/run_back.tres"), Vector2(2, -1), 4)

# 设置混合位置
$AnimationTree.set("parameters/blend_position", Vector2(1.5, 0))  # 混合 walk 和 run 动画
```

### 7.2 1D 混合空间

`AnimationNodeBlendSpace1D` 用于基于一维参数混合多个动画：

```gdscript
# 创建 1D 混合空间
var blend_space = AnimationNodeBlendSpace1D.new()

# 设置混合空间范围
blend_space.min_space = 0
blend_space.max_space = 2

# 添加动画点
blend_space.add_blend_point(preload("res://animations/idle.tres"), 0, 0)
blend_space.add_blend_point(preload("res://animations/walk.tres"), 1, 1)
blend_space.add_blend_point(preload("res://animations/run.tres"), 2, 2)

# 设置混合值
$AnimationTree.set("parameters/blend_value", 1.5)  # 混合 walk 和 run 动画
```

## 8. 动画脚本控制

### 8.1 动画事件

动画事件允许在动画播放到特定帧时触发回调。

```gdscript
# 在动画编辑器中添加事件
# 选择时间轴上的帧，点击 "Add Event"
# 输入事件名称，如 "attack_hit"

# 连接动画事件信号
$AnimationPlayer.animation_finished.connect(_on_animation_finished)

# 或者在动画播放器中连接事件
$AnimationPlayer.connect("animation_finished", self, "_on_animation_finished")

# 处理动画事件
func _on_animation_finished(anim_name):
    print("Animation finished: " + anim_name)

# 或者使用动画事件轨道
# 在动画编辑器中添加 "AnimationTrackType.EVENT" 轨道
# 在关键帧中设置事件数据
```

### 8.2 程序化动画

使用代码创建和控制动画：

```gdscript
# 创建动画
var animation = Animation.new()
animation.length = 2.0

# 添加属性轨道
var track_index = animation.add_track(Animation.TYPE_VALUE)
animation.track_set_path(track_index, NodePath("Node2D:position"))
animation.track_insert_key(track_index, 0.0, Vector2(0, 0))
animation.track_insert_key(track_index, 1.0, Vector2(100, 0))
animation.track_insert_key(track_index, 2.0, Vector2(0, 0))

# 添加动画到播放器
$AnimationPlayer.add_animation("bounce", animation)

# 播放动画
$AnimationPlayer.play("bounce")
```

## 9. 动画系统最佳实践

### 9.1 性能优化

- **使用动画树**: 对于复杂的动画系统，使用 `AnimationTree` 管理动画状态
- **减少关键帧**: 只在必要的地方添加关键帧，减少动画数据大小
- **使用循环动画**: 对于重复的动画，使用循环播放
- **避免过多的动画层**: 合理使用动画，避免过多的动画导致性能下降
- **使用动画缓存**: 对于频繁使用的动画，使用缓存提高性能

### 9.2 动画设计

- **分层动画**: 将动画分解为多个层，如基础动画、表情动画、武器动画等
- **使用混合空间**: 对于连续的动画，如行走、跑步，使用混合空间实现平滑过渡
- **状态机设计**: 合理设计状态机，减少不必要的状态转换
- **动画事件**: 使用动画事件触发游戏逻辑，如攻击命中、脚步声等
- **动画曲线**: 使用动画曲线控制动画的缓动效果，使动画更自然

### 9.3 常见问题

- **动画卡顿**: 检查动画帧率和关键帧数量，优化动画数据
- **动画过渡不自然**: 使用适当的过渡时间和混合参数
- **骨骼动画变形**: 检查骨骼权重和蒙皮设置
- **动画同步问题**: 使用动画事件和时间轴同步游戏逻辑
- **性能问题**: 减少动画复杂度，使用动画树优化动画管理

## 10. 实际应用示例

### 10.1 示例 1：角色动画系统

```gdscript
# CharacterAnimation.gd
extends CharacterBody2D

@onready var animation_tree = $AnimationTree
@onready var animation_state = animation_tree.get("parameters/playback")

var speed = 0.0
var direction = Vector2.ZERO

func _process(delta):
    # 获取输入
    direction = Vector2(
        Input.get_axis("ui_left", "ui_right"),
        Input.get_axis("ui_up", "ui_down")
    )

    # 计算速度
    speed = direction.length()

    # 更新动画参数
    animation_tree.set("parameters/speed", speed)
    animation_tree.set("parameters/direction", direction.x)

    # 处理移动
    if speed > 0:
        velocity = direction.normalized() * 200
    else:
        velocity = Vector2.ZERO

func _physics_process(delta):
    move_and_slide()
```

### 10.2 示例 2：敌人 AI 动画

```gdscript
# EnemyAI.gd
extends CharacterBody2D

@onready var animation_player = $AnimationPlayer

var state = "idle"
var target = null

func _ready():
    # 寻找玩家
    target = get_tree().get_root().get_node("Player")

func _process(delta):
    if target:
        var distance = position.distance_to(target.position)

        if distance < 100:
            # 攻击状态
            if state != "attack":
                state = "attack"
                animation_player.play("attack")
        elif distance < 300:
            # 追逐状态
            if state != "run":
                state = "run"
                animation_player.play("run")

            # 移动向玩家
            var direction = (target.position - position).normalized()
            velocity = direction * 150
        else:
            # idle 状态
            if state != "idle":
                state = "idle"
                animation_player.play("idle")
                velocity = Vector2.ZERO

func _physics_process(delta):
    move_and_slide()
```

### 10.3 示例 3：UI 动画

```gdscript
# UIAnimation.gd
extends Control

@onready var animation_player = $AnimationPlayer

func show():
    # 显示动画
    animation_player.play("show")

func hide():
    # 隐藏动画
    animation_player.play("hide")

func _on_animation_finished(anim_name):
    if anim_name == "hide":
        # 隐藏完成后禁用节点
        visible = false
```

## 11. 总结

Godot 的动画系统是一个功能强大、灵活的工具，它提供了多种动画类型和控制方式，满足不同游戏场景的需求。动画系统的核心优势包括：

- **丰富的动画类型**: 支持属性动画、骨骼动画、精灵动画等多种类型
- **强大的动画控制**: 通过 AnimationPlayer 和 AnimationTree 提供精细的动画控制
- **灵活的动画状态管理**: 使用动画状态机管理复杂的动画状态和过渡
- **平滑的动画混合**: 支持多种动画混合方式，创造自然的动画效果
- **脚本化控制**: 通过代码可以灵活控制动画的播放和参数
通过掌握动画系统的使用方法和最佳实践，开发者可以创建生动、自然的游戏动画，提升游戏的视觉效果和用户体验。动画系统是 Godot 引擎的核心特性之一，也是游戏开发中不可或缺的工具。
