## 1. 网络系统概述

Godot 引擎提供了强大的网络系统，用于创建 multiplayer 游戏和网络应用。网络系统基于 ENet 库，提供了可靠的 UDP 通信。
### 1.1 核心概念

- **服务器 (Server)**: 负责管理游戏状态，处理客户端连接和同步
- **客户端 (Client)**: 连接到服务器，发送玩家输入，接收游戏状态更新
- **对等体 (Peer)**: 网络中的一个节点，可以是服务器或客户端
- **远程过程调用 (RPC)**: 从一个对等体调用另一个对等体上的方法
- **网络同步**: 保持不同对等体之间游戏状态的一致性
### 1.2 网络架构

Godot 的网络系统支持两种主要的网络架构：
- **客户端-服务器架构**: 一个服务器管理多个客户端，客户端之间不直接通信
- **对等架构**: 所有对等体都是平等的，直接相互通信
### 1.3 网络协议

Godot 的网络系统使用以下协议：

- **UDP**: 用于快速、不可靠的通信
- **ENet**: 基于 UDP 的可靠通信层，提供可靠的数据包传输
## 2. 网络通信

### 2.1 服务器与客户端
#### 服务器创建
```gdscript
# 创建服务器
var server = ENetMultiplayerPeer.new()
var error = server.create_server(4242, 4)  # 端口 4242，最多 4 个客户端
if error != OK:
    print("Failed to create server")
else:
    multiplayer.multiplayer_peer = server
    print("Server created successfully")
```

#### 客户端连接
```gdscript
# 连接到服务器
var client = ENetMultiplayerPeer.new()
var error = client.create_client("127.0.0.1", 4242)  # 连接到本地服务器，端口 4242
if error != OK:
    print("Failed to connect to server")
else:
    multiplayer.multiplayer_peer = client
    print("Connected to server")
```

### 2.2 网络连接

- **连接事件**: 当客户端连接或断开时触发
- **连接状态**: 客户端的连接状态，如连接中、已连接、已断开
```gdscript
# 连接事件
multiplayer.peer_connected.connect(_on_peer_connected)
multiplayer.peer_disconnected.connect(_on_peer_disconnected)

func _on_peer_connected(peer_id):
    print("Peer connected: " + str(peer_id))

func _on_peer_disconnected(peer_id):
    print("Peer disconnected: " + str(peer_id))
```

### 2.3 数据传输

- **RPC**: 远程过程调用，用于调用其他对等体上的方法
- **网络变量**: 自动同步的变量
```gdscript
# RPC 示例
@rpc
func send_message(message: String):
    print("Received message: " + message)

# 调用 RPC
rpc("send_message", "Hello from client")
```

### 2.4 网络事件

- **peer_connected**: 当新的对等体连接时触发
- **peer_disconnected**: 当对等体断开连接时触发
- **connection_failed**: 当连接失败时触发
- **server_disconnected**: 当服务器断开连接时触发
## 3. 远程过程调用 (RPC)

### 3.1 RPC 基础

RPC 允许从一个对等体调用另一个对等体上的方法。
```gdscript
# 定义 RPC 方法
@rpc
func update_position(new_position: Vector3):
    position = new_position

# 调用 RPC
rpc("update_position", position)
```

### 3.2 RPC 类型

Godot 支持多种 RPC 类型：
- **rpc()**: 调用所有对等体（包括自己）上的方法
- **rpc_id()**: 调用特定对等体上的方法
- **rpc_group()**: 调用特定组中的对等体上的方法
```gdscript
# 调用所有对等体
rpc("update_position", position)

# 调用特定对等体
rpc_id(1, "update_position", position)  # 调用 ID 为 1 的对等体

# 调用组中的对等体
rpc_group("players", "update_position", position)  # 调用 "players" 组中的对等体
```

### 3.3 RPC 参数

RPC 方法可以接受各种类型的参数：

- **基本类型**: int, float, bool, String
- **复合类型**: Vector2, Vector3, Color, Dictionary, Array
- **对象引用**: NodePath, Resource
```gdscript
@rpc
func update_player_data(name: String, position: Vector3, health: int):
    player_name = name
    position = position
    player_health = health

# 调用 RPC 并传递多个参数
rpc("update_player_data", "Player1", position, 100)
```

### 3.4 RPC 最佳实践
- **减少 RPC 调用**: 避免过于频繁的 RPC 调用，减少网络流量
- **批量更新**: 将多个小更新合并为一个 RPC 调用
- **使用可靠 RPC**: 对于重要数据，使用可靠的 RPC
- **避免发送大型数据**: 尽量减少 RPC 参数的大小
## 4. 网络同步

### 4.1 状态同步
状态同步是保持不同对等体之间游戏状态一致性的过程。
- **权威服务器**: 服务器是游戏状态的权威来源，客户端只接收状态更新
- **状态插值**: 使用插值平滑状态更新，减少网络延迟的影响
```gdscript
# 服务器端更新位置
@rpc
func update_position(new_position: Vector3):
    position = new_position

# 客户端接收位置更新
var last_position = position
var target_position = position
var interpolation_time = 0.1
var elapsed_time = 0.0

func _process(delta):
    if multiplayer.is_server():
        # 服务器更新位置
        position += velocity * delta
        rpc("update_position", position)
    else:
        # 客户端插值位置
        elapsed_time += delta
        var t = min(elapsed_time / interpolation_time, 1.0)
        position = last_position.linear_interpolate(target_position, t)

func update_position(new_position: Vector3):
    last_position = position
    target_position = new_position
    elapsed_time = 0.0
```

### 4.2 输入同步

输入同步是处理玩家输入并在网络上同步的过程：

- **客户端预测**: 客户端预测输入的结果，减少延迟感
- **服务器验证**: 服务器验证客户端的输入，防止作弊
```gdscript
# 客户端发送输入
func _process(delta):
    var input = get_input()
    rpc("process_input", input, Time.get_time_dict_from_system())
    # 客户端预测
    process_input_locally(input)

# 服务器处理输入
@rpc
func process_input(input, timestamp):
    # 验证输入
    if validate_input(input):
        # 处理输入
        process_input_server(input)
        # 广播结果
        rpc("update_state", get_state())
```

### 4.3 实体同步

实体同步是同步游戏中实体的状态：

- **实体创建**: 当新实体创建时，通知所有客户端
- **实体销毁**: 当实体销毁时，通知所有客户端
- **实体状态**: 同步实体的状态，如位置、健康值等
```gdscript
# 服务器创建实体
func create_enemy():
    var enemy = preload("res://scenes/enemy.tscn").instantiate()
    add_child(enemy)
    # 通知客户端
    rpc("spawn_enemy", enemy.position, enemy.rotation)

# 客户端生成实体
@rpc
func spawn_enemy(position: Vector3, rotation: Vector3):
    var enemy = preload("res://scenes/enemy.tscn").instantiate()
    enemy.position = position
    enemy.rotation = rotation
    add_child(enemy)
```

### 4.4 同步策略

- **全状态同步**: 同步所有游戏状态，简单但网络流量大
- **增量同步**: 只同步变化的状态，减少网络流量
- **事件同步**: 只同步游戏事件，客户端根据事件重建状态
## 5. 多线程编程
### 5.1 线程基础

多线程允许在后台执行任务，避免阻塞主线程。
- **主线程**: 处理游戏逻辑、渲染和输入
- **工作线程**: 处理耗时任务，如资源加载、网络通信等
### 5.2 线程创建与管理
```gdscript
# 创建线程
var thread = Thread.new()

# 启动线程
thread.start(_thread_function, "Hello from thread")

# 线程函数
func _thread_function(data):
    print("Thread started with data: " + str(data))
    # 执行耗时任务
    for i in range(1000000):
        var value = sqrt(i)
    print("Thread finished")

# 等待线程完成
func _ready():
    thread.wait_to_finish()
    print("Thread has finished")
```

### 5.3 线程安全

线程安全是确保多个线程同时访问数据时不会导致数据损坏。
- **互斥锁 (Mutex)**: 确保一次只有一个线程访问共享数据
- **信号量 (Semaphore)**: 控制对资源的访问数量
```gdscript
# 创建互斥锁
var mutex = Mutex.new()
var shared_data = 0

# 线程函数
func _thread_function(data):
    for i in range(1000):
        mutex.lock()
        shared_data += 1
        mutex.unlock()

# 启动多个线程
for i in range(10):
    var thread = Thread.new()
    thread.start(_thread_function, i)
```

### 5.4 线程通信

线程通信是在不同线程之间传递数据：

- **队列 (Queue)**: 用于在线程之间安全地传递数据
- **信号 (Signal)**: 用于通知主线程任务完成
```gdscript
# 创建队列
var queue = Queue.new()

# 线程函数
func _thread_function(data):
    # 执行任务
    var result = heavy_computation()
    # 将结果添加到队列
    queue.push_back(result)
    # 发送信号
    emit_signal("task_completed")

# 主线程处理结果
func _on_task_completed():
    while not queue.is_empty():
        var result = queue.pop_front()
        process_result(result)
```

## 6. 网络与多线程最佳实践
### 6.1 性能优化

- **减少网络流量**: 只发送必要的数据，使用压缩和增量更新
- **优化线程使用**: 只在必要时使用线程，避免过多的线程创建和销毁
- **使用线程池**: 重用线程，避免频繁创建线程的开销
- **异步操作**: 使用异步操作处理网络和 I/O 任务
### 6.2 安全考虑

- **输入验证**: 验证客户端输入，防止作弊
- **数据加密**: 加密敏感数据，如用户凭据和支付信息
- **防 DDoS**: 实现防 DDoS 措施，如连接限制和速率限制
- **安全通信**: 使用 HTTPS 或其他安全协议进行网络通信
### 6.3 错误处理

- **网络错误**: 处理网络连接失败、超时等错误
- **线程错误**: 捕获和处理线程中的错误，避免应用崩溃
- **异常处理**: 使用 try-except 块处理异常
- **错误日志**: 记录错误信息，便于调试和分析
## 7. 实际应用示例

### 7.1 示例 1：简单的多人游戏

```gdscript
# NetworkManager.gd
extends Node

static var instance: NetworkManager = null
var server = ENetMultiplayerPeer.new()
var client = ENetMultiplayerPeer.new()

func _ready():
    instance = self
    # 连接网络事件
    multiplayer.peer_connected.connect(_on_peer_connected)
    multiplayer.peer_disconnected.connect(_on_peer_disconnected)

func start_server(port: int = 4242, max_players: int = 4):
    var error = server.create_server(port, max_players)
    if error != OK:
        print("Failed to start server")
        return false
    else:
        multiplayer.multiplayer_peer = server
        print("Server started on port " + str(port))
        return true

func connect_to_server(address: String = "127.0.0.1", port: int = 4242):
    var error = client.create_client(address, port)
    if error != OK:
        print("Failed to connect to server")
        return false
    else:
        multiplayer.multiplayer_peer = client
        print("Connected to server at " + address + ":" + str(port))
        return true

func _on_peer_connected(peer_id):
    print("Player " + str(peer_id) + " connected")

func _on_peer_disconnected(peer_id):
    print("Player " + str(peer_id) + " disconnected")

@rpc
func spawn_player(peer_id: int, position: Vector3):
    var player = preload("res://scenes/player.tscn").instantiate()
    player.position = position
    player.name = "Player " + str(peer_id)
    add_child(player)
```

### 7.2 示例 2：多线程资源加载

```gdscript
# ResourceLoaderThread.gd
extends Node

var thread = Thread.new()
var queue = Queue.new()
var is_loading = false

func load_resource(path: String, callback: Callable):
    # 将加载任务添加到队列
    queue.push_back({path = path, callback = callback})
    
    # 如果线程未运行，启动线程
    if not is_loading:
        is_loading = true
        thread.start(_load_resources, null)

func _load_resources(data):
    while not queue.is_empty():
        var task = queue.pop_front()
        var resource = load(task.path)
        # 在主线程中调用回调
        call_deferred("_on_resource_loaded", resource, task.callback)
    is_loading = false

func _on_resource_loaded(resource: Resource, callback: Callable):
    callback.call(resource)
```

### 7.3 示例 3：网络同步
```gdscript
# NetworkSync.gd
extends CharacterBody3D

var sync_rate = 10  # 每秒同步次数
var time_since_last_sync = 0

func _ready():
    if multiplayer.is_server():
        # 服务器定期同步位置
        set_process(true)
    else:
        # 客户端接收位置更新
        multiplayer.peer_connected.connect(_on_peer_connected)

func _process(delta):
    if multiplayer.is_server():
        time_since_last_sync += delta
        if time_since_last_sync >= 1.0 / sync_rate:
            # 同步位置
            rpc("update_position", position, velocity)
            time_since_last_sync = 0

@rpc
func update_position(new_position: Vector3, new_velocity: Vector3):
    # 客户端更新位置
    position = new_position
    velocity = new_velocity

func _on_peer_connected(peer_id):
    # 新客户端连接时，发送当前状态
    if multiplayer.is_server():
        rpc_id(peer_id, "update_position", position, velocity)
```

## 8. 总结

Godot 的网络与多线程系统是强大的工具，用于创建 multiplayer 游戏和处理耗时任务。网络系统的核心优势包括：
- **基于 ENet**: 提供可靠的 UDP 通信，适合游戏开发
- **RPC 系统**: 简单易用的远程过程调用机制
- **网络同步**: 支持多种同步策略，保持游戏状态的一致性
- **多线程支持**: 允许在后台执行耗时任务，提高游戏性能
通过掌握网络与多线程系统的使用方法和最佳实践，开发者可以创建高性能、可靠的 multiplayer 游戏和应用。网络与多线程系统是 Godot 引擎的高级特性，对于开发复杂的游戏和应用至关重要。
