# 类型提示与注解系统 | Type Hints & Annotation System

> @Author: Anonymous
> @Category: GDScript Advanced
> @Description: 强类型编程、类型注解、泛型、自定义类型。 | Strong typing, type annotations, generics, custom types.

## 1. 类型注解 | Type Annotations

在 GDScript 4 中，类型注解更加严格和有用：

```gdscript
var health: int = 100

# 可以在参数和返回值上
func take_damage(amount: int) -> void:
    health -= amount
```

### 1.1 类型安全 | Type Safety

启用类型检查（在编辑器设置中）可以在运行前捕获错误：

```gdscript
# 如果向 int 变量赋 String 值，会在编辑器中报错
var health: int = "100"  # 错误!
```

## 2. 常见内建类型 | Common Built-in Types

### 2.1 集合类型 | Collection Types

```gdscript
var my_array: Array[int] = [1, 2, 3]  # 泛型数组
var my_dict: Dictionary[String, int] = {"a": 1, "b": 2}  # 泛型字典
```

### 2.2 Godot 特定类型 | Godot-specific Types

```gdscript
var sprite: Sprite2D = $Sprite2D
var timer: Timer = $Timer
```

### 2.3 空类型 | Nullable Types

使用 `?` 表示可以为空：

```gdscript
var enemy: Node2D? = get_node_or_null("Enemy")

if enemy:
    enemy.visible = true
```

---

## 3. 枚举作为类型 | Enums as Types

枚举可以直接作为类型使用：

```gdscript
enum State { IDLE, WALK, JUMP }

var current_state: State = State.IDLE
```

---

## 4. 类型转换 | Type Casting

### 4.1 as 关键字 | as Keyword

安全的类型转换：

```gdscript
var node: Node = get_node("Character")
var player: Character? = node as Character

if player:
    player.jump()
```

## 5. 注解 | Annotations

GDScript 4 使用注解（Annotations）来添加元数据：

### 5.1 常见注解 | Common Annotations

| 注解 | 说明 | 示例 |
| :--- | :--- | :--- |
| `@onready` | 准备好后初始化 | `@onready var sprite = $Sprite` |
| `@export` | 在编辑器中暴露变量 | `@export var speed: float = 5.0` |
| `@tool` | 在编辑器中运行脚本 | `@tool` (放在脚本顶部) |
| `@rpc` | 网络 RPC 标记 | `@rpc func foo():` |
| `@warning_ignore` | 忽略警告 | `@warning_ignore("unused_parameter")` |
| `@deprecated` | 标记已弃用 | `@deprecated func old_func():` |

### 5.2 @onready | @onready

在 `_ready()` 之前运行，常用于获取子节点引用：

```gdscript
extends Node2D

@onready var sprite: Sprite2D = $Sprite
@onready var collision: CollisionShape2D = $CollisionShape2D
```

等价于：

```gdscript
var sprite: Sprite2D

func _ready():
    sprite = $Sprite
```

### 5.3 @export | @export

把变量暴露在编辑器检查器中：

```gdscript
@export var health: int = 100
@export var speed: float = 200.0
@export var color: Color = Color.WHITE
@export var texture: Texture2D

# 分组导出
@export_group("Combat")
@export var damage: int = 10
@export var defense: int = 5

# 使用枚举
enum State { IDLE, WALK }
@export var initial_state: State = State.IDLE
```

### 5.4 @tool | @tool

让脚本在编辑器中也能运行：

```gdscript
@tool
extends Node2D

@export var radius: float = 50.0

func _process(_delta: float):
    # 在编辑器中也会调用
    queue_redraw()

func _draw():
    draw_circle(Vector2.ZERO, radius, Color.RED)
```

## 6. 泛型 | Generics

GDScript 4 支持泛型集合：

```gdscript
var items: Array[Item] = [sword, shield]
var name_to_id: Dictionary[String, int] = {"Player": 1}
```

---

## 7. 自定义类型 | Custom Types

使用 `class_name`：

```gdscript
class_name Item
extends Resource

@export var item_name: String
@export var description: String
@export var price: int
```

---

## 8. 小结 | Summary

1. GDScript 4 有强大的类型系统
2. 使用 `:` 注解变量类型
3. 使用 `@onready` 初始化节点引用
4. 使用 `@export` 在编辑器中显示变量
5. 使用 `@tool` 在编辑器中运行脚本
