## 1. 信号高级用法 | Advanced Signals

### 1.1 一次性连接 | One-time Connections

使用 `once` 连接，触发后自动断开：

```gdscript
some_signal.connect(_on_some_signal, CONNECT_ONESHOT)
```

或者使用 `await`（见下节）。

### 1.2 延迟连接 | Deferred Connections

使用 `CONNECT_DEFERRED` 让信号在下一帧处理：

```gdscript
some_signal.connect(_on_some_signal, CONNECT_DEFERRED)
```

### 1.3 带参数绑定 | Argument Binding

在连接时绑定额外参数：

```gdscript
button.pressed.connect(_on_button_clicked.bind("Start"))
button2.pressed.connect(_on_button_clicked.bind("Options"))

func _on_button_clicked(action: String):
    print("Action clicked: ", action)
```

### 1.4 自定义信号参数 | Custom Signal Arguments

信号可以有多个参数：

```gdscript
signal player_moved(old_pos: Vector2, new_pos: Vector2)

func move(new_pos: Vector2):
    var old_pos: Vector2 = position
    position = new_pos
    player_moved.emit(old_pos, new_pos)
```

## 2. 异步编程 | Async Programming

### 2.1 await 关键字 | await Keyword

GDScript 4 引入了 `await` 关键字，让异步编程更简单：

```gdscript
func _ready():
    print("Starting...")
    await get_tree().create_timer(2.0).timeout
    print("2 seconds passed")
```

### 2.2 await 信号 | await a Signal

可以 `await` 任意信号，直到信号被触发：

```gdscript
func show_dialog(text: String) -> void:
    $Label.text = text
    $Button.show()
    await $Button.pressed
    $Button.hide()
```

### 2.3 异步函数 | Async Functions

带有 `await` 的函数自动变成异步函数，调用它的地方也可以 `await`：

```gdscript
func load_game_async() -> void:
    var thread: Thread = Thread.new()
    thread.start(_load_game_thread)
    
    # 等待加载完成信号
    await _game_loaded
    
func _load_game_thread():
    # 模拟耗时操作
    OS.delay_msec(3000)
    _game_loaded.emit()
```

## 3. 协程与 yield (Godot 3.x 方式) | Coroutines & yield (Godot 3.x Style)

虽然在 Godot 4.x 推荐用 `await`，但 `yield` 仍然可用：

```gdscript
func my_coroutine():
    print("Step 1")
    yield(get_tree().create_timer(1.0), "timeout")
    
    print("Step 2")
    yield(get_tree().create_timer(1.0), "timeout")
    
    print("Done")
```

## 4. 线程 | Threads

### 4.1 使用 Thread 类 | Using Thread Class

对于真正耗时的任务，使用线程：

```gdscript
extends Node

var thread: Thread
var is_running: bool = false

func start_heavy_task():
    if is_running:
        return
    
    is_running = true
    thread = Thread.new()
    thread.start(_heavy_task)

func _heavy_task():
    # 这里的代码在后台线程运行
    # 注意：不要在这里直接操作场景树！
    
    for i in range(100):
        OS.delay_msec(50)  # 模拟工作
        call_deferred("_update_progress", i)
    
    is_running = false

func _update_progress(value: int):
    # 更新 UI（必须在主线程，所以用 call_deferred）
    $ProgressBar.value = value
```

### 4.2 线程安全 | Thread Safety

- 场景树操作**不是线程安全的**
- 如果需要从另一个线程操作场景树，使用 `call_deferred`
- 使用 `Mutex` 保护共享数据

```gdscript
var mutex: Mutex = Mutex.new()
var shared_data: int = 0

func thread_safe_increment():
    mutex.lock()
    shared_data += 1
    mutex.unlock()
```

## 5. 信号模式 | Signal Patterns

### 5.1 事件总线模式 | Event Bus Pattern

使用一个单例作为全局事件总线：

```gdscript
# EventBus.gd (Autoload)
extends Node

signal player_health_changed(amount: int)
signal item_collected(item: Item)
signal game_over()
```

在任何地方发送：

```gdscript
EventBus.player_health_changed.emit(-10)
```

在任何地方接收：

```gdscript
func _ready():
    EventBus.player_health_changed.connect(_on_player_health_changed)
```

### 5.2 属性变化通知 | Property Change Notifications

```gdscript
class_name Player

signal health_changed(old_health: int, new_health: int)

var health: int = 100:
    set(value):
        var old: int = health
        health = value
        health_changed.emit(old, health)
```

## 6. 小结 | Summary

1. 使用 `await` 等待信号或定时器（Godot 4.x）
2. 使用 `signal_name.connect(...)` 建立连接
3. 使用 `bind` 绑定额外参数
4. 使用 `call_deferred` 从线程回调主线程
5. 推荐使用单例作为全局事件总线
