## 1. Godot 引擎简介 | Godot Engine Overview

### 1.1 什么是 Godot | What is Godot

Godot 是一个免费、开源、跨平台的游戏引擎，由阿根廷程序员 Juan Linietsky 于 2001 年开始开发，2014 年在 GitHub 上开源。它采用宽松的 MIT 许可证，允许完全免费用于任何用途，包括商业用途。

Godot 支持 2D 和 3D 游戏开发，提供了完整的游戏开发工具链，包括场景编辑器、动画编辑器、脚本编辑器、物理引擎、音频系统、渲染系统等。它使用自己的编程语言 GDScript（基于 Python），同时也支持 C#、C++、Visual Script 等。

### 1.2 核心特性 | Core Features

- **跨平台开发**：支持 Windows、macOS、Linux、Android、iOS、Web、ChromeOS、UWP 等多个平台
- **2D/3D 双重支持**：完整的 2D 和 3D 游戏开发功能
- **GDScript 编程语言**：简单易用的脚本语言，基于 Python 语法
- **强大的编辑器**：直观的场景编辑器、动画编辑器、节点系统
- **基于节点的架构**：高度灵活的组件化开发模式
- **完整的文档**：详细的官方文档和活跃的社区
- **MIT 开源**：完全免费，无任何限制

### 1.3 版本历史 | Version History

Godot 经过了多个大版本的发展，目前有两个主要版本在维护：

| 版本 | 发布时间 | 主要特性 |
| :--- | :--- | :--- |
| Godot 1.x | 2014-2016 | 初始开源版本，基础功能 |
| Godot 2.x | 2016-2018 | 重大改进，编辑器优化 |
| Godot 3.x | 2018-2023 | 完全重写，全新渲染系统，重大改进 |
| Godot 4.x | 2023-至今 | 全新架构，Vulkan 渲染，性能大幅提升 |

目前，Godot 4.x 是推荐使用的最新版本，而 Godot 3.x 也会继续得到维护和安全更新。

## 2. 安装与环境配置 | Installation & Environment Setup

### 2.1 下载 Godot | Download Godot

#### 官方下载方式 | Official Downloads

访问 Godot 官网下载页面：

- 官网地址：<https://godotengine.org/download>
- GitHub 发布页：<https://github.com/godotengine/godot/releases>

#### 选择版本 | Choose a Version

下载时需要注意：
1. **选择 Godot 4.x 还是 3.x**：推荐使用最新的 Godot 4.x
2. **选择标准版本还是 Mono 版本**：标准版本默认使用 GDScript，Mono 版本额外支持 C#
3. **选择编辑器运行平台**：选择你正在使用的系统（Windows、macOS、Linux）
4. **选择导出模板**：如果你需要导出游戏到其他平台，需要下载对应的导出模板

#### 下载内容包括：

1. **Godot 编辑器**：用于开发游戏的主程序
2. **导出模板**：用于将游戏打包发布到各个平台的模板文件
3. **可选**：C# 支持的 Mono 版本

### 2.2 编辑器安装 | Editor Installation

#### Windows 安装

1. 下载 .zip 或 .7z 文件
2. 解压到任意目录（建议使用英文路径）
3. 直接运行 godot.windows.editor.x86_64.exe 即可
4. （可选）创建桌面快捷方式

#### macOS 安装

1. 下载 .dmg 或 .zip 文件
2. 解压后拖拽 Godot 到 Applications 文件夹
3. 从应用程序文件夹运行 Godot

#### Linux 安装

1. 下载 .tar.xz 或 .zip 文件
2. 解压到任意目录
3. 直接运行 godot.linux.editor.x86_64 即可
4. （可选）创建 .desktop 文件用于启动器

### 2.3 导出模板配置 | Export Templates Setup

#### 下载导出模板

1. 下载与编辑器版本相同的导出模板包
2. 文件名通常类似：Godot_v.x.x-stable_export_templates.tpz
3. 导入模板：
   - 方法一：在编辑器中打开 "编辑" → "管理导出模板" → "从本地文件导入"
   - 方法二：手动解压到用户目录下的 Godot 模板文件夹

#### 模板目录位置

- **Windows**：`%APPDATA%\Godot\export_templates\<版本>`
- **macOS**：`~/Library/Application Support/Godot/export_templates/<版本>`
- **Linux**：`~/.local/share/godot/export_templates/<版本>`

### 2.4 配置 VS Code 支持（可选） | VS Code Setup (Optional)

虽然 Godot 自带编辑器，但很多开发者更喜欢使用 VS Code：

#### 步骤

1. 安装 VS Code
2. 在扩展商店搜索 "Godot" 相关插件
3. 推荐插件：
   - GDScript 语法高亮
   - Godot Tools（提供更好的集成支持）
4. 配置工程与 VS Code 的通信

## 3. 项目结构 | Project Structure

### 3.1 创建新项目 | Create New Project

1. 打开 Godot 编辑器
2. 点击 "新建项目"
3. 选择项目保存位置（建议全英文路径）
4. 输入项目名称
5. 选择渲染器（Godot 4.x 推荐 Vulkan 或 Compatibility）
6. 点击 "创建并编辑"

### 3.2 默认项目结构 | Default Project Structure

创建好的项目包含以下结构：

```
你的项目/
├── .godot/            # Godot 自动生成的配置和缓存文件夹
│   ├── editor/        # 编辑器配置
│   └── import/        # 导入的资源缓存
├── .gitignore         # 如果使用 Git，会有此文件
├── project.godot      # 项目配置文件（重要）
└── icon.svg           # 项目默认图标
```

#### project.godot 文件

这是 Godot 的项目配置文件，包含：
- 项目名称
- 渲染器设置
- 主场景
- 导入设置
- 其他配置

#### icon.svg

默认的项目图标，你可以替换为自己的图标（.png 或 .svg 格式）。

### 3.3 推荐的资源组织方式 | Recommended Resource Organization

随着项目规模增长，建议按以下方式组织资源：

```
你的项目/
├── scenes/             # 场景文件
│   ├── game/          # 游戏相关场景
│   ├── ui/            # UI 相关场景
│   └── test/          # 测试场景
├── scripts/            # 脚本文件
│   ├── controllers/   # 控制器脚本
│   ├── managers/      # 管理器脚本
│   └── utils/         # 工具脚本
├── assets/             # 资源文件夹
│   ├── images/        # 图片资源
│   │   ├── ui/
│   │   └── textures/
│   ├── audio/         # 音频资源
│   │   ├── music/
│   │   └── sounds/
│   └── models/        # 3D 模型
├── themes/             # 主题样式
└── config/             # 配置文件
```

## 4. Godot 编辑器界面介绍 | Editor Interface Tour

### 4.1 主界面布局 | Main Layout

Godot 编辑器主要分为以下区域：

1. **顶部主菜单**：包含项目、编辑、调试等菜单
2. **左侧面板**：包含文件系统、场景树、导入等面板
3. **中心工作区**：场景编辑器（2D、3D、脚本）
4. **底部面板**：调试、输出、文件系统、动画、音频等
5. **右侧面板**：检查器（Inspector）、历史记录、组、节点、主题预览

### 4.2 常用工作空间 | Workspaces

编辑器顶部有工作空间选项卡，可以切换：
- **2D**：2D 游戏场景编辑
- **3D**：3D 游戏场景编辑
- **脚本**：脚本编辑模式
- **动画**：动画编辑模式
- **音频**：音频编辑模式
- **本地化**：多语言文本编辑
- **组**：组管理

### 4.3 文件系统面板 | File System Panel

位于左下角，用于浏览和管理项目中的资源文件。
- 右键点击可以创建新文件/文件夹
- 双击文件可以在相应编辑器中打开
- 拖拽文件到场景可以创建实例

## 5. 核心概念：节点与场景 | Core Concepts: Nodes & Scenes

### 5.1 节点 | Nodes

节点是 Godot 的基本构建块。每个节点都有特定的功能，例如：
- **Sprite2D**：显示 2D 图片
- **Label**：显示文本
- **Area2D**：检测碰撞区域
- **Camera2D**：相机
- **CharacterBody2D**：角色控制器

每个节点都有属性（通过检查器面板设置）和方法（通过脚本调用）。

### 5.2 场景 | Scenes

场景是节点的树形结构（hierarchy），可以保存为 .tscn（文本场景）或 .scn（二进制场景）文件。
- 一个项目可以包含多个场景
- 场景可以实例化（instantiate）到其他场景中，就像搭积木一样
- 一个场景可以作为另一个场景的子节点

例如：
- 场景 A：玩家角色，包含 Sprite、碰撞体、脚本
- 场景 B：游戏主场景，包含背景、多个实例化的场景 A、UI

### 5.3 主场景 | Main Scene

在 project.godot 中配置的入口场景，启动游戏时会自动加载的第一个场景。

## 6. 第一个 Hello World 项目 | First Hello World Project

### 6.1 创建场景 | Create a Scene

1. 点击左侧面板的 "场景" 标签
2. 点击 "新建场景"
3. 在场景树中点击 "添加子节点"
4. 搜索并添加 "ColorRect"（用于设置背景）
5. 选择 ColorRect，在右侧检查器中设置颜色
6. 再次点击 "添加子节点"
7. 搜索并添加 "Label"（用于显示文本）
8. 选择 Label，设置文本为 "Hello, Godot!"
9. 调整 Label 的位置和大小

### 6.2 设置主场景 | Set Main Scene

1. 保存场景为 main.tscn
2. 点击顶部菜单 "项目" → "项目设置"
3. 找到 "应用程序" → "运行" → "主场景"
4. 点击文件夹图标，选择刚才保存的 main.tscn

### 6.3 运行项目 | Run Project

1. 按 F5 或点击右上角的 "运行项目" 按钮
2. 恭喜，你看到了第一个 Godot 程序！

## 7. 导入资源 | Importing Resources

### 7.1 支持的资源格式 | Supported Formats

Godot 支持以下类型资源：

| 类型 | 常见格式 | 说明 |
| :--- | :--- | :--- |
| 图片 | .png, .jpg, .jpeg, .webp, .bmp, .tga, .svg | 2D 图片资源 |
| 音频 | .wav, .ogg, .mp3, .flac | 音效和音乐 |
| 3D 模型 | .glb, .gltf, .obj, .dae, .fbx (部分支持) | 3D 模型文件 |
| 字体 | .ttf, .otf, .woff, .woff2 | 字体文件 |
| 视频 | .ogv, .webm | 视频文件 |
| 着色器 | .glsl, .gdshader | 着色器代码 |
| 其他 | .json, .csv, .xml | 数据文件 |

### 7.2 资源导入流程 | Import Pipeline

当你把文件放入项目文件夹后：
1. Godot 会自动检测到新文件
2. Godot 会将其导入为内部格式并缓存（在 .godot/import/ 文件夹）
3. 原始文件保留在原地，你可以使用外部编辑器修改，Godot 会重新导入

你可以在"导入"面板调整每个资源的导入设置，如纹理压缩方式、是否生成 mipmap 等。

## 8. 项目设置 | Project Settings

### 8.1 打开项目设置 | Access Project Settings

点击顶部菜单："项目" → "项目设置"

### 8.2 常用设置项 | Common Settings

- **应用程序/运行**：设置主场景
- **应用程序/配置**：设置游戏名称、版本等
- **显示/窗口**：设置窗口大小、标题、是否允许窗口调整
- **渲染/质量**：渲染质量、帧率限制
- **输入**：输入映射（按键绑定）
