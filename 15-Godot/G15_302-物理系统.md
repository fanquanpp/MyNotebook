## 1. 物理系统概述

Godot 引擎提供了强大的 2D 和 3D 物理系统，用于模拟真实世界的物理现象，如重力、碰撞、摩擦力等。物理系统是游戏开发中不可或缺的一部分，它使得游戏对象的运动和交互更加真实和自然。

### 1.1 核心概念

- **物理体 (Physics Body)**: 具有物理属性的游戏对象，如质量、速度、加速度等
- **碰撞形状 (Collision Shape)**: 定义物理体的碰撞边界
- **物理材质 (Physics Material)**: 定义物理体的摩擦系数和 restitution（弹性）
- **碰撞检测 (Collision Detection)**: 检测物理体之间的碰撞
- **物理世界 (Physics World)**: 物理体所在的空间，包含物理规则和参数

### 1.2 物理引擎

Godot 引擎内置了两种物理引擎：

- **Godot Physics**: Godot 自己开发的物理引擎，默认用于 2D 物理
- **Bullet Physics**: 第三方物理引擎，用于 3D 物理

### 1.3 坐标系

- **2D 坐标系**: 以屏幕左上角为原点，x 轴向右，y 轴向下
- **3D 坐标系**: 以屏幕中心为原点，x 轴向右，y 轴向上，z 轴向前

## 2. 2D 物理系统

### 2.1 2D 物理体

Godot 提供了以下 2D 物理体类型：

- **StaticBody2D**: 静态物理体，不会受到力的影响，用于创建静止的障碍物
- **RigidBody2D**: 刚体物理体，完全受物理引擎控制，用于模拟真实的物理运动
- **CharacterBody2D**: 角色物理体，用于玩家或 AI 控制的角色，提供更灵活的控制
- **KinematicBody2D**: 运动学物理体，由代码控制运动，用于需要精确控制的对象
- **Area2D**: 区域物理体，用于检测进入/退出事件，不参与物理碰撞

### 2.2 2D 碰撞形状

2D 碰撞形状定义了物理体的碰撞边界：

- **RectangleShape2D**: 矩形碰撞形状
- **CircleShape2D**: 圆形碰撞形状
- **CapsuleShape2D**: 胶囊碰撞形状
- **ConvexPolygonShape2D**: 凸多边形碰撞形状
- **ConcavePolygonShape2D**: 凹多边形碰撞形状
- **SegmentShape2D**: 线段碰撞形状

### 2.3 2D 物理材质

2D 物理材质定义了物理体的摩擦系数和 restitution（弹性）：

- **Friction**: 摩擦系数，影响物体滑动时的阻力
- **Restitution**: 弹性系数，影响物体碰撞时的反弹程度

### 2.4 2D 碰撞检测

2D 碰撞检测用于检测物理体之间的碰撞：

- **碰撞信号**: 物理体碰撞时会发射信号，如 `body_entered`、`body_exited`
- **碰撞回调**: 可以通过重写 `_on_body_entered`、`_on_body_exited` 等方法处理碰撞
- **射线检测**: 使用 `RayCast2D` 节点检测射线与物体的碰撞
- **形状检测**: 使用 `Area2D` 节点检测形状与物体的碰撞

## 4. 物理世界设置

### 4.1 2D 物理世界

2D 物理世界的设置可以通过 `PhysicsServer2D` 进行：

```gdscript
# 设置 2D 物理世界重力
PhysicsServer2D.body_set_direct_state(physics_body_rid, PhysicsServer2D.body_get_direct_state(physics_body_rid))

# 设置 2D 物理世界参数
var params = PhysicsServer2D.get_world_2d_parameters()
params.gravity = Vector2(0, 980)  # 设置重力
params.default_contact_monitor = true  # 启用接触监控
PhysicsServer2D.set_world_2d_parameters(params)
```

### 4.2 3D 物理世界

3D 物理世界的设置可以通过 `PhysicsServer3D` 进行：

```gdscript
# 设置 3D 物理世界重力
var params = PhysicsServer3D.get_world_3d_parameters()
params.gravity = Vector3(0, -9.8, 0)  # 设置重力
params.default_contact_monitor = true  # 启用接触监控
PhysicsServer3D.set_world_3d_parameters(params)
```

## 5. 物理体属性

### 5.1 质量与密度

- **质量 (Mass)**: 物体的质量，影响物体的惯性和对力的响应
- **密度 (Density)**: 物体的密度，与体积一起决定质量

```gdscript
# 设置 RigidBody2D 的质量
$RigidBody2D.mass = 10.0

# 设置 RigidBody2D 的密度
$RigidBody2D.density = 1.0
```

### 5.2 摩擦力与 restitution

- **Friction**: 摩擦系数，影响物体滑动时的阻力
- **Restitution**: 弹性系数，影响物体碰撞时的反弹程度

```gdscript
# 创建物理材质
var material = PhysicsMaterial2D.new()
material.friction = 0.5
material.restitution = 0.3

# 应用到物理体
$RigidBody2D.physics_material_override = material
```

### 5.3 线性与角速度

- **Linear Velocity**: 线性速度，物体在直线方向上的速度
- **Angular Velocity**: 角速度，物体绕自身轴旋转的速度

```gdscript
# 设置 RigidBody2D 的线性速度
$RigidBody2D.linear_velocity = Vector2(100, 0)

# 设置 RigidBody2D 的角速度
$RigidBody2D.angular_velocity = 1.0  # 弧度/秒
```

### 5.4 碰撞层与碰撞掩码

- **Collision Layer**: 物体所在的碰撞层
- **Collision Mask**: 物体可以与哪些碰撞层发生碰撞

```gdscript
# 设置碰撞层和碰撞掩码
$RigidBody2D.collision_layer = 1  # 第 1 层
$RigidBody2D.collision_mask = 1 | 2  # 与第 1 层和第 2 层碰撞
```

## 6. 碰撞检测与响应

### 6.1 碰撞信号

物理体碰撞时会发射信号，可以通过连接这些信号来处理碰撞：

```gdscript
# 连接碰撞信号
$Area2D.body_entered.connect(_on_body_entered)
$Area2D.body_exited.connect(_on_body_exited)

# 处理碰撞
func _on_body_entered(body):
    print("Body entered: " + body.name)

func _on_body_exited(body):
    print("Body exited: " + body.name)
```

### 6.2 碰撞回调

可以通过重写物理体的回调方法来处理碰撞：

```gdscript
# 重写碰撞回调方法
func _on_body_entered(body):
    print("Body entered: " + body.name)

func _on_body_exited(body):
    print("Body exited: " + body.name)
```

### 6.3 射线检测

使用 `RayCast2D` 或 `RayCast3D` 节点进行射线检测：

```gdscript
# 设置射线检测
$RayCast2D.target_position = Vector2(100, 0)
$RayCast2D.force_raycast_update()

# 检测碰撞
if $RayCast2D.is_colliding():
    var collider = $RayCast2D.get_collider()
    var collision_point = $RayCast2D.get_collision_point()
    print("Collided with: " + collider.name)
    print("Collision point: " + str(collision_point))
```

### 6.4 形状检测

使用 `Area2D` 或 `Area3D` 节点进行形状检测：

```gdscript
# 设置形状检测
$Area2D.monitorable = true
$Area2D.body_entered.connect(_on_body_entered)

# 处理碰撞
func _on_body_entered(body):
    print("Body entered area: " + body.name)
```

## 7. 物理优化

### 7.1 性能优化

- **减少物理体数量**: 只在必要时使用物理体，避免过多的物理体导致性能下降
- **使用简单碰撞形状**: 优先使用简单的碰撞形状，如圆形、矩形，避免复杂的多边形
- **合理设置碰撞层**: 使用碰撞层和碰撞掩码减少不必要的碰撞检测
- **使用睡眠模式**: 对于静止的物理体，启用睡眠模式以减少计算
- **避免在 _process 中修改物理属性**: 尽量在 `_physics_process` 中修改物理属性

### 7.2 内存优化

- **重用碰撞形状**: 对于相同的碰撞形状，重用同一个实例
- **合理设置物理材质**: 重用物理材质实例，避免重复创建
- **及时清理物理体**: 对于不再需要的物理体，及时从场景树中移除

### 7.3 常见问题

- **穿透问题**: 快速移动的物体可能会穿过薄的碰撞体。解决方案：使用连续碰撞检测或增加碰撞体厚度
- **抖动问题**: 物理体在碰撞时可能会产生抖动。解决方案：调整物理参数或使用插值
- **性能问题**: 过多的物理体或复杂的碰撞形状可能导致性能下降。解决方案：优化物理体数量和碰撞形状

## 8. 实际应用示例

### 8.1 示例 1：2D 平台游戏玩家控制器

```gdscript
# PlayerController.gd
extends CharacterBody2D

export var speed = 300
export var jump_force = 500
var velocity = Vector2()

func _process(delta):
    # 处理输入
    var input_dir = Vector2()
    if Input.is_action_pressed("ui_right"):
        input_dir.x += 1
    if Input.is_action_pressed("ui_left"):
        input_dir.x -= 1
    
    # 归一化输入向量
    input_dir = input_dir.normalized()
    
    # 计算水平速度
    velocity.x = input_dir.x * speed

func _physics_process(delta):
    # 应用重力
    velocity.y += 9.8 * 100 * delta
    
    # 处理跳跃
    if Input.is_action_just_pressed("ui_accept") and is_on_floor():
        velocity.y = -jump_force
    
    # 应用移动
    move_and_slide()
```

### 8.2 示例 2：3D 物理对象

```gdscript
# PhysicsObject.gd
extends RigidBody3D

export var initial_force = Vector3(10, 0, 0)

func _ready():
    # 施加初始力
    apply_central_impulse(initial_force)

func _on_body_entered(body):
    # 处理碰撞
    print("Collided with: " + body.name)
    
    # 施加反作用力
    var direction = (position - body.position).normalized()
    apply_central_impulse(direction * 10)
```

### 8.3 示例 3：射线检测

```gdscript
# RaycastExample.gd
extends Node2D

@onready var raycast = $RayCast2D

func _process(delta):
    # 更新射线方向
    var mouse_pos = get_global_mouse_position()
    var direction = (mouse_pos - position).normalized()
    raycast.target_position = direction * 1000
    raycast.force_raycast_update()
    
    # 检测碰撞
    if raycast.is_colliding():
        var collider = raycast.get_collider()
        var collision_point = raycast.get_collision_point()
        print("Collided with: " + collider.name)
        print("Collision point: " + str(collision_point))
```

## 9. 总结

Godot 的物理系统是一个功能强大、灵活的工具，它提供了 2D 和 3D 物理模拟能力，使得游戏对象的运动和交互更加真实和自然。物理系统的核心优势包括：

- **全面的物理体类型**: 提供了多种物理体类型，满足不同的游戏需求
- **丰富的碰撞形状**: 支持多种碰撞形状，适应不同的游戏场景
- **灵活的物理参数**: 可以调整物理体的质量、摩擦力、弹性等参数
- **强大的碰撞检测**: 提供了多种碰撞检测方法，如碰撞信号、射线检测等
- **性能优化**: 支持多种性能优化手段，如睡眠模式、碰撞层等
通过掌握物理系统的使用方法和最佳实践，开发者可以构建物理效果逼真、性能优化的游戏。物理系统是 Godot 引擎的核心特性之一，也是游戏开发中最常用的工具之一。
