## 1. 场景树概述 | Scene Tree Overview

### 1.1 什么是场景树 | What is the Scene Tree

Godot 是基于节点树的架构。所有的内容都在场景树中：
- **根节点**：通常是 `MainLoop` 或 `SceneTree`
- **子节点**：每个节点可以有多个子节点
- **父节点**：每个节点都有一个父节点（除了根）

### 1.2 节点与层级 | Nodes & Hierarchy

```
Root (Viewport)
 ├── GameManager
 └── World (Node2D)
      ├── Player (CharacterBody2D)
      │   ├── Sprite2D
      │   └── CollisionShape2D
      └── Enemy (CharacterBody2D)
          ├── Sprite2D
          └── CollisionShape2D
```

## 2. 节点生命周期 | Node Lifecycle

### 2.1 生命周期回调顺序 | Lifecycle Order

当节点进入场景树时，回调按以下顺序触发：

1. `_init()`：对象初始化，在创建时立即调用
2. `_enter_tree()`：进入场景树时
3. `_ready()`：进入场景树且所有子节点都就绪时（**最常用**）
4. `_process(delta)`：每帧调用
5. `_physics_process(delta)`：每个物理帧调用
6. `_exit_tree()`：离开场景树时

### 2.2 _ready vs _enter_tree

| 方法 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| `_enter_tree` | 进入场景树立即调用 | 早初始化、不依赖子节点 |
| `_ready` | 所有子节点都准备好了才调用 | 获取子节点引用 (`@onready`) |

### 2.3 _process & _physics_process

| 方法 | 说明 | 频率 | 用途 |
| :--- | :--- | :--- | :--- |
| `_process(delta)` | 每帧调用 | 不固定（取决于FPS） | 输入、动画、非物理逻辑 |
| `_physics_process(delta)` | 物理帧调用 | 固定（默认60/秒） | 物理运动、角色移动 |

## 3. 实例化场景 | Instantiating Scenes

### 3.1 从文件实例化 | Instantiate from File

```gdscript
func spawn_enemy():
    var enemy_scene: PackedScene = preload("res://Enemy.tscn")
    var enemy: Node = enemy_scene.instantiate()
    add_child(enemy)
```

### 3.2 添加到特定位置 | Add to Specific Parent

```gdscript
var world: Node2D = get_node_or_null("/root/World")
world.add_child(enemy)
```

### 3.3 删除节点 | Removing Nodes

```gdscript
# 删除自身
queue_free()

# 删除其他节点
enemy.queue_free()
```

注意：使用 `queue_free()` 而不是 `free()`，前者更安全，等待当前帧结束才释放。

## 4. 获取节点 | Getting Nodes

### 4.1 常用方法 | Common Methods

| 方法 | 说明 | 示例 |
| :--- | :--- | :--- |
| `$` | 简写，获取子节点 | `$Sprite` |
| `get_node(path)` | 按路径获取 | `get_node("Sprite")` |
| `get_node_or_null(path)` | 找不到返回 null | `get_node_or_null("Sprite")` |
| `%` | 唯一名称访问 | `%Player` |
| `find_child(name)` | 递归查找子节点 | `find_child("Player", true)` |
| `get_parent()` | 获取父节点 | `get_parent()` |
| `get_tree()` | 获取场景树 | `get_tree()` |

### 4.2 使用唯一名称 | Using Unique Names (%)

给节点设置唯一名称，更安全：

```gdscript
# 在编辑器中给节点起个唯一名称，例如：%Player
func _ready():
    var player: Node = %Player
```

## 5. 节点分组 | Node Groups

### 5.1 添加到组 | Add to Group

```gdscript
func _ready():
    add_to_group("enemies")
```

### 5.2 检查是否在组中 | Check Group Membership

```gdscript
if is_in_group("enemies"):
    print("I'm an enemy")
```

### 5.3 遍历组中的所有节点 | Iterate Over Group

```gdscript
func hit_all_enemies():
    for enemy in get_tree().get_nodes_in_group("enemies"):
        enemy.take_damage(10)
```

## 6. 场景树切换 | Scene Tree Switching

### 6.1 改变场景 | Change Scene

```gdscript
func goto_game():
    get_tree().change_scene_to_file("res://Game.tscn")
```

### 6.2 带参数 | With Parameters

```gdscript
func goto_level(level: int):
    # 方法一：使用单例存储
    Global.level = level
    get_tree().change_scene_to_file("res://Level.tscn")
    
    # 方法二：在新场景 _ready 中读取
```

## 7. 场景继承 | Scene Inheritance

Godot 支持场景继承（类似类继承），在编辑器中右键场景 → "新建继承场景"。

## 8. 场景自动加载 (Autoload) | Autoload

在"项目设置" → "自动加载"中添加，这些单例在整个游戏中都能访问：

```gdscript
# Global.gd (Autoload)
extends Node

var score: int = 0

# 在其他脚本中直接使用
Global.score += 10
```

## 9. 小结 | Summary

1. 理解节点生命周期：`_init()` → `_enter_tree()` → `_ready()` → 循环 → `_exit_tree()`
2. 使用 `@onready` 获取子节点引用
3. 使用 `$` 或 `%` 获取节点
4. 使用分组管理类似的节点
5. 使用 `queue_free()` 删除节点
6. 使用 Autoload 做全局单例
