## 1. 什么是资源 | What is a Resource

资源是数据容器，继承自 `Resource` 类。与节点不同，资源不包含在场景树中，但可以被节点引用。

常见资源类型：
- `Texture2D`：图片
- `Font`：字体
- `AudioStream`：音频
- `PackedScene`：场景文件
- `Animation`：动画
- 等等...

## 2. preload 与 load | preload vs load

### 2.1 preload (预加载)

```gdscript
var my_texture: Texture2D = preload("res://icon.svg")
```

- 在编译时加载
- 更高效
- 路径必须是常量字符串

### 2.2 load (动态加载)

```gdscript
var path: String = "res://" + file_name
var my_texture: Texture2D = load(path)
```

- 在运行时加载
- 可以使用变量
- 稍慢，但更灵活

### 2.3 加载场景 | Loading Scenes

```gdscript
var scene_file: PackedScene = preload("res://Enemy.tscn")
var enemy: Node = scene_file.instantiate()
add_child(enemy)
```

## 3. 自定义资源 | Custom Resources

继承 `Resource` 并使用 `class_name`：

```gdscript
class_name Item
extends Resource

@export var item_name: String
@export var description: String
@export var icon: Texture2D
@export var price: int
```

创建一个 `Item.tres` 资源文件，填充数据，然后：

```gdscript
var sword: Item = preload("res://Items/Sword.tres")
print(sword.item_name)
```

## 4. ResourceLoader (底层 API) | ResourceLoader (Low-level API)

```gdscript
var res: Texture2D = ResourceLoader.load("res://icon.svg") as Texture2D

# 异步加载
ResourceLoader.load_threaded_request("res://BigResource.tres")

# 检查加载进度
var state: int = ResourceLoader.load_threaded_get_status("res://BigResource.tres")
```

## 5. 资源文件路径 | Resource Paths

| 前缀 | 说明 | 示例 |
| :--- | :--- | :--- |
| `res://` | 项目根目录 | `res://Player.tscn` |
| `user://` | 用户数据目录 | `user://savegame.sav` |

---

## 6. 导入设置 | Import Settings

在文件系统面板选择资源，在"导入"选项卡可以调整导入设置：
- 纹理压缩
- 音频采样率
- 模型缩放
- 等等...

---

## 7. 小结 | Summary

1. `preload` 用于常量路径，`load` 用于变量路径
2. 使用 `class_name X extends Resource` 创建自定义资源
3. 自定义资源非常适合做游戏数据（物品、技能等）
4. `res://` 是项目根，`user://` 是用户数据目录
