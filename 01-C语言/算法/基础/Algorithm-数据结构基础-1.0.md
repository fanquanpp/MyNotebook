# 00-数据结构基础（Data Structures Basics）

数据结构是算法的基石。本笔记涵盖了最常用的线性与非线性数据结构及其 C 语言实现。

## 1. 核心概念

- **数据结构定义**：计算机中组织和存储数据的方式，决定了数据的访问效率和操作便利性。
- **复杂度分析**：
    - **时间复杂度**：算法运行所需的时间（通常用大 O 表示法）。
    - **空间复杂度**：算法运行所需的内存空间。

## 2. 线性结构

### 2.1 数组（Array）
- **定义**：连续内存空间存储相同类型元素。
- **特点**：随机访问快 $O(1)$，插入删除慢 $O(n)$。

### 2.2 链表（Linked List）
- **定义**：通过指针连接的非连续内存块。
- **特点**：动态大小，插入删除快 $O(1)$（已知位置），访问慢 $O(n)$。

### 2.3 栈（Stack）
- **定义**：后进先出（LIFO）的线性表。
- **操作**：Push, Pop, Peek。
- **复杂度**：所有操作均为 $O(1)$。

### 2.4 队列（Queue）
- **定义**：先进先出（FIFO）的线性表。
- **操作**：Enqueue, Dequeue。
- **复杂度**：所有操作均为 $O(1)$。

## 3. 非线性结构

### 3.1 树（Tree）
- **二叉树**：每个节点最多两个子节点。
- **遍历方式**：前序、中序、后序、层序。
- **平衡树（AVL/红黑树）**：确保查找效率在 $O(\log n)$。

### 3.2 图（Graph）
- **表示法**：邻接矩阵、邻接表。
- **遍历**：DFS（深度优先）、BFS（广度优先）。

### 3.3 哈希表（Hash Table）
- **原理**：通过哈希函数将 Key 映射到数组索引。
- **冲突处理**：链地址法、开放寻址法。
- **复杂度**：平均查找时间为 $O(1)$。

## 4. 实战代码（C 语言摘要）

## 5. 最佳实践

- **空间换时间**：如哈希表。
- **选择合适结构**：频繁插入删除选链表，频繁随机访问选数组。
