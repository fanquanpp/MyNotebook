## 2 . 状态管理

状态管理是 Git 最基本的操作之一，用于查看当前仓库的状态：

```bash
# 查看当前仓库状态
git status

# 查看详细状态（包括忽略的文件）
git status --ignored

# 以短格式查看状态
git status -s  # 或 git status --short
```

状态输出说明：
- `A`：新添加到暂存区的文件
- `M`：修改过的文件
- `D`：删除的文件
- `??`：未跟踪的文件
- `!!`：被忽略的文件

## 3 . 暂存与提交

<a id="3.1"></a>
### 3.1 暂存操作

暂存操作将工作区的修改添加到暂存区：

```bash
# 添加单个文件到暂存区
git add <文件名>

# 添加所有文件到暂存区
git add .

# 添加指定类型的文件
git add *.md

# 添加目录到暂存区
git add <目录名>/

# 从暂存区移除文件
git reset HEAD <文件名>

# 从暂存区中移除文件但保留工作区文件
git rm --cached <文件名>
```

<a id="3.2"></a>
### 3.2 提交操作

提交操作将暂存区的内容提交到本地仓库：

```bash
# 提交到本地仓库
git commit -m "提交说明"

# 提交时自动添加修改的文件
git commit -am "提交说明"

# 修改最近一次提交信息
git commit --amend -m "新的提交说明"

# 向最近一次提交添加更多修改
git add <文件名>
git commit --amend --no-edit
```

<a id="4"></a>
## 4 . 查看历史记录

查看提交历史是了解项目变更的重要方式：

```bash
# 查看提交历史
git log

# 查看简洁的提交历史
git log --oneline

# 查看指定数量的提交历史
git log -n 5

# 查看提交历史并显示每次提交的变更内容
git log -p

# 查看指定文件的提交历史
git log --oneline <文件名>

# 查看指定作者的提交历史
git log --author="作者名"

# 查看指定时间段的提交历史
git log --since="2026-04-01" --until="2026-04-30"

# 查看分支合并历史
git log --graph --oneline
```

<a id="5"></a>
## 5 . 差异比较

差异比较用于查看文件的变化：

```bash
# 查看工作区与暂存区的差异
git diff

# 查看暂存区与本地仓库的差异
git diff --cached

# 查看工作区与指定提交的差异
git diff <提交哈希>

# 查看两次提交之间的差异
git diff <提交 1> <提交 2>

# 查看指定文件在两次提交之间的差异
git diff <提交 1> <提交 2> <文件名>
```

## 6 . 撤销操作

<a id="6.1"></a>
### 6.1 工作区撤销

撤销工作区中的修改：

```bash
# 撤销工作区中单个文件的修改
git checkout -- <文件名>

# 撤销工作区中所有文件的修改
git checkout .
```

<a id="6.2"></a>
### 6.2 暂存区撤销

撤销暂存区中的修改：

```bash
# 撤销暂存区中单个文件的修改
git reset HEAD <文件名>

# 撤销暂存区中所有文件的修改
git reset HEAD .
```

<a id="6.3"></a>
### 6.3 提交撤销

撤销已经提交的修改：

```bash
# 撤销最近一次提交（保留修改在暂存区）
git reset --soft HEAD~1

# 撤销最近一次提交（保留修改在工作区）
git reset --mixed HEAD~1  # 或 git reset HEAD~1

# 撤销最近一次提交（丢弃所有修改）
git reset --hard HEAD~1

# 撤销指定提交（保留之后的修改）
git reset --soft <提交哈希>

# 撤销指定提交（丢弃所有修改）
git reset --hard <提交哈希>

# 撤销指定提交（创建新的提交）
git revert <提交哈希>
```

<a id="7"></a>
## 7 . 实用技巧

<a id="7.1"></a>
### 7.1 忽略文件

创建 `.gitignore` 文件，添加需要忽略的文件或目录：

```gitignore
# 忽略 node_modules 目录
node_modules/

# 忽略日志文件
*.log

# 忽略编译产物
dist/
build/

# 忽略环境变量文件
.env
.env.local

# 忽略 IDE 配置文件
.vscode/
.idea/

# 忽略操作系统文件
.DS_Store
Thumbs.db

# 忽略临时文件
*.tmp
*.temp

# 忽略测试覆盖率文件
coverage/

# 忽略打包文件
*.zip
*.tar.gz
```

<a id="7.2"></a>
### 7.2 别名设置

设置常用命令的别名，提高操作效率：

```bash
# 设置常用别名
git config --global alias.st status
git config --global alias.ci commit
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.unstage "reset HEAD --"
git config --global alias.last "log -1 --stat"
git config --global alias.logg "log --oneline --graph --all"
git config --global alias.df "diff"
git config --global alias.dfc "diff --cached"
git config --global alias.cp "cherry-pick"
```

## 8. 完整操作流程

<a id="81-新项目初始化与推送"></a>
### 8.1 新项目初始化与推送

```bash
# 1. 全局配置（仅第一次）
git config --global user.name "用户名称"
git config --global user.email "GitHub 邮箱地址"

# 2. 新建/进入项目文件夹并初始化
mkdir my-project
cd my-project

# 3. 初始化 Git 仓库
git init -b main

# 4. 创建 .gitignore 文件
cat > .gitignore << EOF
# 操作系统文件
.DS_Store
Thumbs.db

# 编辑器文件
.vscode/
.idea/

# 编译产物
build/
dist/

# 依赖包
node_modules/

# 环境变量文件
.env
EOF

# 5. 创建初始文件
echo "# My Project" > README.md
echo "console.log('Hello, Git!');" > index.js

# 6. 添加并提交
git add .
git commit -m "chore: 初始化项目"

# 7. 关联远程仓库（替换为你的地址）
git remote add origin https://github.com/xxx/my-project.git

# 8. 首次推送（带 -u）
git push -u origin main
```

<a id="82-日常开发流程"></a>
### 8.2 日常开发流程

```bash
# --- 日常开发 ---
# 1. 拉取远程更新
git pull

# 2. 创建并切换到新分支
git checkout -b feature/login

# 3. 编写代码...

# 4. 查看修改状态
git status

# 5. 添加修改
git add .

# 6. 提交修改
git commit -m "feat(auth): 添加用户登录功能"

# 7. 再次拉取远程更新（防止冲突）
git pull origin main --rebase

# 8. 推送到远程仓库
git push origin feature/login

# 9. 合并到主分支（在 GitHub/GitLab 上创建 PR/MR）
# 10. 切换回主分支
git checkout main

# 11. 拉取最新代码
git pull

# 12. 删除本地分支
git branch -d feature/login
```

<a id="83-代码审查流程"></a>
### 8.3 代码审查流程

```bash
# 1. 从主分支创建特性分支
git checkout -b feature/new-feature

# 2. 进行开发和提交
git add .
git commit -m "feat: 添加新功能"

# 3. 推送到远程
git push origin feature/new-feature

# 4. 在 GitHub/GitLab 上创建 PR/MR
# 5. 进行代码审查
# 6. 根据反馈进行修改
git add .
git commit -m "fix: 修复代码审查反馈"
git push origin feature/new-feature

# 7. 审查通过后，合并到主分支
# 8. 切换回主分支并拉取最新代码
git checkout main
git pull

# 9. 删除本地特性分支
git branch -d feature/new-feature

# 10. 删除远程特性分支
git push origin --delete feature/new-feature
```

<a id="9-最佳实践"></a>
## 9. 最佳实践

### 9.1 提交最佳实践

- **提交频率**：频繁提交，每次提交只包含相关的修改
- **提交信息**：使用清晰、描述性的提交信息，遵循提交规范
- **提交大小**：每次提交的修改不宜过大，便于代码审查和回滚
- **提交顺序**：先提交核心功能，再提交辅助文件和测试

### 9.2 状态管理最佳实践

- **定期查看状态**：使用 `git status` 定期查看仓库状态
- **合理使用暂存区**：使用暂存区组织提交内容，确保提交的原子性
- **避免提交无关文件**：使用 `.gitignore` 忽略不需要版本控制的文件

### 9.3 撤销操作最佳实践

- **谨慎使用 `git reset --hard`**：此命令会永久删除修改，使用前确保备份
- **推送前检查**：在推送到远程前，仔细检查提交内容
- **使用 `git revert` 撤销已推送的提交**：避免使用 `git reset` 撤销已推送的提交

### 9.4 历史记录最佳实践

- **保持提交历史清晰**：避免不必要的合并提交，使用 `git rebase` 整理提交历史
- **使用标签标记重要版本**：如发布版本、里程碑等
- **定期清理分支**：删除已合并的分支，保持仓库整洁

<a id="10-实际应用示例"></a>
## 10. 实际应用示例

### 10.1 示例 1：修复 bug

```bash
# 1. 切换到主分支
git checkout main

# 2. 拉取最新代码
git pull

# 3. 创建 bug 修复分支
git checkout -b fix/login-error

# 4. 修复 bug...

# 5. 提交修复
git add .
git commit -m "fix(auth): 修复登录失败问题"

# 6. 推送到远程
git push origin fix/login-error

# 7. 创建 PR/MR 进行代码审查
# 8. 合并到主分支后，删除本地分支
git checkout main
git pull
git branch -d fix/login-error
```

### 10.2 示例 2：合并多个提交

```bash
# 1. 查看最近的提交
git log --oneline -5

# 2. 合并最近 3 个提交
git rebase -i HEAD~3

# 在打开的编辑器中，将需要合并的提交的 pick 改为 squash
# 保存并退出编辑器

# 3. 编辑合并后的提交信息
# 保存并退出编辑器

# 4. 推送到远程（需要强制推送）
git push -f origin feature/branch
```

### 10.3 示例 3：使用 stash 暂存修改

```bash
# 1. 在当前分支有未提交的修改
git status

# 2. 暂存修改
git stash save "暂存当前修改"

# 3. 切换到其他分支
git checkout main

# 4. 处理其他任务...

# 5. 切换回原分支
git checkout feature/branch

# 6. 恢复暂存的修改
git stash apply

# 7. 查看暂存列表
git stash list

# 8. 删除暂存
git stash drop
```

### 10.4 示例 4：查找特定提交

```bash
# 1. 查找包含特定关键词的提交
git log --grep="登录功能"

# 2. 查找修改特定文件的提交
git log --oneline -- <文件名>

# 3. 查找特定作者的提交
git log --author="John"

# 4. 查找特定时间段的提交
git log --since="2026-04-01" --until="2026-04-30"
```

<a id="11-总结"></a>
## 11. 总结

Git 的基本操作是使用 Git 的核心，掌握这些操作可以有效地管理代码的版本和变更。

- **工作区、暂存区、本地仓库**：理解 Git 的三个核心概念
- **状态管理**：使用 `git status` 查看仓库状态
- **暂存与提交**：使用 `git add` 和 `git commit` 管理代码变更，遵循提交规范
- **查看历史**：使用 `git log` 查看提交历史，了解项目演进
- **差异比较**：使用 `git diff` 查看文件变化，确保代码质量
- **撤销操作**：使用 `git checkout`、`git reset` 和 `git revert` 撤销修改，根据场景选择合适的方法
- **实用技巧**：使用 `.gitignore` 忽略文件，设置别名提高效率，掌握其他实用命令
- **完整流程**：遵循标准化的开发流程，包括项目初始化、日常开发和代码审查
- **最佳实践**：保持提交历史清晰，合理使用 Git 功能，避免常见错误

通过熟练掌握这些基本操作，可以为后续的分支管理、远程操作等高级功能打下基础，提高开发效率和代码质量。
