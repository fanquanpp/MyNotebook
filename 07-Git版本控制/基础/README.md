## 4. 核心知识点 | Core Knowledge Points

### 4.1 Git 概述

- Git 的发展历史和特点
- Git 的三棵树模型（工作区、暂存区、版本库）
- Git 的常用术语（提交、分支、合并等）
- Git 的最小工作流
- Git 与其他版本控制系统的对比

### 4.2 环境配置与初始化

- Git 的安装和配置
- Git 全局配置和局部配置
- 仓库的初始化
- .gitignore 文件的配置
- Git 基础排错

### 4.3 基本操作

- add 命令（将文件添加到暂存区）
- commit 命令（将暂存区内容提交到版本库）
- log 命令（查看提交历史）
- diff 命令（查看文件差异）
- 撤销操作（checkout、reset）
- 回退操作（reset --hard）
- 提交信息规范（Conventional Commits）

### 4.4 分支管理

- 分支的概念和作用
- 分支的创建和切换
- 分支的合并（merge）
- 分支的变基（rebase）
- 冲突的解决
- 分支策略（Git Flow、GitHub Flow 等）
- 分支管理的最佳实践

### 4.5 远程仓库操作

- 远程仓库的概念
- clone 命令（克隆远程仓库）
- fetch 命令（获取远程更新）
- pull 命令（获取并合并远程更新）
- push 命令（推送本地更新到远程）
- 远程分支的管理
- 团队协作流程
- 远程仓库的权限管理

## 5. 学习建议 | Learning Suggestions

1. **循序渐进**：按照学习路线的顺序学习，从 Git 概述开始，逐步掌握 Git 的各种操作
2. **实践为主**：多进行实际操作，通过实际项目练习加深对 Git 概念的理解
3. **重点关注**：特别关注分支管理和远程协作，这是 Git 的核心功能
4. **查阅文档**：遇到问题时，参考 Git 官方文档和相关资源
5. **代码规范**：遵循 Git 提交规范，养成良好的版本控制习惯
6. **团队协作**：参与团队项目，实践 Git 的协作功能
7. **工具使用**：学习使用 Git 图形界面工具，提高工作效率
