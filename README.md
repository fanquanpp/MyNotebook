## 规范文档

### 1. 文件命名规范

#### 1.1 文件名格式
采用以下命名格式：

```
[数字].[数字]-[知识点分类]-[核心主题].md
```

#### 1.2 前缀使用规则
- 第一个数字：表示大类别
  - 0：基础/入门
  - 1：进阶/详细
  - 2：核心/最佳实践
- 第二个数字：表示小类别（在大类别内的序号）

#### 1.3 关键词选择标准
- 准确反映内容主题
- 使用简洁明了的词汇
- 避免模糊或重复的描述
- 优先使用技术领域的标准术语

#### 1.4 命名一致性要求
- 同一技术领域的文件命名风格保持一致
- 避免使用不同的术语描述相同的概念
- 文件名中的单词首字母大写（除了冠词和介词）

### 2. 文件创建规范

#### 2.1 标准内容结构模板
每个笔记文件应包含以下模块：

1. **标题**：清晰的文件标题
2. **目录**：自动生成的内容目录
3. **引言**：简要介绍本知识点的重要性和应用场景
4. **主体内容**：详细的知识点讲解
5. **总结**：对知识点的概括和重点回顾
6. **参考资料**：引用的文献或资源

#### 2.2 标题层级规范
严格遵循Markdown标题层级：
- # 一级标题（文件主标题）
- ## 二级标题（主要章节）
- ### 三级标题（子章节）
- #### 四级标题（详细说明）

#### 2.3 必备模块要求
- 每个文件必须包含标题和主体内容
- 建议包含目录、引言、总结和参考资料
- 代码示例必须使用Markdown代码块并指定语言类型

### 3. 内容编写规范

#### 3.1 语言风格
- 采用简洁专业的技术文档风格
- 使用准确的技术术语
- 避免使用口语化或模糊的表达
- 保持语言的一致性和连贯性

#### 3.2 代码示例格式
- 使用Markdown代码块并指定语言类型
- 为代码添加必要的注释
- 提供完整的可运行示例
- 代码示例应具有代表性和实用性

#### 3.3 引用标注方法
- 引用来源应明确标注
- 采用以下格式标注引用：
  ```
  [引用编号]: 引用来源链接 "引用来源标题"
  ```
- 在正文中使用 `[引用编号]` 引用

### 4. 目录组织原则

#### 4.1 现有目录结构
- 按技术领域划分为不同的文件夹
- 每个文件夹对应一个技术类别
- 所有文件直接存放在对应技术分类文件夹下
- 严格禁止创建新的子文件夹

#### 4.2 各技术分类文件夹的内容范围
- **CSS3**：CSS3相关的语法、布局技术、样式技巧等
- **Git**：Git版本控制的命令、操作、最佳实践等
- **HTML5**：HTML5的标签、语义、API等
- **Java**：Java语言的语法、数据结构、核心概念等
- **Javascript**：JavaScript的语法、DOM操作、框架等
- **Markdown**：Markdown的语法、格式、最佳实践等
- **Mysql**：MySQL数据库的安装、配置、SQL语法等

## 如何使用

1. **按技术领域浏览**：根据需要选择对应的技术文件夹
2. **按知识点查找**：通过README.md中的表格快速定位所需知识点
3. **阅读笔记内容**：使用支持Markdown的编辑器打开笔记文件
4. **参考规范文档**：遵循项目规范进行内容更新和维护
