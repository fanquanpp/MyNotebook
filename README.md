## 规范文档

### 1. 文件命名规范

#### 1.1 文件名格式

采用以下命名格式：

```
[数字].[数字]-[知识点分类]-[核心主题].md
```

#### 1.2 前缀使用规则

| 前缀位置 | 含义 | 取值范围及说明 |
|---------|------|---------------|
| 第一个数字 | 大类别 | 0：基础/入门<br>1：进阶/详细<br>2：核心/最佳实践 |
| 第二个数字 | 小类别 | 在大类别内的序号 |

#### 1.3 关键词选择标准

| 标准  | 说明            |
| --- | ------------- |
| 准确性 | 准确反映内容主题      |
| 简洁性 | 使用简洁明了的词汇     |
| 唯一性 | 避免模糊或重复的描述    |
| 规范性 | 优先使用技术领域的标准术语 |

#### 1.4 命名一致性要求

| 要求   | 说明                    |
| ---- | --------------------- |
| 风格一致 | 同一技术领域的文件命名风格保持一致     |
| 术语统一 | 避免使用不同的术语描述相同的概念      |
| 格式规范 | 文件名中的单词首字母大写（除了冠词和介词） |

### 2. 文件创建规范

#### 2.1 标准内容结构模板

| 模块   | 说明                |
| ---- | ----------------- |
| 标题   | 清晰的文件标题           |
| 目录   | 自动生成的内容目录         |
| 引言   | 简要介绍本知识点的重要性和应用场景 |
| 主体内容 | 详细的知识点讲解          |
| 总结   | 对知识点的概括和重点回顾      |
| 参考资料 | 引用的文献或资源          |

#### 2.2 标题层级规范

| 层级   | 格式   | 说明    |
| ---- | ---- | ----- |
| 一级标题 | #    | 文件主标题 |
| 二级标题 | ##   | 主要章节  |
| 三级标题 | ###  | 子章节   |
| 四级标题 | #### | 详细说明  |

#### 2.3 必备模块要求

| 要求   | 说明                         |
| ---- | -------------------------- |
| 必须包含 | 每个文件必须包含标题和主体内容            |
| 建议包含 | 目录、引言、总结和参考资料              |
| 代码格式 | 代码示例必须使用Markdown代码块并指定语言类型 |

### 3. 内容编写规范

#### 3.1 语言风格

| 要求  | 说明            |
| --- | ------------- |
| 风格  | 采用简洁专业的技术文档风格 |
| 术语  | 使用准确的技术术语     |
| 表达  | 避免使用口语化或模糊的表达 |
| 一致性 | 保持语言的一致性和连贯性  |

#### 3.2 代码示例格式

| 要求  | 说明                   |
| --- | -------------------- |
| 格式  | 使用Markdown代码块并指定语言类型 |
| 注释  | 为代码添加必要的注释           |
| 完整性 | 提供完整的可运行示例           |
| 代表性 | 代码示例应具有代表性和实用性       |

#### 3.3 引用标注方法

| 要求 | 说明                                   |
| -- | ------------------------------------ |
| 标注 | 引用来源应明确标注                            |
| 格式 | 采用以下格式标注引用：`[引用编号]: 引用来源链接 "引用来源标题"` |
| 使用 | 在正文中使用 `[引用编号]` 引用                   |

### 4. 目录组织原则

#### 4.1 现有目录结构

| 原则   | 说明                  |
| ---- | ------------------- |
| 分类方式 | 按技术领域划分为不同的文件夹      |
| 对应关系 | 每个文件夹对应一个技术类别       |
| 存放位置 | 所有文件直接存放在对应技术分类文件夹下 |
| 结构限制 | 严格禁止创建新的子文件夹        |

#### 4.2 各技术分类文件夹的内容范围

| 文件夹        | 内容范围                    |

| CSS3       | CSS3相关的语法、布局技术、样式技巧等    |
| Git        | Git版本控制的命令、操作、最佳实践等     |

| Javascript | JavaScript的语法、DOM操作、框架等 |

## 如何使用

1. **按技术领域浏览**：根据需要选择对应的技术文件夹
2. **按知识点查找**：通过README.md中的表格快速定位所需知识点
3. **阅读笔记内容**：使用支持Markdown的编辑器打开笔记文件
4. **参考规范文档**：遵循项目规范进行内容更新和维护
