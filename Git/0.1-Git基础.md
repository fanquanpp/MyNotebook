## 1. 什么是 Git

Git 是一个分布式版本控制系统，用于跟踪项目中文件的变化。它可以帮助你：

- 记录代码的历史变更
- 协作开发时管理不同版本
- 回滚到之前的版本
- 分支管理，实现并行开发
- 远程仓库同步，方便代码共享

Git 的核心概念包括：
- **仓库（Repository）**：存储项目代码和历史记录的地方
- **提交（Commit）**：代码的快照，包含修改的内容和提交信息
- **分支（Branch）**：独立的开发线路，用于功能开发或 bug 修复
- **远程仓库（Remote）**：存储在网络或其他位置的仓库副本

## 2. 首次使用 Git：全局配置

在任何 Git 操作前，需要先告诉 Git 你是谁，这将用于记录提交者信息。

### 2.1 配置步骤

```bash
# 1. 设置用户名（替换为你的名字）
git config --global user.name "你的用户名（可用中文，或者与 GitHub 相同）"

# 2. 设置邮箱（替换为你的邮箱）
git config --global user.email "你的邮箱地址（与 GitHub 相同）"

# 3. (可选但推荐) 解决 Windows 下中文文件名乱码问题
git config --global core.quotepath false
```

### 2.2 配置验证

配置完成后，你可以通过以下命令验证配置是否成功：

```bash
# 查看所有配置
git config --list

# 查看特定配置
git config user.name
git config user.email
```

## 3. 本地仓库基础操作

### 3.1 初始化仓库

初始化一个新的 Git 仓库：

```bash
# 进入项目目录
cd <项目目录>

# 初始化 Git 仓库
git init
```

执行后，会在当前目录创建一个 `.git` 文件夹，用于存储 Git 相关的信息。

### 3.2 克隆远程仓库

从远程服务器克隆一个已有的仓库：

```bash
# 克隆默认分支
git clone <仓库地址>

# 克隆指定分支
git clone -b <分支名> <仓库地址>
```

### 3.3 提交代码到仓库

基本提交流程：

```bash
# 1. 查看当前状态
git status

# 2. 将所有修改添加到暂存区
git add .

# 3. 将暂存区的内容提交到本地仓库（引号内写本次提交的说明）
git commit -m "描述你做了什么修改"
```

### 3.4 基本操作流程

日常开发中的基本操作流程：

```bash
# 1. 添加修改到暂存区
git add .

# 2. 提交到本地仓库
git commit -m "描述"

# 3. 推送到远程仓库
git push
```

## 4. 关联远程仓库与推送

### 4.1 首次推送（建立关联）

如果是**第一次**把本地代码推送到远程仓库，需要先关联并设置“上游分支”。

```bash
# 1. (如果还没关联远程仓库) 关联远程仓库（替换为你的仓库地址）
git remote add origin <你的远程仓库URL>

# 2. 首次推送到远程 main 分支，并关联上游（关键：-u 参数）
# 作用：以后直接 git push 即可，不用再写分支名
git push -u origin main
```

> **注意**：旧版默认分支常用 `master`，如果默认分支是 `master`，就把 `main` 换成 `master`。现在基本都是 `main`。

### 4.2 后续推送（日常提交）

首次推送成功后，**以后的日常推送**就不需要 `-u` 参数。

```bash
# 1. 拉取远程最新代码（防止冲突，建议每次 push 前先 pull）
git pull

# 2. 直接推送到远程（因为首次已经用 -u 关联过了）
git push
```

### 4.3 常见推送问题及解决方案

1. **fatal: No configured push destination**
   - 原因：未关联远程仓库
   - 解决方案：先执行 `git remote add origin <仓库地址>` 关联远程仓库

2. **error: failed to push some refs to**
   - 原因：远程仓库有本地没有的提交
   - 解决方案：先执行 `git pull` 拉取远程代码，解决冲突后再推送

3. **fatal: refusing to merge unrelated histories**
   - 原因：本地仓库和远程仓库历史不相关
   - 解决方案：执行 `git pull --allow-unrelated-histories` 允许合并不相关历史

## 5. 常用查看与辅助命令

```bash
# 1. 查看提交历史（按 q 退出）
git log

# 2. 查看当前分支
git branch

# 3. 查看远程仓库信息
git remote -v

# 4. 查看工作区和暂存区的差异
git diff

# 5. 查看暂存区和本地仓库的差异
git diff --cached
```

## 6. 完整流程示例

### 6.1 新项目初始化与推送

```bash
# 1. 全局配置（仅第一次）

# 2. 新建/进入项目文件夹并初始化

git push -u origin main
```

### 7.2 完整操作流程总结

```bash
# 初始化本地仓库
git init

# 添加文件到暂存区
git add .

# 提交更改
git commit -m "Initial commit"

# 添加远程仓库
git remote add origin <远程仓库地址>

# 拉取远程更改（如果远程仓库已有内容）
git pull origin main --allow-unrelated-histories

# 推送到远程仓库
git push -u origin main
```

## 8. 关键总结

- **`git push -u origin main`**：仅用于**第一次**推送，目的是“绑定”本地和远程分支。
- **`git push`**：绑定之后，所有后续推送都使用这个命令。
- **好习惯**：推送前先 `git pull`，避免代码冲突。
- **提交信息**：提交信息应该清晰明了，描述本次修改的内容。
- **分支管理**：合理使用分支可以更好地管理代码，避免直接在主分支上开发。

## 9. 分支管理基础

分支是 Git 中非常重要的概念，它允许你在独立的环境中开发新功能或修复 bug。

### 9.1 分支操作基础

```bash
# 查看本地分支
git branch

# 创建新分支
git branch <分支名>

# 切换分支
git checkout <分支名>
# 或 Git 2.23+ 推荐
git switch <分支名>

# 创建并切换分支
git checkout -b <分支名>
# 或 Git 2.23+ 推荐
git switch -c <分支名>

# 合并分支到当前分支
git merge <分支名>

# 删除分支（已合并）
git branch -d <分支名>

# 强制删除分支（未合并）
git branch -D <分支名>
```

### 9.2 分支命名规范

- **feature/功能名**：用于开发新功能
- **bugfix/问题描述**：用于修复 bug
- **hotfix/紧急修复**：用于紧急修复生产环境问题
- **release/版本号**：用于发布版本

## 9. 常见问题与解决方案

### 9.1 推送失败问题

**问题描述**：执行 `git push` 时出现错误。

**常见原因**：
- 未关联远程仓库
- 远程仓库有本地没有的提交
- 本地与远程历史不相关

**解决方案**：
- 关联远程仓库：`git remote add origin <仓库地址>`
- 拉取远程代码：`git pull`
- 允许合并不相关历史：`git pull --allow-unrelated-histories`

### 9.2 合并冲突问题

**问题描述**：执行 `git merge` 时出现冲突。

**解决方案**：
1. 查看冲突文件：`git diff`
2. 手动编辑冲突文件，解决冲突
3. 添加解决后的文件：`git add .`
4. 完成合并：`git commit`

### 9.3 误操作恢复

**问题描述**：误删除文件或提交错误内容。

**解决方案**：
- 恢复工作区文件：`git checkout -- <文件名>`
- 撤销暂存：`git reset HEAD <文件名>`
- 撤销提交：`git reset --soft HEAD~1`

### 9.4 仓库清理问题

**问题描述**：仓库中有很多未跟踪文件。

**解决方案**：
- 清理未跟踪文件：`git clean -f`
- 清理未跟踪文件和目录：`git clean -fd`

## 10. 总结与最佳实践

### 10.1 核心概念

- **仓库**：存储代码和历史记录的地方
- **提交**：代码的快照，包含修改内容和提交信息
- **分支**：独立的开发线路
- **远程仓库**：网络上的仓库副本

### 10.2 最佳实践

1. **配置规范**
   - 设置清晰的用户名和邮箱
   - 配置合适的编辑器和差异工具

2. **提交规范**
   - 定期提交，保持提交粒度适中
   - 提交信息清晰明了，描述具体修改内容
   - 使用语义化提交信息

3. **分支规范**
   - 主分支保持稳定
   - 使用功能分支开发新特性
   - 定期合并主分支到功能分支

4. **远程操作规范**
   - 推送前先拉取，避免冲突
   - 合理使用 `git fetch` 和 `git pull`
   - 定期备份远程仓库

5. **日常操作流程**
   - 开始工作：`git pull`
   - 开发过程：`git add` 和 `git commit`
   - 完成工作：`git pull` 和 `git push`
