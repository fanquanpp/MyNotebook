# 模板：技术栈笔记结构

## 0. 文件头（所有 `.md` 建议统一）

在正文最上方使用 HTML 注释块（不渲染到 HTML 预览中）：

```html
<!--
文件名称：<中文名称>
原名称：<若无则写「无」>
用途描述：<问题、读者、前置知识、关联文件>
创建/更新：<日期>
-->
```

## 1. 核心概念
- **概念定义**：技术的基本定义和作用
- **历史演进**：技术的发展历程
- **应用场景**：主要使用场景和适用范围
- **与其他技术的关系**：与相关技术的对比和联系

## 2. 语法基础
- **基本语法**：核心语法结构和规则
- **数据类型**：支持的数据类型及其特性
- **流程控制**：条件、循环等控制结构
- **函数/方法**：函数定义和调用方式
- **类与对象**：面向对象编程相关内容

## 3. 实战应用
- **常用代码片段**：高频使用的代码示例
- **算法实现**：经典算法的实现
- **设计模式**：常见设计模式的应用
- **项目案例**：实际项目中的应用案例

## 4. 最佳实践
- **编码规范**：推荐的代码风格和规范
- **性能优化**：性能提升的方法和技巧
- **调试技巧**：常见问题的调试方法
- **安全注意事项**：安全编程的最佳实践

## 5. 延伸阅读
- **官方文档**：官方文档链接
- **推荐资源**：高质量的学习资源
- **相关工具**：常用工具和库

## 6. 更新日志
- **日期**：修改日期
- **作者**：修改作者
- **变更摘要**：主要变更内容
- **Commit ID**：相关的版本控制提交ID
