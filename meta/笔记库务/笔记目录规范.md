# 笔记目录与命名规范

## 1. 一级目录（领域模块）

形式：**`序号-技术领域`**，序号用于在资源管理器与 `SUMMARY.md` 中稳定排序；名称在「可检索」与「长度适中」之间折中，**不建议**在未批量更新链接的前提下随意缩短。

| 路径 | 英文（中文） | 说明 |
| :--- | :--- | :--- |
| `笔记库务/` | Meta（元数据） | 维护总览、术语、模板、本规范 |
| `00-Github完整教程/` | GitHub（Git 托管平台） | 平台操作与协作，与 `Git` 命令行笔记互补 |
| `01-C语言/` | C（C 语言） | 系统与底层基础 |
| `02-CSS布局/` | CSS（层叠样式表）布局 | 样式与版面 |

## 2. 统一组织方式（推荐）

多数技术栈采用下列**逻辑层级**（具体文件名以各目录实际为准）：

| 层级 | 含义 | 典型编号 |
| :--- | :--- | :--- |
| 入门 | 环境、工具链、最小示例 | `00-` |
| 核心 | 语法、概念、API | `01-` 及专题编号 |
| 实战 | 片段、案例、项目 | `02-`、`04-` |
| 调试与优化 | 排错、性能、工具 | `03-` |

**例外**：专题拆分（如 `05-HTML5` 下按「标签 / 表单 / Canvas」分文件）、`11-Renpy视觉小说` 下按 `核心|语法|视觉|引擎|工作流` 分栏，在对应 `README.md` 中单独说明。

## 3. 术语与文件头

- 正文首次出现的英文专有名词，建议采用 **English（中文）** 格式，例如：`PR（Pull Request，拉取请求）`。
- 新文档建议使用 [`笔记模板.md`](笔记模板.md) 顶部 HTML 注释头。

## 4. 链接与格式

- 内部链接使用相对路径，修改目录后需更新 `SUMMARY.md` 与相关 `README.md`。
- 根目录工具配置说明见 [`仓库维护总览.md`](仓库维护总览.md) 第 7 节。
