## 4. 质量门禁与 CI / 本地命令

| 门禁项 | 实现方式 | 说明 |
| :--- | :--- | :--- |
| Markdown 行长 | 根目录 `.editorconfig` 不限制 `*.md` 行长 | 长表格与教程可扩展 |
| Markdown 风格 | `markdownlint-cli` + 根目录 `.markdownlint.json` | `verify.yml` 与本地脚本均使用 `-c` 指向该文件 |
| 内部链接 | `markdown-link-check` + `.markdown-link-check.json` | CI 中扫描；外链受网络影响可能偶发超时；主仓库主页可在配置中 `ignorePatterns` 跳过以避免偶发 **Status: 0** |
| 本地一键校验 | `bash scripts/validate.sh`（Git Bash / WSL） | 依赖本机安装 `markdownlint` 等与原文一致 |

**GitHub Actions 报告**：仓库 **Actions → Verify** 工作流最近一次运行日志。

**常用本地命令（在仓库根目录执行）**：

```bash
npx markdownlint-cli "**/*.md" -c .markdownlint.json --ignore node_modules --ignore .git
npx markdown-link-check SUMMARY.md -c .markdown-link-check.json
```

## 7. 根目录配置文件说明（为何不能「合成一个文件」）

以下文件**必须**留在仓库**根目录**（或约定路径），命令行工具才会自动读取；**不能**合并为单个 JSON——工具彼此不兼容。

| 文件 | 作用 |
| :--- | :--- |
| `.editorconfig` | 编辑器统一缩进、换行、`*.md` 不限行长 |
| `.markdownlint.json` | `markdownlint-cli` 规则 |
| `.markdown-link-check.json` | `markdown-link-check` 超时与 HTTP 状态 |
| `scripts/validate.sh` | 本地多语言语法与 Markdown 检查（先 `cd` 到根目录再执行） |

若需「一处阅读全部含义」，以**本节 + 各文件内注释**为准。
