Markdown基础

一、标题语法

Markdown支持6级标题，通过#的数量区分层级，#后必须加1个空格，否则语法不生效。

语法示例

# 一级标题（对应HTML h1，文档主标题）
## 二级标题（对应HTML h2，一级章节）
### 三级标题（对应HTML h3，二级章节）
#### 四级标题
##### 五级标题
###### 六级标题

渲染效果

二级标题

三级标题

四级标题

五级标题
六级标题

二、段落与换行

核心规则

  1. 段落分隔：两个段落之间必须用1个空行分隔，无空行的连续文本会被合并为同一段落
  2. 强制换行：行尾添加2个及以上空格后按回车，实现同一段落内的换行（全平台通用标准写法）

语法示例

这是第一个段落的第一行
这一行和上一行无空行、无行尾空格，会被合并为同一段落

这是第二个段落（上一行是空行，成功分段）
这一行末尾加了2个空格  
这一行成功换行，和上一行属于同一段落

四、列表语法

1. 无序列表

使用-/*/+ + 空格开头，三者效果完全一致，推荐统一使用-，嵌套列表需缩进4个空格/1个制表符。

语法示例

- 无序列表一级项1
- 无序列表一级项2
    - 无序列表二级嵌套项1
    - 无序列表二级嵌套项2
        - 无序列表三级嵌套项
- 无序列表一级项3

渲染效果

  • 无序列表一级项1
  • 无序列表一级项2
    • 无序列表二级嵌套项1
    • 无序列表二级嵌套项2
      • 无序列表三级嵌套项
  • 无序列表一级项3

2. 有序列表

使用数字 + 英文句号 + 空格开头，数字顺序不影响最终渲染结果，推荐按顺序书写，嵌套需缩进4个空格/1个制表符。

语法示例

1. 有序列表一级项1
2. 有序列表一级项2
    1. 有序列表二级嵌套项1
    2. 有序列表二级嵌套项2
3. 有序列表一级项3

渲染效果

  1. 有序列表一级项1
  2. 有序列表一级项2
    1. 有序列表二级嵌套项1
    2. 有序列表二级嵌套项2
  3. 有序列表一级项3

3. 任务列表(GFM扩展)

待办清单专用语法，[ ]表示未完成，[x]表示已完成，符号后必须加空格。

语法示例

- [ ] 未完成的待办任务
- [x] 已完成的待办任务
- [ ] 可嵌套待办
    - [x] 子任务1
    - [ ] 子任务2

渲染效果

  • 未完成的待办任务
  • 已完成的待办任务
  • 可嵌套待办
    • 子任务1
    • 子任务2

七、链接语法

标准格式：[链接显示文本](链接地址 "链接可选标题")

语法示例

打开豆包

用<>包裹网址/邮箱，快速生成链接，适合直接展示完整地址。

十一、转义字符

使用反斜杠\转义Markdown特殊字符，使其正常显示，避免被解析为语法标记。

可转义的核心特殊字符

\   反斜杠
*   星号
_   下划线
#   井号
+   加号
-   减号
.   英文句号
!   感叹号
|   竖线
`   反引号
[ ] 方括号
( ) 圆括号

语法示例

\# 转义后正常显示#号，不会被解析为标题
\* 转义后正常显示*号，不会被解析为列表

十三、常见问题

  1. 语法标记后必须加空格：标题、列表、引用、任务列表等语法的标记符后，必须加1个空格，否则无法正常渲染
  2. 段落必须用空行分隔：无空行的连续文本会被合并为同一段落，无法实现分段
  3. 嵌套元素必须规范缩进：列表嵌套、引用嵌套必须缩进4个空格/1个制表符，否则会出现渲染异常
  4. 特殊字符必须转义：文本中包含Markdown专用特殊字符时，必须用\转义，避免被错误解析
  5. 跨平台优先用标准语法：不同平台对扩展语法支持度不同，跨平台写作优先使用通用标准语法
