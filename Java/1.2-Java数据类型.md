## 4 . 基本数据类型

<a id="4.1"></a>
### 4.1 整数类型

| 类型 | 字节数 | 取值范围 | 默认值 |
| :--- | :--- | :--- | :--- |
| byte | 1 | -128 ~ 127 | 0 |
| short | 2 | -32768 ~ 32767 | 0 |
| int | 4 | -2147483648 ~ 2147483647 | 0 |
| long | 8 | -9223372036854775808 ~ 9223372036854775807 | 0L |

**注意事项：**
- 定义 long 类型变量时，在数值后面加一个 L
- 整数默认类型为 int

**示例：**

```java
byte b = 100;
short s = 1000;
int i = 100000;
long l = 10000000000L; // 注意加 L
```

<a id="4.2"></a>
### 4.2 浮点类型

| 类型 | 字节数 | 取值范围 | 默认值 |
| :--- | :--- | :--- | :--- |
| float | 4 | -3.4028235E+38 ~ 3.4028235E+38 | 0.0f |
| double | 8 | -1.7976931348623157E+308 ~ 1.7976931348623157E+308 | 0.0d |

**注意事项：**
- 定义 float 类型变量时，在数值后面加一个 F
- 浮点数默认类型为 double

**示例：**

```java
float f = 3.14F; // 注意加 F
double d = 3.1415926;
```

## 6 . 类型转换

<a id="6.1"></a>
### 6.1 自动类型转换

**规则：** 小范围类型向大范围类型转换，系统自动完成。

**转换顺序：** byte → short → int → long → float → double

**示例：**

```java
byte b = 10;
int i = b; // 自动转换：byte → int

double d = i; // 自动转换：int → double

char c = 'A';
int ascii = c; // 自动转换：char → int，获取字符的 ASCII 码
```

<a id="6.2"></a>
### 6.2 强制类型转换

**规则：** 大范围类型向小范围类型转换，需要手动强制转换。

**语法：** `(目标类型) 表达式`

**示例：**

```java
int i = 100;
byte b = (byte) i; // 强制转换：int → byte

double d = 3.14;
int num = (int) d; // 强制转换：double → int，小数部分会被截断

int large = 1000;
byte small = (byte) large; // 强制转换：int → byte，可能导致数据溢出
```

<a id="6.3"></a>
### 6.3 类型转换注意事项

1. **数据溢出**：当大范围类型的值超出小范围类型的取值范围时，会发生数据溢出
   ```java
   int i = 130;
   byte b = (byte) i; // 结果为 -126，因为 byte 的范围是 -128~127
   ```

2. **精度丢失**：浮点数转换为整数时，小数部分会被截断
   ```java
   double d = 3.999;
   int i = (int) d; // 结果为 3，不是 4
   ```

3. **布尔类型**：布尔类型不能与其他类型相互转换
   ```java
   boolean flag = true;
   int i = (int) flag; // 编译错误
   ```

4. **字符串转换**：需要使用包装类的方法
   ```java
   String str = "123";
   int i = Integer.parseInt(str);
   double d = Double.parseDouble(str);
   ```

## 7 . 常见问题与解决方案

<a id="7.1"></a>
### 7.1 数据类型相关问题

**问题描述：** 定义 long 类型变量时出现编译错误。

**原因分析：** 整数默认类型为 int，当数值超过 int 范围时需要显式声明为 long。

**解决方案：** 在数值后面加 L 或 l，如`long num = 10000000000L;`

<a id="7.2"></a>
### 7.2 类型转换问题

**问题描述：** 强制类型转换导致数据丢失。

**原因分析：** 大范围类型向小范围类型转换时，超出小范围类型取值范围的数据会被截断。

**解决方案：**
- 确保转换后的数据在目标类型的取值范围内
- 使用合适的数据类型，避免不必要的类型转换

<a id="7.3"></a>
### 7.3 浮点精度问题

**问题描述：** 浮点数计算时出现精度误差。

**原因分析：** 浮点数在计算机中以二进制表示，无法精确表示某些十进制小数。

**解决方案：**
- 对于需要精确计算的场景，使用 BigDecimal 类
- 避免直接比较浮点数是否相等

## 8 . 总结与最佳实践

<a id="8.1"></a>
### 8.1 核心概念

- **基本数据类型**：byte, short, int, long, float, double, char, boolean
- **引用数据类型**：类, 接口, 数组
- **类型转换**：自动类型转换, 强制类型转换

<a id="8.2"></a>
### 8.2 最佳实践

1. **数据类型选择**
   - 根据实际需求选择合适的数据类型
   - 优先使用基本数据类型，减少内存占用
   - 对于大整数，使用 long 类型
   - 对于小数，根据精度要求选择 float 或 double

2. **类型转换**
   - 尽量使用自动类型转换，避免强制类型转换
   - 强制类型转换前检查数据范围，避免数据丢失
   - 注意浮点数到整数的转换会截断小数部分

3. **代码风格**
   - 变量声明时初始化
   - 一行只声明一个变量
   - 合理使用空格和缩进，提高代码可读性

<a id="8.3"></a>
### 8.3 个人实践总结
- 根据实际需求选择合适的数据类型
- 遵循命名规范，提高代码可读性
- 注意类型转换的安全性，避免数据丢失
- 正确处理浮点数精度问题
- 保持代码风格一致，使用适当的缩进和空格
