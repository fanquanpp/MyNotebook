## 1. 计算机的存储规则

### 1.1 文本文字
- **二进制**：由0与1组成，代码中以0b开头
- **十进制**：由0-9组成，无前缀
- **八进制**：由0-7组成，代码中以0开头
- **十六进制**：由0-9与a-f组成，代码中以0x开头

*以上特性为JDK开始存在的特性，JDK8开始支持*

**示例：**

```java
public class Demo {
    public static void main(String[] args) {
        int a = 0b1010; // 二进制
        int b = 010; // 八进制
        int c = 0x10; // 十六进制
        System.out.println(a);
        System.out.println(b);
        System.out.println(c);
    }
}
```

### 1.2 任意进制转换为十进制

**公式：** 系数*基数的权次幂（相加）

```
a*b^n + a*b^(n-1) + ... + a*b^1 + a*b^0
```

- **系数**：每一位上的数字
- **基数**：当前进制数
- **权**：从右往左依次递增（0、1、2、3、4、5…………）

**示例：**
- 101（二进制）-->5(十进制)
  ```
  1               0               1
（1*2^2）        (0*2^1)          （1*2^0）
  |               |               |
  4         +     0        +      1   =   5( 十进制)
  ```
- 101（八进制）-->65(十进制)
- abc（十六进制）-->2748(十进制)

### 1.3 二进制转十进制（8421快速转化法）

**原理：** 每一位二进制值的1都对应一个固定的值，将每一位1代表的固定值相加，即为十进制的值

```
二进制：1    1   1   1    1   1   1   1
对应值：128  64  32  16   8   4   2   1
十进制：（271）
```

**示例：**
- 二进制：1101-->(8、4、0、1)
- 十进制：13  （8+4+0+1=13）

### 1.4 十进制转换成其他禁止（除基取余法）

**原理：** 不断除以基数（几进制基数就是几）得到余数，直到商为0，再将余数倒序拼接即可

1. 除以基数，取余数
2. 将余数保存起来
3. 除以基数，取余数
4. 将余数保存起来
5. 重复3、4步骤，直到商为0
6. 将余数保存的数组倒序输出

**示例：**
- 10（十进制）-->1010（二进制）
  ```
  10/2 = 5   10%2 = 0
  5/2 = 2   5%2 = 1
  2/2 = 1   2%2 = 0
  1/2 = 0   1%2 = 1
  
  0   1   1   0
  ```
- 10（十进制）-->1001（八进制）
- 11（十进制）-->1011 (二进制)
- 7（十进制）--> 111（二进制）

### 1.5 图片（Image）

## 3. Java开发环境搭建

### 3.1 JDK的下载与安装

1. **下载JDK**
   - 访问[Oracle官网](https://www.oracle.com/java/technologies/downloads/)或[OpenJDK官网](https://openjdk.java.net/)
   - 选择适合你操作系统的JDK版本（推荐Java 8或更高版本）
   - 下载并安装JDK

2. **配置环境变量**
   - **Windows系统**：
     - 右键"此电脑" → "属性" → "高级系统设置" → "环境变量"
     - 在系统变量中添加`JAVA_HOME`，值为JDK安装目录
     - 在系统变量`Path`中添加`%JAVA_HOME%\bin`
   - **Mac/Linux系统**：
     - 编辑`~/.bash_profile`或`~/.bashrc`文件
     - 添加`export JAVA_HOME=/path/to/jdk`
     - 添加`export PATH=$JAVA_HOME/bin:$PATH`

3. **验证安装**
   - 打开命令行工具
   - 输入`java -version`，查看Java版本信息
   - 输入`javac -version`，查看Java编译器版本信息

### 3.2 开发工具选择

- **Eclipse**：开源的集成开发环境
- **IntelliJ IDEA**：功能强大的商业IDE，有社区版
- **VSCode**：轻量级编辑器，需要安装Java插件

## 4. 第一个Java程序

### 4.1 编写HelloWorld程序

1. **创建Java文件**
   - 创建一个名为`HelloWorld.java`的文件
   - 写入以下代码：

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

2. **编译程序**
   - 打开命令行工具
   - 进入文件所在目录
   - 执行命令：`javac HelloWorld.java`
   - 生成`HelloWorld.class`字节码文件

3. **运行程序**
   - 执行命令：`java HelloWorld`
   - 控制台输出：`Hello, World!`

### 4.2 代码解析

- **public class HelloWorld**：定义一个公共类，类名必须与文件名相同
- **public static void main(String[] args)**：主方法，是程序的入口点
- **System.out.println("Hello, World!")**：输出语句，打印字符串到控制台

## 5. 常见问题与解决方案

### 5.1 编译错误

**问题描述**：执行`javac`命令时出现编译错误。

**常见原因**：
- 代码语法错误
- 文件名与类名不一致
- JDK未正确安装或环境变量未配置

**解决方案**：
- 检查代码语法，确保没有拼写错误或语法问题
- 确保文件名与类名完全一致（包括大小写）
- 验证JDK安装和环境变量配置

### 5.2 运行错误

**问题描述**：执行`java`命令时出现运行错误。

**常见原因**：
- 类名错误
- 类路径问题
- 运行时异常

**解决方案**：
- 确保使用正确的类名（不包含`.class`后缀）
- 检查类路径设置
- 查看错误信息，定位并修复运行时异常

### 5.3 环境变量问题

**问题描述**：无法识别`java`或`javac`命令。

**原因分析**：环境变量未正确配置。

**解决方案**：
- 重新配置环境变量
- 确保`JAVA_HOME`指向正确的JDK目录
- 确保`Path`中包含`%JAVA_HOME%\bin`（Windows）或`$JAVA_HOME/bin`（Mac/Linux）

## 6. 总结与最佳实践

### 6.1 核心概念

- **Java**：一种面向对象的编程语言
- **JDK**：Java开发工具包，包含编译器和运行时环境
- **JVM**：Java虚拟机，负责执行Java字节码
- **字节码**：Java源代码编译后的中间代码

### 6.2 最佳实践

1. **代码规范**
   - 类名使用大驼峰命名法（如`HelloWorld`）
   - 方法名和变量名使用小驼峰命名法（如`printMessage`）
   - 常量使用全大写字母，单词间用下划线分隔（如`MAX_VALUE`）
   - 代码缩进使用4个空格或1个制表符

2. **开发环境**
   - 使用集成开发环境（IDE）提高开发效率
   - 定期更新JDK到最新稳定版本
   - 配置合理的开发环境变量

3. **学习方法**
   - 从基础语法开始，循序渐进
   - 多写代码，实践出真知
   - 阅读官方文档和优质教程
   - 参与开源项目，积累实战经验

4. **调试技巧**
   - 使用`System.out.println()`打印变量值

   - 阅读错误信息，定位问题所在
