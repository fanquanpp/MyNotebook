# TypeScript-专有名词注释查阅表 | TypeScript

## 1. 基础概念类

### 1.1 TypeScript

**名称**：TypeScript 语言（TypeScript）

**首次出现位置**：C11_101-概述与环境.md 第1章

**定义**：
TypeScript 是 JavaScript 的超集，由微软开发，于 2012 年首次发布。它在 JavaScript 的基础上增加了静态类型系统和其他高级特性，最终通过编译器转换为纯 JavaScript 代码运行。

### 1.2 静态类型系统

**名称**：静态类型系统（Static Type System）

**首次出现位置**：C11_101-概述与环境.md 第1.1章

**定义**：
静态类型系统是指在编译时进行类型检查的类型机制，TypeScript 通过静态类型系统可以在开发阶段发现潜在错误，如拼写错误、类型不匹配等。

### 1.3 tsconfig.json

**名称**：TypeScript 配置文件（TypeScript Configuration File）

**首次出现位置**：C11_101-概述与环境.md 第3章

**定义**：
tsconfig.json 是 TypeScript 项目的配置文件，用于指定编译选项和项目设置，包括目标 JavaScript 版本、模块系统、严格模式等配置。

### 1.4 编译时类型检查

**名称**：编译时类型检查（Compile-time Type Checking）

**首次出现位置**：C11_101-概述与环境.md 第1.1章

**定义**：
编译时类型检查是指在代码编译阶段进行的类型验证，能够在代码运行前发现类型相关的错误，提高代码可靠性。

### 1.5 超集

**名称**：超集（Superset）

**首次出现位置**：C11_101-概述与环境.md 第1.2章

**定义**：
超集是指一种语言包含另一种语言的所有特性。TypeScript 是 JavaScript 的超集，意味着所有 JavaScript 代码都是有效的 TypeScript 代码。

## 2. 类型系统类

### 2.1 原始类型

**名称**：原始类型（Primitive Types）

**首次出现位置**：C11_102-基础类型系统.md 第1.1章

**定义**：
原始类型是 TypeScript 中的基本数据类型，包括 boolean、number、string、symbol 和 bigint 五种类型。

### 2.2 元组

**名称**：元组类型（Tuple）

**首次出现位置**：C11_102-基础类型系统.md 第1.2.2章

**定义**：
元组是固定长度和类型的数组，每个位置的类型可以不同。例如 `[string, number]` 表示第一个元素是字符串，第二个是数字。

### 2.3 枚举

**名称**：枚举类型（Enum）

**首次出现位置**：C11_102-基础类型系统.md 第1.2.3章

**定义**：
枚举是一组命名的常量，默认从 0 开始递增。TypeScript 支持数字枚举和字符串枚举两种形式。

### 2.4 联合类型

**名称**：联合类型（Union Types）

**首次出现位置**：C11_102-基础类型系统.md 第3.1章

**定义**：
联合类型使用 `|` 符号，表示值可以是其中之一。例如 `string | number` 表示值可以是字符串或数字。

### 2.5 交叉类型

**名称**：交叉类型（Intersection Types）

**首次出现位置**：C11_102-基础类型系统.md 第3.2章

**定义**：
交叉类型使用 `&` 符号，表示值必须同时满足所有类型。例如 `Person & Serializable` 表示同时具有 Person 和 Serializable 的属性。

### 2.6 类型别名

**名称**：类型别名（Type Alias）

**首次出现位置**：C11_102-基础类型系统.md 第4章

**定义**：
类型别名使用 `type` 关键字为类型创建一个新名称，可以为原始类型、联合类型、元组等创建别名。

### 2.7 字面量类型

**名称**：字面量类型（Literal Types）

**首次出现位置**：C11_102-基础类型系统.md 第5章

**定义**：
字面量类型表示具体的值，而不是类型范围。例如 `"North" | "South"` 表示只能是这两个字符串值之一。

### 2.8 any 类型

**名称**：任意类型（Any Type）

**首次出现位置**：C11_102-基础类型系统.md 第2.1章

**定义**：
any 类型会绕过所有类型检查，任何值都可以赋值给 any 类型。使用时需谨慎，因为它会失去类型安全保障。

### 2.9 unknown 类型

**名称**：未知类型（Unknown Type）

**首次出现位置**：C11_102-基础类型系统.md 第2.2章

**定义**：
unknown 是安全的 any 类型，任何值都可以赋值给 unknown 类型，但在使用前必须进行类型缩小检查。

### 2.10 never 类型

**名称**：永远不返回类型（Never Type）

**首次出现位置**：C11_102-基础类型系统.md 第2.4章

**定义**：
never 表示永远不会有值的类型，如抛出异常的函数或无限循环的函数。

### 2.11 void 类型

**名称**：空类型（Void Type）

**首次出现位置**：C11_102-基础类型系统.md 第2.3章

**定义**：
void 表示没有返回值的函数返回类型。

## 3. 接口与类型类

### 3.1 接口

**名称**：接口（Interface）

**首次出现位置**：C11_103-接口与类型别名.md 第1章

**定义**：
接口是 TypeScript 中用于定义对象结构的重要工具，它描述了对象应该具有的属性和方法。

### 3.2 可选属性

**名称**：可选属性（Optional Properties）

**首次出现位置**：C11_103-接口与类型别名.md 第1.2章

**定义**：
使用 `?` 标记的属性为可选属性，表示该属性可以有值也可以没有。

### 3.3 只读属性

**名称**：只读属性（Readonly Properties）

**首次出现位置**：C11_103-接口与类型别名.md 第1.3章

**定义**：
使用 `readonly` 标记的属性为只读属性，这些属性只能在初始化时赋值，之后不能修改。

### 3.4 接口继承

**名称**：接口继承（Interface Inheritance）

**首次出现位置**：C11_103-接口与类型别名.md 第2章

**定义**：
接口可以继承其他接口，实现代码复用。使用 `extends` 关键字进行继承。

### 3.5 声明合并

**名称**：声明合并（Declaration Merging）

**首次出现位置**：C11_103-接口与类型别名.md 第4.2章

**定义**：
接口支持声明合并，多个同名接口会自动合并为一个。

### 3.6 索引签名

**名称**：索引签名（Index Signatures）

**首次出现位置**：C11_103-接口与类型别名.md 第1.5章

**定义**：
使用索引签名可以定义任意属性名的类型，例如 `[key: string]: string` 表示键为字符串，值为字符串的对象类型。

## 4. 函数与泛型类

### 4.1 函数重载

**名称**：函数重载（Function Overloading）

**首次出现位置**：C11_104-函数与泛型.md 第1章

**定义**：
函数重载允许为同一个函数提供多个类型定义，根据传入的参数类型和数量来选择合适的类型定义。

### 4.2 泛型

**名称**：泛型（Generics）

**首次出现位置**：C11_104-函数与泛型.md 第2章

**定义**：
泛型是 TypeScript 中一种强大的类型系统特性，允许我们编写可以处理多种类型的代码，而不是仅限于单一类型。

### 4.3 泛型约束

**名称**：泛型约束（Generic Constraints）

**首次出现位置**：C11_104-函数与泛型.md 第3章

**定义**：
泛型约束允许我们限制泛型类型参数的范围，确保它们具有某些特定的属性或方法。

### 4.4 泛型类

**名称**：泛型类（Generic Classes）

**首次出现位置**：C11_104-函数与泛型.md 第4章

**定义**：
泛型类允许我们创建可以处理不同类型数据的类，类名后使用 `<T>` 声明类型参数。

### 4.5 keyof

**名称**：键of运算符（keyof Operator）

**首次出现位置**：C11_104-函数与泛型.md 第3.3章

**定义**：
keyof 运算符用于获取一个类型的所有键名，返回一个联合类型。

### 4.6 泛型工具类型

**名称**：泛型工具类型（Generic Utility Types）

**首次出现位置**：C11_104-函数与泛型.md 第6章

**定义**：
TypeScript 提供了一系列内置的泛型工具类型，用于常见的类型转换场景，如 Partial、Readonly、Pick、Omit 等。

## 5. 类与装饰器类

### 5.1 访问修饰符

**名称**：访问修饰符（Access Modifiers）

**首次出现位置**：C11_105-类与装饰器.md 第1章

**定义**：
TypeScript 提供了四种访问修饰符：public（公共）、private（私有）、protected（受保护）、readonly（只读），用于控制类成员的访问权限。

### 5.2 抽象类

**名称**：抽象类（Abstract Classes）

**首次出现位置**：C11_105-类与装饰器.md 第3章

**定义**：
抽象类是一种不能直接实例化的类，主要用于作为其他类的基类，使用 `abstract` 关键字声明。

### 5.3 抽象方法

**名称**：抽象方法（Abstract Methods）

**首次出现位置**：C11_105-类与装饰器.md 第3章

**定义**：
抽象方法使用 `abstract` 关键字声明，没有具体实现，必须在子类中实现。

### 5.4 静态成员

**名称**：静态成员（Static Members）

**首次出现位置**：C11_105-类与装饰器.md 第4章

**定义**：
静态成员属于类本身，而不是类的实例，可以通过类名直接访问，使用 `static` 关键字声明。

### 5.5 装饰器

**名称**：装饰器（Decorators）

**首次出现位置**：C11_105-类与装饰器.md 第6章

**定义**：
装饰器是 TypeScript 的一个实验性特性，允许我们修改类、方法、属性和参数的行为，使用 `@` 符号后面跟着装饰器函数名。

### 5.6 类装饰器

**名称**：类装饰器（Class Decorators）

**首次出现位置**：C11_105-类与装饰器.md 第6.3章

**定义**：
类装饰器应用于类声明，接收一个参数，即目标类的构造函数。

### 5.7 方法装饰器

**名称**：方法装饰器（Method Decorators）

**首次出现位置**：C11_105-类与装饰器.md 第6.4章

**定义**：
方法装饰器应用于类方法，接收三个参数：目标对象、方法名、方法属性描述符。

### 5.8 存取器

**名称**：存取器（Getters and Setters）

**首次出现位置**：C11_105-类与装饰器.md 第5章

**定义**：
存取器允许我们控制对类属性的访问和修改，使用 get 和 set 关键字定义。

## 6. 高级类型类

### 6.1 类型断言

**名称**：类型断言（Type Assertions）

**首次出现位置**：C11_106-高级类型与演算.md 第1章

**定义**：
类型断言允许我们手动告诉编译器一个值的具体类型，使用 `as` 语法或尖括号语法。

### 6.2 非空断言

**名称**：非空断言（Non-null Assertion）

**首次出现位置**：C11_106-高级类型与演算.md 第2章

**定义**：
非空断言操作符 `!` 告诉编译器一个值不为 null 或 undefined。

### 6.3 类型守卫

**名称**：类型守卫（Type Guards）

**首次出现位置**：C11_106-高级类型与演算.md 第3章

**定义**：
类型守卫是一种运行时检查，用于确定变量的具体类型，帮助编译器进行类型缩小。

### 6.4 typeof 类型守卫

**名称**：typeof类型守卫（typeof Type Guards）

**首次出现位置**：C11_106-高级类型与演算.md 第3.1.1章

**定义**：
使用 typeof 操作符在运行时检查变量类型。

### 6.5 instanceof 类型守卫

**名称**：instanceof类型守卫（instanceof Type Guards）

**首次出现位置**：C11_106-高级类型与演算.md 第3.1.2章

**定义**：
使用 instanceof 操作符检查对象是否为某个类的实例。

### 6.6 自定义类型守卫

**名称**：自定义类型守卫（Custom Type Guards）

**首次出现位置**：C11_106-高级类型与演算.md 第3.2章

**定义**：
自定义类型守卫使用 `is` 关键字来定义一个函数，该函数返回一个布尔值，用于确定变量的类型。

### 6.7 映射类型

**名称**：映射类型（Mapped Types）

**首次出现位置**：C11_106-高级类型与演算.md 第4章

**定义**：
映射类型允许我们根据现有类型创建新类型，通过遍历现有类型的属性并应用转换。

### 6.8 条件类型

**名称**：条件类型（Conditional Types）

**首次出现位置**：C11_106-高级类型与演算.md 第5章

**定义**：
条件类型允许我们根据类型之间的关系创建新类型，语法为 `T extends U ? X : Y`。

### 6.9 infer 关键字

**名称**：infer推断关键字（infer Keyword）

**首次出现位置**：C11_106-高级类型与演算.md 第5.2章

**定义**：
infer 关键字允许我们在条件类型中推断类型，通常用于从复杂类型中提取部分类型。

### 6.10 分发条件类型

**名称**：分发条件类型（Distributive Conditional Types）

**首次出现位置**：C11_106-高级类型与演算.md 第5.3章

**定义**：
当条件类型的左侧是一个联合类型时，条件类型会自动分发到联合类型的每个成员上。

### 6.11 递归类型

**名称**：递归类型（Recursive Types）

**首次出现位置**：C11_106-高级类型与演算.md 第7.2章

**定义**：
递归类型是指在类型定义中引用自身的类型，常用于描述树形结构等嵌套数据。
