## 1. 基础概念类

### 1.1 Vue3

**名称**：Vue3 框架（Vue.js 3）

**首次出现位置**：C12_100-快速入门.md 第1章

**定义**：
Vue3 是 Vue.js 框架的第三个主要版本，是一个用于构建用户界面的渐进式 JavaScript 框架，采用自底向上增量开发的设计哲学。

### 1.2 渐进式框架

**名称**：渐进式框架（Progressive Framework）

**首次出现位置**：C12_100-快速入门.md 第1章

**定义**：
渐进式框架是指可以逐步引入的框架，开发者可以从一个简单的视图层开始，逐步添加更多功能模块，而不需要一开始就使用完整的框架。

### 1.3 单文件组件

**名称**：单文件组件（Single File Component (SFC)）

**首次出现位置**：C12_100-快速入门.md 第3.1章

**定义**：
单文件组件是 Vue 中一种将模板、脚本和样式封装在单个 .vue 文件中的组件格式。

### 1.4 虚拟 DOM

**名称**：虚拟文档对象模型（Virtual DOM）

**首次出现位置**：C12_102-组合式API.md

**定义**：
虚拟 DOM 是真实 DOM 的 JavaScript 对象表示，Vue 使用虚拟 DOM 来高效地计算需要更新的最小 DOM 操作。

### 1.5 响应式

**名称**：响应式系统（Reactivity）

**首次出现位置**：C12_103-响应式系统.md 第1章

**定义**：
响应式是指当数据变化时，视图会自动更新的一种机制，Vue3 提供了更高效的响应式系统。

## 2. 组件系统类

### 2.1 组件

**名称**：组件（Component）

**首次出现位置**：C12_100-快速入门.md 第3.1章

**定义**：
组件是 Vue 应用中的基本构建块，每个组件都封装了自己的模板、逻辑和样式，可以重复使用。

### 2.2 父组件

**名称**：父组件（Parent Component）

**首次出现位置**：C12_100-快速入门.md 第4.3章

**定义**：
父组件是指包含其他组件的组件，父组件可以通过 props 向子组件传递数据。

### 2.3 子组件

**名称**：子组件（Child Component）

**首次出现位置**：C12_100-快速入门.md 第4.3章

**定义**：
子组件是指被其他组件包含的组件，子组件可以通过 emits 向父组件发送消息。

### 2.4 Props

**名称**：属性（Props）

**首次出现位置**：C12_100-快速入门.md 第4.3章

**定义**：
Props 是父组件向子组件传递数据的方式，是组件的输入 API，使用 defineProps 宏在子组件中声明。

### 2.5 Emits

**名称**：事件（Emits）

**首次出现位置**：C12_100-快速入门.md 第4.3章

**定义**：
Emits 是子组件向父组件发送事件的方式，使用 defineEmits 宏在子组件中声明。

### 2.6 插槽

**名称**：插槽（Slots）

**首次出现位置**：C12_104-组件系统.md

**定义**：
插槽是 Vue 中用于分发内容的机制，允许父组件向子组件的特定位置插入 HTML 内容。

### 2.7 依赖注入

**名称**：依赖注入（Provide / Inject）

**首次出现位置**：C12_104-组件系统.md

**定义**：
Provide / Inject 是 Vue 中用于在组件树中传递数据的高级特性，适合在深层嵌套组件间共享数据。

## 3. 组合式 API 类

### 3.1 组合式 API

**名称**：组合式 API（Composition API）

**首次出现位置**：C12_100-快速入门.md 第4.1章

**定义**：
组合式 API 是 Vue3 引入的一套新的 API 组织方式，允许使用导入的函数来组织组件逻辑，更好地复用状态逻辑。

### 3.2 setup

**名称**：设置函数（Setup）

**首次出现位置**：C12_102-组合式API.md

**定义**：
setup 是 Vue3 组合式 API 的入口函数，在组件实例创建之前执行，this 在 setup 中不可用。

### 3.3 script setup

**名称**：脚本设置语法（Script Setup）

**首次出现位置**：C12_100-快速入门.md 第3.1章

**定义**：
script setup 是 Vue3.2+ 引入的编译时语法糖，使得组合式 API 的代码更加简洁。

### 3.4 ref

**名称**：响应式引用（Ref）

**首次出现位置**：C12_100-快速入门.md 第4.2章

**定义**：
ref 是 Vue3 的组合式 API 函数，用于创建响应式的基本类型数据和对象引用，访问时需要通过 .value 属性。

### 3.5 reactive

**名称**：响应式对象（Reactive）

**首次出现位置**：C12_100-快速入门.md 第4.2章

**定义**：
reactive 是 Vue3 的组合式 API 函数，用于创建深层响应式的对象，访问时不需要 .value。

### 3.6 toRefs

**名称**：解构响应式（To Refs）

**首次出现位置**：C12_100-快速入门.md 第4.2章

**定义**：
toRefs 是 Vue3 的组合式 API 函数，用于将 reactive 对象转换为普通对象，对象的每个属性都是响应式的 ref。

### 3.7 computed

**名称**：计算属性（Computed）

**首次出现位置**：C12_100-快速入门.md 第4.1章

**定义**：
computed 是 Vue3 的组合式 API 函数，用于创建基于响应式数据的计算属性，具有缓存特性。

### 3.8 watch

**名称**：监听器（Watch）

**首次出现位置**：C12_102-组合式API.md

**定义**：
watch 是 Vue3 的组合式 API 函数，用于监听响应式数据的变化并执行相应的回调。

### 3.9 生命周期钩子

**名称**：生命周期钩子（Lifecycle Hooks）

**首次出现位置**：C12_100-快速入门.md 第4.1章

**定义**：
生命周期钩子是组件在特定阶段自动执行的函数，如 onMounted、onUpdated、onUnmounted 等。

### 3.10 Composables

**名称**：可组合函数（Composables）

**首次出现位置**：C12_100-快速入门.md 第8章

**定义**：
Composables 是使用组合式 API 封装的可复用状态逻辑的函数，类似于其他框架中的自定义 hooks 或服务。

## 4. 响应式系统类

### 4.1 响应式数据

**名称**：响应式数据（Reactive Data）

**首次出现位置**：C12_103-响应式系统.md 第1章

**定义**：
响应式数据是 Vue 自动追踪其变化并在变化时更新 DOM 的数据。

### 4.2 ref 和 reactive

**名称**：ref和reactive（Ref and Reactive）

**首次出现位置**：C12_103-响应式系统.md 第2章

**定义**：
ref 用于基本类型数据，reactive 用于对象类型数据，两者都是 Vue3 响应式系统的核心。

### 4.3 深层响应式

**名称**：深层响应式（Deep Reactivity）

**首次出现位置**：C12_103-响应式系统.md 第2章

**定义**：
深层响应式是指 Vue 会递归地将对象的所有嵌套属性都转换为响应式的，确保深层数据变化也能被追踪。

### 4.4 响应式副作用

**名称**：响应式副作用（Reactivity Side Effects）

**首次出现位置**：C12_103-响应式系统.md 第3章

**定义**：
响应式副作用是指当响应式依赖变化时自动运行的副作用操作，如 DOM 更新、异步请求等。

### 4.5 触发更新

**名称**：触发更新（Triggering Updates）

**首次出现位置**：C12_103-响应式系统.md 第3章

**定义**：
触发更新是指当响应式数据发生变化时，Vue 自动执行与该数据相关的所有更新操作。

## 5. 模板语法类

### 5.1 模板

**名称**：模板（Template）

**首次出现位置**：C12_105-模板语法.md 第1章

**定义**：
模板是 Vue 组件中用于声明式地将底层数据渲染到 DOM 的声明式语法。

### 5.2 插值

**名称**：插值（Interpolation）

**首次出现位置**：C12_105-模板语法.md 第2章

**定义**：
插值是指在模板中使用双花括号语法 `{{ expression }}` 来显示动态数据。

### 5.3 指令

**名称**：指令（Directives）

**首次出现位置**：C12_106-指令系统.md 第1章

**定义**：
指令是 Vue 模板中以 v- 前缀开头的特殊 attribute，用于在模板中添加响应式行为。

### 5.4 v-bind

**名称**：属性绑定指令（V-Bind）

**首次出现位置**：C12_106-指令系统.md 第2章

**定义**：
v-bind 是用于响应式地将 HTML attribute 绑定到表达式，语法糖为冒号 `:`

### 5.5 v-on

**名称**：事件绑定指令（V-On）

**首次出现位置**：C12_106-指令系统.md 第3章

**定义**：
v-on 是用于监听 DOM 事件并执行相应逻辑的指令，语法糖为 `@`

### 5.6 v-if

**名称**：条件渲染指令（V-If）

**首次出现位置**：C12_106-指令系统.md 第4章

**定义**：
v-if 是用于根据表达式真假值来条件渲染元素的指令。

### 5.7 v-show

**名称**：显示切换指令（V-Show）

**首次出现位置**：C12_106-指令系统.md 第4章

**定义**：
v-show 是用于切换元素 display 属性的指令，与 v-if 不同，v-show 不支持 template 元素。

### 5.8 v-for

**名称**：列表渲染指令（V-For）

**首次出现位置**：C12_106-指令系统.md 第5章

**定义**：
v-for 是用于基于源数据多次渲染元素的指令，需要配合 :key 属性。

### 5.9 v-model

**名称**：双向绑定指令（V-Model）

**首次出现位置**：C12_106-指令系统.md 第6章

**定义**：
v-model 是用于在表单元素上创建双向数据绑定的指令。

### 5.10 v-slot

**名称**：插槽指令（V-Slot）

**首次出现位置**：C12_106-指令系统.md 第7章

**定义**：
v-slot 是用于提供和传递插槽内容的指令，语法糖为 `#`

### 5.11 v-memo

**名称**：记忆化指令（V-Memo）

**首次出现位置**：C12_100-快速入门.md 第8章

**定义**：
v-memo 是用于缓存模板的指令，当依赖项没有变化时跳过重新渲染。

### 5.12 v-once

**名称**：一次性渲染指令（V-Once）

**首次出现位置**：C12_106-指令系统.md

**定义**：
v-once 是用于只渲染元素和组件一次的指令，之后的重新渲染将被跳过。

## 6. 路由与状态管理类

### 6.1 Vue Router

**名称**：Vue路由（Vue Router）

**首次出现位置**：C12_100-快速入门.md 第5.1章

**定义**：
Vue Router 是 Vue.js 的官方路由管理器，用于构建单页面应用，实现页面间的导航和视图切换。

### 6.2 路由

**名称**：路由（Route）

**首次出现位置**：C12_100-快速入门.md 第5.1章

**定义**：
路由是指 URL 路径与组件之间的映射关系，定义了应用的导航结构。

### 6.3 动态路由

**名称**：动态路由（Dynamic Route）

**首次出现位置**：C12_204-Vue Router.md

**定义**：
动态路由是指使用参数化的路径，如 `/user/:id`，来匹配不同但结构相同的路由。

### 6.4 嵌套路由

**名称**：嵌套路由（Nested Routes）

**首次出现位置**：C12_204-Vue Router.md

**定义**：
嵌套路由是指在父路由组件内部渲染子路由组件的路由结构。

### 6.5 Pinia

**名称**：Pinia状态管理（Pinia）

**首次出现位置**：C12_100-快速入门.md 第5.2章

**定义**：
Pinia 是 Vue3 的官方状态管理库，提供了更简单的 API 和更好的 TypeScript 支持。

### 6.6 Store

**名称**：状态存储（Store）

**首次出现位置**：C12_205-Pinia 状态管理.md

**定义**：
Store 是 Pinia 中用于存储应用状态的核心概念，每个 Store 都是一个独立的状态管理单元。

### 6.7 State

**名称**：状态（State）

**首次出现位置**：C12_205-Pinia 状态管理.md

**定义**：
State 是 Store 中存储的响应式数据，类似于组件中的 data。

### 6.8 Getters

**名称**：计算属性（Getters）

**首次出现位置**：C12_205-Pinia 状态管理.md

**定义**：
Getters 是 Store 中的计算属性，用于从 state 派生出新的数据。

### 6.9 Actions

**名称**：操作方法（Actions）

**首次出现位置**：C12_205-Pinia 状态管理.md

**定义**：
Actions 是 Store 中用于执行同步或异步操作的方法，类似于组件中的 methods。

### 7.1 Teleport

**名称**：传送门组件（Teleport）

**首次出现位置**：C12_203-高级组件特性.md

**定义**：
Teleport 是 Vue3 提供的高级组件，用于将组件的 DOM 移动到其他位置，如 body。

## 8. 性能优化类

### 8.1 虚拟滚动

**名称**：虚拟滚动（Virtual Scrolling）

**首次出现位置**：G12_201-性能优化.md

**定义**：
虚拟滚动是一种只渲染可见区域内容的技术，用于优化长列表的渲染性能。

### 8.2 懒加载

**名称**：懒加载（Lazy Loading）

**首次出现位置**：G12_201-性能优化.md

**定义**：
懒加载是一种延迟加载非关键资源的优化技术，如图片懒加载、路由懒加载等。

### 8.3 代码分割

**名称**：代码分割（Code Splitting）

**首次出现位置**：G12_201-性能优化.md

**定义**：
代码分割是将应用拆分为多个小块，按需加载的技术，可以减少首屏加载时间。

### 8.4 缓存

**名称**：缓存（Caching）

**首次出现位置**：G12_201-性能优化.md

**定义**：
缓存是指存储计算结果以避免重复计算的技术，computed 属性就是 Vue 中的缓存机制。
