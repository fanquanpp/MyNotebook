## 1. 场景系统概述

场景系统是 Godot 引擎的核心特性之一，它采用基于节点的层次结构来组织游戏对象和逻辑。场景系统提供了一种直观、灵活的方式来构建和管理游戏世界。
### 1.1 核心概念

- **节点 (Node)**: 游戏对象的基本单位，具有特定功能和属性
- **场景 (Scene)**: 可重用的节点组合，是 Godot 中的基本构建块
- **场景树 (Scene Tree)**: 游戏世界的层次结构，由节点组成
- **根节点 (Root Node)**: 场景的顶层节点，每个场景有且只有一个根节点
### 1.2 场景树结构
场景树是一个树形结构，其中每个节点可以有多个子节点，但只能有一个父节点。这种层次结构使得游戏对象的组织和管理更加直观和高效。
- **根节点**: 场景树的顶部节点，通常是 `Node`、`Node2D` 或 `Node3D`
- **子节点**: 附加到父节点的节点，继承父节点的变换和属性
- **兄弟节点**: 具有相同父节点的节点
### 1.3 场景与节点的关系

- **场景包含节点**: 每个场景由一个或多个节点组成，形成一个层次结构
- **场景可以实例化**: 一个场景可以被多次实例化，每个实例都是独立的
- **场景可以组合**: 场景可以作为子场景添加到其他场景中，形成更复杂的结构
- **场景可以继承**: 场景可以继承自其他场景，扩展或修改其功能
## 2. 节点类型

Godot 提供了丰富的节点类型，每种类型都有特定的功能和用途。以下是一些常用的节点类型：
### 2.1 基础节点

- **Node**: 最基本的节点类型，没有特定功能，用作其他节点的容器
- **Node2D**: 2D 游戏的基础节点，具有位置、旋转和缩放属性
- **Node3D**: 3D 游戏的基础节点，具有位置、旋转和缩放属性
- **Control**: UI 元素的基础节点，具有布局和交互功能
### 2.2 2D 节点

- **Sprite2D**: 显示 2D 纹理的节点
- **AnimatedSprite2D**: 显示动画纹理的节点
- **TileMap**: 用于创建 2D 网格地图的节点
- **CharacterBody2D**: 2D 物理角色控制器
- **RigidBody2D**: 2D 物理刚体
- **Area2D**: 2D 区域检测节点
- **Camera2D**: 2D 相机节点，控制视图
### 2.3 3D 节点

- **MeshInstance3D**: 显示 3D 网格的节点
- **CharacterBody3D**: 3D 物理角色控制器
- **RigidBody3D**: 3D 物理刚体
- **Area3D**: 3D 区域检测节点
- **Camera3D**: 3D 相机节点，控制视图
- **Light3D**: 3D 光源节点
- **DirectionalLight3D**: 方向光节点
- **PointLight3D**: 点光源节点
- **SpotLight3D**: 聚光灯节点
### 2.4 UI 节点

- **Label**: 显示文本的节点
- **Button**: 可点击的按钮节点
- **LineEdit**: 单行文本输入节点
- **TextEdit**: 多行文本输入节点
- **Panel**: 面板节点，用于 UI 布局
- **HBoxContainer**: 水平布局容器
- **VBoxContainer**: 垂直布局容器
- **GridContainer**: 网格布局容器
- **ScrollContainer**: 滚动容器，用于显示超出视口的内容
### 2.5 其他节点

- **Timer**: 定时器节点，用于定时执行操作
- **AudioStreamPlayer**: 音频播放节点
- **AnimationPlayer**: 动画播放节点
- **ParticleSystem2D/3D**: 粒子系统节点，用于创建特效
- **Path2D/3D**: 路径节点，用于定义运动路径
- **RemoteTransform2D/3D**: 远程变换节点，用于同步变换
## 3. 场景创建与编辑
### 3.1 创建场景

1. **通过编辑器创建**:
   - 点击 `Scene` -> `New Scene`
   - 选择根节点类型（如 `Node2D`、`Node3D` 或 `Control`）
   - 点击 `Scene` -> `Save Scene`，为场景命名并保存
2. **通过代码创建**:
   ```gdscript
   # 创建一个新的场景
   var scene = PackedScene.new()
   
   # 创建根节点
   var root_node = Node2D.new()
   root_node.name = "MyScene"
   
   # 添加子节点
   var sprite = Sprite2D.new()
   root_node.add_child(sprite)
   
   # 将节点树打包到场景中
   scene.pack(root_node)
   
   # 保存场景
   ResourceSaver.save("res://scenes/my_scene.tscn", scene)
   ```

### 3.2 编辑场景

- **添加节点**: 点击场景树面板中的 `+` 按钮，选择要添加的节点类型
- **删除节点**: 选中节点，按 `Delete` 键或右键点击选择 `Delete`
- **移动节点**: 拖动节点在场景树中调整位置，或在视图中拖动节点
- **修改属性**: 在 Inspector 面板中修改节点的属性
- **连接信号**: 在 Node 面板中连接节点的信号到其他节点的方法
### 3.3 场景继承

场景继承允许你基于现有场景创建新场景，继承其所有节点和属性，同时可以添加、修改或删除节点。
1. **创建继承场景**:
   - 点击 `Scene` -> `New Inherited Scene`
   - 选择要继承的场景文件
   - 编辑继承的场景，添加或修改节点
2. **代码中的场景继承**:
   ```gdscript
   # 加载基础场景
   var base_scene = load("res://scenes/base_scene.tscn")
   
   # 实例化基础场景
   var instance = base_scene.instantiate()
   
   # 添加新节点
   var new_node = Node2D.new()
   instance.add_child(new_node)
   ```

### 3.4 场景组合

场景组合是将一个场景作为子场景添加到另一个场景中，形成更复杂的结构。
1. **通过编辑器组合场景**:
   - 点击场景树面板中的 `+` 按钮
   - 选择 `Instance Child Scene`
   - 选择要添加的场景文件
2. **通过代码组合场景**:
   ```gdscript
   # 加载子场景
   var child_scene = load("res://scenes/child_scene.tscn")
   
   # 实例化子场景
   var child_instance = child_scene.instantiate()
   
   # 添加到当前场景
   add_child(child_instance)
   ```

## 4. 场景实例化
场景实例化是创建场景的一个实例，每个实例都是独立的，可以有不同的属性和状态。
### 4.1 编辑器中实例化
1. **添加子场景**:
   - 点击场景树面板中的 `+` 按钮
   - 选择 `Instance Child Scene`
   - 选择要实例化的场景文件
2. **修改实例属性**:
   - 选中实例节点
   - 在 Inspector 面板中修改实例的属性
   - 点击 `Override Local` 按钮可以覆盖继承的属性
### 4.2 代码中实例化

```gdscript
# 加载场景
var scene = load("res://scenes/player.tscn")

# 实例化场景
var player_instance = scene.instantiate()

# 设置实例属性
player_instance.position = Vector2(100, 200)
player_instance.name = "Player"

# 添加到当前场景
add_child(player_instance)

# 或者添加到场景树
get_tree().root.add_child(player_instance)
```

### 4.3 场景实例的参数化

可以通过脚本为场景实例设置参数，使其更加灵活。
```gdscript
# 玩家场景脚本
class_name Player extends CharacterBody2D

export var player_name = "Player"
export var health = 100
export var speed = 200

func _ready():
    print("Player " + player_name + " spawned with " + str(health) + " health")

# 实例化时设置参数
var player_scene = load("res://scenes/player.tscn")
var player = player_scene.instantiate()
player.player_name = "Hero"
player.health = 150
player.speed = 250
add_child(player)
```

## 5. 场景管理

### 5.1 场景切换

场景切换是在游戏运行时从一个场景切换到另一个场景。
```gdscript
# 加载并切换到新场景
func change_scene():
    var next_scene = load("res://scenes/level2.tscn")
    get_tree().change_scene_to_packed(next_scene)

# 异步加载并切换场景
func change_scene_async():
    var next_scene = load("res://scenes/level2.tscn")
    get_tree().change_scene_to_packed(next_scene)
```

### 5.2 场景加载

场景加载可以是同步的或异步的，异步加载不会阻塞游戏主线程。
```gdscript
# 同步加载场景
func load_scene_sync():
    var scene = load("res://scenes/level.tscn")
    var instance = scene.instantiate()
    add_child(instance)

# 异步加载场景
func load_scene_async():
    var scene_path = "res://scenes/level.tscn"
    var scene_loader = ResourceLoader.load_threaded_get(scene_path)
    
    # 检查加载状态
    while not ResourceLoader.load_threaded_get_status(scene_path) == ResourceLoader.THREAD_LOAD_COMPLETE:
        await get_tree().process_frame
    
    var scene = ResourceLoader.load_threaded_get(scene_path)
    var instance = scene.instantiate()
    add_child(instance)
```

### 5.3 场景卸载

场景卸载是从场景树中移除场景实例，释放其占用的资源。
```gdscript
# 卸载场景实例
func unload_scene(instance):
    if instance and is_instance_valid(instance):
        instance.queue_free()

# 卸载所有子场景
func unload_all_children():
    for child in get_children():
        child.queue_free()
```

### 5.4 场景状态管理
场景状态管理是保存和恢复场景的状态，用于游戏存档和读档。
```gdscript
# 保存场景状态
func save_scene_state():
    var state = {}
    state["position"] = position
    state["health"] = health
    state["inventory"] = inventory
    
    # 保存到文件
    var file = FileAccess.open("user://savegame.save", FileAccess.WRITE)
    if file:
        file.store_string(JSON.stringify(state))
        file.close()

# 加载场景状态
func load_scene_state():
    var file = FileAccess.open("user://savegame.save", FileAccess.READ)
    if file:
        var content = file.get_as_text()
        var state = JSON.parse_string(content)
        position = state["position"]
        health = state["health"]
        inventory = state["inventory"]
        file.close()
```

## 6. 最佳实践
### 6.1 场景设计

- **模块化设计**: 将游戏分解为多个小的、可重用的场景
- **职责单一**: 每个场景只负责一个特定的功能或游戏元素
- **层次清晰**: 合理组织场景的层次结构，避免过深的节点树
- **命名规范**: 使用清晰、描述性的名称命名场景和节点
### 6.2 节点组织

- **按功能组织**: 将相关的节点组织在一起，形成逻辑组
- **使用空节点作为容器**: 使用 `Node` 或 `Node2D`/`Node3D` 作为逻辑组的容器
- **避免冗余节点**: 只添加必要的节点，避免过多的空节点
- **使用标签和组**: 为节点添加标签和组，方便查找和管理
### 6.3 性能优化

- **减少节点数量**: 合理使用节点，避免过多的节点导致性能下降
- **使用对象池**: 对于频繁创建和销毁的对象，使用对象池
- **优化绘制**: 使用适当的绘制方法，避免不必要的绘制
- **使用可见性设置**: 对于不在视口中的节点，设置其可见性为 false
- **避免在 _process 中进行复杂计算**: 将复杂计算移到适当的时机执行
## 7. 实际应用示例

### 7.1 示例 1：玩家场景
```gdscript
# Player.gd
extends CharacterBody2D

export var speed = 200
export var jump_force = 500
var velocity = Vector2()

func _process(delta):
    # 处理输入
    var input_dir = Vector2()
    if Input.is_action_pressed("ui_right"):
        input_dir.x += 1
    if Input.is_action_pressed("ui_left"):
        input_dir.x -= 1
    
    # 归一化输入向量
    input_dir = input_dir.normalized()
    
    # 计算水平速度
    velocity.x = input_dir.x * speed

func _physics_process(delta):
    # 应用重力
    velocity.y += 9.8 * 100 * delta
    
    # 处理跳跃
    if Input.is_action_just_pressed("ui_accept") and is_on_floor():
        velocity.y = -jump_force
    
    # 应用移动
    move_and_slide()
```

### 7.2 示例 2：敌人场景
```gdscript
# Enemy.gd
extends CharacterBody2D

export var speed = 100
export var patrol_points = [Vector2(0, 0), Vector2(200, 0)]
var current_point = 0
var velocity = Vector2()

func _process(delta):
    # 获取目标点
    var target = patrol_points[current_point]
    
    # 计算方向
    var direction = (target - position).normalized()
    
    # 计算速度
    velocity = direction * speed

func _physics_process(delta):
    # 应用移动
    move_and_slide()
    
    # 检查是否到达目标点
    var target = patrol_points[current_point]
    if position.distance_to(target) < 10:
        # 移动到下一个点
        current_point = (current_point + 1) % patrol_points.size()
```

### 7.3 示例 3：场景切换管理器

```gdscript
# SceneManager.gd
extends Node

static var instance: SceneManager = null

func _ready():
    instance = self

func change_scene(scene_path: String):
    # 加载场景
    var scene = load(scene_path)
    if scene:
        # 切换场景
        get_tree().change_scene_to_packed(scene)
    else:
        print("Failed to load scene: " + scene_path)

func load_scene_async(scene_path: String, on_loaded: Callable = null):
    # 异步加载场景
    var scene_loader = ResourceLoader.load_threaded_get(scene_path)
    
    # 创建加载协程
    var load_task = load_scene_task(scene_path, on_loaded)
    load_task.resume()

func load_scene_task(scene_path: String, on_loaded: Callable):
    # 等待加载完成
    while not ResourceLoader.load_threaded_get_status(scene_path) == ResourceLoader.THREAD_LOAD_COMPLETE:
        await get_tree().process_frame
    
    # 获取加载的场景
    var scene = ResourceLoader.load_threaded_get(scene_path)
    if scene:
        # 切换场景
        get_tree().change_scene_to_packed(scene)
        
        # 调用回调
        if on_loaded:
            on_loaded.call()
    else:
        print("Failed to load scene: " + scene_path)
```

## 8. 总结

Godot 的场景系统是一个强大、灵活的工具，它通过基于节点的层次结构来组织游戏对象和逻辑。场景系统的核心优势包括：
- **直观的层次结构**: 基于节点的场景树使得游戏对象的组织和管理更加直观
- **高度模块化**: 场景可以被重用、组合和继承，提高代码的可维护性
- **灵活性**: 可以通过代码动态创建、修改和管理场景
- **性能优化**: 场景系统提供了多种性能优化手段，如对象池、可见性设置等
通过掌握场景系统的使用方法和最佳实践，开发者可以构建结构清晰、性能优化的游戏。场景系统是 Godot 引擎的核心特性之一，也是游戏开发中最常用的工具之一。
