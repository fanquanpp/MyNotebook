## 1. 输入系统概述

**Input** 是处理输入的单例，用于处理键盘按键、鼠标按钮及移动、游戏手柄、输入动作等。动作以及对应的事件可以在 **项目 > 项目设置** -> **输入映射** 选项卡中设置，也可以使用 **InputMap** 类设置。
**注意:** **Input** 的方法反映的是全局输入状态，不受 **Control.accept_event()** 或 **Viewport.set_input_as_handled()** 的影响，因为这两个方法处理的是输入在 **SceneTree** 中传播的方式。
### 1.1 核心概念

- **输入事件 (InputEvent)**: 表示来自输入设备的输入，如按键、鼠标移动、触摸等
- **输入映射 (InputMap)**: 将输入事件映射到游戏中的动作，如 "move_left"、"jump" 等
- **输入处理 (Input Handling)**: 处理输入事件并执行相应的游戏逻辑
- **输入设备 (Input Device)**: 如键盘、鼠标、游戏手柄、触摸屏幕等
- **输入捕获 (Input Capture)**: 捕获和处理特定的输入事件
### 1.2 输入类型

Godot 支持多种输入类型。
- **键盘输入**: 按键按下、释放等事件
- **鼠标输入**: 鼠标移动、点击、滚轮等事件
- **游戏手柄输入**: 摇杆、按钮、扳机等事件
- **触摸输入**: 触摸屏幕的位置、移动、抬起等事件
- **自定义输入**: 通过脚本创建的自定义输入
## 2. 输入映射

### 2.1 输入映射概述

输入映射是将物理输入（如按键、摇杆）映射到游戏中的动作（如 "move_left"、"jump"）的系统。通过输入映射，开发者可以创建可配置的控制方案，允许玩家自定义控制键位。
### 2.2 配置输入映射

1. **在编辑器中配置输入映射**:
   - 点击 `Project` -> `Project Settings` -> `Input Map`
   - 点击 `Add Action` 添加新动作，如 "move_left"、"jump"
   - 点击 `Add Event` 为动作添加输入事件，如键盘按键、游戏手柄按钮等
2. **通过代码配置输入映射**:

```gdscript
# 添加输入动作
InputMap.add_action("move_left")

# 添加输入事件
InputMap.action_add_event("move_left", InputEventKey.new()
    .set_keycode(KEY_A))

# 添加游戏手柄输入
var event = InputEventJoypadButton.new()
event.button_index = JOY_BUTTON_0
event.pressed = true
InputMap.action_add_event("jump", event)

# 移除输入事件
InputMap.action_erase_event("move_left", event)

# 移除输入动作
InputMap.erase_action("move_left")

# 获取动作的所有事件
var events = InputMap.action_get_events("move_left")
```

### 2.3 输入映射使用

```gdscript
# 检查动作是否被按下
if Input.is_action_pressed("move_left"):
    # 处理向左移动
    pass

# 检查动作是否刚刚被按下

# 检查动作是否刚刚被释放
```

## 3. 输入事件

### 3.1 输入事件类型

Godot 提供了多种输入事件类型：

- **InputEventKey**: 键盘按键事件
- **InputEventMouseButton**: 鼠标按钮事件
- **InputEventMouseMotion**: 鼠标移动事件
- **InputEventJoypadButton**: 游戏手柄按钮事件
- **InputEventJoypadMotion**: 游戏手柄摇杆事件
- **InputEventScreenTouch**: 屏幕触摸事件
- **InputEventScreenDrag**: 屏幕拖动事件
- **InputEventMagnifyGesture**: 放大手势事件
- **InputEventPanGesture**: 平移手势事件
### 3.2 输入事件处理

```gdscript
# 在节点的 _input 方法中处理输入事件
func _input(event):
    # 处理键盘事件
    if event is InputEventKey:
        if event.pressed:
            print("Key pressed: " + str(event.keycode))
        else:
            print("Key released: " + str(event.keycode))

    # 处理鼠标事件
    elif event is InputEventMouseButton:
        if event.pressed:
            print("Mouse button pressed: " + str(event.button_index))
        else:
            print("Mouse button released: " + str(event.button_index))

    # 处理鼠标移动事件
    elif event is InputEventMouseMotion:
        print("Mouse moved: " + str(event.relative))

    # 处理触摸事件
    elif event is InputEventScreenTouch:
        if event.pressed:
            print("Touch started: " + str(event.position))
        else:
            print("Touch ended: " + str(event.position))
```

### 3.3 输入事件捕获

```gdscript
# 捕获输入事件
func _input(event):
    # 处理输入事件
    if event is InputEventMouseButton:
        # 捕获鼠标事件，不再传递给其他节点
        accept_event()

# 捕获特定输入
func _input(event):
    if event.is_action_pressed("pause"):
        # 处理暂停
        accept_event()
```

## 6. 输入系统最佳实践
### 6.1 输入映射设计

- **使用输入映射**: 始终使用输入映射而不是直接检查按键，便于玩家自定义控制
- **命名规范**: 使用清晰、描述性的名称命名输入动作，如 "move_left"、"jump" 等
- **多平台支持**: 为不同平台（如 PC、移动设备）配置不同的输入映射
- **默认映射**: 提供合理的默认输入映射，同时允许玩家自定义
- **输入提示**: 在游戏中显示当前的输入映射，帮助玩家了解控制方式
### 6.2 输入处理

- **使用动作强度**: 对于模拟输入（如摇杆），使用 `get_action_strength` 获取输入强度
- **输入缓冲**: 实现输入缓冲，处理玩家的快速输入
- **输入延迟**: 最小化输入延迟，提高游戏的响应速度
- **输入合并**: 合并相似的输入事件，减少处理开销
- **输入状态管理**: 跟踪输入状态，处理连续输入和组合输入
### 6.3 多设备支持
- **设备检测**: 检测连接的输入设备，适应不同的控制方式
- **自动切换**: 根据连接的设备自动切换控制方案
- **设备优先级**: 为不同设备设置优先级，如游戏手柄优先于键盘
- **设备配置**: 允许玩家配置不同设备的控制方案
- **设备反馈**: 为不同设备提供适当的反馈，如游戏手柄振动
### 6.4 常见问题

- **输入冲突**: 避免不同动作使用相同的输入事件
- **输入延迟**: 检查输入处理代码，避免在输入处理中执行耗时操作
- **输入丢失**: 确保输入事件被正确处理，避免事件丢失
- **跨平台兼容性**: 测试不同平台的输入处理，确保一致性
- **性能问题**: 优化输入处理代码，避免过多的输入事件处理
## 7. 实际应用示例

### 7.1 示例 1：玩家控制器

```gdscript
# PlayerController.gd
extends CharacterBody2D

@export var speed = 300
@export var jump_force = 500

var velocity = Vector2()

func _process(delta):
    # 获取输入
    var input_dir = Vector2()
    input_dir.x = Input.get_action_strength("move_right") - Input.get_action_strength("move_left")
    input_dir.y = Input.get_action_strength("move_down") - Input.get_action_strength("move_up")

    # 归一化输入向量
    input_dir = input_dir.normalized()

    # 计算水平速度
    velocity.x = input_dir.x * speed

    # 处理跳跃
    if Input.is_action_just_pressed("jump") and is_on_floor():
        velocity.y = -jump_force

func _physics_process(delta):
    # 应用重力
    velocity.y += 9.8 * 100 * delta

    # 应用移动
    move_and_slide()
```

### 7.2 示例 2：相机控制器
```gdscript
# CameraController.gd
extends Camera2D

@export var speed = 10
@export var zoom_speed = 0.1

func _process(delta):
    # 键盘控制相机移动
    var input_dir = Vector2()
    input_dir.x = Input.get_action_strength("camera_right") - Input.get_action_strength("camera_left")
    input_dir.y = Input.get_action_strength("camera_down") - Input.get_action_strength("camera_up")

    # 移动相机
    position += input_dir * speed

    # 鼠标滚轮控制缩放
    if Input.is_action_pressed("camera_zoom_in"):
        zoom *= 1.0 - zoom_speed
    elif Input.is_action_pressed("camera_zoom_out"):
        zoom *= 1.0 + zoom_speed

    # 限制缩放范围
    zoom = Vector2(clamp(zoom.x, 0.5, 2.0), clamp(zoom.y, 0.5, 2.0))
```

## 8. 总结

Godot 的输入系统是一个功能强大、灵活的工具，它提供了统一的输入处理方式，支持多种输入设备和输入类型。输入系统的核心优势包括：
- **统一的输入映射**: 通过输入映射将物理输入映射到游戏动作，便于玩家自定义控制
- **丰富的输入事件**: 支持键盘、鼠标、游戏手柄、触摸等多种输入事件
- **灵活的输入处理**: 提供多种输入处理方法，如 `_input`、`_unhandled_input` 等
- **多设备支持**: 支持多种输入设备，适应不同的游戏平台
- **输入状态管理**: 提供输入状态查询和管理功能
通过掌握输入系统的使用方法和最佳实践，开发者可以创建响应灵敏、控制舒适的游戏，提升玩家的游戏体验。输入系统是 Godot 引擎的核心特性之一，也是游戏开发中不可或缺的工具。
