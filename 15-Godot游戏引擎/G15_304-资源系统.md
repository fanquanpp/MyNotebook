## 1. 资源系统概述

Godot 的资源系统是一个强大的工具，用于管理游戏中的各种资源，如纹理、模型、音频、脚本等。资源系统提供了一种统一的方式来加载、管理和使用游戏资源。
### 1.1 核心概念

- **资源 (Resource)**: 游戏中可重用的资产，如纹理、模型、音频等
- **资源路径 (Resource Path)**: 资源在项目中的唯一标识符，如 `res://textures/player.png`
- **资源加载 (Resource Loading)**: 从磁盘或内存中加载资源到游戏中
- **资源卸载 (Resource Unloading)**: 从内存中释放不再需要的资源
- **资源缓存 (Resource Cache)**: 存储已加载资源的缓存，避免重复加载
### 1.2 资源类型

Godot 支持多种资源类型，包括：

- **纹理 (Texture)**: 2D 和 3D 纹理
- **模型 (Mesh)**: 3D 模型
- **音频 (Audio)**: 音效和音乐
- **场景 (Scene)**: 游戏场景
- **脚本 (Script)**: GDScript、C# 等脚本
- **材质 (Material)**: 2D 和 3D 材质
- **动画 (Animation)**: 动画资源
- **字体 (Font)**: 字体资源
- **配置 (Config)**: 配置文件
### 1.3 资源路径

资源路径是资源在项目中的唯一标识符，使用 `res://` 前缀表示项目资源目录，使用 `user://` 前缀表示用户数据目录。
- **res://**: 项目资源目录，包含游戏的所有资源文件
- **user://**: 用户数据目录，用于存储游戏的保存数据、配置文件等
## 2. 资源类型

### 2.1 内置资源

Godot 内置了多种资源类型，用于满足不同的游戏需求：

- **Texture2D**: 2D 纹理资源
- **Texture3D**: 3D 纹理资源
- **Mesh**: 3D 网格资源
- **AudioStream**: 音频流资源
- **PackedScene**: 打包的场景资源
- **Script**: 脚本资源
- **Material**: 材质资源
- **Animation**: 动画资源
- **Font**: 字体资源
- **ConfigFile**: 配置文件资源
### 2.2 自定义资源
可以创建自定义资源类型，用于存储游戏特定的数据：

```gdscript
# 创建自定义资源
class_name ItemData extends Resource

export var name: String = "Item"
export var description: String = "An item"
export var value: int = 0
export var weight: float = 1.0
export var texture: Texture2D

# 使用自定义资源
var item_data = ItemData.new()
item_data.name = "Health Potion"
item_data.description = "Restores health"
item_data.value = 50
item_data.weight = 0.5
item_data.texture = load("res://textures/potion.png")

# 保存自定义资源
ResourceSaver.save("res://data/items/health_potion.tres", item_data)
```

### 2.3 资源继承

资源可以继承自其他资源，扩展其功能：

```gdscript
# 基础资源
class_name BaseItem extends Resource

export var name: String = "Item"
export var value: int = 0

# 派生资源
class_name WeaponItem extends BaseItem

export var damage: int = 10
export var durability: int = 100

# 使用派生资源
var sword = WeaponItem.new()
sword.name = "Iron Sword"
sword.value = 100
sword.damage = 20
sword.durability = 50
```

## 3. 资源管理

### 3.1 资源加载

资源加载是将资源从磁盘或内存中加载到游戏中的过程。
```gdscript
# 同步加载资源
var texture = load("res://textures/player.png")
var scene = load("res://scenes/player.tscn")
var script = load("res://scripts/player.gd")

# 异步加载资源
func load_resource_async(path: String, callback: Callable):
    var resource_loader = ResourceLoader.load_threaded_get(path)
    
    # 检查加载状态
    while not ResourceLoader.load_threaded_get_status(path) == ResourceLoader.THREAD_LOAD_COMPLETE:
        await get_tree().process_frame
    
    var resource = ResourceLoader.load_threaded_get(path)
    callback.call(resource)

# 使用异步加载
load_resource_async("res://textures/large_texture.png", func(resource):
    print("Resource loaded: " + str(resource))
)
```

### 3.2 资源卸载

资源卸载是从内存中释放不再需要的资源的过程：

```gdscript
# 手动卸载资源
func unload_resource(path: String):
    ResourceLoader.unload(path)

# 清理未使用的资源
func cleanup_unused_resources():
    ResourceLoader.cleanup_unused_resources()
```

### 3.3 资源缓存

资源缓存存储已加载的资源，避免重复加载：

```gdscript
# 检查资源是否已缓存
var is_cached = ResourceLoader.has_cached("res://textures/player.png")
print("Is cached: " + str(is_cached))

# 清除资源缓存
ResourceLoader.clear_cache()
```

### 3.4 资源引用

资源引用是指向资源的指针，用于在不同的地方使用同一个资源：

```gdscript
# 创建资源引用
var texture_ref = preload("res://textures/player.png")

# 使用资源引用
var sprite = Sprite2D.new()
sprite.texture = texture_ref
add_child(sprite)
```

## 4. 资源导入与导出
### 4.1 资源导入

资源导入是将外部文件导入到 Godot 项目中的过程。
- **通过编辑器导入**: 将文件拖放到 Godot 编辑器的文件系统面板中
- **通过代码导入**: 使用 `ResourceImporter` 类导入资源
```gdscript
# 通过代码导入资源
var importer = ResourceImporter.new()
var error = importer.import("path/to/external/texture.png", "res://textures/imported_texture.png")
if error == OK:
    print("Resource imported successfully")
else:
    print("Failed to import resource")
```

### 4.2 资源导出

资源导出是将 Godot 项目中的资源导出为可分发的格式的过程。
- **通过编辑器导出**: 使用 Godot 编辑器的导出功能
- **通过代码导出**: 使用 `ResourceSaver` 类导出资源
```gdscript
# 导出资源
var resource = load("res://scenes/player.tscn")
var error = ResourceSaver.save("user://player_backup.tres", resource)
if error == OK:
    print("Resource exported successfully")
else:
    print("Failed to export resource")
```

### 4.3 资源预设

资源预设是预配置的资源，用于快速创建相似的资源。
- **创建资源预设**: 在编辑器中创建资源，然后保存为预设
- **使用资源预设**: 通过预设创建新的资源实例
```gdscript
# 加载资源预设
var preset = load("res://presets/enemy_preset.tres")

# 创建资源实例
var enemy = preset.duplicate()
enemy.name = "Goblin"
enemy.health = 50
enemy.damage = 10
```

## 5. 资源预加载
### 5.1 预加载语法
预加载是在脚本编译时加载资源，避免运行时加载的延迟：

```gdscript
# 预加载资源
@onready var player_texture = preload("res://textures/player.png")
@onready var player_scene = preload("res://scenes/player.tscn")
@onready var player_script = preload("res://scripts/player.gd")

func _ready():
    # 使用预加载的资源
    var sprite = Sprite2D.new()
    sprite.texture = player_texture
    add_child(sprite)
    
    var player = player_scene.instantiate()
    add_child(player)
```

### 5.2 预加载时机
预加载发生在脚本编译时，而不是运行时。
- **脚本编译时**: 预加载的资源在脚本编译时加载到内存中
- **运行时**: 预加载的资源可以直接使用，不需要额外的加载时间
### 5.3 预加载优化
预加载可以优化游戏的性能，减少运行时的加载延迟：

- **预加载常用资源**: 预加载游戏中常用的资源，如玩家纹理、UI 元素等
- **避免预加载大型资源**: 对于大型资源，如高分辨率纹理、复杂模型等，应在需要时再加载
- **使用资源池**: 对于频繁使用的资源，使用资源池来管理
## 6. 资源系统最佳实践
### 6.1 资源组织

- **合理组织资源目录**: 将资源按类型组织到不同的目录中，如 `textures/`、`models/`、`audio/` 等
- **使用资源文件夹**: 使用资源文件夹来组织相关的资源
- **命名规范**: 使用清晰、描述性的名称命名资源文件
- **版本控制**: 使用版本控制系统管理资源文件
### 6.2 性能优化

- **预加载常用资源**: 预加载游戏中常用的资源，减少运行时的加载延迟
- **异步加载大型资源**: 对于大型资源，使用异步加载，避免阻塞游戏主线程
- **使用资源池**: 对于频繁创建和销毁的资源，使用资源池来管理
- **减少资源大小**: 优化资源大小，如压缩纹理、减少模型多边形等
### 6.3 内存管理

- **及时卸载资源**: 对于不再需要的资源，及时从内存中卸载
- **使用弱引用**: 对于可能被释放的资源，使用弱引用
- **监控内存使用**: 定期监控游戏的内存使用情况，避免内存泄漏
- **清理未使用的资源**: 定期清理未使用的资源，释放内存
## 7. 实际应用示例

### 7.1 示例 1：资源管理器

```gdscript
# ResourceManager.gd
extends Node

static var instance: ResourceManager = null
var loaded_resources: Dictionary = {}

func _ready():
    instance = self

func load_resource(path: String) -> Resource:
    if loaded_resources.has(path):
        # 资源已加载，返回缓存的资源
        return loaded_resources[path]
    else:
        # 加载新资源
        var resource = load(path)
        if resource:
            loaded_resources[path] = resource
        return resource

func unload_resource(path: String):
    if loaded_resources.has(path):
        loaded_resources.erase(path)
        ResourceLoader.unload(path)

func unload_all_resources():
    for path in loaded_resources.keys():
        ResourceLoader.unload(path)
    loaded_resources.clear()

func cleanup():
    ResourceLoader.cleanup_unused_resources()
```

### 7.2 示例 2：预加载管理器
```gdscript
# PreloadManager.gd
extends Node

static var instance: PreloadManager = null
var preloaded_resources: Dictionary = {}

func _ready():
    instance = self

func preload_resources():
    # 预加载常用资源
    preloaded_resources["player_texture"] = preload("res://textures/player.png")
    preloaded_resources["enemy_texture"] = preload("res://textures/enemy.png")
    preloaded_resources["player_scene"] = preload("res://scenes/player.tscn")
    preloaded_resources["enemy_scene"] = preload("res://scenes/enemy.tscn")
    preloaded_resources["ui_theme"] = preload("res://themes/ui_theme.tres")

func get_resource(name: String) -> Resource:
    if preloaded_resources.has(name):
        return preloaded_resources[name]
    else:
        return null
```

## 8. 总结

Godot 的资源系统是一个强大、灵活的工具，它提供了一种统一的方式来管理游戏中的各种资源。资源系统的核心优势包括：
- **统一的资源管理**: 提供了统一的方式来加载、管理和使用游戏资源
- **丰富的资源类型**: 支持多种资源类型，满足不同的游戏需求
- **灵活的资源加载**: 支持同步和异步加载资源
- **资源缓存**: 存储已加载的资源，避免重复加载
- **自定义资源**: 支持创建自定义资源类型，存储游戏特定的数据
通过掌握资源系统的使用方法和最佳实践，开发者可以有效地管理游戏资源，提高游戏的性能和可维护性。资源系统是 Godot 引擎的核心特性之一，也是游戏开发中最常用的工具之一。
