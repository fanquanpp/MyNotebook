## 3 . 选择器概述

CSS 选择器用于指定要应用样式的 HTML 元素，它是 CSS 规则的重要组成部分。CSS3 提供了丰富的选择器类型，使得开发者可以更加精确地选择元素。

<a id="4"></a>
## 4 . 基础选择器

基础选择器是 CSS 中最基本、最常用的选择器类型。

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| 元素选择器 | 选择指定类型的元素 | `div { color: red; }` |
| ID 选择器 | 选择指定 ID 的元素 | `#header { background: blue; }` |
| 类选择器 | 选择指定类的元素 | `.container { width: 100%; }` |
| 通用选择器 | 选择所有元素 | `* { margin: 0; padding: 0; }` |
| 属性选择器 | 选择具有指定属性的元素 | `[type="text"] { border: 1px solid #ccc; }` |

<a id="4.1"></a>
### 4.1 属性选择器的扩展

CSS3 扩展了属性选择器，提供了更多的匹配方式：

```css
/* 包含指定属性 */
[attribute] { color: red; }

/* 属性值等于指定值 */
[attribute="value"] { color: red; }

/* 属性值以指定值开头 */
[attribute^="value"] { color: red; }

/* 属性值以指定值结尾 */
[attribute$="value"] { color: red; }

/* 属性值包含指定值 */
[attribute*="value"] { color: red; }

/* 属性值包含指定单词 */
[attribute~="value"] { color: red; }

/* 属性值以指定值开头，后跟连字符 */
[attribute|="value"] { color: red; }
```

<a id="5"></a>
## 5 . 组合选择器

组合选择器用于选择满足多个条件的元素。

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| 后代选择器 | 选择元素的后代元素 | `div p { color: red; }` |
| 子选择器 | 选择元素的直接子元素 | `div > p { color: red; }` |
| 相邻兄弟选择器 | 选择元素的下一个兄弟元素 | `div + p { color: red; }` |
| 通用兄弟选择器 | 选择元素的所有兄弟元素 | `div ~ p { color: red; }` |

<a id="6"></a>
## 6 . 伪类选择器

伪类选择器用于选择处于特定状态的元素。

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| `:hover` | 鼠标悬停时 | `a:hover { color: red; }` |
| `:active` | 元素被激活时 | `button:active { background: blue; }` |
| `:focus` | 元素获得焦点时 | `input:focus { border: 2px solid blue; }` |
| `:first-child` | 第一个子元素 | `li:first-child { font-weight: bold; }` |
| `:last-child` | 最后一个子元素 | `li:last-child { color: red; }` |
| `:nth-child(n)` | 第 n 个子元素 | `li:nth-child(2) { color: blue; }` |
| `:nth-child(even)` | 偶数位置的子元素 | `li:nth-child(even) { background: #f0f0f0; }` |
| `:nth-child(odd)` | 奇数位置的子元素 | `li:nth-child(odd) { background: #e0e0e0; }` |
| `:nth-child(3n)` | 第 3、6、9...个子元素 | `li:nth-child(3n) { color: green; }` |
| `:not(selector)` | 排除指定选择器 | `p:not(.special) { color: gray; }` |
| `:first-of-type` | 同类型的第一个元素 | `p:first-of-type { font-weight: bold; }` |
| `:last-of-type` | 同类型的最后一个元素 | `p:last-of-type { color: red; }` |
| `:nth-of-type(n)` | 同类型的第 n 个元素 | `p:nth-of-type(2) { color: blue; }` |
| `:only-child` | 唯一的子元素 | `div:only-child { color: red; }` |
| `:only-of-type` | 同类型的唯一元素 | `p:only-of-type { color: red; }` |
| `:empty` | 没有子元素的元素 | `div:empty { display: none; }` |
| `:root` | 文档的根元素 | `:root { --primary-color: #3498db; }` |
| `:target` | 当前活动的锚点元素 | `:target { background: yellow; }` |

<a id="6.1"></a>
### 6.1 链接伪类

```css
/* 未访问的链接 */
a:link { color: blue; }

/* 已访问的链接 */
a:visited { color: purple; }

/* 鼠标悬停的链接 */
a:hover { color: red; }

/* 激活的链接 */
a:active { color: green; }

/* 注意顺序：link → visited → hover → active */
```

<a id="7"></a>
## 7 . 伪元素选择器

伪元素选择器用于选择元素的特定部分。

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| `::before` | 在元素内容前插入内容 | `p::before { content: "前缀"; }` |
| `::after` | 在元素内容后插入内容 | `p::after { content: "后缀"; }` |
| `::first-letter` | 选择元素的第一个字母 | `p::first-letter { font-size: 2em; }` |
| `::first-line` | 选择元素的第一行 | `p::first-line { font-weight: bold; }` |
| `::selection` | 选择用户选中的内容 | `::selection { background: yellow; }` |
| `::placeholder` | 选择输入框的占位符文本 | `input::placeholder { color: #999; }` |
| `::backdrop` | 选择对话框的背景 | `dialog::backdrop { background: rgba(0,0,0,0.5); }` |

<a id="8"></a>
## 8 . 选择器优先级

CSS 选择器的优先级决定了当多个选择器应用于同一元素时，哪个选择器的样式会被优先使用。

<a id="8.1"></a>
### 8.1 优先级规则

从高到低：
1. **!important 声明**：最高优先级
2. **内联样式**：使用 style 属性的样式
3. **ID 选择器**：以#开头的选择器
4. **类选择器、属性选择器、伪类选择器**：以.开头的选择器等
5. **元素选择器、伪元素选择器**：如 div、p 等
6. **通用选择器**：* 选择器

<a id="8.2"></a>
### 8.2 优先级计算

- **内联样式**：1000
- **ID 选择器**：100
- **类选择器、属性选择器、伪类选择器**：10
- **元素选择器、伪元素选择器**：1
- **通用选择器**：0

计算示例：
- `#header .nav a`：100 + 10 + 1 = 111
- `.container div`：10 + 1 = 11
- `div`：1

<a id="9"></a>
## 9 . 选择器最佳实践

<a id="9.1"></a>
### 9.1 性能优化

- **避免使用过于复杂的选择器**：复杂的选择器会增加浏览器的渲染时间
- **优先使用类选择器**：类选择器比 ID 选择器和元素选择器更灵活，比复杂的后代选择器性能更好
- **避免使用通用选择器**：通用选择器会匹配所有元素，影响性能
- **避免使用过多的 ID 选择器**：ID 选择器优先级太高，不利于样式的复用

<a id="9.2"></a>
### 9.2 代码组织

- **使用语义化的类名**：类名应该反映元素的用途，而不是样式
- **使用 BEM 命名规范**：Block、Element、Modifier 的命名方式，提高代码的可维护性
- **避免嵌套过深**：选择器嵌套过深会降低代码的可读性和性能

<a id="9.3"></a>
### 9.3 示例

```css
/* 不好的做法 */
body > div > header > nav > ul > li > a {
  color: red;
}

/* 好的做法 */
.nav-link {
  color: red;
}

/* BEM 命名规范 */
.block {
  /* 块样式 */
}

.block__element {
  /* 元素样式 */
}

.block--modifier {
  /* 修改器样式 */
}
```

## 10 . 总结

CSS3 的选择器系统提供了丰富的选择元素的方式，使得开发者可以更加精确地控制样式的应用范围。通过掌握各种选择器的用法和优先级规则，你可以编写更加高效、可维护的 CSS 代码。

在实际应用中，你应该根据具体的需求选择合适的选择器，同时注意选择器的性能和代码的可维护性。通过不断学习和实践，你将能够熟练使用 CSS3 的选择器系统，创建出更加美观、响应式的网页设计。
