## 1. 基础概念类

### 1.1 C语言

**名称**：C语言（C Programming Language）

**首次出现位置**：C02_101-概述.md 第1章

**定义**：
C语言是由 Dennis Ritchie 于 1972 年在贝尔实验室开发的编译型编程语言，最初用于重写 UNIX 操作系统。它以其高效、灵活和可移植性著称，是现代编程语言的重要基础。

**详解**：
C语言的核心特点包括：简洁高效的语法（仅 32 个关键字）、直接内存访问能力（指针操作）、结构化编程支持、优秀的可移植性。C语言标准经历了 C89/C90、C99、C11、C17、C23 等多个版本的演进。几乎所有现代编程语言的运行时或虚拟机都是用 C/C++ 编写的。

---

### 1.2 Dennis Ritchie（丹尼斯·里奇）

**名称**：丹尼斯·里奇（Dennis Ritchie）

**首次出现位置**：C02_101-概述.md 第1章

**定义**：
Dennis Ritchie 是美国计算机科学家，于 1972 年在贝尔实验室发明了 C语言，并与 Brian Kernighan 合著了《The C Programming Language》（K&R C）。

**详解**：
Dennis Ritchie 与 Ken Thompson（UNIX 创始人）共同工作，C语言最初是为了重写 UNIX 操作系统而设计的。他的贡献奠定了现代软件工程的基础，C语言至今仍是系统编程、嵌入式开发等领域的首选语言。

---

### 1.3 编译器

**名称**：编译器（Compiler）

**首次出现位置**：C02_102-程序结构与语法.md 第5章

**定义**：
编译器是将源代码（如 C语言程序）转换为目标代码（机器码或字节码）的程序。GCC、Clang、MSVC 是常用的 C语言编译器。

**详解**：
编译过程包括四个阶段：预处理（处理 #include、#define 等指令）、编译（转换为汇编代码）、汇编（转换为机器码）、链接（合并目标文件生成可执行文件）。常用命令：`gcc -E` 预处理、`gcc -S` 编译、`gcc -c` 汇编、`gcc` 链接。

---

### 1.4 预处理

**名称**：预处理（Preprocessing）

**首次出现位置**：C02_102-程序结构与语法.md 第5章

**定义**：
预处理是编译的第一阶段，由预处理器执行，处理以 `#` 开头的指令，如 #include、#define、条件编译指令等。

**详解**：
预处理的主要任务：插入头文件内容（#include）、替换宏定义（#define）、条件编译（#if/#ifdef/#ifndef）、删除注释、添加行号和文件名标识。预处理输出通常保存为 `.i` 文件。预处理是纯文本替换，不进行语法检查。

---

### 1.5 链接

**名称**：链接（Linking）

**首次出现位置**：C02_102-程序结构与语法.md 第5章

**定义**：
链接是将多个目标文件（.o 或 .obj）合并成最终可执行文件的过程，解析外部符号引用，连接标准库和第三方库。

**详解**：
链接分为静态链接和动态链接。静态链接在编译时将库代码完整复制到最终程序，动态链接则只在运行时加载库文件。链接器（linker）的主要任务：符号解析（将符号引用与定义匹配）、重定位（调整符号地址）、创建可执行文件格式。链接错误如 "undefined reference" 表示有符号未定义。

---

### 1.6 头文件

**名称**：头文件（Header File）

**首次出现位置**：C02_102-程序结构与语法.md 第1章

**定义**：
头文件是以 `.h` 为扩展名的文件，包含函数声明、宏定义、结构体定义等，供其他源文件通过 #include 指令包含使用。

**详解**：
头文件的主要作用是声明接口（函数原型、类型定义），实现细节放在 `.c` 源文件中。头文件保护（#ifndef/#define/#endif）防止重复包含。常见头文件：stdio.h（标准输入输出）、stdlib.h（stdlib 函数）、string.h（字符串操作）、math.h（数学函数）。

---

### 1.7 标识符

**名称**：标识符（Identifier）

**首次出现位置**：C02_102-程序结构与语法.md 第3章

**定义**：
标识符是用于命名变量、函数、数组、结构体等程序元素的符号名称。命名规则：只能由字母、数字和下划线组成，首字符不能是数字，区分大小写，不能使用关键字。

**详解**：
命名约定：变量和函数用 snake_case（如 `user_age`、`calculate_area`）、常量全大写（如 `MAX_SIZE`、`PI`）、结构体用 PascalCase（如 `Student`、`Point`）、自定义类型加 `_t` 后缀（如 `uint8_t`）。好的命名应清晰描述用途，避免单字母（循环计数器 i/j/k 除外）和模糊缩写。

---

### 1.8 关键字

**名称**：关键字（Keyword）

**首次出现位置**：C02_102-程序结构与语法.md 第4章

**定义**：
关键字是 C语言预定义的具有特殊意义的保留词，不能用作标识符。C89 有 32 个关键字，C99 新增 5 个，C11 新增 7 个，C23 新增 3 个。

**详解**：
基本关键字分类：类型关键字（int、char、float、double、void 等）、控制流关键字（if、else、switch、case、for、while、do、break、continue、return、goto）、存储类关键字（auto、register、static、extern、const、volatile）、复合类型关键字（struct、union、enum、typedef）、其他（sizeof）。C99 新增：inline、restrict、_Bool、_Complex、_Imaginary。C11 新增：_Alignas、_Alignof、_Atomic、_Generic、_Noreturn、_Static_assert、_Thread_local。

---

## 2. 数据类型类

### 2.1 基本数据类型

**名称**：基本数据类型（Basic Data Types）

**首次出现位置**：C02_103-数据类型.md 第1章

**定义**：
C语言的基本数据类型包括整型（char、short、int、long、long long）和浮点型（float、double、long double），用于存储不同大小的数值数据。

**详解**：
整型大小：char（1字节）、short（2字节）、int（4字节）、long（4或8字节）、long long（8字节）。浮点型大小：float（4字节）、double（8字节）、long double（8-16字节）。格式说明符：%d（int）、%ld（long）、%lld（long long）、%f（float）、%lf（double）。类型修饰符 signed/unsigned 可改变有符号/无符号属性。

---

### 2.2 指针

**名称**：指针（Pointer）

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
指针是存储内存地址的变量。通过指针可以直接操作内存，实现动态内存分配、函数参数传递、数组操作等高级功能。

**详解**：
声明：`int *ptr;` ptr 是指向 int 类型的指针。取地址：`ptr = &x;` 获取变量 x 的地址。解引用：`*ptr = 20;` 通过指针修改所指向的内存。指针运算：`ptr + 1` 移动到下一个存储单元。空指针：`NULL` 或 `nullptr`（C23）。指针常用于：动态内存管理（malloc/free）、数组和字符串处理、函数参数传递（引用调用）、链表等数据结构。

---

### 2.3 数组

**名称**：数组（Array）

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
数组是相同类型数据的有序集合，通过索引（下标）访问元素。声明格式：`type name[size];`。

**详解**：
一维数组：`int nums[5] = {1, 2, 3, 4, 5};`。二维数组：`int matrix[3][3];`。数组名是指向首元素的常量指针。数组下标从 0 开始。数组作为函数参数时会退化为指针。数组不能直接赋值给另一数组，需要逐元素复制。C99 支持变长数组（VLA），C11 起变长数组变为可选特性。

---

### 2.4 结构体

**名称**：结构体（Structure）

**缩写**：struct

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
结构体是不同类型数据的组合，构成用户自定义的复合数据类型。用于表示具有多个属性的实体，如学生信息、坐标点等。

**详解**：
定义：`struct Point { int x; int y; };`。使用 typedef 为结构体起别名：`typedef struct { int x; int y; } Point;`。访问成员：`.` 操作符（对象）或 `->` 操作符（指针）。结构体内存布局遵循内存对齐规则。结构体可以嵌套定义，也可以包含数组和指针。

---

### 2.5 联合体

**名称**：联合体（Union）

**缩写**：union

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
联合体是一种特殊的数据类型，所有成员共享同一块内存空间。修改一个成员会影响其他所有成员。

**详解**：
定义：`union Data { int i; float f; char str[20]; };`。联合体大小等于最大成员的大小。适用于：节省内存（同一位置存储不同类型）、数据类型转换、网络协议解析（同一字节的不同位含义）。使用联合体时要注意当前活跃成员，避免读取未写入的成员。

---

### 2.6 枚举

**名称**：枚举（Enumeration）

**缩写**：enum

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
枚举为整数常量定义有意义的名称，提高代码可读性。默认从 0 开始，也可以显式指定值。

**详解**：
定义：`enum Weekday { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY };`。使用时 `enum Weekday today = WEDNESDAY;`。可以显式指定值：`enum Color { RED = 1, GREEN = 2, BLUE = 4 };`。枚举常量本质是 int 类型，可以直接作为整数使用。适用于定义一组相关的常量，如星期、月份、错误码等。

---

### 2.7 void 类型

**名称**：空类型（Void Type）

**首次出现位置**：C02_103-数据类型.md 第5章

**定义**：
void 表示无类型或空类型，用法包括：指定函数无返回值、指定函数无参数、声明通用指针（void*）。

**详解**：
无返回值函数：`void print_hello() { printf("Hello!\n"); }`。无参数函数：`int main(void) { ... }`。通用指针：`void *generic_ptr;` 可以指向任何类型，使用前需强制类型转换。void* 不能直接解引用，需要先转换。sizeof(void) 在 C中是未定义行为。

---

### 2.8 typedef

**名称**：类型定义（typedef）

**首次出现位置**：C02_103-数据类型.md 第7章

**定义**：
typedef 为现有类型创建别名，提高代码可读性和可移植性。常用于简化复杂类型定义、创建平台无关类型。

**详解**：
基本用法：`typedef unsigned int uint;`。为结构体创建别名：`typedef struct { int x; int y; } Point;`。标准库中的例子：size_t（unsigned int）、intptr_t（整数指针类型）。typedef 不创建新类型，只是现有类型的别名。与 #define 的区别：typedef 正确处理指针、语法分析，而 #define 是文本替换。

---

### 2.9 sizeof

**名称**：sizeof 运算符（sizeof）

**首次出现位置**：C02_103-数据类型.md 第3章

**定义**：
sizeof 是编译时运算符，计算数据类型或变量的字节大小。返回 size_t 类型（通常是 unsigned int 或 unsigned long）。

**详解**：
用法：`sizeof(int)`、`sizeof(x)`、`sizeof(array) / sizeof(array[0])`（计算数组元素个数）。sizeof 在编译时求值（对于固定大小类型），不会执行表达式。注意：strlen() 计算字符串长度，不包括末尾的 '\0'，而 sizeof 会包含 '\0'。

## 3. 内存管理类

### 3.1 内存对齐

**名称**：内存对齐（Memory Alignment）

**首次出现位置**：C02_103-数据类型.md 第3章

**定义**：
内存对齐是指变量按其类型大小的整数倍地址存放的规则。编译器会自动进行对齐优化，提高内存访问效率。

**详解**：
对齐规则：变量的地址是其对齐要求的倍数。char 可以任何地址，short 是 2 的倍数，int 通常是 4 的倍数。结构体总大小是其最大成员对齐要求的倍数。手动控制对齐：`#pragma pack(1)` 取消对齐，或 `__attribute__((aligned(n)))` 指定对齐。内存对齐是平台相关的，跨平台编程时需注意。

---

### 3.2 堆内存

**名称**：堆（Heap Memory）

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
堆是程序运行时动态分配的内存区域，通过 malloc/free（或 new/delete）手动管理，大小可在程序运行时决定。

**详解**：
分配函数：`malloc(size_t size)` 分配指定字节的内存，`calloc()` 分配并初始化为 0，`realloc()` 调整已分配内存大小。释放函数：`free(ptr)`。内存泄漏：malloc 后未 free 会导致内存逐渐耗尽。悬挂指针：free 后指针未置 NULL，再次使用会产生未定义行为。双重释放：同一块内存 free 两次会导致程序崩溃。

---

### 3.3 栈内存

**名称**：栈（Stack Memory）

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
栈是存储局部变量和函数调用信息的内存区域，由编译器自动管理，函数返回时自动释放。

**详解**：
栈的特点：LIFO（后进先出）结构、分配/释放速度快、空间有限（通常几MB）。局部变量（包括数组）存储在栈上。递归调用过深会导致栈溢出（stack overflow）。栈帧包含：返回地址、参数、局部变量、上一个栈帧指针。与堆相比，栈内存自动管理，无需手动释放。

---

### 3.4 内存泄漏

**名称**：内存泄漏（Memory Leak）

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
内存泄漏是指程序动态分配内存后，未能正确释放且丢失引用，导致内存无法回收，造成资源浪费。

**详解**：
常见原因：malloc 后忘记 free、函数返回栈地址（返回局部指针）、指针赋值覆盖导致原内存不可访问。检测工具：Valgrind（Linux）、AddressSanitizer（gcc/clang 内置）。预防措施：遵循谁分配谁释放原则、使用智能指针（C++）、建立清晰的内存管理规范。长期运行的程序尤其要注意内存泄漏。

---

## 4. 运算符与表达式类

### 4.1 算术运算符

**名称**：算术运算符（Arithmetic Operators）

**首次出现位置**：C02_105-运算符与表达式.md 第1章

**定义**：
算术运算符包括：加（+）、减（-）、乘（*）、除（/）、取模（%）。用于执行基本的数学运算。

**详解**：
整数除法（/）会截断小数部分，如 5/2 = 2（不是 2.5）。取模（%）要求两边操作数均为整数，结果与被除数符号相同。C语言没有指数运算符，需使用 pow() 函数（math.h）。算术表达式遵循优先级规则：乘除取模高于加减，括号可改变优先级。

---

### 4.2 关系运算符

**名称**：关系运算符（Relational Operators）

**首次出现位置**：C02_105-运算符与表达式.md 第1章

**定义**：
关系运算符包括：小于（<）、小于等于（<=）、大于（>）、大于等于（>=）、等于（==）、不等于（!=）。用于比较两个值的大小关系。

**详解**：
关系运算符的结果是 1（真）或 0（假）。注意 == 与 = 的区别：== 是比较，= 是赋值。常见错误：`if (x = 5)` 实际是赋值 5 给 x，而非比较。浮点数比较应使用 epsilon 范围判断，因浮点数精度问题直接比较可能不可靠。

---

### 4.3 逻辑运算符

**名称**：逻辑运算符（Logical Operators）

**首次出现位置**：C02_105-运算符与表达式.md 第1章

**定义**：
逻辑运算符包括：逻辑与（&&）、逻辑或（||）、逻辑非（!）。用于组合布尔条件。

**详解**：
&& 和 || 具有短路特性：&& 左边为假则不计算右边，|| 左边为真则不计算右边。非（!）将非零值转为 0，零值转为 1。注意 & 与 && 的区别：& 是按位与，&& 是逻辑与。类似地 | 与 || 的区别也是按位或与逻辑或。

---

### 4.4 位运算符

**名称**：位运算符（Bitwise Operators）

**首次出现位置**：C02_105-运算符与表达式.md 第1章

**定义**：
位运算符直接操作二进制位：按位与（&）、按位或（|）、按位异或（^）、取反（~）、左移（<<）、右移（>>）。

**详解**：
位运算常用于：设置/清除/测试特定位、位掩码操作、高效乘除（x<<1 等价于 x*2）、交换两数（不需临时变量：`a ^= b; b ^= a; a ^= b;`）。有符号数右移行为取决于实现：算术右移（符号位填充）或逻辑右移（0填充）。unsigned 类型建议使用左移右移，避免未定义行为。

---

### 4.5 条件运算符

**名称**：条件运算符（Conditional Operator）

**首次出现位置**：C02_105-运算符与表达式.md 第1章

**定义**：
条件运算符（?:）是 C语言中唯一的三目运算符，语法：`expr1 ? expr2 : expr3`。如果 expr1 为真（非零）则执行 expr2，否则执行 expr3。

**详解**：
可以嵌套使用，但影响可读性。适用于简单条件赋值：`max = a > b ? a : b;`。expr2 和 expr3 类型不同时，结果类型是两者中较"普通"的那个（整数提升、寻常算术转换规则）。不建议在表达式内部修改变量，如 `a > 0 ? ++b : --b`。

---

### 4.6 赋值运算符

**名称**：赋值运算符（Assignment Operators）

**首次出现位置**：C02_105-运算符与表达式.md 第1章

**定义**：
赋值运算符（=）将右侧值赋给左侧变量。复合赋值运算符包括：+=、-=、*=、/=、%=、<<=、>>=、&=、|=、^=。

**详解**：
赋值表达式有返回值，即赋的值本身，支持链式赋值：`a = b = c = 5;`。复合赋值如 `a += 3;` 等价于 `a = a + 3;`，但更简洁且可能更高效。赋值运算符左值不能是表达式（如数组名），只能是可修改的变量。

---

## 5. 控制流类

### 5.1 if-else 语句

**名称**：if-else 语句（if-else Statement）

**首次出现位置**：C02_106-控制流.md 第1章

**定义**：
if-else 是基本的选择结构，根据条件真伪决定执行哪个代码块。

**详解**：
语法：`if (condition) statement;` 或 `if (condition) statement1; else statement2;`。else 总是与最近的 if 配对（除非使用花括号明确分组）。条件判断非零即为真，零为假。建议始终使用花括号，即使只有一条语句，提高可读性和可维护性。

---

### 5.2 switch 语句

**名称**：switch 语句（switch Statement）

**首次出现位置**：C02_106-控制流.md 第1章

**定义**：
switch 是多分支选择结构，根据表达式的值跳转到匹配的 case 分支执行。

**详解**：
语法：switch 表达式必须是整数类型（char、int、enum），case 标签必须是常量表达式。break 使程序跳出 switch，否则会"穿透"到下一个 case。建议每个 case 结尾都有 break，包括最后一个。default 分支处理未匹配的情况，可放置在任意位置。

---

### 5.3 while 循环

**名称**：while 循环（while Loop）

**首次出现位置**：C02_106-控制流.md 第2章

**定义**：
while 循环在条件为真时重复执行代码块，先判断条件后执行，可能一次都不执行。

**详解**：
语法：`while (condition) { statements; }`。循环前需确保变量已正确初始化。避免死循环：检查循环体内是否有改变条件判断表达式的语句。常见模式：`while (scanf("%d", &num) == 1)` 持续读取直到输入非数字。

---

### 5.4 for 循环

**名称**：for 循环（for Loop）

**首次出现位置**：C02_106-控制流.md 第2章

**定义**：
for 循环是功能最强的循环结构，包含初始化、条件判断、更新三个部分，适合计数循环。

**详解**：
语法：`for (init; condition; update) { statements; }`。三个部分均可省略，分号不能省。init 中可同时声明循环变量（C99）：`for (int i = 0; i < n; i++)`。循环变量建议使用短整型或整型，避免浮点数作为循环变量（精度问题）。C99 支持复杂循环：`for (int i = 0, j = n-1; i < j; i++, j--)`。

---

### 5.5 do-while 循环

**名称**：do-while 循环（do-while Loop）

**首次出现位置**：C02_106-控制流.md 第2章

**定义**：
do-while 是后测试循环，先执行循环体后判断条件，保证循环体至少执行一次。

**详解**：
语法：`do { statements; } while (condition);`。注意条件后有分号。适用场景：显示菜单（先显示后获取选择）、读取输入直到有效值。与 while 循环的区别是 do-while 至少执行一次，while 可能一次都不执行。

---

### 5.6 break 语句

**名称**：break 语句（break Statement）

**首次出现位置**：C02_106-控制流.md 第3章

**定义**：
break 用于提前退出循环（for、while、do-while）或 switch 语句。

**详解**：
break 只退出最内层循环，多层循环嵌套时需使用 goto 或设置标志变量。注意 break 在 switch 中的作用是退出 switch 而非整个循环。死循环常用形式：`while (1) { ... if (condition) break; ... }`。

---

### 5.7 continue 语句

**名称**：continue 语句（continue Statement）

**首次出现位置**：C02_106-控制流.md 第3章

**定义**：
continue 跳过本次循环剩余语句，直接进入下一次循环判断。

**详解**：
在 for 循环中，continue 后会执行更新表达式；在 while/do-while 中会执行条件判断。常用于跳过某些特定值：`for (i = 0; i < n; i++) { if (i == special_value) continue; /* 处理其他值 */ }`。与 break 的区别：break 完全退出循环，continue 仅跳过本次迭代。

---

### 5.8 goto 语句

**名称**：goto 语句（goto Statement）

**首次出现位置**：C02_106-控制流.md 第3章

**定义**：
goto 是无条件跳转语句，跳转到同一函数内用标签标记的位置。现代 C编程中 goto 的使用有争议。

**详解**：
语法：标签定义 `label:`，跳转 `goto label;`。合理用途：跳出多层嵌套循环（goto 比多个 break 清晰）、错误处理（集中清理资源）。不良用途：代替正常控制结构导致 spaghetti code。C标准禁止跨函数跳转。多数情况下可以用 if-else、循环、函数调用替代 goto。

---

## 6. 函数类

### 6.1 函数

**名称**：函数（Function）

**首次出现位置**：C02_107-函数.md 第1章

**定义**：
函数是组织代码的单元，完成特定任务，可被多次调用。C程序由函数组成，main 是程序入口。

**详解**：
函数定义包含：返回类型、函数名、参数列表、函数体。参数传递是值传递，函数内部修改不影响实参（除非传递指针或数组）。无参函数参数列表应写 void 表示无参数。函数应先声明后使用，或将声明放在头文件中。模块化设计：每个函数尽量只完成一个功能。

---

### 6.2 函数声明

**名称**：函数声明（Function Declaration）

**首次出现位置**：C02_107-函数.md 第1章

**定义**：
函数声明（函数原型）告诉编译器函数的接口：返回类型、函数名、参数类型列表，但不包含函数体。

**详解**：
语法：`return_type function_name(parameter_list);`。参数列表只需类型，不需要参数名：`int func(int, float);`。函数声明让编译器检查调用是否正确，提供类型安全。头文件中通常存放函数声明，实现在源文件中。全局函数应在使用前声明。

---

### 6.3 函数调用

**名称**：函数调用（Function Call）

**首次出现位置**：C02_107-函数.md 第1章

**定义**：
函数调用是执行函数代码的指令，语法：`function_name(arg1, arg2, ...)`。

**详解**：
调用过程：实参求值 → 参数传递 → 进入函数体执行 → 返回值（如有）→ 继续执行调用后代码。递归调用：函数调用自身，需有终止条件防止无限递归。函数指针：`int (*func_ptr)(int, int);` 可存储函数地址，实现回调或动态调用。

---

### 6.4 递归

**名称**：递归（Recursion）

**首次出现位置**：C02_107-函数.md 第4章

**定义**：
递归是函数调用自身的技术，每一层调用都有独立的参数和局部变量，适合处理分形结构和递归定义的问题（如汉诺塔、树的遍历）。

**详解**：
递归必须包含：递归调用（收敛向基本情况）和基本情况（终止条件）。递归比循环效率低（函数调用开销），但有时更简洁易懂。尾递归优化：某些编译器可将尾递归转为循环。阶乘示例：`int factorial(int n) { return n <= 1 ? 1 : n * factorial(n-1); }`。

---

### 6.5 指针函数

**名称**：指针函数（Pointer Function）

**首次出现位置**：C02_107-函数.md 第3章

**定义**：
指针函数是返回指针的函数，而非普通返回值。常用于返回字符串、动态分配的内存块，或在函数中返回数组。

**详解**：
声明：`char *strcpy(char *dest, const char *src);`。返回指针的函数，返回的必须是：静态局部变量的地址、全局变量的地址、通过参数传入的指针、动态分配的内存（由调用者负责释放）。返回局部变量地址是错误：`int *func() { int x; return &x; }` 局部变量在函数返回后失效。

---

### 6.6 函数指针

**名称**：函数指针（Function Pointer）

**首次出现位置**：C02_107-函数.md 第3章

**定义**：
函数指针是指向函数的指针变量，可以存储函数的地址，通过指针调用函数，实现回调机制和多态。

**详解**：
声明：`int (*func_ptr)(int, int);` func_ptr 是指向返回 int、接受两个 int 参数的函数的指针。赋值：`func_ptr = some_function;` 或 `func_ptr = &some_function;`。调用：`(*func_ptr)(a, b);` 或 `func_ptr(a, b);`。用途：回调函数、函数表、命令模式、延迟绑定。

---

## 7. 文件操作类

### 7.1 文件 I/O

**名称**：文件输入输出（File Input/Output）

**首次出现位置**：C02_111-文件IO.md 第1章

**定义**：
C语言通过标准库 stdio.h 提供文件 I/O 功能，使用 FILE 结构体操作文件，支持文本和二进制模式。

**详解**：
打开文件：`FILE *fopen(const char *filename, const char *mode);`。模式：r（读）、w（写，清空）、a（追加）、rb/wb/ab（ binary 模式）。关闭文件：`int fclose(FILE *stream);`。常用函数：fprintf()、fscanf()（格式化）、fgets()、fputs()（文本行）、fread()、fwrite()（二进制数据块）。

---

### 7.2 标准输入输出

**名称**：标准输入输出（Standard Input/Output）

**缩写**：stdio

**首次出现位置**：C02_111-文件IO.md 第1章

**定义**：
stdio 是 C语言标准库的一部分，提供 printf()、scanf()、getchar()、putchar() 等函数，用于控制台 I/O 操作。

**详解**：
stdout 是标准输出流（默认屏幕），stderr 是标准错误流。C语言 I/O 支持格式化输出：%d（十进制整数）、%s（字符串）、%f（浮点数）、%c（字符）、%p（指针地址）。getchar() 从 stdin 读取一个字符，返回 int（EOF 时返回 -1）。建议使用 fgets() 替代 gets()，避免缓冲区溢出。

---

### 7.3 fprintf/fscanf

**名称**：格式化文件输入输出（File Printf/Scanf）

**首次出现位置**：C02_111-文件IO.md 第2章

**定义**：
fprintf() 将格式化数据写入文件，fscanf() 从文件读取格式化数据。

**详解**：
语法：`int fprintf(FILE *stream, const char *format, ...);`、`int fscanf(FILE *stream, const char *format, ...);`。示例：`fprintf(fp, "Name: %s, Age: %d\n", name, age);`。返回值是写入/读取的字符数，出错返回 EOF。格式化字符串与 printf/scanf 相同。

---

### 7.4 fread/fwrite

**名称**：二进制文件读写（File Read/Write）

**首次出现位置**：C02_111-文件IO.md 第2章

**定义**：
fread() 从文件读取数据块，fwrite() 向文件写入数据块，用于二进制文件操作。

**详解**：
语法：`size_t fread(void *ptr, size_t size, size_t nmemb, FILE *stream);`、`size_t fwrite(const void *ptr, size_t size, size_t nmemb, FILE *stream);`。参数：ptr 是缓冲区指针、size 是每个元素大小、nmemb 是元素数量。返回实际读取/写入的元素数量（非字节数）。适合读写结构体数组或二进制数据。

---

## 8. 存储类与修饰符类

### 8.1 auto

**名称**：自动存储类（auto）

**首次出现位置**：C02_102-程序结构与语法.md 第4章

**定义**：
auto 是默认的局部变量存储类，表明变量存储在栈上，函数返回时自动销毁。

**详解**：
现代 C编程中 auto 关键字几乎不使用，因为局部变量默认就是 auto。auto 仅用于明确表达意图，与 C++ 中的 auto（类型推导）完全不同。auto 变量在进入作用域时创建，退出作用域时销毁，每次调用函数都会重新初始化。

---

### 8.2 register

**名称**：寄存器存储类（register）

**首次出现位置**：C02_103-数据类型.md 第2章

**定义**：
register 是建议编译器将变量存储在寄存器中以提高访问速度，适用于频繁访问的变量如循环计数器。

**详解**：
现代编译器会自动优化寄存器使用，register 关键字更多是表达"频繁访问"的语义。register 变量不能取地址（&运算符不能用于寄存器变量）。仍可使用：register int i; for (i = 0; i < n; i++)。不能用于全局变量或函数参数。

---

### 8.3 static

**名称**：静态存储类（static）

**首次出现位置**：C02_102-程序结构与语法.md 第4章

**定义**：
static 关键字根据上下文改变变量或函数的作用域和生命周期：局部变量使变量生命周期为整个程序运行期，文件级变量使作用域限于当前文件。

**详解**：
局部静态变量：函数内声明，程序运行期间存在，只初始化一次，函数调用间保持值。全局静态变量/函数：文件内可见，不可被其他文件通过 extern 访问，用于信息隐藏。静态变量默认初始化为 0。静态内部链接：仅在定义它的翻译单元中可见。

---

### 8.4 extern

**名称**：外部链接（extern）

**首次出现位置**：C02_102-程序结构与语法.md 第4章

**定义**：
extern 用于声明变量或函数具有外部链接（可被其他文件访问），表示在别处定义。

**详解**：
extern 变量声明：`extern int global_var;` 表示在别处定义的全局变量。函数声明默认具有 extern 链接，可省略。extern 不分配存储空间，只是声明。使用：其他文件定义的全局变量/函数，通过 extern 声明后可在当前文件使用。避免重复定义：定义只能一次，声明可以多次。

---

### 8.5 const

**名称**：常量（const）

**首次出现位置**：C02_103-数据类型.md 第2章

**定义**：
const 修饰符声明变量为只读，不可修改。用于定义程序中不应改变的值，提高代码可读性和安全性。

**详解**：
基本用法：`const int MAX = 100;` 或 `int const MAX = 100;`。const 与指针：const int *p（p 指向的 int 不可变）、int * const p（p 指针不可变）、const int * const p（两者都不可变）。const 修饰函数参数：`void func(const char *str)` 表明函数不会修改 str 指向的内容。const 变量必须在声明时初始化。

---

### 8.6 volatile

**名称**：易变（volatile）

**首次出现位置**：C02_103-数据类型.md 第2章

**定义**：
volatile 修饰符告诉编译器不要对该变量进行优化，因为它的值可能在意料之外被改变（如硬件寄存器、多线程、中断服务程序）。

**详解**：
常见用途：硬件寄存器映射的变量、多线程共享变量、中断服务程序中的变量。volatile 确保每次访问都从内存读取最新值，而不是使用寄存器缓存。示例：`volatile uint8_t *reg = (volatile uint8_t *)0x40021000;`。可以与 const 一起用：`volatile const int sensor_value;`。

---

## 9. 高级特性类

### 9.1 变长参数

**名称**：可变参数（Variable Arguments）

**首次出现位置**：C02_107-函数.md 第3章

**定义**：
变长参数允许函数接受不确定数量的参数，通过 stdarg.h 中的宏（va_start、va_arg、va_end）访问。

**详解**：
头文件：stdarg.h。声明：`int sum(int count, ...);`。使用步骤：定义 va_list 类型变量、va_start(list, 最后一个固定参数)、va_arg(list, 类型)、va_end(list) 清理。必须至少有一个固定参数来确定参数列表起始位置。va_copy 复制 va_list。注意可变参数类型安全由调用者保证。

---

### 9.2 命令行参数

**名称**：命令行参数（Command Line Arguments）

**首次出现位置**：C02_102-程序结构与语法.md 第6章

**定义**：
C语言 main 函数可以接受命令行参数：`int main(int argc, char *argv[])`。argc 是参数个数，argv 是参数数组。

**详解**：
argv[0] 是程序名称，argv[1] 到 argv[argc-1] 是实际参数，argv[argc] 是 NULL。Windows 命令行参数通过运行时库传递。常见用法：文件处理（`int main(int argc, char *argv[])`）、开关选项解析。示例：`int main(int argc, char *argv[]) { for (int i = 0; i < argc; i++) printf("%s\n", argv[i]); }`。

---

### 9.3 字符串

**名称**：字符串（String）

**首次出现位置**：C02_103-数据类型.md 第4章

**定义**：
C语言中字符串是以 '\0'（空字符）结尾的字符数组。字符串字面量（如 "hello"）存储在常量区，不可修改。

**详解**：
字符串声明：`char str[] = "hello";`（可修改副本）、`char *str = "hello";`（指向常量区的指针）。字符串处理函数（string.h）：strlen()（长度，不含\0）、strcpy()（复制）、strcat()（追加）、strcmp()（比较）、strchr()（查找字符）、strstr()（查找子串）。推荐使用 strncpy、strncat、strncmp 指定长度，避免缓冲区溢出。

---

### 9.4 作用域

**名称**：作用域（Scope）

**首次出现位置**：C02_107-函数.md 第2章

**定义**：
作用域是程序中标识符（变量、函数名）可见的区域。C语言有四种作用域：块作用域、函数作用域、函数原型作用域、文件作用域。

**详解**：
块作用域：花括号内的区域，if、for、while 等语句也创建块作用域。函数作用域：goto 标签在函数内任何地方都可见。函数原型作用域：函数原型中的参数名可省略。C语言没有嵌套函数作用域（函数内不能定义函数）。同名标识符内层遮蔽外层。
