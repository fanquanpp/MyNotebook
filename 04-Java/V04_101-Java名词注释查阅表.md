## 1. 基础概念类

### 1.1 Java

**名称**：Java（爪哇）

**首次出现位置**：C04_101-概述与环境.md 第1章

**定义**：
Java 是由 Sun Microsystems 于 1995 年发布的面向对象编程语言，具有"一次编写，到处运行"的跨平台特性。

**详解**：
Java 核心特性：面向对象（一切皆对象）、强类型、平台无关（JVM）、自动内存管理（GC）、异常处理、多线程、安全性。版本演进：Java 1.0 → Java 8（lambda、Stream）→ Java 11（模块化）→ Java 17（LTS）→ Java 21（虚拟线程）。应用领域：企业级应用（Spring）、Android 开发、大数据（Hadoop/Spark）、Web 后端。Java 属于编译型+解释型，源码编译为字节码，JVM 解释执行。

---

### 1.2 James Gosling（詹姆斯·高斯林）

**名称**：詹姆斯·高斯林（James Gosling）

**首次出现位置**：C04_101-概述与环境.md 第1章

**定义**：
James Gosling 是加拿大计算机科学家，Java 语言的创始人，被称为"Java 之父"，主导了 Java 的设计和实现。

**详解**：
高斯林于 1991 年在 Sun Microsystems 开始开发 Java 的前身 Oak 语言，1995 年正式发布 Java。他对 Java 的设计理念：简单（比 C++ 简单）、面向对象、分布式、健壮、安全、体系结构中立、可移植、高性能、多线程、动态性。2006 年 Sun 将 Java 开源（OpenJDK）。

---

### 1.3 JDK（Java 开发工具包）

**名称**：Java 开发工具包（Java Development Kit）

**缩写**：JDK

**首次出现位置**：C04_101-概述与环境.md 第3章

**定义**：
JDK 是 Java 开发者使用的软件开发工具包，包含 JRE（Java 运行时环境）、编译器（javac）、调试器（jdb）、文档生成器（javadoc）等开发工具。

**详解**：
JDK 组成：javac（编译器）、java（启动器）、javadoc（文档生成）、jar（打包）、javap（反汇编）、jdb（调试器）。组件：JRE（Java 运行时环境）+ 开发工具。版本示例：JDK 17、JDK 21（LTS 版本）。JDK 下载地址：Oracle 官网、Adoptium（开源）。环境变量：JAVA_HOME 指向 JDK 根目录，PATH 添加 %JAVA_HOME%\bin。

---

### 1.4 JRE（Java 运行时环境）

**名称**：Java 运行时环境（Java Runtime Environment）

**缩写**：JRE

**首次出现位置**：C04_101-概述与环境.md 第3章

**定义**：
JRE 是运行 Java 程序所需的最小环境，包含 JVM（Java 虚拟机）和 Java 类库，不包含开发工具。

**详解**：
JRE = JVM + Java 类库。用户只需运行 Java 程序（不开发）只需安装 JRE。JVM 是 Java 跨平台的核心，字节码在不同平台的 JVM 上运行。JVM 本身不是跨平台的（不同平台有不同 JVM 实现）。Server JRE vs JRE：Server JRE 针对服务器优化。

---

### 1.5 JVM（Java 虚拟机）

**名称**：Java 虚拟机（Java Virtual Machine）

**缩写**：JVM

**首次出现位置**：C04_101-概述与环境.md 第3章

**定义**：
JVM 是运行 Java 字节码的虚拟计算机，提供内存管理、线程管理、即时编译（JIT）、垃圾回收等运行时功能。

**详解**：
JVM 组成：类加载器（ClassLoader）、运行时数据区（Method Area、Heap、Java Stack、PC Register、Native Method Stack）、执行引擎（Interpreter、JIT Compiler、GC）。JIT 编译：热点代码即时编译为本地机器码，提高执行效率。常见 JVM：HotSpot（Oracle）、OpenJ9（IBM）、GraalVM。JVM 参数：-Xms（初始堆）、-Xmx（最大堆）、-XX:+UseG1GC（G1 垃圾回收器）。

---

### 1.6 字节码

**名称**：字节码（Bytecode）

**首次出现位置**：C04_101-概述与环境.md 第3章

**定义**：
字节码是 Java 编译器生成的中间代码，存储在 .class 文件中，由 JVM 解释执行或 JIT 编译后执行。

**详解**：
编译流程：.java 源码 → javac 编译 → .class 字节码 → JVM 执行。字节码是平台无关的指令集，类似于汇编但更抽象。每个 class 文件包含：魔数（0xCAFEBABE）、版本信息、常量池、类信息、字段表、方法表、属性表。字节码查看：javap -c ClassName 反编译。ASM、CGLIB 等库直接操作字节码实现 AOP 和动态代理。

---

### 1.7 API（应用程序接口）

**名称**：应用程序接口（Application Programming Interface）

**缩写**：API

**首次出现位置**：C04_101-概述与环境.md 第5章

**定义**：
API 是应用程序向外提供的编程接口，Java API 是 Java 平台提供的预定义类和接口，供开发者使用。

**详解**：
Java API 文档（JavaDoc）是学习和使用 Java 的核心参考资料。常用 API 包：java.lang（核心类，String、Object、System 自动导入）、java.util（集合框架、工具类）、java.io（输入输出）、java.net（网络编程）、java.time（日期时间，Java 8+）、java.stream（Stream API）。API 版本：关注新增和弃用 API。Guava、Apache Commons 是常用第三方 API 库。

---

### 1.8 垃圾回收器（GC）

**名称**：垃圾回收器（Garbage Collector）

**缩写**：GC

**首次出现位置**：C04_115-JVM内存模型.md 第4章

**定义**：
GC 是 JVM 的自动内存管理机制，自动回收不再使用的对象占用的内存，防止内存泄漏。

**详解**：
GC 算法：标记-清除（Mark-Sweep）、复制（Copying）、标记-整理（Mark-Compact）、分代收集（Generational）。分代假说：大多数对象朝生夕灭。堆分区：Young（Eden + Survivor）、Old/Tenured、Metaspace（非堆）。GC 类型：Minor GC（年轻代）、Major GC（老年代）、Full GC（整个堆）。回收器：Serial、Parallel、CMS、G1（默认）、ZGC、Shenandoah。GC 日志：-XX:+PrintGCDetails。

## 2. 数据类型类

### 2.1 基本数据类型

**名称**：基本数据类型（Primitive Data Types）

**首次出现位置**：C04_103-数据类型与类型转换.md 第1章

**定义**：
Java 有 8 种基本数据类型：byte、short、int、long、float、double、boolean、char，用于存储简单数值。

**详解**：
整数类型：byte（1字节 -128~127）、short（2字节）、int（4字节，约21亿）、long（8字节）。浮点类型：float（4字节单精度）、double（8字节双精度）。布尔类型：boolean（true/false，JVM 实现决定大小）。字符类型：char（2字节 Unicode）。默认值：byte/short/int/long=0、float=0.0f、double=0.0d、boolean=false、char='\u0000'。基本类型 vs 包装类：int vs Integer。

---

### 2.2 引用数据类型

**名称**：引用数据类型（Reference Data Types）

**首次出现位置**：C04_103-数据类型与类型转换.md 第1章

**定义**：
引用类型存储对象的内存地址（或引用），而非对象本身，包括类、接口、数组、枚举、字符串等。

**详解**：
引用存储在 JVM 栈帧的局部变量表中，对象实例存储在堆中。引用类型变量默认值是 null。String 是引用类型，存储字符串内容但行为类似基本类型。数组是对象：`int[] arr = new int[5];`。泛型类型参数必须是引用类型：List<int> 非法，需 List<Integer>。自动装箱/拆箱简化基本类型和包装类转换。

---

### 2.3 包装类

**名称**：包装类（Wrapper Classes）

**首次出现位置**：C04_103-数据类型与类型转换.md 第2章

**定义**：
包装类是基本类型的对象封装类，提供将基本类型转为对象的方法，以及常量和工具方法。

**详解**：
对应关系：byte→Byte、short→Short、int→Integer、long→Long、float→Float、double→Double、boolean→Boolean、char→Character。常用方法：Integer.parseInt(str)、Integer.valueOf(str)（缓存）、Integer.toBinaryString()、Character.isDigit()、Character.isLetter()。自动装箱/拆箱：Integer i = 10;（装箱）、int n = i;（拆箱）。常量：Integer.MAX_VALUE、Integer.MIN_VALUE、Double.NaN、Double.POSITIVE_INFINITY。

---

### 2.4 类型转换

**名称**：类型转换（Type Casting）

**首次出现位置**：C04_103-数据类型与类型转换.md 第3章

**定义**：
类型转换是将一种数据类型转换为另一种，包括自动类型转换（隐式）和强制类型转换（显式）。

**详解**：
自动转换（隐式）：小类型→大类型、单一方向（byte→short→int→long→float→double）。精度问题：int→float/long→float/long→double 可能丢失精度。强制转换（显式）：`(targetType) value`，如 `int i = (int) 3.14`（截断为 3）。字符串转换：String.valueOf()、Integer.toString()、""+value。向上转型/向下转型：父类引用指向子类对象（自动）/强制转换回子类。instanceof 运算符在转型前检查类型。

---

### 2.5 字符串（String）

**名称**：字符串（String）

**首次出现位置**：C04_102-程序结构与基础语法.md 第5章

**定义**：
String 是 Java 表示文本的数据类型，本质上是由字符数组实现的对象，存储在字符串常量池中。

**详解**：
创建方式：`String s1 = "hello";`（字符串常量池）、`String s2 = new String("hello");`（堆内存）。不可变性：String 是不可变类（final），每次修改创建新对象。常用方法：length()、charAt()、substring()、indexOf()、lastIndexOf()、equals()（比较内容）、compareTo()（字典序）、split()、trim()、replace()、toLowerCase()、toUpperCase()。字符串拼接：+、concat()、StringBuilder（高效）、StringBuffer（线程安全）。

---

### 2.6 常量池

**名称**：字符串常量池（String Constant Pool / Intern Pool）

**首次出现位置**：C04_102-程序结构与基础语法.md 第5章

**定义**：
字符串常量池是 JVM 堆内存中的特殊区域，存储字符串字面量，相同内容的字符串字面量共享同一引用。

**详解**：
字符串字面量（如 "hello"）直接存在常量池。new String() 每次创建新对象。intern() 方法将字符串放入常量池并返回池中引用。好处：节省内存、提高性能。示例：`String a = "hello"; String b = "hello"; a == b` 为 true（共享池中同一对象）。不同 JDK 版本常量池位置可能变化：Java 7 之前在 PermGen，Java 7+ 在堆中。

---

### 2.7 关键字

**名称**：关键字（Keyword）

**首次出现位置**：C04_102-程序结构与基础语法.md 第3章

**定义**：
关键字是 Java 语言预定义的具有特殊含义的保留词，不能用作标识符。

**详解**：
访问修饰符：public、protected、private。类、方法、变量修饰符：class、interface、enum、abstract、static、final、transient、volatile、synchronized、native。流程控制：if、else、switch、case、default、for、while、do、break、continue、return。异常处理：try、catch、finally、throw、throws、assert。包相关：package、import。其他：new、this、super、void、instanceof、const、goto（const 和 goto 是保留字但未使用）。true、false、null 也是保留字。

---

### 2.8 标识符

**名称**：标识符（Identifier）

**首次出现位置**：C04_102-程序结构与基础语法.md 第3章

**定义**：
标识符是用于命名包、类、接口、方法、变量等的名称，必须以字母、_ 或 $ 开头，后续字符可以是字母、数字、_、$。

**详解**：
命名规范（驼峰命名法）：类/接口：首字母大写，如 UserService。变量/方法：首字母小写，如 userName、getUserById()。常量：全大写下划线分隔，如 MAX_SIZE、TIMEOUT_MS。包名：全小写，如 com.example.project。命名建议：见名知意、避免缩写、避免数字开头和 Java 关键字。Java 9+ 允许使用 Unicode 字符命名，但建议 ASCII 字符。

## 3. 面向对象类

### 3.1 类（class）

**名称**：类（Class）

**首次出现位置**：C04_109-面向对象基础.md 第1章

**定义**：
类是 Java 面向对象的核心，是创建对象的模板或蓝图，定义对象的属性（字段）和行为（方法）。

**详解**：
类的组成：字段（属性/状态）、方法（行为）、构造器、初始化块、内部类、注解。定义语法：`[修饰符] class 类名 [extends 父类] [implements 接口] { 成员 }`。类的访问修饰符：public（公共）、默认（包级私有）。每个.java 文件最多一个 public 类，类名与文件名一致。对象通过 new 和构造器创建。

---

### 3.2 对象（object）

**名称**：对象（Object）

**首次出现位置**：C04_109-面向对象基础.md 第1章

**定义**：
对象是类的实例，具有状态（属性值）、行为（方法）和唯一身份（内存地址）。

**详解**：
对象存储在堆内存中，引用存储在栈内存中。创建对象：`new ClassName()`，调用构造器。对象生命周期：创建（new）→ 使用 → 无引用（GC 回收）。Object 是所有类的父类，提供 toString()、equals()、hashCode()、getClass() 等方法。对象比较：== 比较引用、equals() 比较内容（需重写）。

---

### 3.3 构造器（constructor）

**名称**：构造器（Constructor）

**首次出现位置**：C04_109-面向对象基础.md 第2章

**定义**：
构造器是用于创建和初始化对象的方法，与类名相同，没有返回类型，用于 new 表达式中。

**详解**：
默认构造器：如果没有定义任何构造器，编译器生成无参构造器。构造器重载：一个类可以有多个构造器，参数列表不同。this() 调用：同一类的另一个构造器，必须在第一行。super() 调用：隐式或显式调用父类构造器，必须在构造器第一行。构造器不能被继承、不能被 static/final/subclass 修饰。

---

### 3.4 this 关键字

**名称**：this 引用（this）

**首次出现位置**：C04_109-面向对象基础.md 第2章

**定义**：
this 是隐式引用，指向当前正在执行方法的对象，用于在类内部引用对象本身。

**详解**：
用途：区分字段和局部变量（this.field = param）、在构造器中调用其他构造器（this(args)）、将当前对象作为参数传递（return this）、返回当前对象引用（return this）。static 方法中不能使用 this。this 不能被赋值或声明。构造器中的 this() 和 super() 不能同时出现。

---

### 3.5 super 关键字

**名称**：super 引用（super）

**首次出现位置**：C04_109-面向对象基础.md 第2章

**定义**：
super 是指向父类对象的引用，用于在子类中访问父类的成员（字段、方法、构造器）。

**详解**：
super.field：访问父类字段（当子类遮蔽时）。super.method()：调用父类方法（当子类重写时）。super()：调用父类构造器（必须在构造器第一行）。super(param)：调用父类有参构造器。隐式调用：如果没有显式调用 super()，编译器自动插入无参 super()。this() 和 super() 不能同时出现在构造器第一行。

---

### 3.6 继承（inheritance）

**名称**：继承（Inheritance）

**首次出现位置**：C04_109-面向对象基础.md 第3章

**定义**：
继承是面向对象的重要特性，允许类（子类）继承另一个类（父类）的属性和方法，实现代码复用和类型层次。

**详解**：
关键字：extends（类继承）、implements（接口实现）。单继承：Java 类只能有一个直接父类（Object 是所有类的祖先）。子类可以添加自己的字段和方法。子类可以重写（override）父类方法。子类构造器必须调用父类构造器（显式或隐式）。类型层次：is-a 关系，子类对象也是父类类型。

---

### 3.7 方法重写（override）

**名称**：方法重写（Method Overriding）

**首次出现位置**：C04_109-面向对象基础.md 第3章

**定义**：
方法重写是子类提供父类方法的特定实现，改变父类方法的行为，要求方法签名完全相同。

**详解**：
重写规则：方法名、参数列表、返回类型（协变返回类型允许返回子类类型）必须相同。访问权限：子类方法不能比父类方法访问权限更严格（private 不可重写）。异常：子类方法抛出的异常不能比父类更宽泛。非方法不能重写：static、final、private 方法。重写标记：@Override 注解（建议使用）。方法绑定：virtual method invocation，运行时根据实际类型决定调用哪个方法。

---

### 3.8 多态（polymorphism）

**名称**：多态（Polymorphism）

**首次出现位置**：C04_109-面向对象基础.md 第4章

**定义**：
多态是面向对象的三大特性之一，指相同的方法调用在不同对象上有不同行为，包括编译时多态（重载）和运行时多态（重写/向上转型）。

**详解**：
编译时多态（重载 Overload）：同类方法参数列表不同（类型或个数），编译器根据参数决定。运行时多态（重写 Override）：父类引用指向子类对象，运行时根据实际对象类型调用方法。向上转型：Parent p = new Child(); 父类引用指向子类对象。运行时类型识别：instanceof 检查实际类型、强制转型回子类类型。多态好处：可替换性、可扩展性、接口编程。

---

### 3.9 封装（encapsulation）

**名称**：封装（Encapsulation）

**首次出现位置**：C04_109-面向对象基础.md 第2章

**定义**：
封装是将数据和操作隐藏在类内部，对外提供统一接口，保护对象内部状态，防止外部直接访问和修改。

**详解**：
实现：字段 private（私有）、方法 public（共公）。访问器方法：getter（如 getName()）和 setter（如 setName()）。JavaBean 规范：私有字段 + public getter/setter + 无参构造器。封装好处：数据安全、内部实现可独立改变、简化外部调用。不完全私有：反射可以访问 private 成员（用于框架）。

---

### 3.10 static 关键字

**名称**：静态（static）

**首次出现位置**：C04_109-面向对象基础.md 第5章

**定义**：
static 修饰的成员属于类而非实例，被所有实例共享，包括静态字段、静态方法、静态初始化块、静态内部类。

**详解**：
静态字段：类加载时创建，所有实例共享，可通过 ClassName.field 访问。静态方法：类方法，无 this 引用，只能访问静态成员，用于工具方法。静态初始化块：类加载时执行一次的代码块。静态导入：`import static java.lang.Math.PI;` 直接使用静态成员。静态成员不依赖于对象存在，可通过类名直接访问。

---

### 3.11 final 关键字

**名称**：最终（final）

**首次出现位置**：C04_109-面向对象基础.md 第5章

**定义**：
final 修饰的不可改变，用于定义常量（final 字段）、禁止重写（final 方法）、禁止继承（final 类）。

**详解**：
final 字段：必须在声明时或构造器/初始化块中初始化，之后不能修改。空白 final：声明时未初始化，在构造器中初始化。final 引用：引用不可变（指向的对象可改变），如 final List<String> list = new ArrayList<>()。final 方法：子类不能重写，可被内联优化。final 类：不能被继承，如 String、Math、System。final 参数：方法内部不能修改参数值。

---

### 3.12 abstract 关键字

**名称**：抽象（abstract）

**首次出现位置**：C04_110-抽象类与接口.md 第1章

**定义**：
abstract 用于定义抽象类和抽象方法，抽象类不能实例化，抽象方法没有方法体，由子类实现。

**详解**：
抽象类：`abstract class ClassName`，不能 new 实例，但可以有构造器（供子类 super 调用）。抽象方法：`abstract returnType method(params);`，只有声明没有方法体。子类继承抽象类：必须实现所有抽象方法（除非子类也是抽象类）。抽象类可以有具体方法。抽象类用途：定义公共行为、强制子类实现特定方法、提供部分实现。模板方法模式常用抽象类。

---

### 3.13 接口（interface）

**名称**：接口（Interface）

**首次出现位置**：C04_110-抽象类与接口.md 第2章

**定义**：
接口是抽象方法的集合，定义类必须遵循的契约，Java 8 支持默认方法（default）和静态方法，Java 9 支持私有方法。

**详解**：
定义：`interface InterfaceName { 方法签名; }`。实现：`class ClassName implements Interface1, Interface2`。接口字段：隐式 public static final（常量）。接口方法：Java 8 前全是抽象方法，Java 8+ 支持 default（带实现）、static（静态方法）、private（Java 9+）。接口可以继承其他接口。多实现：类可以实现多个接口。常用接口：Comparable、Runnable、Cloneable、Serializable、Stream（Java 8+ lambda）。

---

### 3.14 默认方法

**名称**：默认方法（Default Method）

**首次出现位置**：C04_110-抽象类与接口.md 第2章

**定义**：
默认方法是 Java 8 引入的接口新特性，使用 default 关键字声明，带有方法体，为接口方法提供默认实现。

**详解**：
语法：`default returnType method() { // 默认实现 }`。优势：不破坏向后兼容，为现有接口添加新方法而不影响实现类。冲突解决：类优先（类优先于接口）、接口冲突必须重写。示例：List 接口的 sort() 方法（Java 8+）是默认方法。Stream() 在 Collection 接口中是默认方法。设计模式：适配器模式可用默认方法简化。

---

### 3.15 多态接口 vs 抽象类

**名称**：接口与抽象类的区别（Interface vs Abstract Class）

**首次出现位置**：C04_110-抽象类与接口.md 第3章

**定义**：
抽象类和接口都用于定义抽象类型，但有重要区别：抽象类可以有状态（字段）和构造函数，接口在 Java 8 前只能是抽象方法。

**详解**：
对比维度：字段（抽象类任意、接口只能是常量）、方法（抽象类任意、Java 8 前只能是抽象方法）、构造函数（抽象类有、接口没有）、多继承（类只能单继承、接口可多实现）、设计意图（抽象类is-a关系、接口has-a能力）。选择原则：需要状态或构造函数用抽象类、需要多实现用接口、行为契约用接口。Java 9 后接口支持 private 方法，更利于代码复用。

## 4. 集合框架类

### 4.1 Collection（集合）

**名称**：集合（Collection）

**首次出现位置**：C04_112-集合框架.md 第1章

**定义**：
Collection 是 Java 集合框架的根接口，定义一组对象（元素）的操作，继承自 Iterable 接口。

**详解**：
Collection 继承结构：Iterable → Collection → List、Set、Queue。核心方法：add()、remove()、contains()、size()、isEmpty()、clear()、iterator()。子接口：List（有序、可重复）、Set（无序、不重复）、Queue（队列，通常FIFO）、Deque（双端队列）。遍历方式：for-each、iterator、forEach()（Java 8+）。注意：Collection 不直接支持随机访问（需要子接口或具体实现）。

---

### 4.2 List（列表）

**名称**：列表（List）

**首次出现位置**：C04_112-集合框架.md 第1章

**定义**：
List 是有序集合（序列），元素有索引，支持随机访问，可以存储重复元素。

**详解**：
实现类：ArrayList（数组实现）、LinkedList（链表实现）、Vector（同步，legacy）、Stack（栈，继承 Vector）。ArrayList：基于动态数组，随机访问 O(1)，尾部操作 O(1)，中间插入 O(n)。LinkedList：基于双向链表，插入/删除 O(1)（已知位置），不支持随机访问。Vector：线程安全（同步），性能较低，Java 5 后被 ArrayList 替代。常用方法：get(int index)、set(int index, E e)、add(E e)、remove(int index)、indexOf()、lastIndexOf()。

---

### 4.3 Set（集合）

**名称**：集合（Set）

**首次出现位置**：C04_112-集合框架.md 第1章

**定义**：
Set 是不包含重复元素的集合，不保证元素顺序（除非使用 LinkedHashSet 或 SortedSet）。

**详解**：
实现类：HashSet（哈希表，最常用）、LinkedHashSet（保持插入顺序）、TreeSet（红黑树，有序）、EnumSet（枚举类型）、CopyOnWriteArraySet（线程安全）。HashSet：基于 HashMap，元素需正确实现 hashCode() 和 equals()。LinkedHashSet：继承 HashSet，额外维护双向链表记录插入顺序。TreeSet：基于 NavigableMap（红黑树），元素需实现 Comparable 或提供 Comparator。添加元素到 Set 前需确保 hashCode() 和 equals() 正确。

---

### 4.4 Map（映射）

**名称**：映射（Map）

**首次出现位置**：C04_112-集合框架.md 第1章

**定义**：
Map 是键值对容器，通过键（Key）快速查找值（Value），键不能重复，每个键最多映射一个值。

**详解**：
实现类：HashMap（哈希表，最常用）、LinkedHashMap（保持插入顺序）、TreeMap（红黑树，有序）、Hashtable（同步，legacy）、ConcurrentHashMap（线程安全）。核心方法：put(K key, V value)、get(Object key)、remove(Object key)、containsKey()、containsValue()、keySet()、values()、entrySet()。遍历：for (Map.Entry<K,V> entry : map.entrySet())。HashMap 允许 null 键和 null 值（非同步），Hashtable 不允许 null。

---

### 4.5 HashMap

**名称**：哈希映射（HashMap）

**首次出现位置**：C04_112-集合框架.md 第2章

**定义**：
HashMap 是基于哈希表的 Map 接口实现，提供 O(1) 平均时间复杂度的 put/get 操作，是最常用的键值对容器。

**详解**：
底层结构：JDK 1.8+ 使用数组+链表+红黑树（链表长度≥8 时树化）。核心概念：容量（bucket 数量）、负载因子（默认 0.75）、初始容量（默认 16）。扩容：当元素数超过 capacity*loadFactor 时扩容 2 倍。键的 hashCode() 和 equals() 决定元素存储位置。线程不安全：多线程环境使用 ConcurrentHashMap。允许 null 键（只能一个）和 null 值（可多个）。HashMap vs Hashtable：Hashtable 线程安全、不允许 null。

---

### 4.6 ArrayList

**名称**：动态数组列表（ArrayList）

**首次出现位置**：C04_112-集合框架.md 第2章

**定义**：
ArrayList 是 List 接口的可变数组实现，支持随机访问，尾部操作高效，中间插入删除效率较低。

**详解**：
底层实现：Object[] elementData，动态扩容（grow() 方法）。初始容量：默认 10，构造器可指定。扩容策略：原容量 + 原容量/2（1.5 倍）。线程不安全：多线程环境用 CopyOnWriteArrayList 或 Collections.synchronizedList()。与 Vector 区别：Vector 同步（早期考虑线程安全），ArrayList 性能更好。常用操作：add(E e)（尾部追加）、add(int index, E e)（插入）、get(int index）（随机访问）、remove(int index)（删除）。

---

### 4.7 迭代器（Iterator）

**名称**：迭代器（Iterator）

**首次出现位置**：C04_112-集合框架.md 第3章

**定义**：
Iterator 是 Java 集合的遍历接口，提供 hasNext()、next()、remove() 方法，替代 Enumeration 遍历集合。

**详解**：
获取：`Iterator<E> it = collection.iterator();`。方法：hasNext() 判断是否有下一元素、next() 返回下一元素并移动指针、remove() 删除上一元素（可选操作）。for-each 循环：编译器将 for-each 转为 iterator。ListIterator：List 的专用迭代器，支持双向遍历和修改。Iterator vs ListIterator：Iterator 只前向，ListIterator 可双向。Iterator 的 remove() 相比 for-each 更安全（避免 ConcurrentModificationException）。

---

### 4.8 泛型（Generics）

**名称**：泛型（Generics）

**首次出现位置**：C04_116-泛型.md 第1章

**定义**：
泛型是 Java 5 引入的参数化类型特性，允许在类、接口、方法中使用类型参数，编译时类型检查，运行时类型擦除。

**详解**：
泛型类：`class Box<T> { private T content; }`。泛型接口：`interface List<E> { void add(E e); E get(int index); }`。泛型方法：`public static <T> T getFirst(List<T> list)`。类型参数：T（类型）、E（元素）、K/V（键值）、N（数字）、?（通配符）。上限通配符：`List<? extends Number>` 读取 Number 及子类。下限通配符：`List<? super Integer>` 写入 Integer 及父类。类型擦除：泛型信息在编译后被擦除，JVM 中是原始类型（Raw Type）。

## 5. 异常处理类

### 5.1 异常（Exception）

**名称**：异常（Exception）

**首次出现位置**：C04_111-异常处理.md 第1章

**定义**：
异常是程序执行时发生的错误或异常情况，Java 通过异常机制处理，而非传统返回码。

**详解**：
异常继承树：Throwable → Error（系统错误，不应捕获）、Exception（应用程序异常）。受检异常（Checked）：编译器强制处理，如 IOException、SQLException，必须捕获或声明 throws。非受检异常（Unchecked）：RuntimeException 及其子类，如 NullPointerException、ArrayIndexOutOfBoundsException，不强制处理。常见异常：NullPointerException、IndexOutOfBoundsException、IllegalArgumentException、IllegalStateException、ClassNotFoundException、NoSuchMethodException。

---

### 5.2 try-catch 语句

**名称**：异常捕获（try-catch Statement）

**首次出现位置**：C04_111-异常处理.md 第1章

**定义**：
try-catch 是 Java 的异常处理结构，try 块包含可能抛出异常的代码，catch 块捕获并处理特定异常。

**详解**：
语法：`try { } catch (ExceptionType1 e) { } catch (ExceptionType2 e) { } finally { }`。多 catch 块：按从具体到一般顺序排列。捕获范围：子类异常放在父类之前。异常参数：catch 块中可使用异常对象（e.getMessage()、e.printStackTrace()）。finally 块：无论是否异常都执行（除非 JVM 退出）。try-with-resources：自动关闭资源（Java 7+）。

---

### 5.3 throws 语句

**名称**：异常声明（throws Clause）

**首次出现位置**：C04_111-异常处理.md 第1章

**定义**：
throws 用于在方法签名中声明该方法可能抛出的异常，调用者必须处理或继续声明。

**详解**：
语法：`returnType method(params) throws ExceptionType1, ExceptionType2 { }`。受检异常必须声明或捕获，非受检异常可选。异常传播：未捕获的异常沿调用栈向上传播，最终被 JVM 捕获导致程序终止。重写方法：子类重写方法不能声明比父类更广泛的受检异常。main 方法的 throws 会导致异常传播到 JVM。RuntimeException 不需要 throws 声明。

---

### 5.4 throw 语句

**名称**：抛出异常（throw Statement）

**首次出现位置**：C04_111-异常处理.md 第2章

**定义**：
throw 语句用于主动抛出异常，中断当前执行流程，创建一个异常对象并抛出。

**详解**：
语法：`throw new ExceptionType("message");`。受检异常必须声明或捕获。常见用法：参数验证失败、状态不合法、业务逻辑错误。自定义异常：extends Exception（受检）或 RuntimeException（非受检）。重新抛出：`throw e;` 传播现有异常。异常链：`throw new NewException("msg", cause);` 保存原始异常。断言：`assert condition : message;` 条件为假时抛出 AssertionError。

---

### 5.5 自定义异常

**名称**：自定义异常（Custom Exception）

**首次出现位置**：C04_111-异常处理.md 第2章

**定义**：
通过继承 Exception 或其子类创建自定义异常类型，表达特定业务场景的错误。

**详解**：
受检异常：`class MyException extends Exception`，调用者必须处理。非受检异常：`class MyRuntimeException extends RuntimeException`，调用者可选处理。构造器：提供多种构造器（无参、消息、原因、消息+原因）。异常命名：以 Exception 结尾。异常类位置：放在合适的包中。可添加额外属性和方法。日志记录：通常在捕获异常时记录，不在异常类中记录。

---

### 5.6 try-with-resources

**名称**：资源尝试语句（Try-with-Resources）

**首次出现位置**：C04_111-异常处理.md 第3章

**定义**：
try-with-resources 是 Java 7 引入的语法，自动关闭实现了 AutoCloseable 接口的资源，无需显式 finally 块。

**详解**：
语法：`try (ResourceType resource = new Resource()) { use resource; }`。自动关闭：资源在 try 块结束后自动 close()，即使异常抛出也关闭。多个资源：用分号分隔：`try (FileInputStream fis = new FileInputStream(f1); FileOutputStream fos = new FileOutputStream(f2))`。隐式 finally：无需 finally 块，代码更简洁。资源关闭顺序：后声明先关闭。常见实现：InputStream、OutputStream、Reader、Writer、Connection、Statement、ResultSet。

---

## 6. 多线程类

### 6.1 线程（Thread）

**名称**：线程（Thread）

**首次出现位置**：C04_113-多线程.md 第1章

**定义**：
线程是程序执行的最小单元，同一进程内的线程共享堆内存和方法区，每个线程有自己的程序计数器和栈。

**详解**：
创建方式：继承 Thread 类（重写 run()）、实现 Runnable 接口（Java 8+ 可用 lambda）、实现 Callable 接口（有返回值）。启动线程：调用 start() 方法（不是 run()）。线程状态：NEW → RUNNABLE → BLOCKED/WAITING/TIMED_WAITING → TERMINATED。常用方法：sleep()（休眠，不释放锁）、yield()（让步，不推荐）、join()（等待其他线程）、interrupt()（中断）。线程优先级：1-10，默认 5，不保证顺序。

---

### 6.2 Runnable 接口

**名称**：可运行接口（Runnable Interface）

**首次出现位置**：C04_113-多线程.md 第1章

**定义**：
Runnable 是 Java 多线程的常用接口，定义了线程执行的任务，run() 方法不含返回值，不能抛出受检异常。

**详解**：
定义：`public interface Runnable { void run(); }`。实现：`class Task implements Runnable { public void run() { } }`。使用：`Thread t = new Thread(new Task()); t.start();`。Lambda 形式：`new Thread(() -> { System.out.println("task"); }).start();`。与 Callable 区别：Runnable 的 run() 无返回值、不能抛受检异常。Callable 有返回值、可抛受检异常，通过 FutureTask 包装使用。

---

### 6.3 synchronized 关键字

**名称**：同步（synchronized）

**首次出现位置**：C04_113-多线程.md 第2章

**定义**：
synchronized 是 Java 的内置锁机制，用于实现线程同步，保证同一时刻只有一个线程执行同步代码块。

**详解**：
用法：修饰方法（实例方法锁定 this，静态方法锁定 Class 对象）、修饰代码块（锁定指定对象）。可重入锁：同一线程可多次获取同一锁。锁粒度：锁定方法（粗粒度）、锁定代码块（细粒度，更灵活）。同步开销：获取/释放锁有性能成本，避免锁粒度过小或过大。monitorenter/monitorexit：字节码层面的锁实现。死锁：多个线程相互等待对方持有的锁。替代方案：java.util.concurrent 包提供更高级的锁（ReentrantLock）。

---

### 6.4 volatile 关键字

**名称**：易变（volatile）

**首次出现位置**：C04_113-多线程.md 第2章

**定义**：
volatile 是 Java 的轻量级同步机制，保证变量的可见性和禁止指令重排序，但不保证原子性。

**详解**：
可见性：一个线程对 volatile 变量的修改，对其他线程立即可见（读取主内存而非 CPU 缓存）。禁止重排序：volatile 变量的读写前后代码不能被 JIT 编译器重排序。应用场景：状态标志、double/long 的线程安全读写（volatile 保证 64 位原子性）。不保证原子性：`volatile int count; count++;` 不是原子操作，仍需 synchronized 或 AtomicInteger。volatile vs synchronized：volatile 是轻量级，只保证可见性；synchronized 保证可见性和原子性。

---

### 6.5 线程池（ThreadPool）

**名称**：线程池（Thread Pool）

**首次出现位置**：C04_113-多线程.md 第3章

**定义**：
线程池是管理线程生命周期的机制，预先创建线程，使用时从池中获取，空闲时归还池中，避免频繁创建销毁线程。

**详解**：
核心类：ExecutorService（接口）、ThreadPoolExecutor（实现）、Executors（工厂类）。线程池参数：corePoolSize（核心线程数）、maximumPoolSize（最大线程数）、keepAliveTime（空闲线程存活时间）、workQueue（任务队列）、threadFactory（线程工厂）、handler（拒绝策略）。 Executors 工厂方法：newFixedThreadPool()（固定大小）、newCachedThreadPool()（弹性大小）、newSingleThreadExecutor()（单线程）、newScheduledThreadPool()（定时任务）。关闭线程池：shutdown()（优雅关闭）、shutdownNow()（立即关闭）。

---

### 6.6 Future

**名称**：异步结果（Future）

**首次出现位置**：C04_113-多线程.md 第3章

**定义**：
Future 是异步计算结果的占位符，提供检查计算是否完成、等待完成、获取结果等方法。

**详解**：
接口方法：get()（阻塞获取结果）、get(long timeout, TimeUnit unit)（超时获取）、isDone()（检查是否完成）、cancel()（取消任务）。实现类：FutureTask（同时实现 Future 和 Runnable）。与 Callable 配合：`Future<V> submit(Callable<V> task);`。异常处理：get() 方法抛出 ExecutionException（包含实际异常）。局限性：不能手动完成、不能组合多个 Future。CompletableFuture（Java 8+）解决这些问题。

---

### 6.7 Lock 接口

**名称**：锁接口（Lock Interface）

**首次出现位置**：C04_113-多线程.md 第2章

**定义**：
Lock 是 java.util.concurrent.locks 包提供的显式锁接口，比 synchronized 更灵活，支持非阻塞获取锁、可超时、公平锁等。

**详解**：
核心方法：lock()（阻塞获取）、tryLock()（非阻塞获取）、tryLock(timeout)（超时获取）、unlock()（释放锁）。实现类：ReentrantLock（可重入锁）、ReadLock/WriteLock（ReentrantReadWriteLock）。try-finally 模式：`Lock lock = new ReentrantLock(); lock.lock(); try { } finally { lock.unlock(); }`。优势：可中断获取锁、公平锁选择、多个条件变量（Condition）。synchronized 是隐式自动释放，Lock 必须显式释放。

---

## 7. I/O 流类

### 7.1 I/O 流

**名称**：输入输出流（I/O Stream）

**首次出现位置**：C04_114-IO流.md 第1章

**定义**：
Java I/O 流是数据序列的抽象，表示从源到目的地的数据流动，包括输入流（读）和输出流（写）。

**详解**：
分类：按方向（输入/输出）、按数据类型（字节/字符）、按功能（节点/处理）。字节流：InputStream/OutputStream（抽象类）。字符流：Reader/Writer（抽象类）。节点流：直接连接数据源，如 FileInputStream。处理流：装饰其他流，如 BufferedInputStream、DataInputStream。标准输入/输出：System.in（InputStream）、System.out（PrintStream）、System.err。关闭流：finally 或 try-with-resources。

---

### 7.2 FileInputStream / FileOutputStream

**名称**：文件输入输出流（File Input/Output Stream）

**首次出现位置**：C04_114-IO流.md 第2章

**定义**：
FileInputStream 和 FileOutputStream 是字节流，用于从文件读取字节和向文件写入字节。

**详解**：
构造器：FileInputStream(String name)、FileInputStream(File file)、FileOutputStream(String name)、FileOutputStream(String name, boolean append)。读取：`int read()` 返回字节（0-255）或 -1（结束）。写入：`void write(int b)` 写入字节、`void write(byte[] b)` 写入字节数组。中文问题：字节流按 byte 处理中文会乱码，应使用字符流。BufferedFileInputStream/BufferedFileOutputStream 提供缓冲提高效率。

---

### 7.3 BufferedReader / BufferedWriter

**名称**：缓冲字符输入输出流（Buffered Reader/Writer）

**首次出现位置**：C04_114-IO流.md 第2章

**定义**：
BufferedReader 和 BufferedWriter 是字符流，提供缓冲功能，提高读写效率，支持按行读取。

**详解**：
BufferedReader：包装 Reader，提供 readLine() 按行读取（返回不含换行符的字符串）、read() 读取单字符。构造器：`BufferedReader(Reader in)`、`BufferedReader(Reader in, int size)`。BufferedWriter：提供 newLine() 写入换行符、flush() 刷新缓冲区。缓冲大小：默认 8KB，可指定。推荐用法：BufferedReader + InputStreamReader（处理字节转字符+缓冲）。示例：`try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) { String line; while ((line = br.readLine()) != null) { } }`

---

### 7.4 对象序列化

**名称**：对象序列化（Object Serialization）

**首次出现位置**：C04_114-IO流.md 第3章

**定义**：
对象序列化是将对象状态转换为字节流（序列化），以及从字节流恢复对象（反序列化）的机制。

**详解**：
关键字：implements Serializable（标记接口）。serialVersionUID：版本号，用于反序列化兼容性验证。transient：标记的字段不序列化（如密码、缓存）。static 字段不序列化。writeObject/readObject：自定义序列化逻辑（可选）。序列化/反序列化：`ObjectOutputStream.writeObject(obj)`、`ObjectInputStream.readObject()`。序列化存储：`ObjectOutputStream` 保存到文件。注意事项：父类实现 Serializable 后子类自动实现、非序列化对象的子类需自己实现、敏感数据需加密或不使用序列化。

## 8. 高级特性类

### 8.1 注解（Annotation）

**名称**：注解（Annotation）

**首次出现位置**：C04_109-面向对象基础.md 第6章

**定义**：
注解是用 @ 符号开头的元数据，标记在程序元素（类、方法、字段、参数）上，供编译器、工具或运行时使用。

**详解**：
内置注解：@Override（重写）、@Deprecated（过时）、@SuppressWarnings（抑制警告）、@FunctionalInterface（函数式接口）。元注解：@Retention（保留策略：SOURCE/CLASS/RUNTIME）、@Target（目标元素）、@Documented、@Inherited。自定义注解：`@interface MyAnnotation { String value(); int count() default 1; }`。注解处理器：编译时（APT）或运行时（反射）处理注解。常用框架：Spring（@Component、@Autowired）、JUnit（@Test）、Hibernate（@Entity）。

---

### 8.2 反射（Reflection）

**名称**：反射（Reflection）

**首次出现位置**：C04_109-面向对象基础.md 第6章

**定义**：
反射是 Java 在运行时动态获取类信息（Class 对象）和操作对象成员（字段、方法、构造器）的能力。

**详解**：
Class 对象获取：Class.forName("com.example.MyClass")、`MyClass.class`、`obj.getClass()`。运行时检查：getName()、getSuperclass()、getInterfaces()、getModifiers()。构造器：getConstructor()、newInstance()。字段：getField()（public）、getDeclaredField()（所有）。方法：getMethod()（public）、getDeclaredMethod()（所有）。调用方法：method.invoke(obj, args)。反射用途：框架（Spring 依赖注入）、IDE（代码补全）、序列化（JDK、JSON 库）、测试。反射性能开销较大，应避免在频繁调用的代码中使用。

---

### 8.3 Lambda 表达式

**名称**：Lambda 表达式（Lambda Expression）

**首次出现位置**：C04_100-快速入门.md 第4章

**定义**：
Lambda 是 Java 8 引入的匿名函数语法，简化函数式接口的实现，用于创建函数式接口的实例。

**详解**：
语法：`(params) -> expression` 或 `(params) -> { statements; }`。类型推断：编译器自动推断参数类型和返回类型。函数式接口：只有一个抽象方法的接口（@FunctionalInterface 标记）。示例：`Runnable r = () -> System.out.println("hello");`。方法引用：`List::size`（类方法引用）、`obj::instanceMethod`（实例方法引用）、`String::new`（构造器引用）。常见用途：Stream API、线程创建、GUI 事件处理、Comparator 实现。

---

### 8.4 Stream API

**名称**：流式API（Stream API）

**首次出现位置**：C04_100-快速入门.md 第4章

**定义**：
Stream API 是 Java 8 引入的数据流操作抽象，支持函数式风格处理集合，提供过滤、映射、归约等操作。

**详解**：
创建：Collection.stream()、Collection.parallelStream()（并行流）、Stream.of()、Arrays.stream()。中间操作：filter(Predicate)、map(Function)、flatMap(Function)、distinct()、sorted()、limit()、skip()。终端操作：collect()（收集到集合）、forEach()、count()、reduce()、max()/min()、anyMatch()/allMatch()/noneMatch()。惰性求值：中间操作惰性，终端操作触发执行。parallelStream()：并行处理，利用多核，提高大数据处理性能。常用收集器：Collectors.toList()、Collectors.groupingBy()、Collectors.joining()。

---

### 8.5 方法引用

**名称**：方法引用（Method Reference）

**首次出现位置**：C04_100-快速入门.md 第4章

**定义**：
方法引用是 Lambda 表达式的简化形式，直接引用已有方法或构造器，用于实现函数式接口。

**详解**：
四种类型：静态方法引用 `ClassName::staticMethod`、实例方法引用 `obj::instanceMethod`、类实例方法引用 `ClassName::instanceMethod`、构造器引用 `ClassName::new`。示例：`List<String>::size` 相当于 `list -> list.size()`。常用场景：`Function.identity()`（返回输入本身）、`Comparator.comparing(Function)`、`Stream.map(ClassName::new)`。方法引用使代码更简洁、可读性更高。

---

### 8.6 Optional

**名称**：可选值（Optional）

**首次出现位置**：C04_100-快速入门.md 第4章

**定义**：
Optional 是 Java 8 引入的容器类，表示值存在或不存在，避免 null 检查的繁琐和空指针异常。

**详解**：
创建：Optional.of(value)（非空）、Optional.ofNullable(value)（可空）、Optional.empty()（空）。判断：isPresent()（是否非空）、ifPresent(Consumer)（非空时执行）、isEmpty()（Java 11+）。获取值：get()（可能抛异常）、orElse(defaultValue)、orElseGet(Supplier)、orElseThrow()。转换：map(Function)、flatMap(Function)、filter(Predicate)。链式调用：`user.flatMap(User::getAddress).flatMap(Address::getCity)` 避免嵌套 null 检查。替代方案：null 对象模式、@NonNull 注解。
