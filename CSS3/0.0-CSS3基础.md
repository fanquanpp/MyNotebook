## 1. CSS基础概念与语法

### 1.1 CSS基本语法

```css
选择器 {
  属性1: 值1;
  属性2: 值2;
  /* 注释 */
}
```

### 1.2 CSS引入方式

1. **内联样式**：直接在HTML元素中使用style属性
   ```html
   <div style="color: red; font-size: 16px;">文本内容</div>
   ```

2. **内部样式**：在HTML文档头部使用style标签
   ```html
   <style>
     div {
       color: red;
       font-size: 16px;
     }
   </style>
   ```

3. **外部样式**：使用link标签引入外部CSS文件
   ```html
   <link rel="stylesheet" href="style.css">
   ```

4. **@import**：在CSS文件中导入其他CSS文件
   ```css
   @import url("other.css");
   ```

### 1.3 CSS优先级

CSS优先级从高到低：
1. !important声明
2. 内联样式
3. ID选择器
4. 类选择器、属性选择器、伪类选择器
5. 元素选择器、伪元素选择器
6. 通用选择器

## 2. 选择器系统

### 2.1 基础选择器

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| 元素选择器 | 选择指定类型的元素 | `div { color: red; }` |
| ID选择器 | 选择指定ID的元素 | `#header { background: blue; }` |
| 类选择器 | 选择指定类的元素 | `.container { width: 100%; }` |
| 通用选择器 | 选择所有元素 | `* { margin: 0; padding: 0; }` |
| 属性选择器 | 选择具有指定属性的元素 | `[type="text"] { border: 1px solid #ccc; }` |

### 2.2 组合选择器

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| 后代选择器 | 选择元素的后代元素 | `div p { color: red; }` |
| 子选择器 | 选择元素的直接子元素 | `div > p { color: red; }` |
| 相邻兄弟选择器 | 选择元素的下一个兄弟元素 | `div + p { color: red; }` |
| 通用兄弟选择器 | 选择元素的所有兄弟元素 | `div ~ p { color: red; }` |

### 2.3 伪类选择器

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| `:hover` | 鼠标悬停时 | `a:hover { color: red; }` |
| `:active` | 元素被激活时 | `button:active { background: blue; }` |
| `:focus` | 元素获得焦点时 | `input:focus { border: 2px solid blue; }` |
| `:first-child` | 第一个子元素 | `li:first-child { font-weight: bold; }` |
| `:last-child` | 最后一个子元素 | `li:last-child { color: red; }` |
| `:nth-child(n)` | 第n个子元素 | `li:nth-child(2) { color: blue; }` |
| `:nth-child(even)` | 偶数位置的子元素 | `li:nth-child(even) { background: #f0f0f0; }` |
| `:nth-child(odd)` | 奇数位置的子元素 | `li:nth-child(odd) { background: #e0e0e0; }` |
| `:not(selector)` | 排除指定选择器 | `p:not(.special) { color: gray; }` |

### 2.4 伪元素选择器

| 选择器 | 描述 | 示例 |
| :--- | :--- | :--- |
| `::before` | 在元素内容前插入内容 | `p::before { content: "前缀"; }` |
| `::after` | 在元素内容后插入内容 | `p::after { content: "后缀"; }` |
| `::first-letter` | 选择元素的第一个字母 | `p::first-letter { font-size: 2em; }` |
| `::first-line` | 选择元素的第一行 | `p::first-line { font-weight: bold; }` |
| `::selection` | 选择用户选中的内容 | `::selection { background: yellow; }` |

## 3. 盒模型原理与应用

### 3.1 盒模型组成

盒模型由以下部分组成：
- **内容区域**：width 和 height 属性定义的区域
- **内边距**：padding 属性定义，出现在内容与盒子边缘之间
- **边框**：border 属性定义
- **外边距**：margin 属性定义，出现在盒子外面

### 3.2 盒模型类型

1. **标准盒模型**：width 和 height 只包括内容区域
2. **IE盒模型**：width 和 height 包括内容区域、内边距和边框

可以通过 `box-sizing` 属性切换盒模型：

```css
/* 标准盒模型 */
.box {
  box-sizing: content-box;
}

/* IE盒模型 */
.box {
  box-sizing: border-box;
}
```

### 3.3 盒模型属性

#### 3.3.1 宽度和高度

```css
.box {
  width: 200px;
  height: 100px;
}
```

#### 3.3.2 内边距（padding）

```css
.box {
  padding: 10px; /* 四个方向都为10px */
  padding: 10px 20px; /* 上下10px，左右20px */
  padding: 10px 20px 30px; /* 上10px，左右20px，下30px */
  padding: 10px 20px 30px 40px; /* 上、右、下、左 */
  
  /* 单独设置 */
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 30px;
  padding-left: 40px;
}
```

#### 3.3.3 边框（border）

```css
.box {
  border: 1px solid #000; /* 宽度、样式、颜色 */
  
  /* 单独设置 */
  border-width: 1px;
  border-style: solid;
  border-color: #000;
  
  /* 单独设置某一边 */
  border-top: 2px dashed red;
  border-right: 1px solid blue;
  border-bottom: 3px double green;
  border-left: 1px dotted yellow;
}
```

#### 3.3.4 外边距（margin）

```css
.box {
  margin: 10px; /* 四个方向都为10px */
  margin: 10px 20px; /* 上下10px，左右20px */
  margin: 10px 20px 30px; /* 上10px，左右20px，下30px */
  margin: 10px 20px 30px 40px; /* 上、右、下、左 */
  
  /* 单独设置 */
  margin-top: 10px;
  margin-right: 20px;
  margin-bottom: 30px;
  margin-left: 40px;
}
```

#### 3.3.5 溢出处理（overflow）

| 属性值 | 效果 |
| :--- | :--- |
| `visible` | 默认值，内容溢出时显示在盒子外面 |
| `hidden` | 溢出隐藏 |
| `scroll` | 溢出滚动（无论是否溢出，都显示滚动条位置） |
| `auto` | 溢出滚动（溢出才显示滚动条位置） |

```css
.box {
  overflow: hidden;
  overflow-x: auto;
  overflow-y: scroll;
}
```

## 4. 布局技术

### 4.1 标准流布局

标准流是HTML元素的默认布局方式，包括：
- 块级元素：独占一行，如div、p、h1-h6等
- 行内元素：在同一行显示，如span、a、img等
- 行内块元素：兼具块级和行内元素特性，如input、button等

### 4.2 浮动布局

浮动元素会脱离标准流，向左或向右移动，直到碰到父元素边界或其他浮动元素。

```css
.box {
  float: left; /* 向左浮动 */
  /* float: right; /* 向右浮动 */
  /* float: none; /* 不浮动 */
}

/* 清除浮动 */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}
```

### 4.3 定位布局

定位元素会脱离标准流，根据指定的参考点进行定位。

| 定位方式 | 描述 | 参考点 |
| :--- | :--- | :--- |
| `static` | 默认值，静态定位 | 无 |
| `relative` | 相对定位 | 元素自身的原始位置 |
| `absolute` | 绝对定位 | 最近的已定位祖先元素 |
| `fixed` | 固定定位 | 浏览器窗口 |
| `sticky` | 粘性定位 | 视口和元素自身位置 |

```css
.box {
  position: relative;
  top: 10px;
  left: 20px;
}

.absolute-box {
  position: absolute;
  top: 0;
  right: 0;
}
```

### 4.4 Flexbox弹性布局

Flexbox是一种一维布局模型，适用于容器内元素的排列和对齐。

#### 4.4.1 容器属性

```css
.container {
  display: flex; /* 启用Flexbox */
  
  /* 主轴方向 */
  flex-direction: row; /* 水平方向，默认 */
  /* flex-direction: column; /* 垂直方向 */
  /* flex-direction: row-reverse; /* 水平方向，反向 */
  /* flex-direction: column-reverse; /* 垂直方向，反向 */
  
  /* 主轴对齐方式 */
  justify-content: flex-start; /* 起点对齐，默认 */
  /* justify-content: flex-end; /* 终点对齐 */
  /* justify-content: center; /* 居中对齐 */
  /* justify-content: space-between; /* 两端对齐，中间均分 */
  /* justify-content: space-around; /* 均匀分布，元素两侧间距相等 */
  /* justify-content: space-evenly; /* 均匀分布，元素与容器间距相等 */
  
  /* 交叉轴对齐方式 */
  align-items: stretch; /* 拉伸填充，默认 */
  /* align-items: flex-start; /* 起点对齐 */
  /* align-items: flex-end; /* 终点对齐 */
  /* align-items: center; /* 居中对齐 */
  /* align-items: baseline; /* 基线对齐 */
  
  /* 换行方式 */
  flex-wrap: nowrap; /* 不换行，默认 */
  /* flex-wrap: wrap; /* 换行 */
  /* flex-wrap: wrap-reverse; /* 反向换行 */
  
  /* 简写属性 */
  flex-flow: row wrap; /* flex-direction + flex-wrap */
}
```

#### 4.4.2 项目属性

```css
.item {
  /* 项目大小 */
  flex: 1; /* 简写属性，flex-grow + flex-shrink + flex-basis */
  
  /* 单独设置 */
  flex-grow: 1; /* 拉伸因子 */
  flex-shrink: 1; /* 收缩因子 */
  flex-basis: auto; /* 初始大小 */
  
  /* 项目排序 */
  order: 0; /* 默认值，数值越小越靠前 */
  
  /* 项目对齐 */
  align-self: auto; /* 继承容器的align-items */
  /* align-self: flex-start; /* 起点对齐 */
  /* align-self: flex-end; /* 终点对齐 */
  /* align-self: center; /* 居中对齐 */
  /* align-self: stretch; /* 拉伸填充 */
  /* align-self: baseline; /* 基线对齐 */
}
```

### 4.5 Grid网格布局

Grid是一种二维布局模型，适用于创建复杂的网格结构。

#### 4.5.1 容器属性

```css
.container {
  display: grid; /* 启用Grid */
  
  /* 网格模板列 */
  grid-template-columns: 100px 200px 100px; /* 固定宽度 */
  /* grid-template-columns: 1fr 2fr 1fr; /* 比例分配 */
  /* grid-template-columns: repeat(3, 1fr); /* 重复3次1fr */
  
  /* 网格模板行 */
  grid-template-rows: 100px 200px;
  
  /* 网格间隙 */
  gap: 10px; /* 行列间隙相同 */
  /* column-gap: 10px; /* 列间隙 */
  /* row-gap: 20px; /* 行间隙 */
  
  /* 网格区域 */
  grid-template-areas: 
    "header header header"
    "sidebar content content"
    "footer footer footer";
}
```

#### 4.5.2 项目属性

```css
.item {
  /* 网格区域 */
  grid-area: header; /* 对应grid-template-areas中的区域 */
  
  /* 网格位置 */
  grid-column-start: 1;
  grid-column-end: 3;
  grid-row-start: 1;
  grid-row-end: 2;
  
  /* 简写属性 */
  grid-column: 1 / 3;
  grid-row: 1 / 2;
  
  /* 对齐方式 */
  justify-self: stretch; /* 水平方向，默认 */
  align-self: stretch; /* 垂直方向，默认 */
}
```

## 7. 动画与过渡效果

### 7.1 过渡效果

过渡效果用于平滑地改变元素的属性值。

```css
.box {
  width: 100px;
  height: 100px;
  background: red;
  
  /* 过渡属性 */
  transition-property: width, height, background;
  /* transition-property: all; /* 所有属性 */
  
  /* 过渡持续时间 */
  transition-duration: 1s;
  
  /* 过渡 timing 函数 */
  transition-timing-function: ease; /* 默认 */
  /* transition-timing-function: linear; /* 线性 */
  /* transition-timing-function: ease-in; /* 缓入 */
  /* transition-timing-function: ease-out; /* 缓出 */
  /* transition-timing-function: ease-in-out; /* 缓入缓出 */
  
  /* 过渡延迟 */
  transition-delay: 0.5s;
  
  /* 简写属性 */
  transition: width 1s ease 0.5s, height 1s ease 0.5s, background 1s ease 0.5s;
  /* transition: all 1s ease 0.5s; /* 所有属性 */
}

.box:hover {
  width: 200px;
  height: 200px;
  background: blue;
}
```

### 7.2 关键帧动画

关键帧动画用于创建更复杂的动画效果。

```css
/* 定义关键帧 */
@keyframes fadeIn {
  0% {
    opacity: 0;
    transform: translateY(20px);
  }
  100% {
    opacity: 1;
    transform: translateY(0);
  }
}

/* 使用动画 */
.box {
  animation-name: fadeIn;
  animation-duration: 1s;
  animation-timing-function: ease;
  animation-delay: 0s;
  animation-iteration-count: 1; /* 动画次数 */
  animation-direction: normal; /* 动画方向 */
  animation-fill-mode: forwards; /* 动画结束后保持状态 */
  
  /* 简写属性 */
  animation: fadeIn 1s ease 0s 1 normal forwards;
}
```

## 8. 响应式设计与媒体查询

### 8.1 媒体查询语法

```css
/* 基本语法 */
@media media-type and (media-feature) {
  /* 样式规则 */
}

/* 示例 */
/* 屏幕宽度小于768px时 */
@media screen and (max-width: 767px) {
  .container {
    width: 100%;
  }
}

/* 屏幕宽度在768px到1199px之间时 */
@media screen and (min-width: 768px) and (max-width: 1199px) {
  .container {
    width: 90%;
  }
}
```

## 13. 常见问题与解决方案

### 13.1 盒模型问题

**问题**：元素实际宽度超出预期

**原因**：标准盒模型中，width和height只包括内容区域，不包括padding和border

**解决方案**：使用box-sizing: border-box

```css
* {
  box-sizing: border-box;
}
```

### 13.2 浮动清除问题

**问题**：父元素高度塌陷

**原因**：浮动元素脱离标准流，父元素无法计算高度

**解决方案**：使用清除浮动技术

```css
/* 方法1：使用clear属性 */
.clear {
  clear: both;
}

/* 方法2：使用伪元素 */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

/* 方法3：使用overflow属性 */
.parent {
  overflow: hidden;
}
```

### 13.3 垂直居中问题

**问题**：元素无法垂直居中

**解决方案**：

```css
/* 方法1：使用Flexbox */
.container {
  display: flex;
  align-items: center;
  justify-content: center;
  height: 300px;
}

/* 方法2：使用Grid */
.container {
  display: grid;
  place-items: center;
  height: 300px;
}

/* 方法3：使用绝对定位 */
.parent {
  position: relative;
  height: 300px;
}

.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

### 13.4 响应式图片问题

**问题**：图片在小屏幕上溢出容器

**解决方案**：

```css
img {
  max-width: 100%;
  height: auto;
}
```

### 13.5 z-index问题

**问题**：元素无法正确显示在其他元素之上

**原因**：z-index只在已定位元素上生效

**解决方案**：

```css
.element {
  position: relative; /* 或其他定位方式 */
  z-index: 10;
}
```

## 14. 最佳实践

### 14.1 代码组织

1. **模块化**：将CSS按功能模块组织

1. **减少HTTP请求**：合并CSS文件

3. **减少选择器复杂度**：避免使用过于复杂的选择器
4. **使用CSS变量**：减少重复代码

6. **使用简写属性**：减少代码量

### 14.4 无障碍访问

1. **颜色对比度**：确保文本与背景的对比度符合WCAG标准
2. **键盘导航**：确保所有元素可以通过键盘访问
