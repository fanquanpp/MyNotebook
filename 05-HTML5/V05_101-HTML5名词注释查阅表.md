## 1. 基础概念类

### 1.1 HTML（超文本标记语言）

**名称**：超文本标记语言（HyperText Markup Language）

**缩写**：HTML

**首次出现位置**：C05_101-概述与语义化.md 第1章

**定义**：
HTML 是用于创建网页的标准标记语言，通过标签（tag）描述网页的结构和内容，由 W3C 制定标准。

**详解**：
HTML 使用标签描述元素，如 `<h1>` 表示标题、`<p>` 表示段落。HTML 文档是纯文本文件，浏览器解析后渲染为可视化页面。HTML5 是最新版本，增加了 Canvas、多媒体、表单增强等特性。HTML 负责结构，CSS 负责样式，JavaScript 负责行为。HTML 标签不区分大小写，但推荐使用小写。

---

### 1.2 W3C（万维网联盟）

**名称**：万维网联盟（World Wide Web Consortium）

**缩写**：W3C

**首次出现位置**：C05_101-概述与语义化.md 第1章

**定义**：
W3C 是国际万维网组织，负责制定和推广 Web 标准，包括 HTML、CSS、DOM、XML、SVG 等标准。

**详解**：
W3C 由 Tim Berners-Lee（万维网发明者）于 1994 年创立。使命：通过开发协议、指南和工具促进 Web 发展。重要标准：HTML 4.01 → XHTML → HTML5、CSS1 → CSS2 → CSS3、DOM Level 1/2/3。WHATWG 是另一个组织，与 W3C 共同制定 HTML5 标准。W3C 标准是浏览器实现的依据。

---

### 1.3 标签（Tag）

**名称**：标签（Tag）

**首次出现位置**：C05_101-概述与语义化.md 第2章

**定义**：
标签是 HTML 的基本单元，由尖括号包围的关键词组成，用于定义网页元素，如 `<div>`、`<span>`、`<a>`。

**详解**：
成对标签：`<tag>内容</tag>`，如 `<p>段落</p>`。自闭合标签：`<br />`、`<img />`、`<input />`。标签属性：`<tag attr="value">`，如 `<a href="url">`。标签可以嵌套，不能交叉嵌套。语义化标签：使用有意义的标签（如 `<article>`、`<nav>`）替代无意义的 `<div>`。HTML5 不强制自闭合斜杠，但 XHTML 要求。

---

### 1.4 元素（Element）

**名称**：元素（Element）

**首次出现位置**：C05_101-概述与语义化.md 第2章

**定义**：
元素是 HTML 文档的基本构建块，由开始标签、内容、结束标签组成，如 `<p>这是一段文本</p>`。

**详解**：
元素分类：块级元素（display: block，如 `<div>`）和行内元素（display: inline，如 `<span>`）。HTML5 新分类：流内容（flow content）、标题内容（heading content）、分段内容（sectioning content）等。元素嵌套：父元素/子元素/兄弟元素。空元素：没有内容的元素，如 `<img>`、`<br>`、`<input>`。元素属性：全局属性（所有元素可用）和特定属性。

---

### 1.5 属性（Attribute）

**名称**：属性（Attribute）

**首次出现位置**：C05_101-概述与语义化.md 第2章

**定义**：
属性提供元素的额外信息，写在开始标签内，以 name="value" 的形式存在。

**详解**：
全局属性：id（唯一标识）、class（样式类）、style（内联样式）、title（提示文本）、data-*（自定义数据）。特定属性：`<a>` 的 href、`<img>` 的 src 和 alt、`<input>` 的 type。布尔属性：只写属性名即可，如 disabled、checked、readonly。属性值引号：推荐使用双引号，单引号也可，HTML5 允许不加引号但不是好习惯。

---

### 1.6 DOCTYPE

**名称**：文档类型声明（Document Type Declaration）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
DOCTYPE 声明位于 HTML 文档第一行，告诉浏览器使用的 HTML 版本，以便正确渲染页面。

**详解**：
HTML5 声明：`<!DOCTYPE html>`（简洁，所有现代浏览器使用标准模式）。HTML 4.01：`<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">`。没有 DOCTYPE 会导致浏览器使用 quirks mode（怪异模式），渲染不一致。HTML5 之前的声明较长且复杂，HTML5 统一使用简短的 `<!DOCTYPE html>`。放置位置：文档最顶部，`<html>` 标签之前。

---

### 1.7 HTML5

**名称**：HTML5（HTML5）

**首次出现位置**：C05_101-概述与语义化.md 第1章

**定义**：
HTML5 是 HTML 的第五个主要版本，引入了语义化标签、视频音频播放（原生支持）、Canvas 绘图、离线存储、WebSocket 等新特性。

**详解**：
新增语义标签：`<header>`、`<nav>`、`<article>`、`<section>`、`<aside>`、`<footer>`、`<main>`。多媒体：`<video>`、`<audio>` 原生支持。图形：`<canvas>` 用于 2D 绘图，WebGL 用于 3D。表单增强：新的 input 类型（email、url、date、range、color）、表单验证属性。API：Geolocation、Web Storage（localStorage/sessionStorage）、WebSocket、Worker、History API。移动端支持：viewport meta 标签、触摸事件。离线应用：manifest 属性。

---

## 2. 语义化标签类

### 2.1 语义化（Semantic）

**名称**：语义化 HTML（Semantic HTML）

**首次出现位置**：C05_101-概述与语义化.md 第1章

**定义**：
语义化是指使用有意义的 HTML 标签描述内容结构，使页面结构清晰，便于开发者阅读和浏览器、搜索引擎理解。

**详解**：
语义化优势：SEO 友好（搜索引擎更好理解内容）、无障碍支持（屏幕阅读器正确朗读）、代码可维护性提升、团队协作更顺畅。语义化标签：`header`（头部）、`nav`（导航）、`main`（主要内容）、`article`（文章）、`section`（章节）、`aside`（侧边栏）、`footer`（底部）。非语义标签：`div`、`span` 无特定含义，用于布局。无语义滥用：避免 `<div>` 嵌套过深。

---

### 2.2 header 元素

**名称**：头部元素（header Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
header 是 HTML5 的语义化标签，用于定义页面或区块的头部区域，通常包含标题、logo、导航等。

**详解**：
header 可以出现多次：每个 section 或 article 都可以有自己的 header。典型内容：网站 logo、标题 `<h1>-<h6>`、导航 `<nav>`、搜索框。header 不等于 `<head>`：head 是文档元数据容器，不显示在页面上。语义：表示内容的介绍性信息。嵌套：`header > nav > ul > li` 常见结构。

---

### 2.3 nav 元素

**名称**：导航元素（nav Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
nav 是 HTML5 语义化标签，用于包裹主要的导航链接区块，帮助浏览器识别页面导航区域。

**详解**：
nav 用途：主站导航、页面内锚点导航、面包屑导航、翻页导航。不是所有链接都需要 nav，只放主要导航。页面可以有多个 nav：主导航（网站级）、侧边栏导航、页脚链接等。nav 内部通常使用 `<ul>` + `<a>` 构建链接列表。语义化：搜索引擎识别 nav 为主导航，提升 SEO。

---

### 2.4 main 元素

**名称**：主要内容元素（main Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
main 是 HTML5 语义化标签，用于包裹页面的主要内容，每个页面应该只有一个 main 元素。

**详解**：
main 特性：页面独一无二，不应重复。内容：文章正文、产品列表、搜索结果等核心内容。不包含：导航、侧边栏、footer、版权信息。全浏览器支持：现代浏览器都支持 main。ARIA 角色：`<main role="main">` 增强无障碍。嵌套结构：`<body><header>...<main>...</main><footer>...</footer></body>`。

---

### 2.5 article 元素

**名称**：文章元素（article Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
article 是 HTML5 语义化标签，用于表示独立的、可以独立分发或复用的内容块，如博客文章、新闻报道、用户评论。

**详解**：
article 特性：独立自包含的文档。典型场景：博客文章、新闻条目、论坛帖子、用户评论、产品卡片。article 可以嵌套：文章内的回复可以使用嵌套 article。内部结构：通常包含 header、段落、footer。article 与 section 区别：article 更强调独立性和可分发性，section 是通用章节。

---

### 2.6 section 元素

**名称**：章节元素（section Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
section 是 HTML5 语义化标签，用于将页面内容分为主题相关的分组，通常有标题。

**详解**：
section 用途：章节、标签页内容、区块。特点：主题相关内容、有逻辑关联、应有标题（`<h2>`-`<h6>`）。section vs div：div 是纯布局容器，section 是语义分组。有序内容用 ol，无序用 ul。section 不适合：单纯作为样式容器的 `<div class="wrapper">` 仍用 div。

---

### 2.7 aside 元素

**名称**：侧边栏元素（aside Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
aside 是 HTML5 语义化标签，用于表示与周围内容间接相关的辅助信息，如侧边栏、引用块、广告区域。

**详解**：
aside 用途：侧边栏（博客文章目录、广告、搜索框）、引用的文段、补充信息。aside 与主内容关联但不属于核心内容。aside 可以放在 article 或 section 内，表示当前文章的辅助信息。常见场景：文章侧边栏的标签云、相关文章推荐、作者信息卡。

---

### 2.8 footer 元素

**名称**：底部元素（footer Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
footer 是 HTML5 语义化标签，用于定义页面或区块的底部区域，通常包含版权、联系方式、相关文档链接。

**详解**：
footer 特性：可以出现多次，每个 section/article 都可以有 footer。典型内容：版权信息（© 2024）、联系方式、站点地图、相关链接、返回顶部。footer 语义：表示结束部分的信息。嵌套结构：`<footer><nav>...</nav><p>© 2024</p></footer>`。

---

### 2.9 figure 和 figcaption

**名称**：图表元素 / 图注元素（figure / figcaption Element）

**首次出现位置**：C05_101-概述与语义化.md 第3章

**定义**：
figure 用于封装媒介内容（图片、图表、代码块），figcaption 用于为其添加图注说明。

**详解**：
figure 用法：`<figure><img src="..."><figcaption>图片说明</figcaption></figure>`。figure 用途：图片、图表、代码块、引用（`<blockquote>`）、诗词。figcaption 位置：通常放在 figure 内容的第一个或最后一个子元素。figure vs img：img 只是图片，figure 是图文组合的语义容器。

---

## 3. 全局属性类

### 3.1 id 属性

**名称**：唯一标识属性（id Attribute）

**首次出现位置**：C05_102-基础标签与全局属性.md 第1章

**定义**：
id 是全局属性，为元素提供在文档中唯一的标识符，用于链接锚点、CSS/JS 操作、辅助技术识别。

**详解**：
命名规则：必须唯一、一个文档中不能有重复 id、以字母开头（可含数字和连字符）、区分大小写。用途：锚点链接 `<a href="#section">`、CSS 选择器 `#myId`、JavaScript `document.getElementById('myId')`。id vs class：id 唯一，class 可重复。id 的特殊性：URL 哈希值（#id）定位、label 的 for 属性关联表单控件。

---

### 3.2 class 属性

**名称**：类属性（class Attribute）

**首次出现位置**：C05_102-基础标签与全局属性.md 第1章

**定义**：
class 是全局属性，为元素指定一个或多个类名，用于 CSS 样式分组和 JavaScript 操作。

**详解**：
多个类：空格分隔，如 `<div class="container main-content">`。命名规范：语义化命名（.btn-primary）、BEM 命名法（.card__title）。class vs id：class 可重复用于多个元素，id 必须唯一。CSS 选择器：`.className`。JavaScript：`document.getElementsByClassName('className')`。classList API：`element.classList.add/remove/toggle/contains()`。

---

### 3.3 style 属性

**名称**：内联样式属性（style Attribute）

**首次出现位置**：C05_102-基础标签与全局属性.md 第1章

**定义**：
style 是全局属性，用于直接在 HTML 元素上定义 CSS 样式，覆盖外部样式表。

**详解**：
语法：`<tag style="property: value; property: value;">`。优先级：内联样式 > id选择器 > class选择器 > 元素选择器。不推荐：大量使用 style 会使 HTML 混乱，难以维护。用途：测试、快速修复、邮件模板（外部样式表支持差）。CSS 简写：`style="margin: 10px; padding: 5px;"`。媒体查询内联：`style="..." @media (...) {...}` 部分支持。

---

### 3.4 data-* 属性

**名称**：自定义数据属性（data-* Attribute）

**首次出现位置**：C05_102-基础标签与全局属性.md 第1章

**定义**：
data-* 是全局属性，允许在 HTML 元素上存储自定义数据，供 JavaScript 读取和使用。

**详解**：
语法：`data-*` 中 *必须是至少一个字符，可包含连字符。示例：`data-user-id="123"`、`data-product-name="手机"`。JavaScript 访问：`element.dataset.userId`（连字符转驼峰）、`element.getAttribute('data-user-id')`。用途：存储组件状态、传递后端数据、客户端模板。无障碍：data-* 不影响无障碍，屏幕阅读器不朗读。

---

### 3.5 title 属性

**名称**：提示文本属性（title Attribute）

**首次出现位置**：C05_102-基础标签与全局属性.md 第1章

**定义**：
title 是全局属性，为元素提供额外的描述性信息，鼠标悬停时显示工具提示。

**详解**：
显示方式：大多数浏览器在鼠标悬停时显示 tooltip。title vs alt：img 的 alt 是图片无法加载时显示的文字，title 是额外说明。title 不是替代：不应以 title 替代必要的可见文本。标题层级：`<h1>` 到 `<h6>` 是内容标题，title 是元素描述。全局可用：所有 HTML 元素都可以有 title 属性。

---

## 4. 基础标签类

### 4.1 h1-h6 标题标签

**名称**：标题标签（Heading Elements）

**首次出现位置**：C05_102-基础标签与全局属性.md 第2章

**定义**：
h1 到 h6 是标题标签，用于定义 HTML 文档的层次结构，h1 是最高级别，h6 是最低级别。

**详解**：
h1：每个页面应该只有一个，表示页面主标题。h2-h6：可以根据需要多次使用，表示副标题、子标题。层级：不要跳级（如 h1 后直接 h4），保持结构清晰。SEO 重要性：搜索引擎用标题理解页面结构，h1 最重要。h1 vs title：h1 是页面内容标题，title 是浏览器标签和搜索结果显示的标题。

---

### 4.2 p 段落标签

**名称**：段落标签（Paragraph Element）

**首次出现位置**：C05_102-基础标签与全局属性.md 第2章

**定义**：
p 标签定义段落，表示一段独立的文本内容，浏览器会自动在段落前后添加间距。

**详解**：
p 特性：块级元素、自动换行、上下外边距。空段落：`<p></p>` 浏览器渲染为空但占用空间，建议用 `<br>` 或 CSS。语义：表示一个独立的文本段落。不要用 p 包裹块级元素如 div。段落内换行：使用 `<br>` 标签。多个空格：HTML 连续空格只显示一个，用 `&nbsp;` 或 `<pre>` 保留。

---

### 4.3 a 超链接标签

**名称**：超链接标签（Anchor Element）

**首次出现位置**：C05_102-基础标签与全局属性.md 第2章

**定义**：
a 标签创建指向其他页面、文件、位置的链接，是 Web 导航的基础元素。

**详解**：
href 属性：链接目标 URL，如 `<a href="https://example.com">`。锚点链接：`<a href="#section-id">` 跳转到当前页面某位置。邮件链接：`href="mailto:test@example.com"`。电话链接：`href="tel:+1234567890"`。target 属性：`_self`（默认，当前窗口）、`_blank`（新窗口）、`_parent`、`_top`。rel 属性：`_blank` 时建议加 `rel="noopener noreferrer"` 防钓鱼。

---

### 4.4 img 图片标签

**名称**：图片标签（Image Element）

**首次出现位置**：C05_102-基础标签与全局属性.md 第2章

**定义**：
img 是自闭合标签，用于在网页中嵌入图片，是非替换元素（inline）。

**详解**：
必需属性：src（图片地址）、alt（替代文本）。其他属性：width/height（宽高）、loading（lazy/加载策略）、srcset（响应式图片）。alt 文本：图片无法显示时显示、屏幕阅读器朗读、SEO。响应式：`max-width: 100%; height: auto;` 防止图片溢出。格式：JPEG（照片）、PNG（透明/图标）、SVG（矢量图）、WebP（现代格式）。图注：结合 figure 和 figcaption 使用。

---

### 4.5 div 标签

**名称**：分区标签（Division Element）

**首次出现位置**：C05_102-基础标签与全局属性.md 第2章

**定义**：
div 是通用的块级容器，没有语义，用于将内容分组进行样式布局，是 HTML 布局的基石。

**详解**：
div 特性：块级元素、无任何语义、纯容器。用途：页面布局区块（配合 CSS）、包裹多个元素统一设置样式、JavaScript 操作。语义化替代：能用 article、section、nav、aside 等语义标签时不用 div。嵌套：可以无限嵌套，但应避免过深（影响性能和维护性）。class 命名：使用语义化类名如 `.header`、`\.content`，避免 `.red-text`。

---

### 4.6 span 标签

## 7. 存储与 API 类

### 7.1 localStorage

**名称**：本地存储（localStorage）

**首次出现位置**：C05_105-存储与WebAPI.md 第1章

**定义**：
localStorage 是 Web Storage API 的一部分，提供在浏览器中持久存储键值对数据的机制，容量约 5-10MB。

**详解**：
API：`localStorage.setItem(key, value)`、`localStorage.getItem(key)`、`localStorage.removeItem(key)`、`localStorage.clear()`。数据类型：只能存储字符串，对象需 JSON.stringify()。持久性：数据永不过期（除非手动清除或清除浏览器数据）。同源策略：仅同源（协议+域名+端口）可访问。容量：各浏览器约 5-10MB。用途：用户偏好设置、登录状态、应用缓存数据。

---

### 7.2 sessionStorage

**名称**：会话存储（sessionStorage）

**首次出现位置**：C05_105-存储与WebAPI.md 第1章

**定义**：
sessionStorage 与 localStorage 类似，但数据仅在当前会话（浏览器标签页）期间有效，关闭标签页或会话结束即清除。

**详解**：
API：与 localStorage 完全相同的方法。生命周期：标签页打开时存在，关闭标签页即清除。页面刷新不清除：只要不关闭标签页。不同标签页：不同标签页的 sessionStorage 独立。用途：表单草稿、临时状态、多步骤向导数据。安全性：敏感数据应加密或不用 Web Storage。

---

### 7.3 Cookie

**名称**：Cookie（Cookie）

**首次出现位置**：C05_105-存储与WebAPI.md 第1章

**定义**：
Cookie 是浏览器存储的小型文本文件，每次 HTTP 请求会自动发送到服务器，用于会话管理、个性化、追踪。

**详解**：
属性：name=value（数据）、expires（过期时间）、path（有效路径）、domain（有效域名）、secure（仅 HTTPS）、HttpOnly（JS 不可访问）。容量：约 4KB。传输：每次 HTTP 请求自动携带。用途：Session ID 存储、记住登录、个性化设置、追踪分析。SameSite 属性：防止 CSRF 攻击（Strict/Lax/None）。与 Web Storage 对比：Cookie 容量小但会随请求发送。

---

### 7.4 IndexedDB

**名称**：索引数据库（IndexedDB）

**首次出现位置**：C05_105-存储与WebAPI.md 第1章

**定义**：
IndexedDB 是浏览器的大容量 NoSQL 数据库系统，用于在客户端存储大量结构化数据，支持索引和事务。

**详解**：
特点：大容量（约 50MB+ 或无限制）、存储结构化数据、支持索引、异步 API（不阻塞 UI）。核心概念：Database（数据库）、Object Store（对象仓库，类比表）、Index（索引）、Transaction（事务）、Cursor（游标）。操作流程：打开数据库 → 创建 Object Store → 增删改查。API：较复杂，通常使用包装库如 Dexie.js。替代：LocalForage（Promise 化 API）。用途：离线应用、大型数据缓存。

---

### 7.5 Fetch API

**名称**：Fetch API（Fetch API）

**首次出现位置**：C05_105-存储与WebAPI.md 第2章

**定义**：
Fetch 是现代浏览器提供的网络请求 API，用于替代 XMLHttpRequest，提供更强大和灵活的请求方式。

**详解**：
基本用法：`fetch(url)` 返回 Promise。Response：`fetch(url).then(res => res.json())` 处理 JSON。POST 请求：`fetch(url, {method: 'POST', body: formData, headers: {...}})`。错误处理：HTTP 错误状态（404、500）不会 reject，需手动检查。终止请求：AbortController + signal。用途：RESTful API 调用、文件上传、跨域请求。Headers：Request/Response 头信息。

---

### 7.6 Geolocation API

**名称**：地理位置 API（Geolocation API）

**首次出现位置**：C05_105-存储与WebAPI.md 第2章

**定义**：
Geolocation API 允许 Web 应用获取用户的地理位置信息（经纬度），需用户授权。

**详解**：
API：`navigator.geolocation.getCurrentPosition(successCallback, errorCallback, options)`。PositionOptions：enableHighAccuracy（高精度）、timeout（超时）、maximumAge（缓存时间）。成功回调：Position 对象包含 coords（经纬度、海拔、速度）和 timestamp。误差：精度从几米到几千米取决于设备。隐私：必须用户授权，浏览器会弹出提示。watchPosition：持续监听位置变化（适合导航应用）。

## 8. Web API 进阶类

### 8.1 Web Workers

**名称**：Web Workers（Web Workers）

**首次出现位置**：C05_105-存储与WebAPI.md 第3章

**定义**：
Web Workers 是在后台线程运行 JavaScript 的机制，用于执行耗时计算，不阻塞主线程和 UI 渲染。

**详解**：
类型：Dedicated Worker（专用，一次性）、Shared Worker（共享，可多窗口共享）、Service Worker（代理，控制网络请求和缓存）。通信：postMessage() 发送消息，onmessage 接收消息。限制：无法访问 DOM、window、document，只能与主线程通信。适用场景：大量数据处理、复杂计算、图片/视频处理。终止：worker.terminate() 手动终止。

---

### 8.2 History API

**名称**：历史记录 API（History API）

**首次出现位置**：C05_105-存储与WebAPI.md 第3章

**定义**：
History API 允许操作浏览器的历史记录，实现 SPA（单页应用）的无刷新路由跳转。

**详解**：
pushState()：添加新历史记录，不刷新页面。replaceState()：替换当前历史记录。popstate 事件：浏览器前进/后退时触发。state 属性：存储与历史记录关联的状态对象。配合：History API + 事件监听 + 动态加载内容 = SPA 路由。示例：`history.pushState({page: 'about'}, '', '/about')`。

---

### 8.3 Drag and Drop API

**名称**：拖放 API（Drag and Drop API）

**首次出现位置**：C05_105-存储与WebAPI.md 第3章

**定义**：
Drag and Drop API 提供原生的拖放交互功能，允许用户拖动元素并放置到目标区域。

**详解**：
事件：draggable 属性启动拖动、dragstart（开始）、dragenter（进入目标）、dragover（悬停）、dragleave（离开）、drop（放置）、dragend（结束）。dataTransfer：传递拖动数据（setData/getData）。dropzone 属性：HTML5 原生放置区支持。默认行为：图片/链接默认拖动，文本默认拖动。常用场景：文件上传（结合文件 API）、列表排序、看板应用。

---

### 8.4 WebSocket

**名称**：WebSocket（WebSocket）

**首次出现位置**：C05_105-存储与WebAPI.md 第3章

**定义**：
WebSocket 是在单个 TCP 连接上提供全双工通信的协议，实现服务器主动向客户端推送数据。

**详解**：
建立连接：`const ws = new WebSocket('ws://example.com/socket')`。事件：onopen、onmessage、onclose、onerror。发送数据：`ws.send('message')`。服务端推送：服务器可以随时发送消息给客户端。应用：实时聊天、股票行情、在线游戏、协作编辑。心跳机制：定期 ping/pong 保持连接。wss：WebSocket Secure，使用 TLS 加密。

---

### 8.5 Intersection Observer API

**名称**：交叉观察者 API（Intersection Observer API）

**首次出现位置**：C05_105-存储与WebAPI.md 第3章

**定义**：
Intersection Observer API 异步检测元素是否进入或离开视口，常用于实现懒加载和无限滚动。

**详解**：
创建观察者：`const observer = new IntersectionObserver(callback, options)`。观察：`observer.observe(element)`。callback 参数：IntersectionObserverEntry 数组，包含 intersectionRatio、isIntersecting、boundingClientRect。options：threshold（触发阈值 0-1）、root（根元素）、rootMargin（根边距）。disconnect()：停止观察。unobserve()：停止观察特定元素。应用：图片懒加载、无限滚动、广告曝光统计、动画触发。

---

### 8.6 Notification API

**名称**：通知 API（Notification API）

**首次出现位置**：C05_105-存储与WebAPI.md 第3章

**定义**：
Notification API 允许 Web 应用向用户发送系统级通知，即使页面不在前台也能显示。

**详解**：
权限：`Notification.requestPermission()` 请求通知权限。创建通知：`new Notification('标题', {body: '内容', icon: '...', tag: '...'})`。生命周期：点击通知触发 onclick 事件。自动关闭：可选 autoClose 毫秒数。事件：onclick、onshow、onerror、onclose。应用：Web 应用消息推送、新邮件提醒、任务完成通知。权限状态：granted（允许）、denied（拒绝）、default（未决定）。

link 标签定义当前文档与外部资源的关系，最常用于引入外部 CSS 样式表。

**详解**：
引入样式表：`<link rel="stylesheet" href="style.css">`。rel 属性：stylesheet（样式表）、icon（网站图标 favicon）、preload（预加载）、alternate（替代版本，如 RSS）。href 属性：资源地址。type 属性：text/css（MIME 类型）。media 属性：`<link media="print" href="print.css">` 媒体查询。preload：用 `as` 指定预加载资源类型提高性能。

### 9.3 script 标签

**名称**：脚本标签（Script Element）

**首次出现位置**：C05_102-基础标签与全局属性.md 第3章

**定义**：
script 标签用于在 HTML 文档中嵌入或引用 JavaScript 代码，是 Web 交互的核心元素。

**详解**：
内联脚本：`<script> console.log('hello'); </script>`。外部脚本：`<script src="app.js"></script>`。async/defer：async（异步下载，立即执行）、defer（延迟执行，DOM 解析完成后按顺序执行）。模块脚本：`<script type="module">` ES6 模块。noscript：`<noscript>请启用 JavaScript</noscript>` 无脚本时显示。位置：通常放在 `<body>` 底部或配合 defer/async。

### 9.4 iframe

**名称**：内联框架（Inline Frame）

**首次出现位置**：C05_102-基础标签与全局属性.md 第3章

**定义**：
iframe 在当前页面嵌入另一个 HTML 页面，实现内容隔离的嵌套浏览上下文。

**详解**：
src 属性：被嵌入页面地址。width/height：宽高（像素或百分比）。sandbox：安全沙盒属性，控制 iframe 内行为（allow-forms、allow-scripts、allow-same-origin 等）。 seamless：无缝嵌入样式（部分支持）。srcdoc：内联 HTML 内容替代 src。用途：嵌入第三方内容（YouTube 视频、地图）、广告隔离、沙盒执行。安全问题：X-Frame-Options 防止点击劫持。

### 9.5 details 和 summary

**名称**：详情折叠元素（Details / Summary Element）

**首次出现位置**：C05_102-基础标签与全局属性.md 第2章

**定义**：
details 创建可折叠的部件，summary 是其标题，点击标题切换展开/收起状态。

**详解**：
details 特性：布尔属性 open（默认展开）、默认收起。summary：可点击的标题，必须有。内容：summary 后可放置任意 HTML 内容。动画：CSS transition 对 height 不生效，可用 grid/clip-path 实现动画。语义：折叠/展开是原生支持的交互，无需 JavaScript。应用：FAQ、手风琴组件、配置面板。
