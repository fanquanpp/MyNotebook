## 1. 基础概念类

### 1.1 CSS（层叠样式表）

**名称**：层叠样式表（Cascading Style Sheets）

**缩写**：CSS

**首次出现位置**：C06_101-概述与语法.md 第1章

**定义**：
CSS 是用于描述 HTML/XML 文档外观和格式的语言，控制网页的颜色、布局、字体等视觉表现，实现内容与表现分离。

**详解**：
CSS 的"Cascading"（层叠）指多个样式规则可以作用于同一元素，按优先级合并。核心概念：选择器（Selector）、属性（Property）、值（Value）。历史版本：CSS1（1996）、CSS2（1998）、CSS3（模块化，2000s-至今）。CSS3 新特性：Flexbox、Grid、动画、过渡、媒体查询、伪元素、滤镜。模块化：Selectors Level 3、Box Model Level 3、Flexbox Level 1、Grid Layout Level 1 等独立模块。

---

### 1.2 选择器（Selector）

**名称**：选择器（Selector）

**首次出现位置**：C06_101-概述与语法.md 第3章

**定义**：
选择器用于定位 HTML 文档中的元素，以便对其应用样式规则。

**详解**：
基础选择器：元素选择器（`p`）、类选择器（`.class`）、ID 选择器（`#id`）、通配符选择器（`*`）。组合选择器：后代（` ` 空格）、子（`>`）、相邻兄弟（`+`）、通用兄弟（`~`）。属性选择器：`[attr]`、`[attr=value]`、`[attr~=value]`、`[attr|=value]`。伪类：`:hover`、`:focus`、`:nth-child()`。伪元素：`::before`、 `::after`、 `::first-line`。优先级：ID > 类/属性/伪类 > 元素/伪元素 > 通配符。

---

### 1.3 属性（Property）

**名称**：CSS 属性（Property）

**首次出现位置**：C06_101-概述与语法.md 第2章

**定义**：
CSS 属性是样式特征名称，如 color、font-size、margin，定义要设置的视觉方面。

**详解**：
属性格式：属性名: 属性值;。常用属性分类：布局（display、position、flex、grid）、盒子（margin、padding、border、width、height）、文本（font-size、color、text-align、line-height）、背景（background-color、background-image）、动画（transition、animation）。属性值类型：关键字（auto、inherit）、数值（px、rem、%、em）、颜色（#hex、rgb()、hsl()）。缩写属性：margin 可缩写四值（top right bottom left）。

---

### 1.4 值（Value）

**名称**：CSS 值（Value）

**首次出现位置**：C06_101-概述与语法.md 第2章

**定义**：
CSS 值是属性的具体设置，如 16px（像素值）、red（颜色值）、bold（关键字值）。

**详解**：
数值单位：px（像素）、em（相对于父元素字体大小）、rem（相对于根元素字体大小）、%（相对于父元素对应属性值）、vw/vh（视口单位）。颜色值：颜色名（red）、十六进制（#fff、#ffffff）、rgb()/rgba()、hsl()/hsla()。函数值：calc()、var()、url()、attr()。关键字：inherit（继承）、initial（初始值）、unset（未设置）、auto、none。属性值可以有多个，通过空格或逗号分隔。

---

### 1.5 声明块（Declaration Block）

**名称**：声明块（Declaration Block）

**首次出现位置**：C06_101-概述与语法.md 第2章

**定义**：
声明块是包含在一对大括号内的样式声明集合，由属性和值组成。

**详解**：
语法： `{ property1: value1; property2: value2; }`。分号分隔：最后一个属性后的分号可选，建议保留。空格处理：大括号内空格通常忽略。缩写属性：`margin: 10px;` 相当于 `margin-top: 10px; margin-right: 10px; margin-bottom: 10px; margin-left: 10px;`。空声明块：`{}` 合法但无效果。

---

### 1.6 规则集（Rule Set）

**名称**：规则集（Rule Set）

**首次出现位置**：C06_101-概述与语法.md 第2章

**定义**：
规则集由选择器和声明块组成，是 CSS 的基本编写单元。

**详解**：
格式：`selector { declarations }`。注释：`/* comment */`。多层规则：`h1, h2, h3 { color: blue; }` 多选择器共用同一声明块。空格：`h1 { color: blue; }` vs `h1{color:blue;}` 空格可选但推荐格式化。CSS 文件：由多个规则集组成。

---

### 1.7 层叠（Cascading）

**名称**：层叠（Cascade）

**首次出现位置**：C06_101-概述与语法.md 第4章

**定义**：
层叠是 CSS 解决多个样式规则应用于同一元素的冲突的机制，按优先级、来源、 specificity 排序。

**详解**：
优先级来源（从低到高）：浏览器默认样式 < 用户代理样式 < 用户普通样式 < 开发者普通样式 < 开发者 !important < 用户 !important。 specificity 计算：内联样式(1000) > ID(100) > 类/属性/伪类(10) > 元素/伪元素(1)。来源：浏览器、用户、开发者。!important：最高优先级，应尽量避免使用。继承：某些属性默认继承父元素。同一优先级：后定义的规则覆盖先定义的。

---

### 1.8 继承（Inheritance）

**名称**：继承（Inheritance）

**首次出现位置**：C06_101-概述与语法.md 第4章

**定义**：
继承允许子元素自动获得父元素的某些样式，如字体、颜色，无需重复定义。

**详解**：
继承属性：typography 相关属性如 color、font-size、font-family、line-height、text-align、visibility。不继承属性：盒模型属性（margin、padding、border、width）、position、display、background。inherit 关键字：强制继承。initial 关键字：设为初始值。unset 关键字：若可继承则 inherit，否则 initial。all 属性：快捷设置所有属性继承/初始/未设置。控制继承：`inherit`、`initial`、`unset`。

---

## 2. 选择器系统类

### 2.1 元素选择器

**名称**：元素选择器（Element Selector / Type Selector）

**首次出现位置**：C06_102-选择器系统.md 第1章

**定义**：
元素选择器通过 HTML 标签名选择所有匹配的元素，如 `p` 选择所有段落。

**详解**：
语法：直接写标签名，不带尖括号。示例：`div { }` 选择所有 div 元素。性能：浏览器从右向左匹配，通配符选择器最慢，元素选择器较快。区分大小写：HTML 标签不区分大小写（XML 区分）。h5 以前的元素：`header`、`nav`、`section`、`article` 等新元素也可作为元素选择器。

---

### 2.2 类选择器

**名称**：类选择器（Class Selector）

**首次出现位置**：C06_102-选择器系统.md 第1章

**定义**：
类选择器通过元素的 class 属性值选择元素，以点号（.）开头。

**详解**：
语法：`.classname`。多类：`p.highlight` 选择同时有 highlight 类的 p 元素。class 属性：一个元素可有多个类（空格分隔）。命名：BEM 规范（block__element--modifier）、语义化命名。性能：比 ID 慢，比元素选择器快。应用：样式分组、组件化。

---

### 2.3 ID 选择器

**名称**：ID 选择器（ID Selector）

**首次出现位置**：C06_102-选择器系统.md 第1章

**定义**：
ID 选择器通过元素的 id 属性值选择元素，以井号（#）开头。

**详解**：
语法：`#idname`。唯一性：id 在文档中应唯一，一个 id 只应有一个元素。优先级：比类选择器高。比类选择器快但差异不大。JavaScript 关联：`getElementById('idname')`。空格处理：id 值不能包含空格，不能以数字开头。ID vs Class：ID 用于唯一元素，Class 用于可复用的样式分组。

---

### 2.4 通配符选择器

**名称**：通配符选择器（Universal Selector）

**首次出现位置**：C06_102-选择器系统.md 第1章

**定义**：
通配符选择器（*）选择所有元素，常用于重置浏览器默认样式。

**详解**：
语法：`* { margin: 0; padding: 0; }` 全局重置。性能：最慢的选择器（从右向左匹配），大文档慎用。组合：`*` 可与其他选择器组合如 `*.warning`（等价于 `.warning`）。CSS 重置：`* { box-sizing: border-box; }` box-sizing 全局设置。继承问题：`*` 不继承，所有元素都匹配。

---

### 2.5 属性选择器

**名称**：属性选择器（Attribute Selector）

**首次出现位置**：C06_102-选择器系统.md 第2章

**定义**：
属性选择器通过元素的属性或属性值选择元素，提供 `[attr]` 形式的多种匹配方式。

**详解**：
匹配类型：`[attr]`（有该属性）、`[attr=value]`（精确匹配）、`[attr~=value]`（空格分隔的词列表中匹配）、`[attr|=value]`（连字符分隔的值）、`[attr^=value]`（开头匹配）、`[attr$=value]`（结尾匹配）、`[attr*=value]`（任意位置包含）。大小写：`[attr=value i]` 忽略大小写（CSS 4）。示例：`a[href^="https"]` 选择所有 HTTPS 链接。

---

### 2.6 伪类（Pseudo-class）

**名称**：伪类（Pseudo-class）

**首次出现位置**：C06_102-选择器系统.md 第3章

**定义**：
伪类是基于元素状态或文档结构特征选择的特殊选择器，以冒号（:）开头。

**详解**：
状态伪类：`:hover`（悬停）、`:focus`（焦点）、`:active`（激活）、`:visited`（已访问）。结构伪类：`:first-child`、 `:last-child`、 `:nth-child(n)`、 `:nth-of-type()`、 `:empty`。否定伪类：`:not(selector)`。表单伪类：`:valid`、 `:invalid`、 `:checked`、 `:disabled`、 `:enabled`。其他：`:lang()`、 `:target`（URL 哈希指向的元素）。语法：`selector:pseudo-class { }`。

---

### 2.7 伪元素（Pseudo-element）

**名称**：伪元素（Pseudo-element）

**首次出现位置**：C06_102-选择器系统.md 第3章

**定义**：
伪元素是在文档中插入虚构的元素，用于样式化部分内容，以双冒号（::）开头。

**详解**：
常用伪元素：`::before`（内容前插入）、`::after`（内容后插入）、`::first-line`（首行）、`::first-letter`（首字母）。content 属性：::before/::after 必须配合 content 属性。旧语法：单冒号 `::before` 在 IE8 支持。::selection：用户选中文本的样式。::placeholder：输入框占位文本样式（需前缀）。::marker：列表项标记样式（li::marker）。

---

### 2.8 组合选择器

**名称**：组合选择器（Combinators）

**首次出现位置**：C06_102-选择器系统.md 第2章

**定义**：
组合选择器通过元素之间的关系（祖先、父子、兄弟）选择元素。

**详解**：
后代选择器（空格）：`div p` 选择 div 内所有 p（子孙）。子选择器（>`）：`div > p` 选择 div 的直接子 p（仅子代）。相邻兄弟（+）：`h1 + p` 选择 h1 后紧跟的 p。通用兄弟（~）：`h1 ~ p` 选择 h1 后所有兄弟 p。组合链：`ul > li:first-child`。空格可省略：`div .class` 等同于 `div .class`。

---

### 2.9 specificity（优先级）

**名称**：优先级计算（Specificity）

**首次出现位置**：C06_102-选择器系统.md 第4章

**定义**：
Specificity 是 CSS 选择器优先级的计算规则，决定当多个规则作用于同一元素时哪个规则生效。

**详解**：
计算方式：`(ID数, 类/属性/伪类数, 元素/伪元素数)`。比较规则：从左到右逐个比较，先比较 ID 数量，相同则比较类，以此类推。内联样式：最高优先级 specificity 为 (1, 0, 0, 0)。!important：覆盖 specificity，但自身仍按 specificity 排序。通配符：`*(0,0,0,0)`。同一优先级：后定义胜出（代码顺序）。伪类如 :nth-child() 算作类。

## 3. 盒模型类

### 3.1 盒模型（Box Model）

**名称**：盒模型（Box Model）

**首次出现位置**：C06_103-盒模型.md 第1章

**定义**：
CSS 盒模型是 CSS 布局的基础概念，每个 HTML 元素都被视为一个矩形盒子，由内容、内边距、边框、外边距组成。

**详解**：
盒子构成（从内到外）：Content（内容）→ Padding（内边距）→ Border（边框）→ Margin（外边距）。默认值：width/height 只作用于 content。计算方式：标准盒模型（width = content）、IE 盒模型（width = content + padding + border）。box-sizing：控制盒模型计算方式。display 属性：决定盒子类型（block/inline/inline-block/flex/none 等）。

---

### 3.2 margin（外边距）

**名称**：外边距（Margin）

**首次出现位置**：C06_103-盒模型.md 第1章

**定义**：
margin 是盒子边框外的空间，用于控制元素与其他元素的距离，是透明背景的区域。

**详解**：
语法：`margin: top right bottom left;`。缩写：1值（四边）、2值（上下、左右）、3值（上、左右、下）、4值（各边）。负值：margin 可以是负数，用于偏移元素。auto：水平居中常用 `margin: 0 auto;`。折叠：相邻元素的垂直 margin 会折叠（collapse）成较大值。margin-top/bottom：对行内元素（inline）无效。

---

### 3.3 padding（内边距）

**名称**：内边距（Padding）

**首次出现位置**：C06_103-盒模型.md 第1章

**定义**：
padding 是内容与边框之间的空间，用于增加内容与边框的距离，padding 区域会显示背景色。

**详解**：
语法：与 margin 相同，支持 1-4 值缩写。背景延伸：元素的背景色/背景图会填充 padding。百分比值：`padding: 10%;` 相对于包含块的宽度。负值：padding 不支持负值。行内元素：padding-top/padding-bottom 对行内元素有效果但不改变行高，可能导致重叠。

---

### 3.4 border（边框）

**名称**：边框（Border）

**首次出现位置**：C06_103-盒模型.md 第1章

**定义**：
border 是盒子边框，围绕内容和 padding 的线框，具有宽度、样式、颜色属性。

**详解**：
border-width：边框宽度（thin/medium/thick/px）。border-style：边框样式（none/solid/dashed/dotted/double/groove/ridge/inset/outset）。border-color：边框颜色，默认 currentColor。缩写：`border: width style color;`。单边：`border-top`、`border-right`、`border-bottom`、`border-left`。透明边框：`border: 10px solid transparent;` 常用技巧。border-radius：圆角边框。

---

### 3.5 width / height

**名称**：宽度 / 高度（Width / Height）

**首次出现位置**：C06_103-盒模型.md 第1章

**定义**：
width 和 height 定义元素内容区域的尺寸，配合 box-sizing 决定盒子大小计算方式。

**详解**：
值类型：长度值（px、em、rem）、百分比（相对于包含块）、auto（默认）、min-content、max-content、fit-content。height 注意事项：默认 auto（由内容决定），设为固定高度可能溢出。max-width/max-height：限制元素最大尺寸，防止溢出。min-width/min-height：设置最小尺寸。溢出处理：overflow 属性控制内容超出时的显示。

---

### 3.6 box-sizing

**名称**：盒尺寸计算（box-sizing）

**首次出现位置**：C06_103-盒模型.md 第2章

**定义**：
box-sizing 属性决定 width 和 height 如何计算盒子的总尺寸，是否包含 padding 和 border。

**详解**：
content-box（默认）：width = content。border-box：width = content + padding + border，包含块更容易控制。更推荐：`*{ box-sizing: border-box; }` 全局应用。padding-box（已废弃）：Firefox 曾支持，现已废弃。布局中的应用：border-box 使 padding 和 border 的增加不会撑大盒子。Flexbox/Grid 子项：自动应用 border-box。

---

### 3.7 display

**名称**：显示属性（display）

**首次出现位置**：C06_103-盒模型.md 第3章

**定义**：
display 属性决定元素的显示类型和布局方式，是 CSS 布局的核心属性。

**详解**：
常见值：none（不显示）、block（块级）、inline（行内）、inline-block（行内块）、flex（弹性盒）、grid（网格）、table（表格）。none vs visibility:hidden：none 不占位，visibility:hidden 占位。inline 特性：无法设置宽高，margin-top/bottom 无效，width 由内容决定。inline-block：行内排列但可设宽高。flex/grid：现代布局方案，功能强大。

---

### 3.8 overflow

**名称**：溢出处理（overflow）

**首次出现位置**：C06_103-盒模型.md 第3章

**定义**：
overflow 属性控制内容超出元素盒子时的显示方式，是剪切、滚动还是显示。

**详解**：
值：visible（默认，显示）、hidden（隐藏）、scroll（始终显示滚动条）、auto（内容溢出时显示滚动条）。单独方向：overflow-x、overflow-y。overflow: hidden 用途：清除浮动、创建裁剪区域。overflow 与 float/absolute：触发 BFC（块格式化上下文）。移动端滚动：iOS 上 `-webkit-overflow-scrolling: touch;` 实现惯性滚动。

---

### 3.9 边距折叠（Margin Collapse）

**名称**：外边距折叠（Margin Collapse）

**首次出现位置**：C06_103-盒模型.md 第2章

**定义**：
相邻块级元素的垂直 margin 在特定条件下会合并为较大值的现象。

**详解**：
折叠条件：块级元素、相邻（中间无 padding/border/行盒/清除浮动分隔）、垂直方向。不会折叠：flex/grid 子项、浮动元素、绝对定位元素、inline-block 元素、根元素 html。折叠结果：取两个 margin 的较大值（非相加）。负 margin：正负 margin 相加。正负相遇：最大正值的绝对值。嵌套关系：父元素与第一个/最后一个子元素也可能折叠。

---

## 4. Flexbox 布局类

### 4.1 Flexbox（弹性盒）

**名称**：弹性盒子布局（Flexible Box / Flexbox）

**首次出现位置**：C06_104-Flex布局.md 第1章

**定义**：
Flexbox 是一维布局模型，用于在主轴和交叉轴方向上排列、对齐、分配弹性项目之间的空间。

**详解**：
容器属性：display: flex（块级弹性容器）、display: inline-flex（行内弹性容器）。主轴：项目排列的主要方向（默认水平，从左到右）。交叉轴：垂直于主轴的方向。优势：垂直居中、等高列、圣杯布局、响应式导航。兼容性：现代浏览器全面支持，IE11 部分支持。

---

### 4.2 flex-direction

**名称**：弹性方向（flex-direction）

**首次出现位置**：C06_104-Flex布局.md 第1章

**定义**：
flex-direction 属性定义弹性容器的主轴方向，决定弹性项目的排列方向。

**详解**：
值：row（默认，水平从左到右）、row-reverse（水平从右到左）、column（垂直从上到下）、column-reverse（垂直从下到上）。主轴方向：flex-direction 决定主轴，flex-wrap 决定是否换行。RTL 支持：direction: rtl 会镜像布局方向。常用组合：`flex-direction: column;` 用于移动端垂直布局。

---

### 4.3 flex-wrap

**名称**：弹性换行（flex-wrap）

**首次出现位置**：C06_104-Flex布局.md 第1章

**定义**：
flex-wrap 属性控制弹性项目是单行排列还是多行排列，以及多行时的排列方向。

**详解**：
值：nowrap（默认，不换行，项目压缩）、wrap（换行，第一行在上）、wrap-reverse（换行，第一行在下）。换行方向：wrap-reverse 沿交叉轴反向排列。多行弹性：每行形成独立的 flex 容器（flex line）。交叉轴对齐：align-content 对多行有效（需要 wrap）。项目压缩：nowrap 时项目会缩小到 min-width 以下则溢出。

---

### 4.4 justify-content

**名称**：主轴对齐（justify-content）

**首次出现位置**：C06_104-Flex布局.md 第2章

**定义**：
justify-content 定义弹性项目在主轴上的对齐方式，分配项目之间及其周围的空间。

**详解**：
值：flex-start（默认，靠主轴起点）、flex-end（靠主轴终点）、center（居中）、space-between（两端对齐，项目间距相等）、space-around（项目两侧间距相等）、space-evenly（项目之间及两侧间距相等）。项目未填满容器时生效。对齐方向受 flex-direction 影响。

---

### 4.5 align-items

**名称**：交叉轴对齐（align-items）

**首次出现位置**：C06_104-Flex布局.md 第2章

**定义**：
align-items 定义弹性项目在交叉轴上的对齐方式，默认 stretch（拉伸）使项目填满容器高度。

**详解**：
值：stretch（默认，拉伸填满）、flex-start（靠交叉轴起点）、flex-end（靠交叉轴终点）、center（居中）、baseline（基线对齐）。baseline：项目第一行文字基线对齐。stretch 限制：项目设置了 height 或 align-self 覆盖则不拉伸。项目对齐：align-self 可单独覆盖单个项目。

---

### 4.6 align-content

**名称**：多行对齐（align-content）

**首次出现位置**：C06_104-Flex布局.md 第2章

**定义**：
align-content 定义多行项目在交叉轴上的对齐方式，仅在 flex-wrap: wrap 且有多行时生效。

**详解**：
值：stretch（默认）、flex-start、flex-end、center、space-between、space-around、space-evenly。与 justify-content 对应，但作用于交叉轴。条件：必须有多行（flex-wrap: wrap）、项目总高度小于容器高度。

---

### 4.7 flex-grow / flex-shrink

**名称**：放大 / 缩小因子（flex-grow / flex-shrink）

**首次出现位置**：C06_104-Flex布局.md 第3章

**定义**：
flex-grow 定义项目放大比例，flex-shrink 定义项目缩小比例，按比例分配容器剩余或不足空间。

**详解**：
flex-grow：默认 0（不放大）。flex-shrink：默认 1（可缩小）。flex-basis：项目在分配多余/不足空间前的基准尺寸。缩写 flex：`flex: grow shrink basis`，如 `flex: 1` 等于 `flex: 1 1 0%`。比例分配：`flex: 1` 和 `flex: 2` 的两个项目分别占 1/3 和 2/3。负值无效。

---

### 4.8 flex-basis

**名称**：基准尺寸（flex-basis）

**首次出现位置**：C06_104-Flex布局.md 第3章

**定义**：
flex-basis 定义在分配多余空间之前，项目占据的主轴空间，类似于元素的初始 width。

**详解**：
值：auto（默认，项目自身尺寸）、<length>（固定值）、content（根据内容）。与 width 关系：flex-basis 为 auto 时，width 生效；flex-basis 为具体值时，width 被忽略。优先级：flex-basis > width > 内容尺寸。flex 缩写：`flex: 1 1 auto`。

---

### 4.9 align-self

**名称**：单独对齐（align-self）

**首次出现位置**：C06_104-Flex布局.md 第3章

**定义**：
align-self 允许单个项目覆盖容器的 align-items 设置，实现独立的交叉轴对齐。

**详解**：
值：auto（默认，继承 align-items）、stretch、flex-start、flex-end、center、baseline。与 align-items 相同的值。应用场景：一行中某项目顶部对齐、其他底部对齐。圣杯布局：中间 flex: 1 自适应，两侧固定宽度。

---

## 5. Grid 布局类

### 5.1 Grid Layout（网格布局）

**名称**：网格布局（Grid Layout）

**首次出现位置**：C06_105-Grid布局.md 第1章

**定义**：
Grid Layout 是二维布局系统，同时控制行和列，实现复杂的网页布局，是当前最强大的 CSS 布局方案。

**详解**：
容器属性：display: grid（块级网格）、display: inline-grid（行内网格）。二维布局：同时处理行和列，不像 Flexbox 是一维。与 Flexbox 配合：Grid 主布局，Flexbox 组件内部。浏览器支持：现代浏览器全面支持，IE11 部分支持（有限语法）。应用场景：页面整体布局、相册、表单、数据展示。

---

### 5.2 grid-template-columns / rows

**名称**：网格轨道模板（grid-template-columns / rows）

**首次出现位置**：C06_105-Grid布局.md 第1章

**定义**：
grid-template-columns 定义网格列轨道，grid-template-rows 定义网格行轨道，决定网格的列数和行高。

**详解**：
单位：px、%、fr（比例单位）、auto、min-content、max-content、minmax()。函数：repeat(n, value) 重复、`minmax(min, max)` 范围、`auto-fill`、`auto-fit`。示例：`grid-template-columns: repeat(3, 1fr)` 三列等宽。响应式：`repeat(auto-fit, minmax(250px, 1fr))` 自动响应式列数。

---

### 5.3 fr 单位

**名称**：比例单位（Fraction Unit）

**首次出现位置**：C06_105-Grid布局.md 第1章

**定义**：
fr 是 Grid 布局专用的比例单位，表示网格容器中可用空间的比例分配。

**详解**：
计算方式：剩余空间 = 容器尺寸 - 固定轨道 - 间距。示例：`1fr 2fr 1fr` 分为 4 份（1+2+1），各占 1/4、2/4、1/4。配合 minmax：`minmax(200px, 1fr)` 至少 200px，可放大。组合：`repeat(auto-fit, minmax(250px, 1fr))`。fr 不适用于 grid-gap。

---

### 5.4 gap（间距）

**名称**：网格间距（Gap）

**首次出现位置**：C06_105-Grid布局.md 第1章

**定义**：
gap 属性设置网格项目之间的间距，简写 grid-gap 已被 gap 取代。

**详解**：
单独设置：row-gap（行间距）、column-gap（列间距）。缩写：`gap: 10px 20px`（10px 行间距，20px 列间距）。缩写：`gap: 10px`（行列间距相同）。注意：gap 只作用于网格项目之间，不作用于容器边缘。gap vs margin：gap 只控制项目间，margin 会折叠。Flexbox 和多列布局也支持 gap。

---

### 5.5 grid-area

**名称**：网格区域（grid-area）

**首次出现位置**：C06_105-Grid布局.md 第2章

**定义**：
grid-area 是网格项目的属性，定义项目在网格中的区域位置，是 grid-row/column 的简写形式。

**详解**：
命名区域：`grid-template-areas: "header header" "sidebar main" "footer footer";`。使用命名：`grid-area: header;`（使用容器定义的区域名）。定位：`grid-area: 1 / 1 / 3 / 2`（row-start / col-start / row-end / col-end）。与 grid-template-areas 配合：先定义区域，再指定项目位置。灵活定位：可以用 grid-area 任意放置项目。

---

### 5.6 grid-template-areas

**名称**：网格区域模板（grid-template-areas）

**首次出现位置**：C06_105-Grid布局.md 第2章

**定义**：
grid-template-areas 通过ASCII艺术形式定义命名网格区域，使网格布局可视化。

**详解**：
语法：`"header header header" "sidebar main main" "footer footer footer"`。命名使用：`.header { grid-area: header; }`。同一区域名连续：占据多个单元格。点号 `.`：空单元格。用处：可视化布局意图、简化网格定位。配合：grid-area 属性使用区域名定位。

---

### 5.7 auto-fit / auto-fill

**名称**：自动填充 / 自动适配（auto-fit / auto-fill）

**首次出现位置**：C06_105-Grid布局.md 第3章

**定义**：
auto-fit 和 auto-fill 是 grid-template-columns/rows 中使用的关键字，自动根据容器宽度填充列/行。

**详解**：
auto-fill：即使没有项目也创建空列/行轨道。auto-fit：没有项目时折叠空轨道，项目占据实际空间。相同点：都创建尽可能多的轨道。不同点：auto-fit 压缩空轨道，auto-fill 保留。响应式：`repeat(auto-fit, minmax(250px, 1fr))` 实现自动响应式网格，无需媒体查询。

---

## 6. 传统布局与定位类

### 6.1 position

**名称**：定位属性（position）

**首次出现位置**：C06_106-传统布局与定位.md 第1章

**定义**：
position 属性决定元素的定位类型，影响元素在文档流中的位置和与其他元素的关系。

**详解**：
值：static（默认，正常流）、relative（相对定位）、absolute（绝对定位）、fixed（固定定位）、sticky（粘性定位）。定位坐标：top、right、bottom、left 属性配合使用。z-index：定位元素叠加顺序。定位上下文：absolute/fixed 的定位参考对象。包含块：relative 的包含块是最近的块级祖先。

---

### 6.2 相对定位（relative）

**名称**：相对定位（Relative Positioning）

**首次出现位置**：C06_106-传统布局与定位.md 第1章

**定义**：
relative 定位相对于元素在正常文档流中的原始位置偏移，元素保留其在文档流中的空间。

**详解**：
特点：保留原位置、偏移不影响其他元素、原始位置作为定位参考。偏移属性：top/right/bottom/left 设置偏移量。用途：微调元素位置、作为 absolute 子元素的包含块。负值：允许负值向相反方向偏移。常配合 z-index：调整叠加顺序。

---

### 6.3 绝对定位（absolute）

**名称**：绝对定位（Absolute Positioning）

**首次出现位置**：C06_106-传统布局与定位.md 第1章

**定义**：
absolute 定位相对于最近的已定位祖先元素偏移，完全脱离文档流，不占据空间。

**详解**：
定位参考：第一个已定位的祖先（relative/absolute/fixed/sticky）。无已定位祖先：相对于初始包含块（html）。脱离文档流：不影响其他元素布局，可能被其他元素覆盖。用途：下拉菜单、弹出层、固定头部、图标叠加。居中技巧：`left: 50%; top: 50%; transform: translate(-50%, -50%);`。

---

### 6.4 固定定位（fixed）

**名称**：固定定位（Fixed Positioning）

**首次出现位置**：C06_106-传统布局与定位.md 第1章

**定义**：
fixed 定位相对于视口（viewport）固定，不随页面滚动而移动，脱离文档流。

**详解**：
参考系：始终相对于视口，不受父元素影响。移动端：iOS Safari 使用 position: fixed 在软键盘弹出时可能失效。脱离文档流：不占据空间，可能覆盖其他内容。常见应用：固定导航栏、固定侧边栏、回到顶部按钮、模态框背景。z-index 注意：fixed 元素的堆叠可能受父元素 transform 影响。

---

### 6.5 粘性定位（sticky）

**名称**：粘性定位（Sticky Positioning）

**首次出现位置**：C06_106-传统布局与定位.md 第1章

**定义**：
sticky 是 relative 和 fixed 的混合，元素在滚动出视口前表现为 relative，之后变为 fixed。

**详解**：
条件：必须指定 top/right/bottom/left 之一才能生效。生效范围：在包含块（父元素）内"粘住"。兼容性：现代浏览器支持，IE 不支持。常见用法：粘性表头、粘性导航、评论列表的用户头像。包含块问题：父元素 overflow: hidden/auto 会阻断 sticky。

---

### 6.2 浮动（float）

**名称**：浮动（float）

**首次出现位置**：C06_106-传统布局与定位.md 第2章

**定义**：
float 属性使元素向左或向右移动，直到触碰包含块边缘或另一个浮动元素，常用于文字环绕图片。

**详解**：
值：left（左浮动）、right（右浮动）、none（不浮动）。文档流：浮动元素脱离正常文档流，影响后续元素位置。清除浮动：clear: both / clearfix 技术 / overflow: hidden。浮动崩塌：父元素高度塌陷（无内容撑开）。替代方案：Flexbox/Grid 布局更现代。典型布局：两栏布局 `float: left` + `float: right`。

---

### 6.3 清除浮动（clear）

**名称**：清除浮动（clear）

**首次出现位置**：C06_106-传统布局与定位.md 第2章

**定义**：
clear 属性指定元素的哪一侧不能与之前的浮动元素相邻，用于消除浮动对后续元素的影响。

**详解**：
值：left（清除左浮动）、right（清除右浮动）、both（清除两侧浮动）、none（默认）。使用方式：在浮动元素后添加空 div `<div style="clear: both;"></div>`。Clearfix：`.clearfix::after { content: ''; display: block; clear: both; }` 推荐方式。overflow 清除：`父元素 { overflow: hidden; }` 触发 BFC。clear 不作用于 flex/grid 项目。

---

### 6.4 BFC（块格式化上下文）

**名称**：块格式化上下文（Block Formatting Context）

**首次出现位置**：C06_106-传统布局与定位.md 第3章

**定义**：
BFC 是页面的块级渲染区域，具有 BFC 特性的元素内部布局与外部隔离，影响浮动元素和清除浮动。

**详解**：
触发条件：根元素、float 不为 none、position 不为 static/relative、display: inline-block/table-cell/flex/grid、overflow 不为 visible。特性：包含内部浮动（清除浮动）、排除外部浮动、阻止 margin 折叠。BFC 用途：两栏自适应布局（overflow: hidden）、清除内部浮动、防止 margin 折叠。 IFC（行内格式化上下文）：内联元素显示区域。

---

### 6.5 z-index

**名称**：堆叠顺序（z-index）

**首次出现位置**：C06_106-传统布局与定位.md 第1章

**定义**：
z-index 属性设置元素的堆叠顺序，值越大越在上层，可以是负数。

**详解**：
默认值：auto（等于 0）。负值：允许负值，在内容后面。定位要求：仅对定位元素（position 非 static）有效。层叠上下文：定位 + z-index 创建新的层叠上下文。层叠顺序（从后到前）：负 z-index → block → float → inline → z-index auto/0 → 正 z-index。父元素比较：子元素无法通过 z-index 超过父元素（除非父元素 z-index 更低）。

---

## 7. 响应式设计类

### 7.1 媒体查询（Media Query）

**名称**：媒体查询（Media Query）

**首次出现位置**：G06_201-响应式设计.md 第1章

**定义**：
媒体查询是 CSS3 的条件规则语法，允许根据设备特性（屏幕宽度、高度、分辨率等）应用不同样式。

**详解**：
语法：`@media screen and (max-width: 768px) { ... }`。媒体类型：screen（屏幕）、print（打印）、all（所有）。媒体特性：width、height、min-width、max-width、orientation（portrait/landscape）、resolution、aspect-ratio。逻辑运算符：and、not、only、逗号（or）。常用断点：320px（手机）、768px（平板）、1024px（桌面）、1440px（大屏）。引入方式：`<link media="screen and (min-width: 768px)" href="tablet.css">`。

---

### 7.2 视口（viewport）

**名称**：视口（Viewport）

**首次出现位置**：G06_201-响应式设计.md 第1章

**定义**：
viewport 是浏览器窗口中用于显示网页内容的区域，移动端通过 meta viewport 控制缩放和尺寸。

**详解**：
meta 标签：`<meta name="viewport" content="width=device-width, initial-scale=1.0">`。属性：width=device-width（视口宽度等于设备宽度）、initial-scale（初始缩放比例）、minimum-scale/maximum-scale（缩放范围）、user-scalable（是否允许缩放）。width =device-width：让 CSS 像素等于设备像素，使媒体查询基于设备宽度生效。禁用缩放：user-scalable=no 影响可访问性，谨慎使用。

---

### 7.3 移动优先（Mobile First）

**名称**：移动优先（Mobile First）

**首次出现位置**：G06_201-响应式设计.md 第2章

**定义**：
移动优先是一种响应式设计策略，先为移动设备编写基础样式，再通过媒体查询为更大屏幕添加增强样式。

**详解**：
优势：移动端代码更简洁、减少不必要的样式覆盖、更好的性能。实现：`/* 基础样式（移动） */ @media (min-width: 768px) { /* 平板及以上 */ }`。min-width vs max-width：移动优先用 min-width，从小到大。内容分块：按重要性排列内容，移动端优先显示核心内容。

---

### 7.4 断点（Breakpoint）

**名称**：断点（Breakpoint）

**首次出现位置**：G06_201-响应式设计.md 第2章

**定义**：
断点是媒体查询中定义的屏幕宽度阈值，用于在不同屏幕尺寸下切换样式。

**详解**：
常见断点：320px-480px（手机）、481px-768px（大手机/小平板）、769px-1024px（平板）、1025px-1200px（笔记本）、1201px+（桌面）。设备分类：small、medium、large（Bootstrap 断点）。命名：small (sm)、medium (md)、large (lg)、extra large (xl)。断点设置原则：基于内容而非设备、内容决定断点。

---

### 7.5 弹性图片

**名称**：弹性图片（Fluid Image）

**首次出现位置**：G06_201-响应式设计.md 第3章

**定义**：
弹性图片通过设置 max-width: 100% 让图片在容器内缩放，不会溢出或变形。

**详解**：
CSS：`img { max-width: 100%; height: auto; }`。max-width: 100%：图片宽度不超过容器。height: auto：保持宽高比。object-fit：裁剪方式（cover/contain/fill）配合 width/height。背景图片：`background-size: cover;`。响应式图片：`<img srcset="small.jpg 480w, large.jpg 1200w" sizes="...">`。

---

### 7.6 srcset

**名称**：图片源集合（srcset）

**首次出现位置**：G06_201-响应式设计.md 第3章

**定义**：
srcset 属性允许为不同屏幕尺寸提供不同分辨率的图片，让浏览器选择最适合的图片。

**详解**：
语法：`<img srcset="img-480w.jpg 480w, img-800w.jpg 800w" sizes="(max-width: 600px) 480px, 800px">`。w 单位：图片物理宽度。sizes：媒体条件与图片槽位。浏览器选择：根据设备像素比和视口宽度选择最合适图片。picture 元素：更灵活的多图片源场景：`<picture><source media="..." srcset="..."><img src="..."></picture>`。

## 8. 动画与过渡类

### 8.1 transition（过渡）

**名称**：CSS 过渡（Transition）

**首次出现位置**：C06_107-Canvas绘图.md 第3章

**定义**：
transition 属性定义属性值变化的过渡效果，使变化不是瞬间完成而是平滑动画。

**详解**：
子属性：transition-property（属性）、transition-duration（时长）、transition-timing-function（缓动）、transition-delay（延迟）。缩写：`transition: all 0.3s ease 0s;`。常用缓动：ease、linear、ease-in、ease-out、ease-in-out、cubic-bezier()。性能：过渡 transform 和 opacity 最流畅（GPU 加速）。触发：伪类（:hover、:focus）、JavaScript 添加类名、媒体查询切换。

---

### 8.2 animation（动画）

**名称**：CSS 动画（Animation）

**首次出现位置**：C06_107-Canvas绘图.md 第3章

**定义**：
animation 属性通过 @keyframes 定义关键帧动画，比 transition 更强大，支持暂停、循环、复杂轨迹。

**详解**：
子属性：animation-name（关键帧名）、animation-duration、animation-timing-function、animation-delay、animation-iteration-count（次数）、animation-direction（方向）、animation-fill-mode（填充）、animation-play-state（播放/暂停）。缩写：`animation: name 1s ease-in-out infinite alternate;`。@keyframes：定义各时间点的状态。iteration-count：infinite（无限循环）。

---

### 8.3 transform（变换）

**名称**：CSS 变换（transform）

**首次出现位置**：C06_107-Canvas绘图.md 第2章

**定义**：
transform 属性在 2D 或 3D 空间内对元素进行旋转、缩放、倾斜、平移等视觉变换。

**详解**：
2D 函数：translate(x, y)、scale(x, y)、rotate(angle)、skew(x, y)、matrix()。3D 函数：rotateX/Y/Z()、translateZ()、perspective()。组合：`transform: translateX(100px) rotate(45deg);`。性能：transform 和 opacity 不触发重排，使用 GPU 加速。transform-origin：变换原点，默认 center。transform-box：参考框（view-box/fill-box）。

---

### 8.4 keyframes

**名称**：关键帧（@keyframes）

**首次出现位置**：C06_107-Canvas绘图.md 第3章

**定义**：
@keyframes 规则定义动画序列的中间步骤，通过百分比或 from/to 关键字指定各时间点的样式。

**详解**：
语法：`@keyframes name { 0% { } 50% { } 100% { } }`。from/to：`from { } to { }` 等同于 `0% { } 100% { }`。百分比：可设置多个中间关键帧。属性限制：不是所有 CSS 属性都可动画。可动画属性：opacity、transform、background-color、width 等。性能考虑：优先动画 transform 和 opacity。

---

### 8.5 will-change

**名称**：变更提示（will-change）

**首次出现位置**：G06_202-CSS-in-JS与高级布局.md 第2章

**定义**：
will-change 属性提示浏览器元素即将发生的属性变化，让浏览器提前做优化准备。

**详解**：
值：auto（默认）、scroll-position（滚动位置）、contents（内容）、transform（变换）、opacity（透明度）。使用：`will-change: transform;`。时机：动画即将开始前添加，动画结束后移除。滥用问题：过多 will-change 消耗内存。替代方案：CSS 动画本身已足够流畅。常用场景：复杂动画、滚动时 GPU 加速。

## 9. 其他重要概念类

### 9.1 CSS 变量（Custom Properties）

**名称**：CSS 自定义属性（CSS Custom Properties）

**首次出现位置**：G06_202-CSS-in-JS与高级布局.md 第1章

**定义**：
CSS 变量是以 -- 开头的命名属性，可在声明后在选择器作用域内复用，支持运行时修改。

**详解**：
声明：`--primary-color: #3498db;`。使用：`color: var(--primary-color);`。回退值：`var(--primary-color, blue)`。作用域：选择器内声明，作用域内生效。:root 声明：全局可用。JavaScript 修改：`element.style.setProperty('--var', 'value')`。主题切换：CSS 变量 + JS 实现主题切换。Sass 变量 vs CSS 变量：Sass 编译时替换，CSS 变量运行时可改。

---

### 9.2 rem / em / vw / vh 单位

**名称**：相对单位（rem / em / vw / vh Units）

**首次出现位置**：G06_201-响应式设计.md 第2章

**定义**：
rem 相对于根元素字体大小，em 相对于当前元素字体大小，vw/vh 是视口宽高的 1%。

**详解**：
rem：根元素（html）字体大小，如 `html { font-size: 16px; }` 则 1rem = 16px。em：相对于当前元素字体大小，嵌套时可能复合。em vs rem：rem 更稳定，推荐用于响应式字体和间距。vw：1vw = 视口宽度的 1%，100vw = 整个视口宽度。vh：1vh = 视口高度的 1%，100vh = 整个视口高度。vmin/vmax：较小/较大的视口尺寸。配合 calc()：`width: calc(100vw - 2rem);`。

---

### 9.3 颜色值

**名称**：CSS 颜色值（Color Values）

**首次出现位置**：C06_101-概述与语法.md 第2章

**定义**：
CSS 支持多种颜色表示方式：颜色名、HEX、RGB、RGBA、HSL、HSLA。

**详解**：
HEX：`#fff`（缩写）、`#ffffff`（全写），可加透明通道 `#ffffff80`（CSS Color Level 4）。RGB：`rgb(255, 0, 0)`、`rgba(255, 0, 0, 0.5)`。HSL：`hsl(0, 100%, 50%)`（色相、饱和度、亮度）。HSLA：`hsla(0, 100%, 50%, 0.5)`。currentColor：关键字，继承 color 属性值。透明色：transparent（rgba(0,0,0,0) 的别名）。

---

### 9.4 opacity / visibility / display

**名称**：显示控制属性（opacity / visibility / display）

**首次出现位置**：G06_201-响应式设计.md 第3章

**定义**：
opacity、visibility、display 都影响元素可见性，但行为和效果不同。

**详解**：
opacity：0-1 透明度，元素仍占据空间、仍可交互（即使不可见）。visibility：visible（可见，默认）或 hidden（隐藏），hidden 时元素占据空间但不可交互、不响应事件。display：none（不显示，不占空间，不响应事件）、block/inline 等显示值。display: none vs visibility: hidden：none 完全不占位，后者占位。display 切换动画：需要配合 opacity 或 height 过渡。

---

### 9.5 calc()

**名称**：计算函数（calc()）

**首次出现位置**：G06_202-CSS-in-JS与高级布局.md 第1章

**定义**：
calc() 函数允许在 CSS 属性值中进行数学运算，支持加减乘除混合计算。

**详解**：
语法：`width: calc(100% - 2rem);`。运算符：+、-、*、/。单位混用：`calc(100vw / 3 - 20px)`。空格：运算符两侧必须有空格。嵌套：`calc(calc(100% / 3) - 10px)`。常见用法：响应式布局、flex 项目尺寸计算、居中留边。浏览器支持：现代浏览器支持，IE9+ 部分支持（需空格）。

---

### 9.6 filter

**名称**：滤镜（filter）

**首次出现位置**：C06_107-Canvas绘图.md 第2章

**定义**：
filter 属性将 CSS 滤镜效果（如模糊、亮度、对比度）应用于元素。

**详解**：
函数：blur()、brightness()、contrast()、grayscale()、hue-rotate()、invert()、opacity()、saturate()、sepia()、drop-shadow()。多滤镜：`filter: blur(2px) brightness(1.2);`。性能：滤镜可能影响性能，大量使用需测试。动画：filter 支持过渡和动画。canvas 滤镜：可通过 CSS filter 或 CanvasRenderingContext2D.filter 实现。

---

### 9.7 backdrop-filter

**名称**：背景滤镜（backdrop-filter）

**首次出现位置**：C06_107-Canvas绘图.md 第2章

**定义**：
backdrop-filter 属性将滤镜效果应用到元素背后的区域（背景），创造毛玻璃等效果。

**详解**：
语法：与 filter 相同。常见用法：`backdrop-filter: blur(10px);` 毛玻璃效果。应用场景：模态框背景模糊、玻璃拟态设计（Glassmorphism）、导航栏半透明。兼容性：现代浏览器支持，Safari 需前缀 -webkit-。性能：比 filter 更耗性能，注意使用场景。背景要求：需要非透明背景才能看到效果。
