# MySQL核心知识点

## 1. MySQL概述

MySQL是一种关系型数据库管理系统（RDBMS），由Oracle公司开发和维护。它是最流行的开源数据库之一，广泛应用于Web应用和企业级系统。

### 1.1 MySQL的核心特点
- **开源免费**：使用GPL许可证，可自由使用和修改
- **跨平台**：支持多种操作系统，如Windows、Linux、Mac OS等
- **高性能**：优化的存储引擎和查询优化器
- **可靠性**：支持事务和崩溃恢复
- **可扩展性**：支持分区、复制和集群
- **丰富的功能**：支持存储过程、触发器、视图等
- **安全性**：提供用户权限管理和加密功能

### 1.2 MySQL的应用场景
- **Web应用**：WordPress、Drupal等内容管理系统
- **企业应用**：ERP、CRM等企业级系统
- **数据分析**：数据仓库和商业智能
- **嵌入式系统**：需要本地数据存储的应用

## 2. MySQL安装与配置

### 2.1 Windows安装

1. **下载安装包**：从MySQL官网下载Windows安装包
2. **运行安装程序**：按照提示完成安装
3. **配置MySQL**：设置root密码、端口等
4. **启动服务**：启动MySQL服务
5. **验证安装**：使用MySQL命令行工具连接数据库

### 2.2 Linux安装

- **使用包管理器**：
  ```bash
  # Ubuntu/Debian
  sudo apt-get update
  sudo apt-get install mysql-server
  
  # CentOS/RHEL
  sudo yum install mysql-server
  ```

- **启动服务**：
  ```bash
  # Ubuntu/Debian
  sudo systemctl start mysql
  
  # CentOS/RHEL
  sudo service mysqld start
  ```

- **设置密码**：
  ```bash
  sudo mysql_secure_installation
  ```

### 2.3 配置文件

MySQL的配置文件通常位于：
- Windows：`C:\ProgramData\MySQL\MySQL Server 8.0\my.ini`
- Linux：`/etc/my.cnf` 或 `/etc/mysql/my.cnf`

常用配置项：

```ini
[mysqld]
# 端口
port = 3306
# 数据目录
datadir = /var/lib/mysql
# 字符集
character-set-server = utf8mb4
# 最大连接数
max_connections = 100
# 缓冲区大小
innodb_buffer_pool_size = 128M
```

## 3. 基本概念

### 3.1 数据库

数据库是存储数据的集合，MySQL中可以创建多个数据库。

### 3.2 表

表是数据库中的基本存储单位，由行和列组成。

### 3.3 行

行（Row）是表中的一条记录。

### 3.4 列

列（Column）是表中的一个字段，有特定的数据类型。

### 3.5 主键

主键（Primary Key）是表中的唯一标识，用于唯一识别每条记录。

### 3.6 外键

外键（Foreign Key）是表中引用其他表主键的字段，用于建立表之间的关系。

### 3.7 索引

索引（Index）是用于加速查询的数据结构。

## 4. SQL语法

### 4.1 数据定义语言（DDL）

#### 4.1.1 创建数据库

```sql
CREATE DATABASE database_name;
```

#### 4.1.2 删除数据库

```sql
DROP DATABASE database_name;
```

#### 4.1.3 创建表

```sql
CREATE TABLE table_name (
    column1 datatype constraints,
    column2 datatype constraints,
    ...
    PRIMARY KEY (column_name)
);
```

#### 4.1.4 修改表

```sql
-- 添加列
ALTER TABLE table_name ADD column_name datatype constraints;

-- 修改列
ALTER TABLE table_name MODIFY column_name datatype constraints;

-- 删除列
ALTER TABLE table_name DROP column_name;

-- 添加主键
ALTER TABLE table_name ADD PRIMARY KEY (column_name);

-- 添加外键
ALTER TABLE table_name ADD FOREIGN KEY (column_name) REFERENCES other_table (other_column);
```

#### 4.1.5 删除表

```sql
DROP TABLE table_name;
```

### 4.2 数据操作语言（DML）

#### 4.2.1 插入数据

```sql
-- 插入单行
INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);

-- 插入多行
INSERT INTO table_name (column1, column2, ...) VALUES
(value1, value2, ...),
(value3, value4, ...),
...;
```

#### 4.2.2 更新数据

```sql
UPDATE table_name SET column1 = value1, column2 = value2, ... WHERE condition;
```

#### 4.2.3 删除数据

```sql
DELETE FROM table_name WHERE condition;
```

#### 4.2.4 查询数据

```sql
SELECT column1, column2, ... FROM table_name WHERE condition;
```

### 4.3 数据查询语言（DQL）

#### 4.3.1 基本查询

```sql
SELECT * FROM table_name;
SELECT column1, column2 FROM table_name;
```

#### 4.3.2 条件查询

```sql
SELECT * FROM table_name WHERE condition;

-- 示例
SELECT * FROM users WHERE age > 18;
SELECT * FROM users WHERE name LIKE '%张%';
SELECT * FROM users WHERE age BETWEEN 18 AND 30;
SELECT * FROM users WHERE age IN (18, 20, 22);
```

#### 4.3.3 排序

```sql
SELECT * FROM table_name ORDER BY column1 ASC, column2 DESC;
```

#### 4.3.4 分组

```sql
SELECT column1, COUNT(*) FROM table_name GROUP BY column1;
```

#### 4.3.5 连接查询

```sql
-- 内连接
SELECT * FROM table1 INNER JOIN table2 ON table1.column = table2.column;

-- 左连接
SELECT * FROM table1 LEFT JOIN table2 ON table1.column = table2.column;

-- 右连接
SELECT * FROM table1 RIGHT JOIN table2 ON table1.column = table2.column;

-- 全连接
SELECT * FROM table1 FULL JOIN table2 ON table1.column = table2.column;
```

#### 4.3.6 子查询

```sql
SELECT * FROM table1 WHERE column IN (SELECT column FROM table2 WHERE condition);
```

#### 4.3.7 聚合函数

```sql
SELECT COUNT(*) FROM table_name;
SELECT SUM(column) FROM table_name;
SELECT AVG(column) FROM table_name;
SELECT MAX(column) FROM table_name;
SELECT MIN(column) FROM table_name;
```

### 4.4 数据控制语言（DCL）

#### 4.4.1 创建用户

```sql
CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';
```

#### 4.4.2 授权

```sql
GRANT privileges ON database.table TO 'username'@'localhost';
```

#### 4.4.3 撤销权限

```sql
REVOKE privileges ON database.table FROM 'username'@'localhost';
```

#### 4.4.4 删除用户

```sql
DROP USER 'username'@'localhost';
```

## 6. 存储引擎

### 6.1 InnoDB

- **特点**：
  - 支持事务
  - 支持行级锁
  - 支持外键
  - 支持崩溃恢复
  - 适合高并发环境

- **使用场景**：需要事务支持的应用，如电商、金融系统

### 6.2 MyISAM

- **特点**：
  - 不支持事务
  - 表级锁
  - 速度快
  - 适合读多写少的场景

- **使用场景**：只读或读多写少的应用，如博客、新闻网站

### 6.3 Memory

- **特点**：
  - 数据存储在内存中
  - 速度极快
  - 数据易失

- **使用场景**：临时表、缓存

### 6.4 CSV

- **特点**：
  - 以CSV格式存储数据
  - 适合数据交换

- **使用场景**：数据导入/导出

## 7. 索引

### 7.1 索引类型

- **主键索引**：唯一标识记录，自动创建
- **唯一索引**：确保列值唯一
- **普通索引**：加速查询
- **全文索引**：用于全文搜索
- **空间索引**：用于地理空间数据

### 7.2 创建索引

```sql
-- 创建表时创建索引
CREATE TABLE table_name (
    column1 datatype PRIMARY KEY,
    column2 datatype,
    INDEX index_name (column2)
);

-- 为现有表创建索引
CREATE INDEX index_name ON table_name (column);
CREATE UNIQUE INDEX index_name ON table_name (column);
CREATE FULLTEXT INDEX index_name ON table_name (column);
```

### 7.3 删除索引

```sql
DROP INDEX index_name ON table_name;
```

### 7.4 索引优化

- **选择合适的列**：选择唯一性高、查询频繁的列
- **复合索引**：为经常一起查询的列创建复合索引
- **避免过度索引**：索引会增加写操作的开销
- **定期重建索引**：优化索引结构

## 8. 事务

### 8.1 事务的特性

- **原子性（Atomicity）**：事务是一个不可分割的工作单位
- **一致性（Consistency）**：事务前后数据保持一致
- **隔离性（Isolation）**：事务之间相互隔离
- **持久性（Durability）**：事务一旦提交，数据永久保存

### 8.2 事务控制

```sql
-- 开始事务
START TRANSACTION;

-- 执行操作
INSERT INTO table_name VALUES (...);
UPDATE table_name SET ... WHERE ...;

-- 提交事务
COMMIT;

-- 回滚事务
ROLLBACK;
```

### 8.3 事务隔离级别

- **READ UNCOMMITTED**：可读取未提交的数据
- **READ COMMITTED**：只能读取已提交的数据
- **REPEATABLE READ**：可重复读，MySQL默认级别
- **SERIALIZABLE**：串行化，最高隔离级别

```sql
-- 设置隔离级别
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
```

## 10. 存储过程与函数

### 10.1 存储过程

```sql
-- 创建存储过程
DELIMITER //
CREATE PROCEDURE procedure_name (IN parameter1 datatype, OUT parameter2 datatype) BEGIN
    -- 操作
    SELECT * FROM table_name WHERE column = parameter1;
    SET parameter2 = 1;
END //
DELIMITER ;

-- 调用存储过程
CALL procedure_name(value1, @value2);
SELECT @value2;
```

### 10.2 函数

```sql
-- 创建函数
DELIMITER //
CREATE FUNCTION function_name (parameter1 datatype) RETURNS datatype BEGIN
    -- 操作
    DECLARE result datatype;
    SELECT COUNT(*) INTO result FROM table_name WHERE column = parameter1;
    RETURN result;
END //
DELIMITER ;

-- 调用函数
SELECT function_name(value1);
```

## 13. 性能优化

### 13.1 查询优化

- **使用索引**：为经常查询的列创建索引

- **优化JOIN操作**：使用合适的连接类型
- **使用EXPLAIN**：分析查询执行计划
- **避免SELECT ***：只选择需要的列

### 13.2 服务器优化

- **配置缓存**：调整innodb_buffer_pool_size
- **优化连接**：调整max_connections
- **日志优化**：适当配置二进制日志
- **硬件优化**：使用SSD、增加内存

### 13.3 存储优化

- **选择合适的存储引擎**：根据应用场景选择
- **分区表**：对大表进行分区
- **压缩表**：使用压缩减少存储空间

## 15. 常见问题与解决方案

### 15.1 连接问题

**问题**：无法连接到MySQL服务器
**解决方案**：
- 检查MySQL服务是否启动
- 检查网络连接

**问题**：查询速度慢
**解决方案**：
- 分析查询执行计划
- 创建适当的索引
- 优化SQL语句
- 调整服务器配置

**问题**：数据不一致
**解决方案**：
- 使用事务
- 合理设置隔离级别
- 避免脏读、幻读等问题

### 15.4 备份恢复问题

**问题**：备份恢复失败
**解决方案**：
