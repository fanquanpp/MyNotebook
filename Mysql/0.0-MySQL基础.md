## 1. MySQL 简介

MySQL 是一种关系型数据库管理系统，由瑞典 MySQL AB 公司开发，目前属于 Oracle 公司。MySQL 是最流行的关系型数据库管理系统之一，在 WEB 应用方面，MySQL 是最好的 RDBMS (Relational Database Management System，关系数据库管理系统) 应用软件之一。

### 1.1 MySQL 的特点

- **开源免费**：MySQL 是开源软件，可以免费使用和修改
- **跨平台**：支持多种操作系统，如 Linux、Windows、macOS 等
- **高性能**：针对不同的使用场景进行了优化
- **可靠性**：具有强大的事务处理能力和数据完整性保证
- **易于使用**：提供了简单易用的命令行工具和图形界面
- **可扩展性**：支持集群和分布式架构

### 1.2 MySQL 的应用场景

- **Web 应用**：作为 Web 应用的后端数据库
- **企业应用**：用于企业内部的数据管理
- **数据仓库**：用于数据分析和报表生成
- **嵌入式系统**：作为嵌入式设备的数据库

## 2. MySQL 安装与配置

### 2.1 Windows 系统安装

1. 下载 MySQL 安装包
2. 运行安装程序，选择自定义安装
3. 配置 MySQL 服务
4. 设置 root 用户密码
5. 完成安装并启动服务

### 2.2 Linux 系统安装

使用包管理器安装：

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install mysql-server

# CentOS/RHEL
sudo yum install mysql-server
```

### 2.3 配置 MySQL

主要配置文件：
- Windows：`my.ini`
- Linux：`my.cnf`

常见配置项：
- `port`：端口号，默认 3306
- `datadir`：数据存储目录
- `character-set-server`：默认字符集
- `max_connections`：最大连接数

## 3. MySQL 基本概念

### 3.1 数据库

数据库是存储数据的容器，MySQL 中可以创建多个数据库。

### 3.2 表

表是数据库中的基本存储单位，由行和列组成。

### 3.3 字段

字段是表中的列，定义了数据的类型和属性。

### 3.4 记录

记录是表中的行，包含了一条完整的数据。

### 3.5 主键

主键是表中唯一标识一条记录的字段或字段组合。

### 3.6 外键

外键是表中引用其他表主键的字段，用于建立表之间的关联关系。

## 4. SQL 基础语法

### 4.1 SQL 分类

- **DDL (Data Definition Language)**：数据定义语言，用于创建、修改和删除数据库对象
- **DML (Data Manipulation Language)**：数据操作语言，用于插入、更新、删除和查询数据
- **DCL (Data Control Language)**：数据控制语言，用于控制用户权限
- **TCL (Transaction Control Language)**：事务控制语言，用于管理事务

### 4.2 DDL 语句

#### 创建数据库

```sql
CREATE DATABASE database_name;
```

#### 使用数据库

```sql
USE database_name;
```

#### 创建表

```sql
CREATE TABLE table_name (
    column1 datatype constraints,
    column2 datatype constraints,
    ...
    PRIMARY KEY (column_name)
);
```

#### 修改表

```sql
ALTER TABLE table_name ADD column_name datatype constraints;
ALTER TABLE table_name MODIFY column_name datatype constraints;
ALTER TABLE table_name DROP column_name;
```

#### 删除表

```sql
DROP TABLE table_name;
```

### 4.3 DML 语句

#### 插入数据

```sql
INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);
```

#### 更新数据

```sql
UPDATE table_name SET column1 = value1, column2 = value2 WHERE condition;
```

#### 删除数据

```sql
DELETE FROM table_name WHERE condition;
```

#### 查询数据

```sql
SELECT column1, column2, ... FROM table_name WHERE condition;
```

### 4.4 DCL 语句

#### 创建用户

```sql
CREATE USER 'username'@'host' IDENTIFIED BY 'password';
```

#### 授予权限

```sql
GRANT privileges ON database_name.table_name TO 'username'@'host';
```

#### 撤销权限

```sql
REVOKE privileges ON database_name.table_name FROM 'username'@'host';
```

#### 删除用户

```sql
DROP USER 'username'@'host';
```

### 4.5 TCL 语句

#### 开始事务

```sql
START TRANSACTION;
```

#### 提交事务

```sql
COMMIT;
```

#### 回滚事务

```sql
ROLLBACK;
```

## 5. MySQL 数据类型

### 5.1 数值类型

- `INT`：整数，4字节
- `BIGINT`：大整数，8字节
- `FLOAT`：单精度浮点数，4字节
- `DOUBLE`：双精度浮点数，8字节
- `DECIMAL`：精确小数，可变长度

### 5.2 字符串类型

- `CHAR`：固定长度字符串，0-255字节
- `VARCHAR`：可变长度字符串，0-65535字节
- `TEXT`：文本，0-65535字节
- `LONGTEXT`：长文本，0-4294967295字节

### 5.3 日期时间类型

- `DATE`：日期，格式 YYYY-MM-DD
- `TIME`：时间，格式 HH:MM:SS
- `DATETIME`：日期时间，格式 YYYY-MM-DD HH:MM:SS
- `TIMESTAMP`：时间戳，自动更新
- `YEAR`：年份，格式 YYYY

### 5.4 其他类型

- `BOOLEAN`：布尔值，0或1
- `ENUM`：枚举类型，只能取预定义的值
- `SET`：集合类型，可以取多个预定义的值

## 6. MySQL 高级特性

### 6.1 索引

索引是提高查询性能的重要手段，常见的索引类型：

- **普通索引**：加速查询
- **唯一索引**：保证数据唯一性
- **主键索引**：特殊的唯一索引
- **复合索引**：基于多个列的索引
- **全文索引**：用于全文搜索

### 6.2 事务

事务是一组原子性的操作，要么全部执行成功，要么全部执行失败。事务具有 ACID 特性：

- **原子性 (Atomicity)**：事务是一个不可分割的工作单位
- **一致性 (Consistency)**：事务前后数据保持一致
- **隔离性 (Isolation)**：事务之间相互隔离
- **持久性 (Durability)**：事务一旦提交，数据就永久保存

### 6.3 视图

视图是基于查询结果的虚拟表，可以简化复杂查询。

```sql
CREATE VIEW view_name AS SELECT column1, column2 FROM table_name WHERE condition;
```

### 6.4 存储过程

存储过程是预编译的 SQL 语句集合，可以提高执行效率。

```sql
DELIMITER //
CREATE PROCEDURE procedure_name (parameters)
BEGIN
    SQL statements;
END //
DELIMITER ;
```

### 6.5 触发器

触发器是在特定事件发生时自动执行的 SQL 语句。

```sql
CREATE TRIGGER trigger_name
AFTER INSERT ON table_name
FOR EACH ROW
BEGIN
    SQL statements;
END;
```

## 7. MySQL 备份与恢复

### 7.1 备份方法

- **mysqldump**：逻辑备份工具
- **mysqlpump**：改进的备份工具
- **物理备份**：直接复制数据文件

### 7.2 恢复方法

- **从 SQL 文件恢复**：
  ```bash
  mysql -u username -p database_name < backup.sql
  ```

- **从物理备份恢复**：直接复制数据文件到数据目录

## 8. MySQL 性能优化

### 8.1 查询优化

- 使用索引
- 避免全表扫描
- 优化 WHERE 子句
- 合理使用 JOIN

### 8.2 服务器优化

- 使用 SSD 存储
- 配置合理的缓冲区大小

### 8.3 架构优化

- 读写分离
- 分库分表
- 集群部署
- 缓存机制

## 9. MySQL 常见问题与解决方案

### 9.1 连接问题

- **问题**：连接被拒绝
  **解决方案**：检查网络连接、防火墙设置、MySQL 服务状态

- **问题**：连接超时
  **解决方案**：调整 `wait_timeout` 参数

### 9.2 性能问题

- **问题**：查询速度慢
  **解决方案**：优化查询语句、添加索引

- **问题**：服务器负载高
  **解决方案**：调整配置参数、优化查询

### 9.3 数据问题

- **问题**：数据丢失
  **解决方案**：定期备份、启用二进制日志

- **问题**：数据不一致
  **解决方案**：使用事务、合理设计表结构
