# MySQL 安装与配置

## 1. 概述

MySQL 的安装与配置是使用 MySQL 的第一步，正确的安装和配置可以确保 MySQL 服务的正常运行和最佳性能。本教程将详细介绍在不同操作系统上安装和配置 MySQL 的方法。

## 2. Windows 系统安装

### 2.1 下载 MySQL 安装包

1. 访问 MySQL 官方网站：https://dev.mysql.com/downloads/installer/
2. 选择适合您系统的安装包（32位或64位）
3. 下载 MySQL Installer 安装程序

### 2.2 运行安装程序

1. 双击下载的安装程序
2. 选择安装类型：
   - **Developer Default**：适合开发者，包含所有组件
   - **Server Only**：仅安装 MySQL 服务器
   - **Client Only**：仅安装客户端工具
   - **Full**：完整安装
   - **Custom**：自定义安装

3. 点击 "Next" 继续

### 2.3 配置 MySQL 服务

1. 选择 "Configure MySQL Server" 选项
2. 配置服务器类型：
   - **Development Machine**：开发机器，适合个人开发
   - **Server Machine**：服务器机器，适合中小型服务器
   - **Dedicated Machine**：专用机器，适合大型服务器

3. 设置端口号（默认 3306）
4. 选择字符集（建议使用 utf8mb4）

### 2.4 设置 root 用户密码

1. 为 root 用户设置强密码
2. 可以选择添加其他用户并设置权限

### 2.5 完成安装并启动服务

1. 点击 "Execute" 执行配置
2. 完成后点击 "Finish" 结束安装
3. 启动 MySQL 服务

## 3. Linux 系统安装

### 3.1 Ubuntu/Debian 系统

使用 apt 包管理器安装：

```bash
# 更新包列表
sudo apt update

# 安装 MySQL 服务器
sudo apt install mysql-server

# 启动 MySQL 服务
sudo systemctl start mysql

# 设置开机自启
sudo systemctl enable mysql

# 运行安全配置向导
sudo mysql_secure_installation
```

### 3.2 CentOS/RHEL 系统

使用 yum 包管理器安装：

```bash
# 安装 MySQL 服务器
sudo yum install mysql-server

# 启动 MySQL 服务
sudo systemctl start mysqld

# 设置开机自启
sudo systemctl enable mysqld

# 运行安全配置向导
sudo mysql_secure_installation
```

### 3.3 手动安装

1. 下载 MySQL 源码或二进制包
2. 解压并编译（如果是源码包）
3. 配置并安装
4. 启动服务

## 4. macOS 系统安装

### 4.1 使用 Homebrew 安装

```bash
# 安装 Homebrew（如果未安装）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 安装 MySQL
brew install mysql

# 启动 MySQL 服务
brew services start mysql

# 运行安全配置向导
mysql_secure_installation
```

### 4.2 使用 DMG 安装包

1. 访问 MySQL 官方网站下载 DMG 安装包
2. 双击安装包并按照提示安装
3. 启动 MySQL 服务

## 5. 配置 MySQL

### 5.1 主要配置文件

- **Windows**：`C:\ProgramData\MySQL\MySQL Server X.X\my.ini`
- **Linux**：`/etc/my.cnf` 或 `/etc/mysql/my.cnf`
- **macOS**：`/usr/local/etc/my.cnf` 或 `~/.my.cnf`

### 5.2 常见配置项

| 配置项 | 描述 | 默认值 | 建议值 |
|-------|------|-------|-------|
| `port` | MySQL 服务端口 | 3306 | 3306（如需修改，确保未被占用） |
| `datadir` | 数据存储目录 | 取决于操作系统 | 保持默认或设置到空间充足的分区 |
| `character-set-server` | 默认字符集 | latin1 | utf8mb4 |
| `max_connections` | 最大连接数 | 151 | 根据服务器资源调整 |
| `innodb_buffer_pool_size` | InnoDB 缓冲池大小 | 128M | 建议设置为服务器内存的 50-80% |
| `innodb_log_file_size` | InnoDB 日志文件大小 | 48M | 根据业务需求调整 |
| `query_cache_size` | 查询缓存大小 | 0（MySQL 8.0 已移除） | 0 |

### 5.3 配置示例

```ini
[mysqld]
# 基本配置
port = 3306
datadir = /var/lib/mysql
socket = /var/lib/mysql/mysql.sock

# 字符集配置
character-set-server = utf8mb4
collation-server = utf8mb4_unicode_ci

# 连接配置
max_connections = 100
wait_timeout = 60

# InnoDB 配置
innodb_buffer_pool_size = 1G
innodb_log_file_size = 256M
innodb_file_per_table = 1

# 日志配置
log_error = /var/log/mysql/error.log
general_log = 0
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow-query.log
long_query_time = 2
```

## 6. 验证安装

### 6.1 检查 MySQL 服务状态

**Windows**：
```cmd
net start MySQL
```

**Linux**：
```bash
sudo systemctl status mysql
```

**macOS**：
```bash
brew services list | grep mysql
```

### 6.2 连接 MySQL

使用命令行工具连接：

```bash
mysql -u root -p
```

输入密码后，若出现 MySQL 提示符，则表示连接成功：


### 6.3 测试查询

在 MySQL 提示符下执行简单查询：

```sql
SHOW DATABASES;
```

若返回数据库列表，则表示 MySQL 运行正常。

## 7. 常见问题与解决方案

### 7.1 连接被拒绝

**问题**：无法连接到 MySQL 服务器，出现 "Access denied" 错误。

**解决方案**：
- 检查用户名和密码是否正确
- 检查 MySQL 服务是否运行
- 检查防火墙是否允许 MySQL 端口（默认 3306）
- 检查用户权限是否正确

### 7.2 服务启动失败

**问题**：MySQL 服务无法启动。

**解决方案**：
- 检查配置文件是否正确
- 检查数据目录权限是否正确
- 检查端口是否被占用
- 查看错误日志获取详细信息

### 7.3 性能问题

**问题**：MySQL 运行速度慢。

**解决方案**：
- 优化配置参数
- 添加适当的索引
- 优化查询语句
- 考虑使用缓存

## 8. 总结

正确的安装和配置是使用 MySQL 的基础，通过本教程的学习，你应该已经掌握了在不同操作系统上安装和配置 MySQL 的方法。

安装和配置 MySQL 时需要注意以下几点：
- 选择适合自己系统的安装方法
- 合理配置 MySQL 参数以获得最佳性能
- 确保 MySQL 服务的安全
- 定期备份数据以防止数据丢失

通过正确的安装和配置，MySQL 将为你的应用提供稳定、高效的数据存储服务。
