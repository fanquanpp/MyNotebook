# MySQL 数据类型

## 1. 概述

MySQL 支持多种数据类型，用于存储不同类型的数据。选择合适的数据类型对于数据库的性能和存储空间至关重要。本教程将详细介绍 MySQL 中常用的数据类型，包括数值类型、字符串类型、日期时间类型和其他类型。

## 2. 数值类型

### 2.1 整数类型

MySQL 提供了多种整数类型，它们的存储空间和取值范围不同：

| 类型 | 存储空间 | 取值范围 | 描述 |
|------|----------|----------|------|
| `TINYINT` | 1字节 | -128 到 127 | 非常小的整数 |
| `SMALLINT` | 2字节 | -32768 到 32767 | 小整数 |
| `MEDIUMINT` | 3字节 | -8388608 到 8388607 | 中等大小的整数 |
| `INT` | 4字节 | -2147483648 到 2147483647 | 整数 |
| `BIGINT` | 8字节 | -9223372036854775808 到 9223372036854775807 | 大整数 |

**示例**：

```sql
-- 创建包含整数类型字段的表
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    age TINYINT,
    salary INT,
    population BIGINT
);
```

### 2.2 小数类型

MySQL 支持两种小数类型：`FLOAT` 和 `DOUBLE`，以及精确小数类型 `DECIMAL`：

| 类型 | 存储空间 | 描述 |
|------|----------|------|
| `FLOAT` | 4字节 | 单精度浮点数 |
| `DOUBLE` | 8字节 | 双精度浮点数 |
| `DECIMAL(M,D)` | 可变 | 精确小数，M 是总位数，D 是小数位数 |

**示例**：

```sql
-- 创建包含小数类型字段的表
CREATE TABLE products (
    id INT PRIMARY KEY,
    price DECIMAL(10,2),  -- 最多10位数字，其中2位是小数
    weight FLOAT,
    height DOUBLE
);
```

### 2.3 位类型

`BIT(M)` 用于存储位值，M 表示位数，范围是 1 到 64：

| 类型 | 描述 |
|------|------|
| `BIT(M)` | 存储 M 位的位值 |

**示例**：

```sql
-- 创建包含位类型字段的表
CREATE TABLE flags (
    id INT PRIMARY KEY,
    permissions BIT(8)  -- 8位权限标志
);
```

## 3. 字符串类型

### 3.1 固定长度字符串

`CHAR(M)` 用于存储固定长度的字符串，M 是字符串长度，范围是 0 到 255：

| 类型 | 存储空间 | 描述 |
|------|----------|------|
| `CHAR(M)` | M 字节 | 固定长度字符串 |

**示例**：

```sql
-- 创建包含固定长度字符串字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    gender CHAR(1),  -- 'M' 或 'F'
    code CHAR(6)     -- 6位验证码
);
```

### 3.2 可变长度字符串

`VARCHAR(M)` 用于存储可变长度的字符串，M 是最大长度，范围是 0 到 65535：

| 类型 | 存储空间 | 描述 |
|------|----------|------|
| `VARCHAR(M)` | 实际长度 + 1-2字节 | 可变长度字符串 |

**示例**：

```sql
-- 创建包含可变长度字符串字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    email VARCHAR(100)
);
```

### 3.3 文本类型

MySQL 提供了多种文本类型，用于存储较长的文本：

| 类型 | 存储空间 | 描述 |
|------|----------|------|
| `TINYTEXT` | 0-255字节 | 非常小的文本 |
| `TEXT` | 0-65535字节 | 文本 |
| `MEDIUMTEXT` | 0-16777215字节 | 中等大小的文本 |
| `LONGTEXT` | 0-4294967295字节 | 长文本 |

**示例**：

```sql
-- 创建包含文本类型字段的表
CREATE TABLE articles (
    id INT PRIMARY KEY,
    title VARCHAR(200),
    content TEXT,
    description TINYTEXT,
    long_content LONGTEXT
);
```

### 3.4 二进制类型

MySQL 提供了多种二进制类型，用于存储二进制数据：

| 类型 | 存储空间 | 描述 |
|------|----------|------|
| `TINYBLOB` | 0-255字节 | 非常小的二进制数据 |
| `BLOB` | 0-65535字节 | 二进制数据 |
| `MEDIUMBLOB` | 0-16777215字节 | 中等大小的二进制数据 |
| `LONGBLOB` | 0-4294967295字节 | 长二进制数据 |

**示例**：

```sql
-- 创建包含二进制类型字段的表
CREATE TABLE files (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    data BLOB
);
```

### 3.5 枚举类型

`ENUM` 用于存储枚举值，只能取预定义的值：

| 类型 | 描述 |
|------|------|
| `ENUM('value1','value2',...)` | 枚举类型，只能取预定义的值 |

**示例**：

```sql
-- 创建包含枚举类型字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    status ENUM('active', 'inactive', 'pending')
);
```

### 3.6 集合类型

`SET` 用于存储集合值，可以取多个预定义的值：

| 类型 | 描述 |
|------|------|
| `SET('value1','value2',...)` | 集合类型，可以取多个预定义的值 |

**示例**：

```sql
-- 创建包含集合类型字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    hobbies SET('reading', 'sports', 'music', 'travel')
);
```

## 4. 日期时间类型

### 4.1 日期类型

`DATE` 用于存储日期，格式为 YYYY-MM-DD：

| 类型 | 存储空间 | 取值范围 | 描述 |
|------|----------|----------|------|
| `DATE` | 3字节 | 1000-01-01 到 9999-12-31 | 日期 |

**示例**：

```sql
-- 创建包含日期类型字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    birthday DATE
);
```

### 4.2 时间类型

`TIME` 用于存储时间，格式为 HH:MM:SS：

| 类型 | 存储空间 | 取值范围 | 描述 |
|------|----------|----------|------|
| `TIME` | 3字节 | -838:59:59 到 838:59:59 | 时间 |

**示例**：

```sql
-- 创建包含时间类型字段的表
CREATE TABLE events (
    id INT PRIMARY KEY,
    start_time TIME,
    end_time TIME
);
```

### 4.3 日期时间类型

`DATETIME` 用于存储日期时间，格式为 YYYY-MM-DD HH:MM:SS：

| 类型 | 存储空间 | 取值范围 | 描述 |
|------|----------|----------|------|
| `DATETIME` | 8字节 | 1000-01-01 00:00:00 到 9999-12-31 23:59:59 | 日期时间 |

**示例**：

```sql
-- 创建包含日期时间类型字段的表
CREATE TABLE orders (
    id INT PRIMARY KEY,
    created_at DATETIME,
    updated_at DATETIME
);
```

### 4.4 时间戳类型

`TIMESTAMP` 用于存储时间戳，格式为 YYYY-MM-DD HH:MM:SS，自动更新：

| 类型 | 存储空间 | 取值范围 | 描述 |
|------|----------|----------|------|
| `TIMESTAMP` | 4字节 | 1970-01-01 00:00:01 UTC 到 2038-01-19 03:14:07 UTC | 时间戳 |

**示例**：

```sql
-- 创建包含时间戳类型字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 4.5 年份类型

`YEAR` 用于存储年份，格式为 YYYY：

| 类型 | 存储空间 | 取值范围 | 描述 |
|------|----------|----------|------|
| `YEAR` | 1字节 | 1901 到 2155 | 年份 |

**示例**：

```sql
-- 创建包含年份类型字段的表
CREATE TABLE movies (
    id INT PRIMARY KEY,
    title VARCHAR(100),
    release_year YEAR
);
```

## 5. 其他类型

### 5.1 布尔类型

`BOOLEAN` 或 `BOOL` 是 `TINYINT(1)` 的别名，用于存储布尔值：

| 类型 | 描述 |
|------|------|
| `BOOLEAN` | 布尔值，0 表示 false，1 表示 true |

**示例**：

```sql
-- 创建包含布尔类型字段的表
CREATE TABLE users (
    id INT PRIMARY KEY,
    is_active BOOLEAN,
    is_admin BOOL
);
```

### 5.2 JSON 类型

MySQL 5.7+ 支持 `JSON` 类型，用于存储 JSON 数据：

| 类型 | 描述 |
|------|------|
| `JSON` | JSON 数据类型 |

**示例**：

```sql
-- 创建包含 JSON 类型字段的表
CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    attributes JSON
);

-- 插入 JSON 数据
INSERT INTO products (name, attributes) VALUES (
    'Smartphone',
    '{"brand": "Apple", "model": "iPhone 13", "specs": {"storage": "128GB", "color": "Midnight"}}'
);

-- 查询 JSON 数据
SELECT attributes->"$.brand" FROM products;
```

### 5.3 空间类型

MySQL 支持空间类型，用于存储地理空间数据：

| 类型 | 描述 |
|------|------|
| `GEOMETRY` | 几何类型的基类 |
| `POINT` | 点 |
| `LINESTRING` | 线 |
| `POLYGON` | 多边形 |
| `MULTIPOINT` | 多点 |
| `MULTILINESTRING` | 多线 |
| `MULTIPOLYGON` | 多多边形 |
| `GEOMETRYCOLLECTION` | 几何集合 |

**示例**：

```sql
-- 创建包含空间类型字段的表
CREATE TABLE locations (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    position POINT
);

-- 插入空间数据
INSERT INTO locations (name, position) VALUES (
    'Beijing',
    POINT(116.4074, 39.9042)
);
```

## 6. 数据类型选择最佳实践

### 6.1 数值类型选择

- 选择最小的合适类型，减少存储空间
- 对于整数，优先使用 `INT`，如果范围较小可以使用 `TINYINT`、`SMALLINT` 或 `MEDIUMINT`
- 对于小数，精度要求高的使用 `DECIMAL`，否则使用 `FLOAT` 或 `DOUBLE`

### 6.2 字符串类型选择

- 对于固定长度的字符串，使用 `CHAR`
- 对于可变长度的字符串，使用 `VARCHAR`
- 对于较长的文本，使用 `TEXT` 类型
- 对于二进制数据，使用 `BLOB` 类型
- 对于有限的预定义值，使用 `ENUM`
- 对于多个预定义值，使用 `SET`

### 6.3 日期时间类型选择

- 只需要日期时，使用 `DATE`
- 只需要时间时，使用 `TIME`
- 需要日期时间且范围较大时，使用 `DATETIME`
- 需要自动更新的时间戳时，使用 `TIMESTAMP`
- 只需要年份时，使用 `YEAR`

### 6.4 其他类型选择

- 对于布尔值，使用 `BOOLEAN`
- 对于 JSON 数据，使用 `JSON` 类型（MySQL 5.7+）
- 对于地理空间数据，使用空间类型

## 7. 示例：综合使用各种数据类型

```sql
CREATE TABLE users (
    -- 整数类型
    id INT PRIMARY KEY AUTO_INCREMENT,
    age TINYINT,
    salary DECIMAL(10,2),
    
    -- 字符串类型
    name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    gender CHAR(1),
    bio TEXT,
    status ENUM('active', 'inactive', 'pending'),
    hobbies SET('reading', 'sports', 'music', 'travel'),
    
    -- 日期时间类型
    birthday DATE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    last_login DATETIME,
    
    -- 其他类型
    is_admin BOOLEAN DEFAULT FALSE,
    preferences JSON
);
```

## 8. 总结

MySQL 提供了丰富的数据类型，选择合适的数据类型对于数据库的性能和存储空间至关重要。通过本教程的学习，你应该已经掌握了 MySQL 中常用的数据类型及其使用方法。

在实际应用中，你需要根据具体的业务需求，选择合适的数据类型：

- 对于整数，选择合适范围的整数类型
- 对于小数，根据精度要求选择 `FLOAT`、`DOUBLE` 或 `DECIMAL`
- 对于字符串，根据长度和使用场景选择 `CHAR`、`VARCHAR` 或 `TEXT`
- 对于日期时间，根据需要选择 `DATE`、`TIME`、`DATETIME` 或 `TIMESTAMP`
- 对于其他特殊数据，使用相应的特殊类型

合理选择数据类型，可以提高数据库的性能，减少存储空间，确保数据的完整性和准确性。
