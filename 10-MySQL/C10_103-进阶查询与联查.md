## 1. 多表联查 (Joins)

### 1.1 基本联查类型

| 联查类型 | 描述 | 语法 |
|---------|------|------|
| **INNER JOIN** | 返回两表匹配的行 | `SELECT * FROM table1 INNER JOIN table2 ON table1.id = table2.id` |
| **LEFT JOIN** | 返回左表所有行，右表不匹配补 NULL | `SELECT * FROM table1 LEFT JOIN table2 ON table1.id = table2.id` |
| **RIGHT JOIN** | 返回右表所有行，左表不匹配补 NULL | `SELECT * FROM table1 RIGHT JOIN table2 ON table1.id = table2.id` |
| **FULL JOIN** | 返回两表所有行，不匹配的补 NULL | MySQL 需通过 `UNION` 模拟 |

### 1.2 联查示例

**示例表结构**:

```sql
-- 部门表
CREATE TABLE departments (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50) NOT NULL
);

-- 员工表
CREATE TABLE employees (
    emp_id INT PRIMARY KEY,
    emp_name VARCHAR(50) NOT NULL,
    dept_id INT,
    salary DECIMAL(10, 2),
    hire_date DATE,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);

-- 插入测试数据
INSERT INTO departments VALUES (1, '技术部'), (2, '市场部'), (3, '财务部');
INSERT INTO employees VALUES 
(1, '张三', 1, 8000, '2020-01-01'),
(2, '李四', 1, 9000, '2020-02-01'),
(3, '王五', 2, 7000, '2020-03-01'),
(4, '赵六', 2, 6000, '2020-04-01'),
(5, '钱七', 3, 10000, '2020-05-01');
```

**INNER JOIN 示例**:

```sql
-- 查询员工及其所属部门
SELECT e.emp_id, e.emp_name, d.dept_name, e.salary
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id;

-- 结果:
-- emp_id | emp_name | dept_name | salary
-- 1      | 张三     | 技术部    | 8000.00
-- 2      | 李四     | 技术部    | 9000.00
-- 3      | 王五     | 市场部    | 7000.00
-- 4      | 赵六     | 市场部    | 6000.00
-- 5      | 钱七     | 财务部    | 10000.00
```

**LEFT JOIN 示例**:

```sql
-- 查询所有部门及其员工（包括没有员工的部门）
SELECT d.dept_id, d.dept_name, e.emp_name, e.salary
FROM departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id;

-- 结果:
-- dept_id | dept_name | emp_name | salary
-- 1      | 技术部    | 张三     | 8000.00
-- 1      | 技术部    | 李四     | 9000.00
-- 2      | 市场部    | 王五     | 7000.00
-- 2      | 市场部    | 赵六     | 6000.00
-- 3      | 财务部    | 钱七     | 10000.00
```

**RIGHT JOIN 示例**:

```sql
-- 查询所有员工及其部门（如果员工没有部门也会显示）
SELECT e.emp_id, e.emp_name, d.dept_name, e.salary
FROM departments d
RIGHT JOIN employees e ON d.dept_id = e.dept_id;

-- 结果:
-- emp_id | emp_name | dept_name | salary
-- 1      | 张三     | 技术部    | 8000.00
-- 2      | 李四     | 技术部    | 9000.00
-- 3      | 王五     | 市场部    | 7000.00
-- 4      | 赵六     | 市场部    | 6000.00
-- 5      | 钱七     | 财务部    | 10000.00
```

**FULL JOIN 模拟**:

```sql
-- 模拟 FULL JOIN
SELECT d.dept_id, d.dept_name, e.emp_name, e.salary
FROM departments d
LEFT JOIN employees e ON d.dept_id = e.dept_id
UNION
SELECT d.dept_id, d.dept_name, e.emp_name, e.salary
FROM departments d
RIGHT JOIN employees e ON d.dept_id = e.dept_id;
```

### 1.3 多表联查

```sql
-- 假设有第三个表：项目表
CREATE TABLE projects (
    project_id INT PRIMARY KEY,
    project_name VARCHAR(50) NOT NULL,
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);

-- 三表联查
SELECT e.emp_name, d.dept_name, p.project_name
FROM employees e
INNER JOIN departments d ON e.dept_id = d.dept_id
INNER JOIN projects p ON d.dept_id = p.dept_id;
```

## 2. 分组统计 (Grouping)

### 2.1 基本分组

使用 `GROUP BY` 配合聚合函数进行分组统计：

```sql
-- 按部门分组，计算每个部门的平均工资
SELECT dept_id, AVG(salary) as avg_salary
FROM employees
GROUP BY dept_id;

-- 结果:
-- dept_id | avg_salary
-- 1      | 8500.00
-- 2      | 6500.00
-- 3      | 10000.00
```

### 2.2 HAVING 子句

`HAVING` 用于对分组后的结果进行过滤，而 `WHERE` 是在分组前过滤：

```sql
-- 查找平均工资大于 7000 的部门
SELECT dept_id, AVG(salary) as avg_salary
FROM employees
GROUP BY dept_id
HAVING AVG(salary) > 7000;

-- 结果:
-- dept_id | avg_salary
-- 1      | 8500.00
-- 3      | 10000.00
```

### 2.3 多列分组

```sql
-- 按部门和入职年份分组，计算平均工资
SELECT dept_id, YEAR(hire_date) as hire_year, AVG(salary) as avg_salary
FROM employees
GROUP BY dept_id, YEAR(hire_date);
```

### 2.4 常用聚合函数

| 聚合函数 | 描述 |
|---------|------|
| `COUNT()` | 计算行数 |
| `SUM()` | 计算总和 |
| `AVG()` | 计算平均值 |
| `MAX()` | 计算最大值 |
| `MIN()` | 计算最小值 |

```sql
-- 计算员工总数、总工资、平均工资、最高工资和最低工资
SELECT 
    COUNT(*) as total_employees,
    SUM(salary) as total_salary,
    AVG(salary) as avg_salary,
    MAX(salary) as max_salary,
    MIN(salary) as min_salary
FROM employees;
```

## 3. 子查询 (Subqueries)

### 3.1 标量子查询

返回单一值的子查询：

```sql
-- 查询工资高于平均工资的员工
SELECT emp_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- 结果:
-- emp_name | salary
-- 李四     | 9000.00
-- 钱七     | 10000.00
```

### 3.2 列子查询

返回一列值的子查询，通常配合 `IN`, `ANY`, `ALL` 使用：

```sql
-- 查询技术部和市场部的员工
SELECT emp_name, dept_id
FROM employees
WHERE dept_id IN (SELECT dept_id FROM departments WHERE dept_name IN ('技术部', '市场部'));

-- 结果:
-- emp_name | dept_id
-- 张三     | 1
-- 李四     | 1
-- 王五     | 2
-- 赵六     | 2
```

### 3.3 行子查询

返回一行多列的子查询：

```sql
-- 查询与张三同部门同工资的员工
SELECT emp_name, dept_id, salary
FROM employees
WHERE (dept_id, salary) = (SELECT dept_id, salary FROM employees WHERE emp_name = '张三');
```

### 3.4 表子查询

返回一个表的子查询，可以作为临时表使用：

```sql
-- 查找每个部门工资最高的员工
SELECT e.emp_name, e.dept_id, e.salary
FROM employees e
JOIN (
    SELECT dept_id, MAX(salary) as max_salary
    FROM employees
    GROUP BY dept_id
) t ON e.dept_id = t.dept_id AND e.salary = t.max_salary;

-- 结果:
-- emp_name | dept_id | salary
-- 李四     | 1      | 9000.00
-- 王五     | 2      | 7000.00
-- 钱七     | 3      | 10000.00
```

### 3.5 相关子查询

子查询中使用了外部查询的列：

```sql
-- 查询每个员工的工资在部门中的排名
SELECT 
    emp_name, 
    dept_id, 
    salary,
    (SELECT COUNT(*) + 1 
     FROM employees e2 
     WHERE e2.dept_id = e1.dept_id AND e2.salary > e1.salary) as rank
FROM employees e1
ORDER BY dept_id, rank;

-- 结果:
-- emp_name | dept_id | salary  | rank
-- 李四     | 1      | 9000.00 | 1
-- 张三     | 1      | 8000.00 | 2
-- 王五     | 2      | 7000.00 | 1
-- 赵六     | 2      | 6000.00 | 2
-- 钱七     | 3      | 10000.00 | 1
```

## 4. 窗口函数 (Window Functions - MySQL 8.0+)

窗口函数允许在不分组的情况下进行聚合计算，为每行数据生成一个结果。

### 4.1 基本语法

```sql
<窗口函数> OVER (
    [PARTITION BY <分区列>]
    [ORDER BY <排序列>]
    [ROWS/RANGE <窗口范围>]
)
```

### 4.2 常用窗口函数

#### 4.2.1 排名函数

| 函数 | 描述 |
|------|------|
| `ROW_NUMBER()` | 为每行分配唯一的序号，即使有相同值 |
| `RANK()` | 相同值会有相同的排名，后续排名会跳过 |
| `DENSE_RANK()` | 相同值会有相同的排名，后续排名不会跳过 |

**示例**:

```sql
-- 按部门对员工工资进行排名
SELECT 
    emp_name, 
    dept_id, 
    salary,
    ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) as row_num,
    RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) as rank,
    DENSE_RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) as dense_rank
FROM employees;

-- 结果:
-- emp_name | dept_id | salary  | row_num | rank | dense_rank
-- 李四     | 1      | 9000.00 | 1       | 1    | 1
-- 张三     | 1      | 8000.00 | 2       | 2    | 2
-- 王五     | 2      | 7000.00 | 1       | 1    | 1
-- 赵六     | 2      | 6000.00 | 2       | 2    | 2
-- 钱七     | 3      | 10000.00 | 1       | 1    | 1
```

#### 4.2.2 聚合函数作为窗口函数

```sql
-- 计算累计工资
SELECT 
    emp_name, 
    dept_id, 
    salary,
    SUM(salary) OVER (PARTITION BY dept_id ORDER BY salary) as cumulative_salary,
    AVG(salary) OVER (PARTITION BY dept_id) as dept_avg_salary,
    MAX(salary) OVER (PARTITION BY dept_id) as dept_max_salary
FROM employees;

-- 结果:
-- emp_name | dept_id | salary  | cumulative_salary | dept_avg_salary | dept_max_salary
-- 张三     | 1      | 8000.00 | 8000.00          | 8500.00         | 9000.00
-- 李四     | 1      | 9000.00 | 17000.00         | 8500.00         | 9000.00
-- 赵六     | 2      | 6000.00 | 6000.00          | 6500.00         | 7000.00
-- 王五     | 2      | 7000.00 | 13000.00         | 6500.00         | 7000.00
-- 钱七     | 3      | 10000.00 | 10000.00         | 10000.00        | 10000.00
```

#### 4.2.3 分析函数

| 函数 | 描述 |
|------|------|
| `LAG()` | 获取前 N 行的值 |
| `LEAD()` | 获取后 N 行的值 |
| `FIRST_VALUE()` | 获取窗口内的第一个值 |
| `LAST_VALUE()` | 获取窗口内的最后一个值 |

**示例**:

```sql
-- 计算员工工资与前一个员工的工资差异
SELECT 
    emp_name, 
    dept_id, 
    salary,
    LAG(salary, 1) OVER (PARTITION BY dept_id ORDER BY salary) as prev_salary,
    salary - LAG(salary, 1) OVER (PARTITION BY dept_id ORDER BY salary) as salary_diff
FROM employees;

-- 结果:
-- emp_name | dept_id | salary  | prev_salary | salary_diff
-- 张三     | 1      | 8000.00 | NULL        | NULL
-- 李四     | 1      | 9000.00 | 8000.00     | 1000.00
-- 赵六     | 2      | 6000.00 | NULL        | NULL
-- 王五     | 2      | 7000.00 | 6000.00     | 1000.00
-- 钱七     | 3      | 10000.00 | NULL        | NULL
```

### 4.3 窗口范围

```sql
-- 使用 ROWS 定义窗口范围
SELECT 
    emp_name, 
    dept_id, 
    salary,
    SUM(salary) OVER (
        PARTITION BY dept_id 
        ORDER BY salary 
        ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING
    ) as moving_sum
FROM employees;

-- 结果:
-- emp_name | dept_id | salary  | moving_sum
-- 张三     | 1      | 8000.00 | 17000.00  -- 8000 + 9000
-- 李四     | 1      | 9000.00 | 17000.00  -- 8000 + 9000
-- 赵六     | 2      | 6000.00 | 13000.00  -- 6000 + 7000
-- 王五     | 2      | 7000.00 | 13000.00  -- 6000 + 7000
-- 钱七     | 3      | 10000.00 | 10000.00  -- 只有自己
```

## 5. 实际应用示例

### 5.1 复杂查询示例

```sql
-- 查询每个部门工资最高的前两名员工
SELECT 
    emp_name, 
    dept_name, 
    salary,
    rank
FROM (
    SELECT 
        e.emp_name, 
        d.dept_name, 
        e.salary,
        ROW_NUMBER() OVER (PARTITION BY e.dept_id ORDER BY e.salary DESC) as rank
    FROM employees e
    JOIN departments d ON e.dept_id = d.dept_id
) t
WHERE rank <= 2;

-- 结果:
-- emp_name | dept_name | salary  | rank
-- 李四     | 技术部    | 9000.00 | 1
-- 张三     | 技术部    | 8000.00 | 2
-- 王五     | 市场部    | 7000.00 | 1
-- 赵六     | 市场部    | 6000.00 | 2
-- 钱七     | 财务部    | 10000.00 | 1
```

### 5.2 性能优化建议

1. **使用索引**: 确保联查的连接列和 WHERE 子句中的列有索引
2. **合理使用子查询**: 避免过于复杂的子查询，考虑使用 JOIN 替代
3. **限制返回数据**: 使用 LIMIT 限制返回行数
4. **避免 SELECT ***: 只选择需要的列
5. **使用 EXPLAIN**: 分析查询执行计划，找出性能瓶颈
