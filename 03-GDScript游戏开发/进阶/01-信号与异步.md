## 1. 信号 (Signals)

信号是 Godot 中实现观察者模式的核心机制，用于对象间的通信，实现松耦合的设计。

### 1.1 定义与发射 (Definition & Emission)

#### 基本定义

```gdscript
# 在脚本顶部定义信号
signal health_changed(new_health)
signal game_over(reason)
signal player_joined(name, level)
```

#### 发射信号

```gdscript
func take_damage(amount: int):
    health -= amount
    # 发射信号并传递参数
    health_changed.emit(health)
    
    if health <= 0:
        # 发射游戏结束信号
        game_over.emit("Player died")
```

### 1.2 信号连接方式

#### 代码连接

```gdscript
# 连接信号到当前节点的方法
player.health_changed.connect(_on_health_changed)

# 连接信号到其他节点的方法
player.health_changed.connect(ui.update_health_bar)

# 断开连接
player.health_changed.disconnect(_on_health_changed)
```

#### 编辑器连接

1. 在场景树中选择发送信号的节点
2. 在 Inspector 面板中切换到 **Node** 选项卡
3. 在 **Signals** 部分找到要连接的信号
4. 点击信号旁边的 **Connect** 按钮
5. 在弹出的对话框中选择接收信号的节点和方法

#### 连接参数

```gdscript
# 连接时传递额外参数
player.health_changed.connect(_on_health_changed.bind("Player"))

# 使用 lambda 表达式
player.health_changed.connect(func(health): print("Current health: ", health))
```

### 1.3 信号的高级使用

#### 信号组

```gdscript
# 定义多个信号
signal enemy_spotted

signal enemy_defeated

signal mission_completed

# 统一处理信号
func setup_signals():
    enemy_spotted.connect(_on_enemy_spotted)
    enemy_defeated.connect(_on_enemy_defeated)
    mission_completed.connect(_on_mission_completed)
```

#### 延迟连接

```gdscript
# 在节点_ready后连接信号
func _ready():
    # 等待其他节点初始化
    await get_tree().create_timer(0.1).timeout
    player.health_changed.connect(_on_health_changed)
```

#### 一次性信号

```gdscript
# 连接信号，只触发一次后自动断开
player.health_changed.connect(_on_health_changed, flags=CONNECT_ONESHOT)
```

### 1.4 信号的最佳实践

1. **命名规范**：使用动词+宾语的形式，如 `health_changed`、`enemy_spawned`
2. **参数设计**：只传递必要的参数，避免传递整个对象
3. **解耦设计**：使用信号将数据层与视图层分离
4. **信号管理**：在 `_ready()` 中连接信号，在 `_exit_tree()` 中断开连接
5. **信号文档**：为信号添加注释，说明其用途和参数

## 2. 异步处理 (Asynchronous - `await`)

Godot 4 引入了 `await` 关键字，替代了 Godot 3 中的 `yield`，提供了更现代、更直观的异步编程方式。

### 2.1 基本使用

```gdscript
func delayed_action():
    print("Starting action...")
    # 等待 1 秒
    await get_tree().create_timer(1.0).timeout
    print("Action done after 1 second")
```

### 2.2 等待信号

```gdscript
func wait_for_player_input():
    print("Waiting for player input...")
    # 等待按钮按下信号
    await $Button.pressed
    print("Button pressed!")
```

### 2.3 异步函数

```gdscript
# 定义异步函数
func async load_level(level_id: int) -> bool:
    print("Loading level ", level_id)
    # 模拟加载时间
    await get_tree().create_timer(2.0).timeout
    print("Level ", level_id, " loaded")
    return true

# 调用异步函数
func start_game():
    var success = await load_level(1)
    if success:
        print("Game started!")
```

### 2.4 并行异步

```gdscript
func load_resources():
    # 并行加载多个资源
    var load_textures = load_textures_async()
    var load_audio = load_audio_async()
    var load_models = load_models_async()
    
    # 等待所有加载完成
    await all([load_textures, load_audio, load_models])
    print("All resources loaded!")

func load_textures_async():
    await get_tree().create_timer(1.0).timeout
    print("Textures loaded")

func load_audio_async():
    await get_tree().create_timer(1.5).timeout
    print("Audio loaded")

func load_models_async():
    await get_tree().create_timer(2.0).timeout
    print("Models loaded")
```

### 2.5 异步错误处理

```gdscript
func risky_operation():
    try:
        await perform_risky_task()
        print("Operation successful")
    except:
        print("Operation failed")

func perform_risky_task():
    await get_tree().create_timer(1.0).timeout
    # 模拟错误
    if randf() < 0.5:
        push_error("Task failed")
        return
    print("Task completed")
```

### 2.6 协程与状态管理

```gdscript
var is_processing = false

func process_data():
    if is_processing:
        return
    
    is_processing = true
    
    try:
        await process_chunk(1)
        await process_chunk(2)
        await process_chunk(3)
        print("All chunks processed")
    finally:
        is_processing = false

func process_chunk(chunk_id: int):
    print("Processing chunk ", chunk_id)
    await get_tree().create_timer(0.5).timeout
    print("Chunk ", chunk_id, " processed")
```

### 2.7 异步的最佳实践

1. **避免阻塞**：使用异步操作处理耗时任务，避免阻塞主线程
2. **错误处理**：使用 try-except 捕获异步操作中的错误
3. **状态管理**：跟踪异步操作的状态，避免重复执行
4. **资源清理**：在异步操作完成后清理资源
5. **用户反馈**：在异步操作期间提供适当的用户反馈

## 3. 注解 (Annotations - Godot 4+)

注解是 Godot 4 引入的新特性，用于向编译器或编辑器提供额外信息，增强代码的可读性和功能。

### 3.1 常用注解

#### `@export` - 暴露变量到编辑器

```gdscript
# 基本使用
@export var speed: float = 100.0

# 限制范围
@export_range(0, 100) var health: int = 100

# 枚举选择
@export_enum("Easy", "Medium", "Hard") var difficulty: String = "Medium"

# 资源引用
@export var texture: Texture2D

# 节点路径
@export var player_path: NodePath
```

#### `@onready` - 自动初始化变量

```gdscript
# 在 _ready() 执行前自动初始化
@onready var player = $Player
@onready var ui = $UI
@onready var camera = $Camera2D

func _ready():
    # 可以直接使用初始化后的变量
    player.speed = 100.0
    ui.show()
    camera.follow(player)
```

#### `@tool` - 编辑器模式运行

```gdscript
@tool
class_name MyCustomNode extends Node

@export var color: Color = Color(1, 0, 0)

# 在编辑器模式下也会执行
func _process(delta):
    if Engine.is_editor_hint():
        # 编辑器模式下的逻辑
        update_editor_preview()
    else:
        # 运行时逻辑
        update_runtime()
```

#### `@rpc` - 远程过程调用

```gdscript
# 定义 RPC 方法
@rpc(any_peer)
func sync_position(new_position: Vector3):
    position = new_position

# 调用 RPC 方法
func update_position():
    rpc("sync_position", position)
```

### 3.2 高级注解

#### `@icon` - 自定义节点图标

```gdscript
@icon("res://icons/my_node_icon.svg")
class_name MyCustomNode extends Node
```

#### `@warning_ignore` - 忽略警告

```gdscript
@warning_ignore("unused_parameter")
func on_button_pressed(_button: Button):
    print("Button pressed")
```

#### `@deprecated` - 标记为已弃用

```gdscript
@deprecated("Use new_method() instead")
func old_method():
    print("This method is deprecated")

func new_method():
    print("This is the new method")
```

### 3.3 注解的最佳实践

1. **合理使用**：只在必要时使用注解，避免过度使用
2. **清晰命名**：使用有意义的变量名和方法名
3. **文档注释**：为使用注解的代码添加适当的注释
4. **版本兼容性**：注意注解在不同 Godot 版本中的兼容性
5. **性能考虑**：某些注解可能会影响性能，如 `@tool`

## 4. 实际应用案例

### 4.1 游戏状态管理

```gdscript
class_name GameManager extends Node

signal game_started
signal game_paused
signal game_resumed
signal game_over(score)

@export var initial_lives: int = 3
var lives: int
var score: int = 0
var is_paused: bool = false

func _ready():
    lives = initial_lives
    # 连接信号
    get_tree().paused.connect(_on_tree_paused)

func start_game():
    lives = initial_lives
    score = 0
    is_paused = false
    game_started.emit()

func pause_game():
    is_paused = true
    get_tree().paused = true
    game_paused.emit()

func resume_game():
    is_paused = false
    get_tree().paused = false
    game_resumed.emit()

func add_score(points: int):
    score += points

func lose_life():
    lives -= 1
    if lives <= 0:
        end_game()

func end_game():
    get_tree().paused = true
    game_over.emit(score)

func _on_tree_paused(paused: bool):
    is_paused = paused
```

### 4.2 异步资源加载

```gdscript
class_name ResourceLoader extends Node

signal load_started
signal load_progress(percentage)
signal load_completed
signal load_failed(error)

@export var resources: Array[Resource] = []
var loaded_resources: Array[Resource] = []

func load_resources_async():
    if resources.is_empty():
        load_completed.emit()
        return
    
    load_started.emit()
    loaded_resources.clear()
    
    var total = resources.size()
    for i in range(total):
        var resource = resources[i]
        try:
            await resource.load_async()
            loaded_resources.append(resource)
            var progress = float(i + 1) / total * 100
            load_progress.emit(progress)
        except:
            load_failed.emit("Failed to load resource: " + str(resource.resource_path))
            return
    
    load_completed.emit()
```

### 4.3 敌人 AI 行为

```gdscript
class_name EnemyAI extends Node

@export var patrol_points: Array[Vector3] = []
@export var patrol_speed: float = 2.0
@export var chase_speed: float = 4.0
@export var detection_range: float = 10.0

var current_state: String = "patrolling"
var current_point: int = 0
var player: CharacterBody3D

func _ready():
    player = get_tree().get_first_node_in_group("player")
    start_patrolling()

func start_patrolling():
    current_state = "patrolling"
    patrol_coroutine()

func patrol_coroutine():
    while current_state == "patrolling":
        if patrol_points.size() > 0:
            var target = patrol_points[current_point]
            await move_to(target, patrol_speed)
            
            current_point = (current_point + 1) % patrol_points.size()
        else:
            await get_tree().create_timer(1.0).timeout
        
        # 检查是否检测到玩家
        if is_player_detected():
            start_chasing()
            return

func start_chasing():
    current_state = "chasing"
    chase_coroutine()

func chase_coroutine():
    while current_state == "chasing":
        if player:
            await move_to(player.position, chase_speed)
        
        # 检查玩家是否超出范围
        if not is_player_detected():
            start_patrolling()
            return

func is_player_detected() -> bool:
    if not player:
        return false
    
    var distance = position.distance_to(player.position)
    return distance <= detection_range

func move_to(target: Vector3, speed: float):
    var direction = (target - position).normalized()
    var distance = position.distance_to(target)
    var duration = distance / speed
    
    var start_time = Time.get_time_dict_from_system()
    var elapsed = 0.0
    
    while elapsed < duration:
        var progress = elapsed / duration
        position = position.linear_interpolate(target, progress)
        
        await get_tree().process_frame
        elapsed = Time.get_time_dict_from_system() - start_time
    
    position = target
```

## 5. 常见问题与解决方案

### 5.1 信号连接问题

**问题**：信号连接后不触发

**解决方案**：
1. 检查信号名称是否正确
2. 检查连接是否在信号发射前完成
3. 检查节点是否被正确引用
4. 检查连接是否被意外断开

### 5.2 异步操作问题

**问题**：异步操作卡住或不执行

**解决方案**：
1. 确保 await 表达式正确
2. 检查是否有无限循环
3. 确保信号能够正常发射
4. 检查错误处理

### 5.3 注解使用问题

**问题**：注解不生效

**解决方案**：
1. 确保使用的是 Godot 4+ 版本
2. 检查注解语法是否正确
3. 检查变量类型是否匹配
4. 重启编辑器以应用更改

### 5.4 性能问题

**问题**：信号或异步操作导致性能下降

**解决方案**：
1. 减少信号的发射频率
2. 避免在每一帧发射信号
3. 合理使用异步操作，避免过多的 await
4. 优化信号连接和断开的时机
