## 1. 什么是 Markdown

Markdown 是一种轻量级标记语言，由 John Gruber 和 Aaron Swartz 在 2004 年创建。它允许人们使用易读易写的纯文本格式编写文档，然后转换为结构完整的 HTML 文档。

### 1.1 Markdown 的特点

- **简单易用**：语法简洁明了，学习成本低
- **跨平台兼容**：几乎所有现代文本编辑器都支持
- **可读性强**：即使不转换为 HTML，纯文本也具有良好的可读性
- **灵活性高**：支持嵌入 HTML 代码，扩展功能
- **广泛应用**：GitHub、GitLab、Bitbucket 等平台都支持 Markdown

### 1.2 Markdown 的应用场景

- **项目文档**：README 文件、API 文档等
- **技术博客**：许多博客平台支持 Markdown
- **笔记整理**：个人学习笔记、会议记录等
- **邮件撰写**：部分邮件客户端支持 Markdown
- **电子书创作**：可以转换为 PDF、EPUB 等格式

## 2. 标题语法

Markdown 支持 6 级标题，通过 `#` 的数量区分层级，**`#` 后必须加 1 个空格**，否则语法不生效。

### 语法示例

```markdown
# 一级标题（对应 HTML h1，文档主标题）
## 二级标题（对应 HTML h2，一级章节）
### 三级标题（对应 HTML h3，二级章节）
#### 四级标题
##### 五级标题
###### 六级标题
```

### 渲染效果

# 一级标题

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题

## 3. 段落与换行

### 核心规则

1. **段落分隔**：两个段落之间必须用**1个空行**分隔，无空行的连续文本会被合并为同一段落
2. **强制换行**：行尾添加**2个及以上空格**后按回车，实现同一段落内的换行（全平台通用标准写法）

### 语法示例

```markdown
这是第一个段落的第一行
这一行和上一行无空行、无行尾空格，会被合并为同一段落

这是第二个段落（上一行是空行，成功分段）
这一行末尾加了2个空格  
这一行成功换行，和上一行属于同一段落
```

## 4. 基础文本格式

### 核心语法

| 格式效果 | 语法写法 | 渲染效果 |
| :--- | :--- | :--- |
| 斜体 | `*斜体文本*` 或 `_斜体文本_` | *斜体文本* |
| 粗体 | `**粗体文本**` 或 `__粗体文本__` | **粗体文本** |
| 粗斜体 | `***粗斜体文本***` 或 `___粗斜体文本___` | ***粗斜体文本*** |
| 删除线(GFM扩展) | `~~删除线文本~~` | ~~删除线文本~~ |

### 语法示例

```markdown
这是*斜体*、**粗体**、***粗斜体***、~~删除线~~文本
```

## 5. 列表语法

### 5.1 无序列表

使用 `-`/`*`/`+` + 空格开头，三者效果完全一致，推荐统一使用 `-`，嵌套列表需缩进**4个空格/1个制表符**。

#### 语法示例

```markdown
- 无序列表一级项1
- 无序列表一级项2
    - 无序列表二级嵌套项1
    - 无序列表二级嵌套项2
        - 无序列表三级嵌套项
- 无序列表一级项3
```

#### 渲染效果

- 无序列表一级项1
- 无序列表一级项2
  - 无序列表二级嵌套项1
  - 无序列表二级嵌套项2
    - 无序列表三级嵌套项
- 无序列表一级项3

### 5.2 有序列表

使用 `数字 + 英文句号 + 空格` 开头，数字顺序不影响最终渲染结果，推荐按顺序书写，嵌套需缩进**4个空格/1个制表符**。

#### 语法示例

```markdown
1. 有序列表一级项1
2. 有序列表一级项2
    1. 有序列表二级嵌套项1
    2. 有序列表二级嵌套项2
3. 有序列表一级项3
```

#### 渲染效果

1. 有序列表一级项1
2. 有序列表一级项2
   1. 有序列表二级嵌套项1
   2. 有序列表二级嵌套项2
3. 有序列表一级项3

### 5.3 任务列表

待办清单专用语法（GFM 扩展），`[ ]` 表示未完成，`[x]` 表示已完成，符号后必须加空格。

#### 语法示例

```markdown
- [ ] 未完成的待办任务
- [x] 已完成的待办任务
- [ ] 可嵌套待办
    - [x] 子任务1
    - [ ] 子任务2
```

#### 渲染效果

- [ ] 未完成的待办任务
- [x] 已完成的待办任务
- [ ] 可嵌套待办
  - [x] 子任务1
  - [ ] 子任务2

## 6. 块引用

使用 `>` + 空格开头，可多层嵌套，也可与标题、列表、代码等其他语法混用。

### 语法示例

```markdown
> 单层块引用，多行内容可只在第一行加>
> 第二行内容，和上一行同属一个引用块

> 多层嵌套引用
> > 第二层嵌套引用
> > > 第三层嵌套引用

> 引用内混用其他语法
> 1. 有序列表项
> 2. 有序列表项
> **粗体文本**
```

### 渲染效果

> 单层块引用，多行内容可只在第一行加>
> 第二行内容，和上一行同属一个引用块

> 多层嵌套引用
>
> > 第二层嵌套引用
>
> > > 第三层嵌套引用

> 引用内混用其他语法
>
> 1. 有序列表项
> 2. 有序列表项
>    **粗体文本**

## 7. 分隔线

使用 3 个及以上的 `-`/`*`/`_` 单独占一行，前后建议保留空行，避免与二级标题语法混淆。

### 语法示例

```markdown
---
***
___
```

### 渲染效果

***

***

## 8. 链接语法

### 8.1 行内式链接

标准格式：`[链接显示文本](链接地址 "链接可选标题")`

- 可选标题：鼠标悬浮时显示的提示文字，可省略

#### 语法示例

```markdown
[GitHub官网](https://github.com "全球最大开源托管平台")
[打开豆包](https://www.doubao.com)
```

#### 渲染效果

[GitHub官网](https://github.com "全球最大开源托管平台")
[打开豆包](https://www.doubao.com)

### 8.2 自动链接

用 `<>` 包裹网址/邮箱，快速生成链接，适合直接展示完整地址。

#### 语法示例

```markdown
<https://github.com>
<fanquanpangpiing@163.com>
```

#### 渲染效果

<https://github.com>
<fanquanpangpiing@163.com>

### 8.3 参考式链接

适合长文档中多次复用同一个链接，文档末尾统一定义地址，便于维护。

#### 语法示例

```markdown
[GitHub官网][github]
[GitHub开源社区][github]

<!-- 文档末尾统一定义 -->
[github]: https://github.com "GitHub官网"
```

## 9. 图片语法

基础格式：`![图片替代文本](图片地址 "图片可选标题")`

- 替代文本：图片加载失败时显示的文字，必填
- 可选标题：鼠标悬浮时显示的提示文字，可省略

### 语法示例

```markdown
![示例图](https://example.com/image.jpg "这是一张示例图片")
```

### 图片带跳转链接

嵌套链接语法，实现点击图片跳转到指定地址。

```markdown
[![示例图片](https://example.com/image.jpg)](https://github.com)
```

## 10. 代码语法

### 10.1 行内代码

用单个反引号 `` ` `` 包裹，适合在行内插入短代码、关键词、命令。

#### 语法示例

```markdown
Python的打印函数是`print("Hello World")`
```

#### 渲染效果

Python的打印函数是`print("Hello World")`

### 10.2 多行代码块

用三组反引号 ``` 包裹，上下单独占一行，支持指定编程语言实现语法高亮。

#### 语法示例

```markdown
```python
# Python代码示例
def hello_markdown():
    print("Hello Markdown!")
    return 0
```
```

#### 渲染效果

```python
# Python代码示例
def hello_markdown():
    print("Hello Markdown!")
    return 0
```

## 11. 表格语法

使用 `|` 分隔单元格，`-` 分隔表头和表体，`:` 定义列对齐方式（GFM 扩展）：

- 左对齐：`:---`（默认）
- 居中对齐：`:---:`
- 右对齐：`---:`

### 语法示例

```markdown
| 左对齐列 | 居中对齐列 | 右对齐列 |
| :--- | :--- | :--- |
| 内容1 | 内容2 | 内容3 |
| 长文本测试 | 居中显示 | 靠右显示 |
```

### 渲染效果

| 左对齐列 | 居中对齐列 | 右对齐列 |
| :--- | :--- | :--- |
| 内容1 | 内容2 | 内容3 |
| 长文本测试 | 居中显示 | 靠右显示 |

## 12. 转义字符

使用反斜杠 `\` 转义 Markdown 特殊字符，使其正常显示，避免被解析为语法标记。

### 可转义的核心特殊字符

```markdown
\   反斜杠
*   星号
_   下划线
#   井号
+   加号
-   减号
.   英文句号
!   感叹号
|   竖线
`   反引号
[ ] 方括号
( ) 圆括号
```

### 语法示例

```markdown
\# 转义后正常显示#号，不会被解析为标题
\* 转义后正常显示*号，不会被解析为列表
```

## 14. 常见问题与最佳实践

### 常见问题

1. **语法标记后必须加空格**：标题、列表、引用、任务列表等语法的标记符后，必须加 1 个空格，否则无法正常渲染
2. **段落必须用空行分隔**：无空行的连续文本会被合并为同一段落，无法实现分段
3. **嵌套元素必须规范缩进**：列表嵌套、引用嵌套必须缩进 4 个空格/1 个制表符，否则会出现渲染异常
4. **特殊字符必须转义**：文本中包含 Markdown 专用特殊字符时，必须用 `\` 转义，避免被错误解析
5. **跨平台优先用标准语法**：不同平台对扩展语法支持度不同，跨平台写作优先使用通用标准语法

### 最佳实践

1. **统一风格**：在同一文档中使用一致的 Markdown 语法风格，如统一使用 `-` 作为无序列表标记
2. **合理使用标题层级**：根据文档结构合理使用标题层级，避免过多或过少的层级
3. **添加适当的空行**：在不同语法元素之间添加空行，提高文档的可读性
4. **使用代码块**：对于代码、命令等内容，使用代码块而非行内代码，提高可读性
5. **添加注释**：对于复杂的文档，可使用 HTML 注释 `<!-- 注释内容 -->` 添加说明
6. **测试渲染效果**：在不同的 Markdown 编辑器中测试渲染效果，确保跨平台兼容性
