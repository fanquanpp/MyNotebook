## 1. 数据库概述 (Overview)
MySQL 是全球最受欢迎的**开源关系型数据库管理系统 (RDBMS)**，由 Oracle 公司维护和开发。

### 1.1 核心概念
- **关系型 (RDBMS)**: 数据存储在表中，表之间通过外键关联，遵循 ACID 特性
- **SQL (Structured Query Language)**: 结构化查询语言，用于管理数据
- **表 (Table)**: 数据的基本存储单元，由行和列组成
- **字段 (Column)**: 表中的列，定义数据类型
- **记录 (Row)**: 表中的行，包含一条完整的数据
- **主键 (Primary Key)**: 唯一标识表中记录的字段
- **外键 (Foreign Key)**: 关联其他表主键的字段

### 1.2 核心特点 (Key Features)
- **高性能**: 优化的查询引擎 (InnoDB)，支持事务和行级锁
- **高可用**: 支持主从复制、集群部署、读写分离
- **安全性**: 完善的权限控制系统，支持 SSL 加密
- **可扩展性**: 支持分区表、分库分表
- **社区活跃**: 丰富的文档和第三方支持
- **跨平台**: 支持 Windows、Linux、macOS 等多种操作系统

### 1.3 MySQL 版本选择
- **Community Edition**: 免费开源版本，适合大多数应用场景
- **Enterprise Edition**: 商业版本，提供更多高级功能和技术支持
- **Cluster CGE**: 集群版本，提供高可用性和横向扩展能力

## 2. 环境搭建 (Environment Setup)

### 2.1 安装方法

#### 2.1.1 Docker 部署 (推荐)
```bash
# 拉取 MySQL 镜像
docker pull mysql:8.0

# 运行容器
docker run --name mysql \
  -e MYSQL_ROOT_PASSWORD=your_password \
  -e MYSQL_DATABASE=mydb \
  -e MYSQL_USER=user \
  -e MYSQL_PASSWORD=user_password \
  -p 3306:3306 \
  -v mysql-data:/var/lib/mysql \
  -d mysql:8.0 --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
```

#### 2.1.2 Windows 安装
1. 从 [MySQL 官网](https://dev.mysql.com/downloads/installer/) 下载 MySQL Installer
2. 运行安装程序，选择 "Developer Default" 或 "Server Only"
3. 按照向导完成安装和配置
4. 启动 MySQL 服务

#### 2.1.3 Linux 安装 (Ubuntu)
```bash
# 更新包列表
sudo apt update

# 安装 MySQL
sudo apt install mysql-server

# 安全配置
sudo mysql_secure_installation

# 启动服务
sudo systemctl start mysql
sudo systemctl enable mysql
```

### 2.2 环境变量配置

#### 2.2.1 Windows
1. 右键 "此电脑" → "属性" → "高级系统设置" → "环境变量"
2. 在 "系统变量" 中找到 "Path"，添加 MySQL 安装目录的 bin 文件夹路径
3. 重启命令行窗口使配置生效

#### 2.2.2 Linux
```bash
# 编辑环境变量文件
sudo nano /etc/profile

# 在文件末尾添加
export PATH=$PATH:/usr/bin/mysql

# 使配置生效
source /etc/profile
```

### 2.3 管理工具

#### 2.3.1 命令行工具
- **mysql**: 官方命令行客户端
  ```bash
  mysql -u root -p
  ```
- **mysqldump**: 数据备份工具
  ```bash
  mysqldump -u root -p database_name > backup.sql
  ```
- **mysqladmin**: 管理工具
  ```bash
  mysqladmin -u root -p status
  ```

#### 2.3.2 GUI 工具
- **MySQL Workbench**: 官方 GUI 工具，功能全面
- **DBeaver**: 开源跨平台数据库工具，支持多种数据库
- **Navicat**: 商业 GUI 工具，界面友好，功能强大
- **phpMyAdmin**: Web 界面管理工具，适合远程管理

## 3. 基本操作 (Basic Ops)

### 3.1 数据库操作
```sql
-- 查看所有数据库
SHOW DATABASES;

-- 创建数据库
CREATE DATABASE mydb 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

-- 删除数据库
DROP DATABASE IF EXISTS mydb;

-- 使用数据库
USE mydb;

-- 查看当前数据库
SELECT DATABASE();
```

### 3.2 表操作
```sql
-- 创建表
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50) NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  age INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 查看表结构
DESCRIBE users;

-- 修改表结构
ALTER TABLE users ADD COLUMN phone VARCHAR(20);

-- 删除表
DROP TABLE IF EXISTS users;
```

### 3.3 数据操作
```sql
-- 插入数据
INSERT INTO users (name, email, age) VALUES 
('张三', 'zhangsan@example.com', 25),
('李四', 'lisi@example.com', 30);

-- 查询数据
SELECT * FROM users;
SELECT name, email FROM users WHERE age > 25;

-- 更新数据
UPDATE users SET age = 26 WHERE name = '张三';

-- 删除数据
DELETE FROM users WHERE id = 1;
```

### 3.4 用户与权限
```sql
-- 创建用户
CREATE USER 'newuser'@'localhost' IDENTIFIED BY 'password';

-- 授予权限
GRANT ALL PRIVILEGES ON mydb.* TO 'newuser'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;

-- 查看用户权限
SHOW GRANTS FOR 'newuser'@'localhost';

-- 撤销权限
REVOKE ALL PRIVILEGES ON mydb.* FROM 'newuser'@'localhost';

-- 删除用户
DROP USER 'newuser'@'localhost';
```

## 4. 数据库设计基础

### 4.1 设计原则
- **范式化**: 遵循数据库设计范式，减少数据冗余
- **主键设计**: 选择合适的主键类型，推荐使用自增整数
- **索引设计**: 为常用查询字段创建索引
- **外键约束**: 建立表之间的关联关系
- **数据类型**: 选择合适的数据类型，优化存储空间

### 4.2 常用数据类型
| 数据类型 | 描述 | 示例 |
| :--- | :--- | :--- |
| INT | 整数类型 | `age INT` |
| VARCHAR | 可变长度字符串 | `name VARCHAR(50)` |
| TEXT | 长文本 | `description TEXT` |
| DATE | 日期 | `birthday DATE` |
| DATETIME | 日期时间 | `created_at DATETIME` |
| DECIMAL |  decimal(10,2) | `price DECIMAL(10,2)` |
| BOOLEAN | 布尔值 | `active BOOLEAN` |

## 5. 性能优化建议

### 5.1 服务器配置
- 调整 `innodb_buffer_pool_size`（建议为内存的 70-80%）
- 设置合适的 `max_connections`
- 配置 `query_cache_size`（适用于读多写少的场景）

### 5.2 查询优化
- 使用索引加速查询
- 避免 SELECT *，只选择需要的字段
- 使用 LIMIT 限制返回行数
- 避免在 WHERE 子句中使用函数
- 合理使用 JOIN 操作

### 5.3 备份策略
- 定期使用 mysqldump 进行全量备份
- 启用二进制日志，用于增量备份
- 考虑使用第三方备份工具，如 Percona XtraBackup

## 6. 安全配置

### 6.1 基础安全
- 设置强密码策略
- 限制 root 用户远程访问
- 定期更新 MySQL 版本
- 移除匿名用户

### 6.2 高级安全
- 启用 SSL 加密连接
- 使用防火墙限制数据库端口访问
- 定期审计用户权限
- 启用慢查询日志，监控异常查询

## 7. 常见问题与解决方案

### 7.1 连接问题
- **问题**: 无法连接到 MySQL 服务器
- **解决方案**: 检查网络连接、防火墙设置、MySQL 服务状态

### 7.2 权限问题
- **问题**: 访问被拒绝
- **解决方案**: 检查用户权限、主机限制

### 7.3 性能问题
- **问题**: 查询速度慢
- **解决方案**: 优化查询语句、添加索引、调整服务器配置

### 7.4 数据丢失
- **问题**: 数据意外删除
- **解决方案**: 从备份恢复、使用二进制日志恢复
